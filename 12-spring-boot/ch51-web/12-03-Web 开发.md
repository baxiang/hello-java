# 12.3 Web 开发：如何开发 RESTful API？

## 一、先问一个问题

**什么是 RESTful API？**

### 生活例子：餐厅菜单

```
传统方式
├── /getUser?id=1（获取用户）
├── /createUser（创建用户）
└── /updateUser（修改用户）

RESTful 方式
├── GET /users/1（获取用户）
├── POST /users（创建用户）
├── PUT /users/1（修改用户）
└── DELETE /users/1（删除用户）
```

**RESTful 原则**：
- 用 HTTP 方法表示操作
- 用 URL 表示资源
- 统一、规范、易理解

## 二、创建 Controller

### 基本结构

```java
package com.example.demo.controller;

import org.springframework.web.bind.annotation.*;
import java.util.*;

@RestController  // 标记这是 REST 控制器
@RequestMapping("/api/users")  // 基础路径
public class UserController {
    
    // 模拟数据库
    private Map<Long, User> db = new HashMap<>();
    
    // 获取所有用户
    @GetMapping
    public List<User> list() {
        return new ArrayList<>(db.values());
    }
    
    // 获取单个用户
    @GetMapping("/{id}")
    public User get(@PathVariable Long id) {
        return db.get(id);
    }
    
    // 创建用户
    @PostMapping
    public User create(@RequestBody User user) {
        db.put(user.getId(), user);
        return user;
    }
    
    // 修改用户
    @PutMapping("/{id}")
    public User update(@PathVariable Long id, @RequestBody User user) {
        db.put(id, user);
        return user;
    }
    
    // 删除用户
    @DeleteMapping("/{id}")
    public void delete(@PathVariable Long id) {
        db.remove(id);
    }
}
```

### User 实体类

```java
package com.example.demo.entity;

import lombok.Data;

@Data  // Lombok 注解，自动生成 getter/setter
public class User {
    private Long id;
    private String username;
    private String email;
    private Integer age;
}
```

### 添加 Lombok 依赖

```xml
<dependency>
    <groupId>org.projectlombok</groupId>
    <artifactId>lombok</artifactId>
    <optional>true</optional>
</dependency>
```

**Lombok 是什么？**
- 自动帮你写 getter/setter
- 自动帮你写构造方法
- 减少代码量

**没有 Lombok**：
```java
public class User {
    private Long id;
    private String username;
    
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }
    public String getUsername() { return username; }
    public void setUsername(String username) { this.username = username; }
    // ... 几十行代码
}
```

**有 Lombok**：
```java
@Data
public class User {
    private Long id;
    private String username;
    // 搞定！
}
```

## 三、请求映射注解

### @RequestMapping

```java
// 类级别：所有方法的基础路径
@RequestMapping("/api/users")

// 方法级别：完整路径
@RequestMapping(value = "/list", method = RequestMethod.GET)
```

### 简化注解

| 注解 | 等价于 | 用途 |
|------|--------|------|
| @GetMapping | @RequestMapping(method=GET) | 查询 |
| @PostMapping | @RequestMapping(method=POST) | 创建 |
| @PutMapping | @RequestMapping(method=PUT) | 更新 |
| @DeleteMapping | @RequestMapping(method=DELETE) | 删除 |
| @PatchMapping | @RequestMapping(method=PATCH) | 部分更新 |

### 示例

```java
@RestController
@RequestMapping("/api/articles")
public class ArticleController {
    
    // GET /api/articles
    @GetMapping
    public List<Article> list() { }
    
    // GET /api/articles/1
    @GetMapping("/{id}")
    public Article get(@PathVariable Long id) { }
    
    // POST /api/articles
    @PostMapping
    public Article create(@RequestBody Article article) { }
    
    // PUT /api/articles/1
    @PutMapping("/{id}")
    public Article update(@PathVariable Long id, @RequestBody Article article) { }
    
    // DELETE /api/articles/1
    @DeleteMapping("/{id}")
    public void delete(@PathVariable Long id) { }
}
```

## 四、参数接收

### 1. 路径参数（@PathVariable）

```java
// URL: /api/users/123
@GetMapping("/users/{id}")
public User get(@PathVariable Long id) {
    // id = 123
}

// 多个路径参数
// URL: /api/users/123/orders/456
@GetMapping("/users/{userId}/orders/{orderId}")
public Order getOrder(
    @PathVariable Long userId,
    @PathVariable Long orderId
) { }
```

### 2. 查询参数（@RequestParam）

```java
// URL: /api/users?page=1&size=10
@GetMapping("/users")
public List<User> list(
    @RequestParam(defaultValue = "1") int page,
    @RequestParam(defaultValue = "10") int size
) { }

// 可选参数
// URL: /api/users?keyword=张三
@GetMapping("/users")
public List<User> search(@RequestParam(required = false) String keyword) { }
```

### 3. 请求体（@RequestBody）

```java
// POST /api/users
// Body: {"username":"张三","email":"zhang@example.com"}
@PostMapping("/users")
public User create(@RequestBody User user) {
    // user.username = "张三"
    // user.email = "zhang@example.com"
}
```

### 4. 请求头（@RequestHeader）

```java
// 获取 Token
@GetMapping("/users/me")
public User getCurrentUser(
    @RequestHeader("Authorization") String token
) { }
```

### 5. Cookie（@CookieValue）

```java
// 获取 Cookie
@GetMapping("/users")
public List<User> list(
    @CookieValue("sessionId") String sessionId
) { }
```

## 五、统一响应格式

### 定义响应类

```java
package com.example.demo.common;

import lombok.Data;
import lombok.AllArgsConstructor;

@Data
@AllArgsConstructor
public class Result<T> {
    private Integer code;      // 状态码
    private String message;    // 消息
    private T data;            // 数据
    
    // 成功响应
    public static <T> Result<T> success(T data) {
        return new Result<>(200, "success", data);
    }
    
    // 成功响应（无数据）
    public static <T> Result<T> success() {
        return new Result<>(200, "success", null);
    }
    
    // 错误响应
    public static <T> Result<T> error(String message) {
        return new Result<>(500, message, null);
    }
    
    // 错误响应（带状态码）
    public static <T> Result<T> error(Integer code, String message) {
        return new Result<>(code, message, null);
    }
}
```

### 使用统一响应

```java
@RestController
@RequestMapping("/api/users")
public class UserController {
    
    @GetMapping
    public Result<List<User>> list() {
        List<User> users = userService.list();
        return Result.success(users);
    }
    
    @GetMapping("/{id}")
    public Result<User> get(@PathVariable Long id) {
        User user = userService.getById(id);
        if (user == null) {
            return Result.error(404, "用户不存在");
        }
        return Result.success(user);
    }
    
    @PostMapping
    public Result<User> create(@RequestBody User user) {
        userService.save(user);
        return Result.success(user);
    }
}
```

### 响应示例

**成功**：
```json
{
    "code": 200,
    "message": "success",
    "data": {
        "id": 1,
        "username": "张三",
        "email": "zhang@example.com"
    }
}
```

**失败**：
```json
{
    "code": 404,
    "message": "用户不存在",
    "data": null
}
```

## 六、数据验证

### 添加依赖

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-validation</artifactId>
</dependency>
```

### 实体类添加验证注解

```java
@Data
public class User {
    
    @NotNull(message = "ID 不能为空")
    private Long id;
    
    @NotBlank(message = "用户名不能为空")
    @Size(min = 2, max = 20, message = "用户名长度 2-20")
    private String username;
    
    @NotBlank(message = "邮箱不能为空")
    @Email(message = "邮箱格式不正确")
    private String email;
    
    @NotNull(message = "年龄不能为空")
    @Min(value = 0, message = "年龄不能小于 0")
    @Max(value = 150, message = "年龄不能大于 150")
    private Integer age;
}
```

### Controller 添加@Valid

```java
@PostMapping
public Result<User> create(@Valid @RequestBody User user) {
    // 如果验证失败，自动抛出异常
    userService.save(user);
    return Result.success(user);
}
```

### 全局异常处理

```java
package com.example.demo.exception;

import com.example.demo.common.Result;
import org.springframework.validation.BindException;
import org.springframework.web.bind.MethodArgumentNotValidException;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.RestControllerAdvice;

@RestControllerAdvice  // 全局异常处理
public class GlobalExceptionHandler {
    
    // 处理参数验证异常
    @ExceptionHandler(MethodArgumentNotValidException.class)
    public Result<Void> handleValidationException(
            MethodArgumentNotValidException e) {
        String message = e.getBindingResult()
            .getFieldErrors()
            .stream()
            .map(error -> error.getField() + ": " + error.getDefaultMessage())
            .findFirst()
            .orElse("参数验证失败");
        return Result.error(400, message);
    }
    
    // 处理业务异常
    @ExceptionHandler(BusinessException.class)
    public Result<Void> handleBusinessException(BusinessException e) {
        return Result.error(e.getCode(), e.getMessage());
    }
    
    // 处理其他异常
    @ExceptionHandler(Exception.class)
    public Result<Void> handleException(Exception e) {
        e.printStackTrace();
        return Result.error(500, "服务器内部错误");
    }
}
```

### 自定义业务异常

```java
package com.example.demo.exception;

import lombok.Getter;

@Getter
public class BusinessException extends RuntimeException {
    
    private final Integer code;
    
    public BusinessException(Integer code, String message) {
        super(message);
        this.code = code;
    }
    
    public BusinessException(String message) {
        super(message);
        this.code = 500;
    }
}
```

### 使用业务异常

```java
@Service
public class UserService {
    
    public User getById(Long id) {
        User user = userMapper.selectById(id);
        if (user == null) {
            throw new BusinessException(404, "用户不存在");
        }
        return user;
    }
    
    public void save(User user) {
        // 检查用户名是否存在
        User exist = userMapper.selectByUsername(user.getUsername());
        if (exist != null) {
            throw new BusinessException("用户名已存在");
        }
        
        userMapper.insert(user);
    }
}
```

## 七、分页查询

### 定义分页参数

```java
@Data
public class PageQuery {
    
    @Min(value = 1, message = "页码最小为 1")
    private Integer page = 1;
    
    @Min(value = 1, message = "每页最小为 1")
    @Max(value = 100, message = "每页最大为 100")
    private Integer size = 10;
    
    private String sortBy = "id";
    private String order = "desc";
}
```

### 定义分页响应

```java
@Data
@AllArgsConstructor
public class PageResult<T> {
    private List<T> list;         // 数据列表
    private Long total;           // 总数
    private Integer page;         // 当前页
    private Integer size;         // 每页数
    private Integer pages;        // 总页数
    
    public static <T> PageResult<T> of(List<T> list, Long total, Integer page, Integer size) {
        int pages = (int) Math.ceil((double) total / size);
        return new PageResult<>(list, total, page, size, pages);
    }
}
```

### Controller 实现分页

```java
@GetMapping
public Result<PageResult<User>> list(PageQuery query) {
    Page<User> page = new Page<>(query.getPage(), query.getSize());
    
    IPage<User> result = userMapper.selectPage(page, null);
    
    PageResult<User> pageResult = PageResult.of(
        result.getRecords(),
        result.getTotal(),
        query.getPage(),
        query.getSize()
    );
    
    return Result.success(pageResult);
}
```

## 八、文件上传

### 单文件上传

```java
@PostMapping("/upload")
public Result<String> upload(@RequestParam("file") MultipartFile file) {
    // 检查文件
    if (file.isEmpty()) {
        return Result.error("文件不能为空");
    }
    
    // 检查文件类型
    String contentType = file.getContentType();
    if (!contentType.startsWith("image/")) {
        return Result.error("只能上传图片");
    }
    
    // 生成文件名
    String fileName = System.currentTimeMillis() + "_" + file.getOriginalFilename();
    
    // 保存文件
    try {
        file.transferTo(new File("/path/to/upload/" + fileName));
    } catch (IOException e) {
        return Result.error("上传失败");
    }
    
    // 返回文件 URL
    String fileUrl = "http://localhost:8080/files/" + fileName;
    return Result.success(fileUrl);
}
```

### 配置文件上传大小

```yaml
spring:
  servlet:
    multipart:
      max-file-size: 10MB
      max-request-size: 10MB
```

### 多文件上传

```java
@PostMapping("/uploads")
public Result<List<String>> uploads(@RequestParam("files") List<MultipartFile> files) {
    List<String> urls = new ArrayList<>();
    
    for (MultipartFile file : files) {
        // 处理每个文件
        String url = uploadFile(file);
        urls.add(url);
    }
    
    return Result.success(urls);
}
```

## 九、跨域配置

### 什么是跨域？

```
前端：http://localhost:3000
后端：http://localhost:8080

浏览器：不同域名，禁止访问！
```

### 全局跨域配置

```java
package com.example.demo.config;

import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.config.annotation.CorsRegistry;
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;

@Configuration
public class CorsConfig implements WebMvcConfigurer {
    
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/**")  // 允许所有路径
            .allowedOrigins("http://localhost:3000")  // 允许的源
            .allowedMethods("GET", "POST", "PUT", "DELETE")  // 允许的方法
            .allowedHeaders("*")  // 允许的头部
            .allowCredentials(true)  // 允许 Cookie
            .maxAge(3600);  // 预检缓存时间
    }
}
```

### 局部跨域配置

```java
@RestController
@CrossOrigin(origins = "http://localhost:3000")
@RequestMapping("/api/users")
public class UserController {
    // 只对这个 Controller 生效
}
```

## 十一、常见问题

### Q1：@RestController 和@Controller 有什么区别？

**@Controller**：
```java
@Controller
public class UserController {
    @GetMapping("/users")
    public String list() {
        return "user-list";  // 返回页面
    }
}
```

**@RestController**：
```java
@RestController
public class UserController {
    @GetMapping("/users")
    public List<User> list() {
        return userList;  // 返回 JSON
    }
}
```

**区别**：
- @Controller 返回页面
- @RestController 返回 JSON（= @Controller + @ResponseBody）

### Q2：如何处理 POST 请求的表单数据？

