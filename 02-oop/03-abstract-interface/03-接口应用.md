# 9.2 接口应用：策略模式与回调（生活类比：万能遥控器）

## 一、先问一个问题

**问题**：如何让同一个按钮控制不同品牌的电视？

### 生活例子

万能遥控器的工作原理：
- 遥控器定义操作规范（开、关、换台）
- 不同品牌的电视实现这些规范
- 用户只需要操作遥控器，不用关心内部实现

这就是接口在代码中的应用！遥控器是"接口"，各种品牌电视是"实现类"。

## 二、策略模式

### 2.1 问题引入

```java
// 不好：硬编码支付方式，扩展困难
public void pay(String type, double amount) {
    if ("alipay".equals(type)) {
        System.out.println("支付宝支付：" + amount);
    } else if ("wechat".equals(type)) {
        System.out.println("微信支付：" + amount);
    } else if ("card".equals(type)) {
        System.out.println("银行卡支付：" + amount);
    }
}
```

每次新增支付方式都要修改原有代码，违反开闭原则。

### 2.2 策略模式实现

```java
// 策略接口
interface PaymentStrategy {
    void pay(double amount);
}

// 支付宝策略
class AlipayStrategy implements PaymentStrategy {
    @Override
    public void pay(double amount) {
        System.out.println("支付宝支付：" + amount);
    }
}

// 微信支付策略
class WechatPayStrategy implements PaymentStrategy {
    @Override
    public void pay(double amount) {
        System.out.println("微信支付：" + amount);
    }
}

// 订单使用策略
class Order {
    private PaymentStrategy strategy;

    public void setStrategy(PaymentStrategy strategy) {
        this.strategy = strategy;
    }

    public void checkout(double amount) {
        strategy.pay(amount);
    }
}

// 测试
public class Main {
    public static void main(String[] args) {
        Order order = new Order();

        order.setStrategy(new AlipayStrategy());
        order.checkout(100);

        order.setStrategy(new WechatPayStrategy());
        order.checkout(200);
    }
}
```

新增支付方式只需添加新策略类，无需修改原有代码。

## 三、回调接口

```java
// 点击回调接口
interface OnClickListener {
    void onClick();
}

// 按钮组件
class Button {
    private OnClickListener listener;

    public void setOnClickListener(OnClickListener listener) {
        this.listener = listener;
    }

    public void click() {
        if (listener != null) {
            listener.onClick();
        }
    }
}

// 使用回调
public class Main {
    public static void main(String[] args) {
        Button button = new Button();

        button.setOnClickListener(new OnClickListener() {
            @Override
            public void onClick() {
                System.out.println("按钮被点击！");
            }
        });

        button.click();
    }
}
```

回调接口的核心思想：**把"做什么"交给使用者决定**。

## 四、Comparable 排序接口

```java
import java.util.*;

class Student implements Comparable<Student> {
    String name;
    int score;

    public Student(String name, int score) {
        this.name = name;
        this.score = score;
    }

    @Override
    public int compareTo(Student other) {
        return Integer.compare(this.score, other.score);
    }

    @Override
    public String toString() {
        return name + "(" + score + ")";
    }
}

public class Main {
    public static void main(String[] args) {
        List<Student> students = Arrays.asList(
            new Student("张三", 85),
            new Student("李四", 92),
            new Student("王五", 78)
        );

        Collections.sort(students);
        System.out.println(students);
        // 输出：[王五(78), 张三(85), 李四(92)]
    }
}
```

实现 `Comparable` 接口后，对象可以直接使用 `Collections.sort()` 排序。

## 五、接口继承

```java
// 基础接口
interface A {
    void methodA();
}

// 接口继承接口
interface B extends A {
    void methodB();
}

// 实现类需要实现所有方法
class C implements B {
    @Override
    public void methodA() {
        System.out.println("方法 A");
    }

    @Override
    public void methodB() {
        System.out.println("方法 B");
    }
}

public class Main {
    public static void main(String[] args) {
        B obj = new C();
        obj.methodA();
        obj.methodB();
    }
}
```

接口可以继承多个接口，这是类无法做到的。

## 六、常见疑问

**Q1：什么时候用策略模式？**
A：当同一个操作有多种算法实现时，比如支付方式、排序算法、数据过滤等。策略之间可以随意切换。

**Q2：回调接口和策略模式有什么区别？**
A：回调通常用于事件处理（按钮点击、网络请求完成），策略用于算法替换。两者本质相同，只是使用场景不同。

**Q3：Comparable 和 Comparator 有什么区别？**
A：`Comparable` 是对象自身具备比较能力（实现接口）；`Comparator` 是外部比较器（单独类）。前者修改类本身，后者不修改。

## 七、小结

| 应用场景 | 使用接口 | 核心作用 |
|----------|----------|----------|
| 策略模式 | PaymentStrategy | 算法可替换 |
| 回调机制 | OnClickListener | 事件处理 |
| 对象排序 | Comparable | 自然排序 |
| 接口扩展 | extends | 能力组合 |
