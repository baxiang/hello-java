# 9.2 接口基础：规范的力量（生活类比：USB 接口）

## 一、先问一个问题
**问题**：如果每个设备都有自己的充电接口，你的手机能充电吗？

### 生活例子
USB 接口就是一种规范：
- 规定了形状、引脚、电压
- 任何设备只要符合规范就能使用
- 不需要知道内部实现

想象一下：
- 你的笔记本电脑有 USB 口
- 你的鼠标也有 USB 口
- 它们来自不同厂商，但能互相配合

Java 接口也是如此！

## 二、接口的概念

**接口（Interface）**：定义一组规范，实现接口的类必须遵守这些规范。

```java
// 接口就像合同
interface Flyable {
    void fly();      // 必须实现飞的方法
    void land();     // 必须实现降落的方法
}
```

接口不包含具体实现，只定义"应该做什么"。

### 2.1 USB 类比映射到代码

```java
// USB 接口规范
interface UsbDevice {
    void connect();    // 连接
    void transfer();   // 传输数据
    void disconnect(); // 断开连接
}

// 鼠标实现 USB 接口
class Mouse implements UsbDevice {
    @Override
    public void connect() {
        System.out.println("鼠标已连接");
    }
    
    @Override
    public void transfer() {
        System.out.println("传输鼠标数据");
    }
    
    @Override
    public void disconnect() {
        System.out.println("鼠标已断开");
    }
}
```

不同厂商的鼠标只要实现 `UsbDevice` 接口，就能在任何电脑上使用——这就是接口规范的力量。

## 三、接口的定义

### 3.1 基本语法

```java
[public] interface 接口名 {
    // 常量（隐式 public static final）
    int MAX_SPEED = 1000;
    
    // 抽象方法（隐式 public abstract）
    void fly();
    
    // 默认方法（Java 8+）
    default void rest() {
        System.out.println("休息一下");
    }
    
    // 静态方法（Java 8+）
    static void showInfo() {
        System.out.println("这是飞行接口");
    }
}
```

### 3.2 实现接口

```java
// 实现接口
class Bird implements Flyable {
    @Override
    public void fly() {
        System.out.println("鸟儿飞翔");
    }
    
    @Override
    public void land() {
        System.out.println("鸟儿降落");
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        Flyable bird = new Bird();
        bird.fly();    // 鸟儿飞翔
        bird.land();   // 鸟儿降落
    }
}
```

**关键点**：
- 使用 `implements` 关键字
- 必须实现接口中的所有抽象方法
- 实现方法必须是 `public` 的

### 3.3 调用默认方法和静态方法

```java
public class Main {
    public static void main(String[] args) {
        Flyable bird = new Bird();
        
        // 调用抽象方法（通过对象）
        bird.fly();    // 鸟儿飞翔
        bird.land();   // 鸟儿降落
        
        // 调用默认方法（通过对象）
        bird.rest();   // 休息一下
        
        // 调用静态方法（通过接口名）
        Flyable.showInfo();  // 这是飞行接口
    }
}
```

**区分两种方法**：
- `default` 方法：通过实现类的**对象**调用，可以被重写
- `static` 方法：通过**接口名**调用，不能被重写

## 四、接口的特点

| 特点 | 说明 | 示例 |
|------|------|------|
| 抽象方法 | 方法默认是抽象的 | `void fly();` |
| 多实现 | 一个类可实现多个接口 | `class A implements B, C` |
| 常量 | 只能定义常量 | `int MAX = 100;` |
| 无构造 | 不能有构造方法 | ❌ `interface A() {}` |

**补充说明**：
- 接口中的常量默认是 `public static final`
- 接口中的抽象方法默认是 `public abstract`
- 接口不能被实例化

## 五、多接口实现

```java
interface Swimmable { void swim(); }

// 一个类实现多个接口
class Duck implements Flyable, Swimmable {
    @Override
    public void fly() {
        System.out.println("鸭子飞");
    }
    
    @Override
    public void land() {
        System.out.println("鸭子降落");
    }
    
    @Override
    public void swim() {
        System.out.println("鸭子游");
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        Duck duck = new Duck();
        duck.fly();    // 鸭子飞
        duck.swim();   // 鸭子游
    }
}
```

## 六、常见疑问

**Q1：接口和抽象类有什么区别？**
A：接口定义"能力"，抽象类定义"是什么"。鸭子"是"动物（extends），同时"有"飞行能力（implements）。

**Q2：一个类可以实现多少个接口？**
A：理论上没有限制，但建议不超过 3-4 个，否则设计可能有问题。

**Q3：接口可以被继承吗？**
A：可以。接口可以继承其他接口，使用 `extends` 关键字。

## 七、小结

| 概念 | 关键字 | 特点 |
|------|--------|------|
| 接口定义 | interface | 定义规范 |
| 接口实现 | implements | 必须实现所有抽象方法 |
| 多实现 | 逗号分隔 | 支持多个接口 |
