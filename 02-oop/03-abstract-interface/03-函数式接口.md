# 函数式接口

## 一、问题引入

一个接口只有一个方法，实现它要写一个独立的 class 吗？

```java
interface Flyable {
    void fly();
}

// ❌ 单独写类太繁琐
class MyBird implements Flyable {
    public void fly() { System.out.println("飞起来了"); }
}
Flyable bird = new MyBird();
```

✅ `new 接口() { ... }` 的匿名内部类已经短一些了，但 Lambda 表达式更简洁：

```java
Flyable bird = () -> System.out.println("飞起来了");
```

只有一个抽象方法的接口叫**函数式接口**，可以用 Lambda 简洁实现。

## 二、核心概念

### 2.1 函数式接口

只包含**一个抽象方法**的接口。`@FunctionalInterface` 注解不是必须的，但建议加上——编译器会帮你检查：

```java
@FunctionalInterface
interface Flyable {
    void fly();
}
// ✅ 只能有一个抽象方法，default 方法可以有多个
```

### 2.2 Lambda 表达式

`(参数) -> { 方法体 }`：

```java
// 无参数
Runnable r = () -> System.out.println("运行");

// 一个参数（括号可省略）
Consumer<String> c = s -> System.out.println(s);

// 多参数
BiFunction<Integer, Integer, Integer> add = (a, b) -> a + b;

// 多行方法体
Function<Integer, Integer> f = n -> {
    int result = n * 2;
    return result;
};
```

### 2.3 方法引用

把已有方法"借"给接口用，是 Lambda 的简化写法：

```java
List<String> names = Arrays.asList("zhang", "li", "wang");

// Lambda
names.forEach(s -> System.out.println(s));

// 方法引用（更简洁）
names.forEach(System.out::println); // 等价于 s -> System.out.println(s)
```

| 形式 | 语法 | 示例 |
|------|------|------|
| 静态方法 | `类名::方法名` | `Integer::parseInt` |
| 实例方法（对象） | `对象::方法名` | `str::length` |
| 实例方法（类） | `类名::方法名` | `String::isEmpty` |
| 构造方法 | `类名::new` | `ArrayList::new` |

## 三、Java 内置函数式接口

位于 `java.util.function` 包：

| 接口 | 方法 | 输入 | 输出 | 用途 |
|------|------|------|------|------|
| `Supplier<T>` | `T get()` | 无 | T | 生产对象 |
| `Consumer<T>` | `void accept(T)` | T | 无 | 消费/操作 |
| `Function<T,R>` | `R apply(T)` | T | R | 转换 |
| `Predicate<T>` | `boolean test(T)` | T | boolean | 判断 |

```java
// Supplier
Supplier<List<String>> factory = ArrayList::new;

// Consumer
names.forEach(System.out::println);

// Function
Function<String, Integer> toLen = String::length;
toLen.apply("hello"); // 5

// Predicate
Predicate<String> isEmpty = String::isEmpty;
isEmpty.test(""); // true

// 组合
Function<String, String> upper = String::toUpperCase;
Function<String, String> addPrefix = s -> "Hi, " + s;
upper.andThen(addPrefix).apply("world"); // Hi, WORLD
```

## 四、代码对比

❌ **用匿名内部类**：

```java
new Thread(new Runnable() {
    @Override public void run() { System.out.println("任务"); }
}).start();
```

✅ **用 Lambda**：

```java
new Thread(() -> System.out.println("任务")).start();
```

❌ **自定义函数式接口用于简单场景**：

```java
@FunctionalInterface
interface MathOp { int op(int a, int b); }
MathOp add = new MathOp() {
    @Override public int op(int a, int b) { return a + b; }
};
```

✅ **用内置 BiFunction**：

```java
BiFunction<Integer, Integer, Integer> add = (a, b) -> a + b;
```

## 五、Q&A

**Q：Lambda 可以访问外部变量吗？**
A：可以，但外部变量必须是 final 或 effectively final（Java 8+）。

**Q：方法引用和 Lambda 有什么区别？**
A：方法引用是 Lambda 的语法糖，方法引用一定是 Lambda，Lambda 不一定是方法引用。

**Q：什么时候用自定义函数式接口？**
A：内置接口覆盖不了的方法签名时（比如需要 3 个参数的 `TriFunction`）再自定义。

**Q：构造函数引用怎么用？**
A：

```java
Supplier<Student> s1 = Student::new;        // 无参构造
Function<String, Student> s2 = Student::new; // 有参构造
Student stu = s2.apply("张三");
```

## 六、小结

| 概念 | 说明 |
|------|------|
| 函数式接口 | 只有一个抽象方法的接口 |
| @FunctionalInterface | 编译时检查，防止误加抽象方法 |
| Lambda 表达式 | `(params) -> expression/block` |
| 方法引用 | `类名::方法` 或 `对象::方法` |
| 内置接口 | Supplier/Consumer/Function/Predicate |
| 构造引用 | `类名::new` |
