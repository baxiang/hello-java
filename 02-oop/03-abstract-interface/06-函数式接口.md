# 9.4 函数式接口：Lambda 的基石（生活类比：自动售货机）

## 一、先问一个问题
**问题**：如果一个接口只有一个方法，能不能用更简洁的方式实现它？

### 生活例子
自动售货机：
- 只有一个功能：给钱→出货
- 投入硬币（输入参数）→ 得到商品（返回值）
- 这种"单一功能"的东西最适合自动化

函数式接口也是如此：
- 只有一个抽象方法
- 接收参数 → 返回结果
- 用 Lambda 表达式简洁实现

## 二、什么是函数式接口

**函数式接口**：只包含一个抽象方法的接口。

```java
// 函数式接口
@FunctionalInterface
interface Flyable {
    void fly();  // 唯一的抽象方法
}

// 使用 Lambda 实现（替代匿名内部类）
Flyable bird = () -> System.out.println("鸟儿飞翔");
bird.fly();
```

### 2.1 @FunctionalInterface 注解

```java
@FunctionalInterface
interface Calculator {
    int calculate(int a, int b);
    
    // ✅ 可以有默认方法
    default void show() {
        System.out.println("计算器");
    }
    
    // ✅ 可以有静态方法
    static void info() {
        System.out.println("函数式接口");
    }
}
```

**作用**：编译时检查接口是否符合函数式接口的要求，避免误加多个抽象方法。

## 三、Java 内置的函数式接口

### 3.1 Supplier（供应者）—— 只管生产

```java
import java.util.function.Supplier;

public class SupplierDemo {
    public static void main(String[] args) {
        // 生成随机数
        Supplier<Double> random = () -> Math.random();
        System.out.println(random.get());
        
        // 创建对象
        Supplier<String> hello = () -> new String("Hello");
        System.out.println(hello.get());
    }
}
```

### 3.2 Consumer（消费者）—— 只管消费

```java
import java.util.function.Consumer;

public class ConsumerDemo {
    public static void main(String[] args) {
        // 打印字符串
        Consumer<String> print = s -> System.out.println(s);
        print.accept("Hello World");
        
        // 链式调用
        Consumer<String> greet = s -> System.out.print("你好，");
        Consumer<String> welcome = s -> System.out.println(s);
        greet.andThen(welcome).accept("张三");
    }
}
```

### 3.3 Function（函数）—— 转换大师

```java
import java.util.function.Function;

public class FunctionDemo {
    public static void main(String[] args) {
        // 字符串转长度
        Function<String, Integer> length = s -> s.length();
        System.out.println(length.apply("Hello"));
        
        // 链式调用（String::toUpperCase 是方法引用语法，我们将在第五节详细介绍）
        Function<String, String> toUpper = String::toUpperCase;
        Function<String, String> addPrefix = s -> "Hi, " + s;
        
        String result = toUpper.andThen(addPrefix).apply("world");
        System.out.println(result);  // 输出: Hi, WORLD
    }
}
```

### 3.4 Predicate（断言）—— 条件判断

```java
import java.util.function.Predicate;

public class PredicateDemo {
    public static void main(String[] args) {
        // 判断是否成年
        Predicate<Integer> isAdult = age -> age >= 18;
        System.out.println(isAdult.test(20));  // true
        System.out.println(isAdult.test(15));  // false
        
        // 组合条件：and
        Predicate<Integer> isStudent = age -> age < 30;
        Predicate<Integer> isAdultStudent = isAdult.and(isStudent);
        System.out.println(isAdultStudent.test(25));  // true
        
        // 组合条件：or
        Predicate<Integer> isWeekend = day -> day == 6 || day == 7;
        Predicate<Integer> isHoliday = day -> day == 1;
        Predicate<Integer> isRest = isWeekend.or(isHoliday);
        System.out.println(isRest.test(6));  // true
        
        // 取反：negate
        Predicate<Integer> isNotAdult = isAdult.negate();
        System.out.println(isNotAdult.test(15));  // true
    }
}
```

## 四、内置接口总结

| 接口 | 方法 | 输入 | 返回 | 用途 |
|------|------|------|------|------|
| Supplier<T> | get() | 无 | T | 生成/提供对象 |
| Consumer<T> | accept(T) | T | void | 消费/处理数据 |
| Function<T,R> | apply(T) | T | R | 转换/映射 |
| Predicate<T> | test(T) | T | boolean | 判断/过滤 |

## 五、方法引用

方法引用是 Lambda 的进一步简化语法。

```java
import java.util.function.*;

public class MethodRefDemo {
    public static void main(String[] args) {
        // 静态方法引用：Math::abs 等价于 x -> Math.abs(x)
        Function<Double, Double> abs = Math::abs;
        System.out.println(abs.apply(-5.0));
        
        // 实例方法引用：str::length 等价于 () -> str.length()
        String str = "Hello";
        Supplier<Integer> length = str::length;
        System.out.println(length.get());
        
        // 构造方法引用：StringBuilder::new 等价于 () -> new StringBuilder()
        Supplier<StringBuilder> sb = StringBuilder::new;
        System.out.println(sb.get().append("World"));
    }
}
```

## 六、常见疑问

**Q1：函数式接口只能有一个方法吗？**
A：只能有一个**抽象方法**。默认方法和静态方法不影响函数式接口的定义。

**Q2：必须加 @FunctionalInterface 注解吗？**
A：不是必须的，但强烈建议加上——编译器会帮你检查接口是否合规。

**Q3：什么时候用内置接口，什么时候自定义？**
A：内置接口能覆盖 80% 的场景。只有当方法签名（参数类型/数量/返回值）不匹配时才自定义。

## 七、小结

| 概念 | 说明 | 示例 |
|------|------|------|
| 函数式接口 | 只有一个抽象方法的接口 | `@FunctionalInterface` |
| Lambda 表达式 | 简洁实现函数式接口的语法 | `() -> {}` |
| 内置接口 | 位于 java.util.function 包 | Supplier/Consumer/Function/Predicate |
| 方法引用 | 用 `::` 简化 Lambda 写法 | `Class::method` |
