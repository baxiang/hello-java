# 抽象类与接口

## 一、问题引入

写一个图形计算器，需要计算 Circle、Rectangle、Triangle 的面积。能不能定义一个"图形"类型，让每种图形都按自己的方式计算面积？

❌ **不用抽象类——父类给个空实现**：

```java
class Shape {
    double getArea() { return 0; } // ❌ 给个默认值，但没意义
}
class Circle extends Shape {
    double getArea() { return Math.PI * r * r; } // 重写了
}
```

子类可能会忘记重写 `getArea()`，而父类的 0 没有任何意义。

✅ **用抽象类——强制子类重写**：

```java
abstract class Shape {
    abstract double getArea(); // ❌ 没有方法体，子类必须重写
}
class Circle extends Shape {
    double r;
    double getArea() { return Math.PI * r * r; } // 不得不重写
}
```

## 二、核心概念

### 2.1 抽象类

用 `abstract` 修饰的类，**不能实例化**，必须被继承：

```java
abstract class Animal {
    abstract void makeSound(); // 抽象方法，没有方法体
    void sleep() { }          // 普通方法，可以有
}
```

**抽象方法**：`abstract` 修饰，没有方法体，子类必须重写（除非子类也是抽象类）。

### 2.2 抽象方法的特点

- 不能是 private（子类要重写，访问权限至少是 default）
- 不能是 static（静态方法不参与重写）
- 不能是 final（final 方法不能被重写）

```java
abstract class Demo {
    abstract void a();       // ✅
    protected abstract void b(); // ✅
    // private abstract void c(); // ❌
    // static abstract void d();  // ❌
    // final abstract void e();   // ❌
}
```

### 2.3 接口

接口定义**规范**（能力），用 `interface` 声明，`implements` 实现：

```java
interface Flyable {
    void fly();               // 抽象方法
    default void land() { }   // 默认方法（Java 8+）
    static void info() { }   // 静态方法（Java 8+）
}

class Bird implements Flyable {
    public void fly() { }     // 必须实现（且必须是 public）
}
```

**接口 vs 抽象类**：

| 特性 | 抽象类 | 接口 |
|------|--------|------|
| 关键字 | `abstract class` | `interface` |
| 实现 | `extends` | `implements` |
| 多继承 | ❌ | ✅（一个类可实现多个接口） |
| 构造方法 | ✅ | ❌ |
| 成员变量 | 任意 | 只能是常量（`public static final` 隐式） |
| 方法 | 抽象+普通 | 默认抽象，可有 default/static（Java 8+） |

### 2.4 设计原则

```
is-a 关系（"是一种"）  →  继承抽象类
has-a 能力（"能做什么"）→  实现接口
```

```java
abstract class Animal { }         // 动物是一种生物，用抽象类
interface Flyable { void fly(); }  // 能飞是一种能力，用接口
interface Swimmable { void swim(); }

class Duck extends Animal implements Flyable, Swimmable {
    // Duck 是一种动物，同时能飞能游
}
```

## 三、代码对比

❌ **用继承来模拟"多能力"**（Java 单继承，不够灵活）：

```java
class Dog extends Animal, Flyable { } // ❌ 错误：Java 不支持多继承
```

✅ **用接口实现多能力**：

```java
class Dog extends Animal implements Flyable, Swimmable { } // ✅
```

❌ **抽象类提供默认实现，但子类不需要**：

```java
abstract class Animal {
    void eat() { } // 给了实现，但 Dog 吃东西方式完全不同
}
```

✅ **用抽象方法强制子类提供自己的实现**：

```java
abstract class Animal {
    abstract void eat(); // 不给默认实现，强制 Dog 重写
}
```

❌ **接口添加新方法导致所有实现类都要改**：

```java
interface A { void old(); }
class B implements A { public void old() { } }
// 现在给 A 加一个方法 new()，B 必须改
```

✅ **Java 8+ 用 default 方法添加新方法而不破坏兼容性**：

```java
interface A {
    void old();
    default void newer() { } // ✅ 已有实现的地方不用改
}
```

## 四、实现方式

### 4.1 模板方法模式（抽象类典型应用）

父类定义算法骨架（用 `final` 防止重写），具体步骤由子类实现：

```java
abstract class Coffee {
    // 模板方法，用 final 防止重写
    public final void make() {
        boil();
        brew();
        pour();
        addCondiments();
    }

    abstract void brew();              // 子类实现
    abstract void addCondiments();

    void boil() { System.out.println("烧水"); }
    void pour() { System.out.println("倒杯子里"); }
}
```

### 4.2 回调与策略模式（接口典型应用）

```java
interface OnClickListener { void onClick(); }

class Button {
    private OnClickListener listener;
    public void setOnClickListener(OnClickListener l) { this.listener = l; }
    public void click() { if (listener != null) listener.onClick(); }
}

button.setOnClickListener(() -> System.out.println("点到了！"));
```

### 4.3 Comparable 接口

```java
class Student implements Comparable<Student> {
    String name;
    int score;

    @Override
    public int compareTo(Student other) {
        return Integer.compare(this.score, other.score);
    }
}

Collections.sort(students); // 实现 Comparable 后可直接排序
```

## 五、Q&A

**Q：抽象类可以有构造方法吗？**
A：可以。抽象类的构造方法供子类调用（通过 `super()`）。

**Q：接口可以有构造方法吗？**
A：不能。接口不是类，不能实例化。

**Q：接口的成员变量是什么类型？**
A：隐式 `public static final`，即使不写也自动加。所以接口的变量**必须是常量**。

**Q：什么时候用抽象类，什么时候用接口？**
A：有共性代码需要复用 → 抽象类；定义能力/规范 → 接口；需要多继承 → 接口。

**Q：Java 9+ 接口可以定义私有方法吗？**
A：可以。私有方法供 default 方法调用，隐藏实现细节。

## 六、小结

| 要点 | 说明 |
|------|------|
| 抽象方法 | `abstract` 修饰，无方法体，子类必须重写 |
| 抽象类 | 不能实例化，必须被继承，构造方法供子类调用 |
| 接口 | 定义规范/能力，`implements` 实现，支持多继承 |
| 多能力 | 一个类可实现多个接口 |
| is-a vs has-a | "是一种"用抽象类，"能做什么"用接口 |
| default 方法 | Java 8+，接口加新方法不破坏兼容性 |
| 模板方法 | 抽象类定义骨架，子类实现具体步骤 |
