# 内部类

## 一、问题引入

只在某个类内部使用的类，有没有必要单独写一个文件？

```java
class Outer {
    class Inner { } // ✅ 只在 Outer 内部用，不单独建文件
}
```

**内部类**：定义在另一个类内部的类。好处是更好的封装（可以访问外部类的私有成员）和逻辑分组。

## 二、核心概念

内部类分为四类：

```
内部类
├── 成员内部类    — 作为外部类的成员
├── 静态内部类    — 用 static 修饰
├── 局部内部类    — 定义在方法内
└── 匿名内部类    — 没有名字的内部类
```

### 2.1 成员内部类

```java
class Outer {
    private int num = 10;

    class Inner {
        void show() {
            System.out.println(num); // ✅ 访问外部类私有成员
            System.out.println(Outer.this.num); // 显式指定
        }
    }
}

// 创建对象：外部类对象.new 内部类()
Outer outer = new Outer();
Outer.Inner inner = outer.new Inner();
```

### 2.2 静态内部类

```java
class Outer {
    static class Inner {
        void show() {
            // 只能访问外部类的静态成员
            // System.out.println(num); // ❌ 不能访问外部类实例成员
        }
    }
}

// 创建对象：不需要外部类实例
Outer.Inner inner = new Outer.Inner();
```

### 2.3 局部内部类

定义在方法内部的类，作用域仅限该方法：

```java
class Outer {
    void method() {
        int localVar = 10; // effectively final

        class LocalInner {
            void show() {
                System.out.println(localVar); // ✅ 可以访问
            }
        }

        LocalInner li = new LocalInner();
        li.show();
    }
}
```

### 2.4 匿名内部类

没有名字的内部类，常用于**一次性实现接口或继承类**：

```java
interface Flyable {
    void fly();
}

// ❌ 不用单独写一个类
// class MyBird implements Flyable { public void fly() { } }

// ✅ 用匿名内部类
Flyable bird = new Flyable() {
    @Override
    public void fly() {
        System.out.println("飞起来了");
    }
};
```

## 三、代码对比

❌ **单独写一个类来实现简单接口**（增加文件数量）：

```java
// MyButtonListener.java
class MyButtonListener implements OnClickListener {
    @Override
    public void onClick() { }
}
button.setOnClickListener(new MyButtonListener());
```

✅ **用匿名内部类替代**（简洁，只用一次时推荐）：

```java
button.setOnClickListener(new OnClickListener() {
    @Override
    public void onClick() { }
});
```

❌ **在 Lambda 可用时不必要地使用匿名内部类**：

```java
button.setOnClickListener(new OnClickListener() {
    @Override
    public void onClick() { System.out.println("clicked"); }
});
```

✅ **如果接口是函数式接口（只有一个抽象方法），用 Lambda**：

```java
button.setOnClickListener(() -> System.out.println("clicked"));
```

## 四、实现方式

### 4.1 匿名内部类实现多接口

```java
interface A { void a(); }
interface B { void b(); }

// ❌ 匿名内部类只能继承一个类或实现一个接口
// ✅ 但可以用成员内部类实现多接口
class Outer {
    class AB implements A, B {
        public void a() { }
        public void b() { }
    }
}
```

### 4.2 匿名内部类访问外部变量

匿名内部类可以访问外部变量，但变量必须是 **final 或 effectively final**（Java 8+）：

```java
int num = 10; // effectively final
Runnable r = new Runnable() {
    public void run() {
        System.out.println(num); // ✅ 可以读取
        // num = 20; // ❌ 不能修改
    }
};
```

### 4.3 实际应用场景

**场景一：事件监听**（Android / GUI 编程）

```java
button.setOnClickListener(e -> System.out.println("点击了"));
```

**场景二：线程任务**

```java
new Thread(() -> System.out.println("任务执行")).start();
```

**场景三：自定义比较器**

```java
Collections.sort(list, (a, b) -> a.compareTo(b));
```

## 五、Q&A

**Q：什么时候用匿名内部类，什么时候用 Lambda？**
A：接口只有一个抽象方法 → 用 Lambda；接口有多个抽象方法或需要 fields → 用匿名内部类。

**Q：匿名内部类属于哪个类？**
A：属于包围它的类。如果在方法中，就是方法所在类的内部类。

**Q：静态内部类和普通内部类的区别？**
A：静态内部类不持有外部类引用，不能访问外部类实例成员；普通内部类持有外部类引用。

**Q：局部内部类能访问方法的参数吗？**
A：能，但参数必须是 final 或 effectively final（Java 8+）。

## 六、小结

| 类型 | 定义位置 | 访问外部类 | 创建方式 |
|------|----------|-----------|----------|
| 成员内部类 | 外部类方法外 | 实例成员（通过 `Outer.this`） | `outer.new Inner()` |
| 静态内部类 | 外部类方法外 | 只能访问静态成员 | `new Outer.Inner()` |
| 局部内部类 | 方法内 | final/effectively final 局部变量 | 方法内 `new` |
| 匿名内部类 | 方法内/表达式中 | final/effectively final 局部变量 | `new 接口/抽象类() { }` |

| 场景 | 推荐类型 |
|------|----------|
| 一次性使用接口 | 匿名内部类（Lambda） |
| 需要多个实例 | 命名内部类 |
| 不需要外部类引用 | 静态内部类 |
| 方法内私有逻辑 | 局部内部类 |
