# 9.2 接口

## 一、接口的概念

接口（Interface）是一种完全抽象的类型，定义了一组规范，实现接口的类必须遵守这些规范。

**比喻**：
```
接口就像合同/协议：
- USB 接口：规定了形状和引脚
- 充电接口：规定了电压和电流
- 实现方只要符合规范即可
```

## 二、接口的定义

### 2.1 基本语法

```java
[public] interface 接口名 {
    // 常量（public static final）
    数据类型 常量名 = 值;
    
    // 抽象方法（public abstract）
    返回值类型 方法名 (参数列表);
    
    // 默认方法（Java 8+）
    default 返回值类型 方法名 (参数列表) {
        // 方法体
    }
    
    // 静态方法（Java 8+）
    static 返回值类型 方法名 (参数列表) {
        // 方法体
    }
}
```

### 2.2 示例

```java
// 定义接口
interface Flyable {
    // 常量
    int MAX_SPEED = 1000;
    
    // 抽象方法
    void fly();
    void land();
}

// 实现接口
class Bird implements Flyable {
    @Override
    public void fly() {
        System.out.println("鸟在飞翔");
    }
    
    @Override
    public void land() {
        System.out.println("鸟降落了");
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        Flyable bird = new Bird();
        bird.fly();
        bird.land();
    }
}
```

## 三、接口的特点

### 3.1 接口的成员

```java
interface MyInterface {
    // 常量（隐式 public static final）
    int NUM = 10;
    public static final String NAME = "测试";
    
    // 抽象方法（隐式 public abstract）
    void method1();
    public abstract void method2();
    
    // 默认方法（Java 8+）
    default void method3() {
        System.out.println("默认方法");
    }
    
    // 静态方法（Java 8+）
    static void method4() {
        System.out.println("静态方法");
    }
    
    // 私有方法（Java 9+）
    private void helper() {
        System.out.println("私有方法");
    }
}
```

### 3.2 接口的继承

```java
// 接口可以继承其他接口
interface A {
    void methodA();
}

interface B extends A {
    void methodB();
}

// 接口可以多继承
interface C {
    void methodC();
}

interface D extends B, C {
    void methodD();
}

// 实现类必须实现所有方法
class E implements D {
    @Override
    public void methodA() { }
    
    @Override
    public void methodB() { }
    
    @Override
    public void methodC() { }
    
    @Override
    public void methodD() { }
}
```

### 3.3 一个类实现多个接口

```java
interface Flyable {
    void fly();
}

interface Swimmable {
    void swim();
}

interface Runnable {
    void run();
}

// 一个类可以实现多个接口
class Duck implements Flyable, Swimmable, Runnable {
    @Override
    public void fly() {
        System.out.println("鸭子飞");
    }
    
    @Override
    public void swim() {
        System.out.println("鸭子游泳");
    }
    
    @Override
    public void run() {
        System.out.println("鸭子跑");
    }
}
```

## 四、接口 vs 抽象类

| 特性 | 接口 | 抽象类 |
|------|------|--------|
| 关键字 | interface | abstract class |
| 继承 | implements | extends |
| 多继承 | 支持 | 不支持 |
| 构造方法 | 无 | 有 |
| 成员变量 | 只能有常量 | 可以有各种变量 |
| 方法 | 默认抽象方法 | 可以有普通方法 |
| 设计目的 | 定义规范 | 代码复用 |

### 4.1 使用场景

```java
// 接口：定义能力/规范
interface Flyable {
    void fly();
}

// 抽象类：提取共性
abstract class Animal {
    String name;
    public void eat() { }
    public abstract void makeSound();
}

// 结合使用
class Bird extends Animal implements Flyable {
    @Override
    public void makeSound() {
        System.out.println("叽叽喳喳");
    }
    
    @Override
    public void fly() {
        System.out.println("鸟儿飞翔");
    }
}
```

## 五、接口的应用

### 5.1 Comparable 接口

```java
// 实现 Comparable 接口进行排序
class Student implements Comparable<Student> {
    String name;
    int age;
    double score;
    
    public Student(String name, int age, double score) {
        this.name = name;
        this.age = age;
        this.score = score;
    }
    
    @Override
    public int compareTo(Student other) {
        // 按成绩排序
        return Double.compare(this.score, other.score);
    }
    
    @Override
    public String toString() {
        return name + "(" + score + ")";
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        List<Student> students = Arrays.asList(
            new Student("张三", 18, 85),
            new Student("李四", 19, 92),
            new Student("王五", 18, 78)
        );
        
        Collections.sort(students);
        System.out.println(students);
    }
}
```

### 5.2 Comparator 接口

```java
class Student {
    String name;
    int age;
    double score;
    
    public Student(String name, int age, double score) {
        this.name = name;
        this.age = age;
        this.score = score;
    }
    
    @Override
    public String toString() {
        return name + "(" + age + "," + score + ")";
    }
}

public class Main {
    public static void main(String[] args) {
        List<Student> students = Arrays.asList(
            new Student("张三", 18, 85),
            new Student("李四", 19, 92),
            new Student("王五", 18, 78)
        );
        
        // 按年龄排序
        Collections.sort(students, new Comparator<Student>() {
            @Override
            public int compare(Student s1, Student s2) {
                return Integer.compare(s1.age, s2.age);
            }
        });
        
        // Lambda 表达式（Java 8+）
        students.sort((s1, s2) -> Double.compare(s2.score, s1.age));
    }
}
```

### 5.3 策略模式

```java
// 策略接口
interface PaymentStrategy {
    void pay(double amount);
}

// 具体策略
class AlipayStrategy implements PaymentStrategy {
    @Override
    public void pay(double amount) {
        System.out.println("使用支付宝支付：" + amount);
    }
}

class WechatPayStrategy implements PaymentStrategy {
    @Override
    public void pay(double amount) {
        System.out.println("使用微信支付：" + amount);
    }
}

class BankCardStrategy implements PaymentStrategy {
    @Override
    public void pay(double amount) {
        System.out.println("使用银行卡支付：" + amount);
    }
}

// 上下文
class Order {
    private PaymentStrategy strategy;
    
    public void setStrategy(PaymentStrategy strategy) {
        this.strategy = strategy;
    }
    
    public void checkout(double amount) {
        strategy.pay(amount);
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        Order order = new Order();
        
        order.setStrategy(new AlipayStrategy());
        order.checkout(100);
        
        order.setStrategy(new WechatPayStrategy());
        order.checkout(200);
        
        order.setStrategy(new BankCardStrategy());
        order.checkout(300);
    }
}
```

### 5.4 回调接口

```java
// 回调接口
interface OnClickListener {
    void onClick();
}

// 按钮
class Button {
    private OnClickListener listener;
    
    public void setOnClickListener(OnClickListener listener) {
        this.listener = listener;
    }
    
    public void click() {
        if (listener != null) {
            listener.onClick();
        }
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        Button button = new Button();
        
        button.setOnClickListener(new OnClickListener() {
            @Override
            public void onClick() {
                System.out.println("按钮被点击了");
            }
        });
        
        button.click();
    }
}
```

## 六、默认方法

### 6.1 默认方法的作用

```java
interface Animal {
    // 抽象方法
    void makeSound();
    
    // 默认方法
    default void sleep() {
        System.out.println("动物睡觉");
    }
}

class Dog implements Animal {
    @Override
    public void makeSound() {
        System.out.println("汪汪叫");
    }
    
    // 可以重写默认方法
    @Override
    public void sleep() {
        System.out.println("狗趴着睡觉");
    }
}
```

### 6.2 默认方法冲突解决

```java
interface A {
    default void show() {
        System.out.println("A");
    }
}

interface B {
    default void show() {
        System.out.println("B");
    }
}

// 实现多个有相同默认方法的接口
class C implements A, B {
    // 必须重写解决冲突
    @Override
    public void show() {
        // 可以选择调用其中一个
        A.super.show();
        // 或者 B.super.show();
        // 或者完全重写
        System.out.println("C");
    }
}
```

## 七、综合示例

```java
// 能力接口
interface Flyable {
    void fly();
}

interface Swimmable {
    void swim();
}

interface Runnable {
    void run();
}

// 抽象类
abstract class Animal {
    protected String name;
    
    public Animal(String name) {
        this.name = name;
    }
    
    public void eat() {
        System.out.println(name + " 吃东西");
    }
    
    public abstract void makeSound();
}

// 具体类
class Duck extends Animal implements Flyable, Swimmable, Runnable {
    public Duck(String name) {
        super(name);
    }
    
    @Override
    public void makeSound() {
        System.out.println(name + " 嘎嘎叫");
    }
    
    @Override
    public void fly() {
        System.out.println(name + " 在天上飞");
    }
    
    @Override
    public void swim() {
        System.out.println(name + " 在水里游");
    }
    
    @Override
    public void run() {
        System.out.println(name + " 在地上跑");
    }
}

class Dog extends Animal implements Runnable {
    public Dog(String name) {
        super(name);
    }
    
    @Override
    public void makeSound() {
        System.out.println(name + " 汪汪叫");
    }
    
    @Override
    public void run() {
        System.out.println(name + " 在地上跑");
    }
}

// 使用
public class Zoo {
    public static void main(String[] args) {
        Animal[] animals = {
            new Duck("唐老鸭"),
            new Dog("旺财")
        };
        
        for (Animal animal : animals) {
            animal.eat();
            animal.makeSound();
            
            if (animal instanceof Flyable) {
                ((Flyable) animal).fly();
            }
            if (animal instanceof Swimmable) {
                ((Swimmable) animal).swim();
            }
            if (animal instanceof Runnable) {
                ((Runnable) animal).run();
            }
            System.out.println();
        }
    }
}
```

## 八、小结

本节要点：
1. **接口**：定义规范，完全抽象
2. **实现**：implements，可以实现多个
3. **成员**：常量、抽象方法、默认方法、静态方法
4. **vs 抽象类**：接口定义能力，抽象类提取共性
5. **应用**：排序、策略模式、回调等
