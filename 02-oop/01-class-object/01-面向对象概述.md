# 面向对象概述

## 一、问题引入

写一个"把大象放进冰箱"的程序，用面向过程和面向对象分别怎么做？

❌ **面向过程**——关注步骤，方法堆砌：

```java
public class PutElephant {
    public static void main(String[] args) {
        openDoor();   // 开门
        putElephant(); // 放大象
        closeDoor();  // 关门
    }
}
```

✅ **面向对象**——找对象，让对象做事：

```java
public class Main {
    public static void main(String[] args) {
        Refrigerator fridge = new Refrigerator();
        Elephant elephant = new Elephant();
        fridge.open();
        fridge.put(elephant);
        fridge.close();
    }
}
```

面向过程像"写菜谱"（一步步指令），面向对象像"找厨师"（找对象，对象自己知道怎么做）。

## 二、核心概念

### 2.1 类与对象

| 概念 | 说明 |
|------|------|
| **类（Class）** | 抽象模板，定义对象的属性和行为，只有一份 |
| **对象（Object）** | 类的具体实例，根据模板创建的实物，可以有多个 |

比喻：类 = 月饼模具，对象 = 用模具压出的月饼。

```java
class Student {           // 类：模板
    String name;
    int age;
    void study() { }
}

Student s = new Student(); // 对象：实例
```

### 2.2 面向对象三大特性

```
封装（Encapsulation）  →  隐藏细节，只暴露接口
    ↓
继承（Inheritance）   →  子类复用父类的代码
    ↓
多态（Polymorphism）  →  同一行为，不同对象有不同表现
```

**封装**：把属性藏起来，通过 getter/setter 有限度地开放。

```java
class Person {
    private int age; // ❌ 外部不能直接访问

    public int getAge() { return age; }    // ✅ 通过方法访问
    public void setAge(int age) {         // ✅ 可以在 setter 里加验证
        if (age >= 0) this.age = age;
    }
}
```

**继承**：子类复用父类的成员，不需要重复写相同代码。

```java
class Animal {
    void eat() { }
}
class Dog extends Animal { // Dog 自动拥有 eat()
    void bark() { }
}
```

**多态**：父类引用可以指向子类对象，调用时执行子类的实现。

```java
Animal a = new Dog(); // 父类引用指向子类对象
a.eat();              // 执行的是 Dog 的 eat()
```

## 三、代码对比

❌ **滥用对象，直接操作属性**：

```java
Student s = new Student();
s.name = "张三";        // 直接操作属性，无法控制
s.age = -100;           // 危险：无效数据长驱直入
```

✅ **封装后，通过方法访问**：

```java
Student s = new Student();
s.setName("张三");
s.setAge(18);           // setter 可以拒绝无效值

// ❌ age < 0 时 setter 拒绝，-100 不会被接受
s.setAge(-100);
```

❌ **用继承实现所有 is-a**：

```java
class Dog extends Animal { }       // ✅ 合理
class Car extends Vehicle { }      // ✅ 合理
class Employee extends Database { } // ❌ 不合理，Employee 不是 Database
```

✅ **is-a 用继承，has-a 用组合**：

```java
class Employee {
    private Database db;  // ✅ 组合：Employee has-a Database
}
```

## 四、实现方式

### 4.1 定义类

```java
public class Student {
    String name;     // 成员变量（属性）
    int age;

    public Student() { }                       // 无参构造
    public Student(String name, int age) {      // 有参构造
        this.name = name;
        this.age = age;
    }

    void study() { }                          // 成员方法（行为）
}
```

### 4.2 创建和使用对象

```java
Student s1 = new Student();             // 无参构造
Student s2 = new Student("张三", 18);     // 有参构造
s1.study();
```

### 4.3 成员变量 vs 局部变量

| 特性 | 成员变量 | 局部变量 |
|------|----------|----------|
| 定义位置 | 类中，方法外 | 方法内 |
| 默认值 | 有（0/false/null） | 无，必须初始化 |
| 生命周期 | 对象存在期间 | 方法执行期间 |

```java
class Demo {
    int a; // ✅ 有默认值 0

    void method() {
        int b;            // ❌ 没有默认值
        // System.out.println(b); // 编译错误
        b = 10;           // 必须先赋值
    }
}
```

## 五、Q&A

**Q：new 关键字做了什么？**
A：分配内存 → 初始化成员变量为默认值 → 调用构造方法 → 返回对象引用。

**Q：类定义了就能用吗？**
A：不能，必须用 `new` 创建对象。类只是模板，对象才是实体。

**Q：static 成员属于谁？**
A：属于类，不属于对象。所有对象共享一份。

**Q：private 字段子类能继承吗？**
A：能继承（子类确实拥有这个字段），但不能直接访问（被访问控制保护）。

## 六、小结

| 要点 | 说明 |
|------|------|
| 类 vs 对象 | 模板 vs 实例 |
| 三大特性 | 封装、继承、多态 |
| 封装 | private + getter/setter |
| 继承 | extends，子类复用父类成员 |
| 多态 | 父类引用指向子类对象 |
| 成员 vs 局部 | 默认值 / 作用域 / 生命周期不同 |
