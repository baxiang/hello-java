# 多态与类型转换

## 一、问题引入

一个动物园喂食程序，Dog、Cat、Bird 都有 `eat()` 方法，怎样用**一套代码**喂所有动物？

❌ **不用多态——写死每种动物**：

```java
void feed(Dog d) { d.eat(); }
void feed(Cat c) { c.eat(); }
void feed(Bird b) { b.eat(); }
```

每新增一种动物就要加一个方法，扩展痛苦。

✅ **用多态——父类引用，一套代码**：

```java
void feed(Animal a) { a.eat(); } // Animal 类型，可以传入任何子类

feed(new Dog());   // 输出：狗吃骨头
feed(new Cat());   // 输出：猫吃鱼
feed(new Bird());  // 输出：鸟吃虫
```

**多态**：同一个父类引用，调用相同方法，执行的是不同子类的实现。

## 二、核心概念

### 2.1 多态的三个前提

```
① 继承关系          →  子类 extends 父类
② 方法重写          →  子类重写父类方法
③ 父类引用指向子类对象 →  Animal a = new Dog()
```

> **类比**：多态就像 **USB 接口**——电脑不关心插的是什么设备（U 盘、鼠标、键盘），只要符合 USB 协议（实现接口/重写方法），就能正常工作。同一个 USB 插口，插不同设备有不同的效果。

### 2.2 向上转型（自动）

子类引用 → 父类引用，**自动转换**，安全：

```java
Animal a = new Dog(); // 向上转型，自动
a.eat();              // ✅ 多态调用：执行 Dog 的 eat()
// a.bark();           // ❌ 编译错误：父类没有 bark()
```

### 2.3 向下转型（强制）

父类引用 → 子类引用，需要**强制转换**，存在风险：

```java
Animal a = new Dog();
Dog d = (Dog) a;  // 向下转型，强制
d.bark();         // ✅ 可以调用子类特有方法
```

### 2.4 instanceof 安全检查

```java
Animal a = new Dog();

// ❌ 危险：a 实际上是 Dog，转成 Cat 会 ClassCastException
// Cat c = (Cat) a;

// ✅ 安全：先判断
if (a instanceof Dog) {
    Dog d = (Dog) a;
    d.bark();
} else if (a instanceof Cat) {
    Cat c = (Cat) a;
    c.meow();
}
```

Java 16+ 支持 **Pattern Matching for instanceof**（类型模式匹配），简化写法：

```java
if (a instanceof Dog d) { // 转型 + 声明一步完成
    d.bark();
} else if (a instanceof Cat c) {
    c.meow();
}
```

### 2.5 为什么成员变量没有多态

方法调用是**动态绑定**（运行时根据实际对象决定），而字段访问是**静态绑定**（编译时根据引用类型决定）。Java 这样设计是因为字段不具备"行为"——不存在"同一个字段有不同的值"的需求，字段就是数据本身。

## 三、代码对比

❌ **不用多态，方法参数写死子类**：

```java
void processDog(Dog d) { d.eat(); }
void processCat(Cat c) { c.eat(); }
// 每加一个子类就要加一个方法
```

✅ **用多态，方法参数用父类**：

```java
void process(Animal a) { a.eat(); }
// 任何 Animal 子类都能传进来
```

❌ **向下转型前不检查类型**：

```java
Animal a = new Dog();
Cat c = (Cat) a; // ❌ ClassCastException
```

✅ **instanceof 检查后再转型**：

```java
if (a instanceof Cat c) {
    c.meow();
}
```

❌ **误以为成员变量也有多态**：

```java
class Parent { int x = 1; }
class Child extends Parent { int x = 2; }

Parent p = new Child();
System.out.println(p.x); // 1，不是 2！成员变量没有多态
```

## 四、实现方式

### 4.1 多态的应用场景

**场景一：统一参数类型**

```java
void printArea(Shape s) { // Shape 是父类，Circle/Rectangle 是子类
    System.out.println(s.getArea());
}
```

**场景二：多态数组**

```java
Animal[] animals = { new Dog(), new Cat(), new Bird() };
for (Animal a : animals) {
    a.eat(); // 多态调用
}
```

**场景三：策略模式（完整示例）**

```java
interface Payment { void pay(double amount); }

class Alipay implements Payment {
    public void pay(double amount) { System.out.println("支付宝支付：" + amount); }
}
class WechatPay implements Payment {
    public void pay(double amount) { System.out.println("微信支付：" + amount); }
}
class BankCard implements Payment {
    public void pay(double amount) { System.out.println("银行卡支付：" + amount); }
}

void checkout(Payment p, double amount) {
    p.pay(amount); // 任何 Payment 实现都可以
}

checkout(new Alipay(), 100);    // 支付宝支付：100.0
checkout(new WechatPay(), 200); // 微信支付：200.0
```

### 4.2 多态的局限性

```java
Animal a = new Dog();
a.eat();      // ✅ 可以：eat 是父类中有的方法
// a.bark();  // ❌ 不行：bark 是 Dog 特有的，父类引用看不到
```

向下转型可以恢复访问子类特有成员，但必须先 instanceof 检查。

## 五、Q&A

**Q：多态调用的是父类方法还是子类方法？**
A：看**实际对象**是谁（new 的是哪个），不看引用类型。`Animal a = new Dog()` → 调用的是 Dog 的实现。

**Q：成员变量有没有多态？**
A：没有。成员变量是**静态绑定**的，看引用类型不看实际对象。

**Q：构造方法有没有多态？**
A：没有多态，但有**构造链**。子类的构造方法中如果调用了被子类重写的方法，会调用子类的重写版本，此时子类字段还没初始化——这是常见的 Bug 来源。

**Q：instanceof 和 getClass() 有什么区别？**
A：`instanceof` 允许子类匹配父类；`getClass()` 严格匹配类型。通常推荐 `getClass()` 避免对称性问题。

## 六、小结

| 要点 | 说明 |
|------|------|
| 多态前提 | 继承 + 重写 + 父类引用指向子类对象 |
| 向上转型 | 子类→父类，自动，安全，访问范围缩小 |
| 向下转型 | 父类→子类，强制，需要 instanceof 检查 |
| instanceof | 判断对象是否是某个类（或其子类）的实例 |
| 成员变量 | 无多态，静态绑定到引用类型 |
| 策略模式 | 多态的典型应用，接口引用 + 不同实现 |
| 局限 | 不能直接访问子类特有成员，需向下转型 |
