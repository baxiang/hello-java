# 10.4 单例设计模式

## 一、设计模式概述

设计模式（Design Pattern）是解决软件设计中常见问题的最佳实践方案。

**单例模式**：确保一个类只有一个实例，并提供全局访问点。

## 二、单例模式的实现

### 2.1 饿汉式（线程安全）

```java
public class Singleton {
    // 静态变量持有唯一实例（类加载时创建）
    private static final Singleton INSTANCE = new Singleton();
    
    // 私有构造，防止外部实例化
    private Singleton() { }
    
    // 公共静态方法获取实例
    public static Singleton getInstance() {
        return INSTANCE;
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        Singleton s1 = Singleton.getInstance();
        Singleton s2 = Singleton.getInstance();
        
        System.out.println(s1 == s2);  // true（同一实例）
    }
}
```

**优点**：
- 线程安全
- 实现简单

**缺点**：
- 类加载时就创建实例，可能浪费资源

### 2.2 懒汉式（线程不安全）

```java
public class Singleton {
    // 静态变量
    private static Singleton instance;
    
    // 私有构造
    private Singleton() { }
    
    // 公共静态方法
    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}
```

**问题**：多线程环境下可能创建多个实例。

### 2.3 懒汉式 + 同步（线程安全）

```java
public class Singleton {
    private static Singleton instance;
    
    private Singleton() { }
    
    // 同步方法
    public static synchronized Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}
```

**缺点**：每次调用都同步，性能低。

### 2.4 双重检查锁定（DCL，推荐）

```java
public class Singleton {
    // volatile 保证可见性和禁止指令重排
    private static volatile Singleton instance;
    
    private Singleton() { }
    
    public static Singleton getInstance() {
        // 第一次检查
        if (instance == null) {
            // 同步块
            synchronized (Singleton.class) {
                // 第二次检查
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}
```

**优点**：
- 线程安全
- 延迟加载
- 性能好

### 2.5 静态内部类（推荐）

```java
public class Singleton {
    // 私有构造
    private Singleton() { }
    
    // 静态内部类
    private static class Holder {
        private static final Singleton INSTANCE = new Singleton();
    }
    
    // 获取实例
    public static Singleton getInstance() {
        return Holder.INSTANCE;
    }
}
```

**优点**：
- 线程安全（类加载机制保证）
- 延迟加载
- 简洁

### 2.6 枚举（最简洁，推荐）

```java
public enum Singleton {
    INSTANCE;
    
    public void doSomething() {
        System.out.println("做事");
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        Singleton s1 = Singleton.INSTANCE;
        Singleton s2 = Singleton.INSTANCE;
        
        System.out.println(s1 == s2);  // true
        s1.doSomething();
    }
}
```

**优点**：
- 线程安全
- 防止反射和序列化攻击
- 最简洁

## 三、单例模式的应用

### 3.1 配置管理器

```java
public class ConfigManager {
    private Properties properties;
    
    private ConfigManager() {
        properties = new Properties();
        try {
            properties.load(new FileInputStream("config.properties"));
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
    
    private static class Holder {
        private static final ConfigManager INSTANCE = new ConfigManager();
    }
    
    public static ConfigManager getInstance() {
        return Holder.INSTANCE;
    }
    
    public String get(String key) {
        return properties.getProperty(key);
    }
}

// 使用
ConfigManager config = ConfigManager.getInstance();
String dbUrl = config.get("database.url");
```

### 3.2 日志记录器

```java
public class Logger {
    private PrintWriter writer;
    
    private Logger() {
        try {
            writer = new PrintWriter(new FileWriter("app.log", true));
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
    
    private static class Holder {
        private static final Logger INSTANCE = new Logger();
    }
    
    public static Logger getInstance() {
        return Holder.INSTANCE;
    }
    
    public void info(String msg) {
        writer.println("[INFO] " + msg);
        writer.flush();
    }
    
    public void error(String msg) {
        writer.println("[ERROR] " + msg);
        writer.flush();
    }
}

// 使用
Logger logger = Logger.getInstance();
logger.info("应用启动");
logger.error("发生错误");
```

### 3.3 线程池

```java
public class ThreadPool {
    private List<Worker> workers;
    private BlockingQueue<Runnable> taskQueue;
    
    private ThreadPool() {
        workers = new ArrayList<>();
        taskQueue = new LinkedBlockingQueue<>();
        
        // 创建固定数量的工作线程
        for (int i = 0; i < 10; i++) {
            Worker worker = new Worker(taskQueue);
            workers.add(worker);
            new Thread(worker).start();
        }
    }
    
    private static class Holder {
        private static final ThreadPool INSTANCE = new ThreadPool();
    }
    
    public static ThreadPool getInstance() {
        return Holder.INSTANCE;
    }
    
    public void execute(Runnable task) {
        taskQueue.offer(task);
    }
    
    private static class Worker implements Runnable {
        private BlockingQueue<Runnable> queue;
        
        public Worker(BlockingQueue<Runnable> queue) {
            this.queue = queue;
        }
        
        @Override
        public void run() {
            while (true) {
                try {
                    Runnable task = queue.take();
                    task.run();
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }
    }
}
```

## 四、单例模式的破坏与防护

### 4.1 反射攻击

```java
// 普通单例可能被反射破坏
Singleton s1 = Singleton.getInstance();

Constructor<Singleton> constructor = Singleton.class.getDeclaredConstructor();
constructor.setAccessible(true);
Singleton s2 = constructor.newInstance();

System.out.println(s1 == s2);  // false（破坏了单例）
```

**防护**：在构造方法中检查

```java
private Singleton() {
    if (instance != null) {
        throw new RuntimeException("不能创建多个实例");
    }
}
```

### 4.2 序列化攻击

```java
// 序列化后反序列化会创建新实例
Singleton s1 = Singleton.getInstance();

ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream("obj.ser"));
oos.writeObject(s1);

ObjectInputStream ois = new ObjectInputStream(new FileInputStream("obj.ser"));
Singleton s2 = (Singleton) ois.readObject();

System.out.println(s1 == s2);  // false（破坏了单例）
```

**防护**：实现 readResolve 方法

```java
private Object readResolve() {
    return getInstance();
}
```

### 4.3 枚举单例的优势

```java
// 枚举单例天然防止反射和序列化攻击
public enum Singleton {
    INSTANCE;
}

// 无法通过反射创建
// 序列化时自动处理
```

## 五、综合示例

```java
/**
 * 运行时配置（单例）
 */
public class RuntimeConfig {
    private Map<String, Object> config = new ConcurrentHashMap<>();
    
    private RuntimeConfig() {
        // 初始化默认配置
        config.put("version", "1.0.0");
        config.put("debug", false);
        config.put("maxConnections", 100);
    }
    
    private static class Holder {
        private static final RuntimeConfig INSTANCE = new RuntimeConfig();
    }
    
    public static RuntimeConfig getInstance() {
        return Holder.INSTANCE;
    }
    
    public void set(String key, Object value) {
        config.put(key, value);
    }
    
    public Object get(String key) {
        return config.get(key);
    }
    
    public <T> T get(String key, Class<T> clazz) {
        return clazz.cast(config.get(key));
    }
    
    public void printAll() {
        config.forEach((k, v) -> System.out.println(k + " = " + v));
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        RuntimeConfig config = RuntimeConfig.getInstance();
        
        config.set("appName", "MyApp");
        config.set("port", 8080);
        
        System.out.println(config.get("version"));
        System.out.println(config.get("appName"));
        System.out.println(config.get("port", Integer.class));
        
        config.printAll();
    }
}
```

## 六、小结

本节要点：
1. **饿汉式**：类加载时创建，线程安全
2. **懒汉式**：延迟加载，需处理线程安全
3. **DCL**：双重检查锁定，推荐
4. **静态内部类**：推荐，简洁且安全
5. **枚举**：最简洁，防止攻击
6. **应用**：配置管理、日志、线程池等
