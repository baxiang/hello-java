# 54.5 实战示例：消息驱动架构（生活类比：物流中心）

## 一、先问一个问题
**问题**：如何用消息队列构建解耦的微服务系统？

### 生活例子
物流中心：
- 仓库发货（订单服务发送消息）
- 分拣中心（消息路由到不同队列）
- 配送员取件（库存、邮件服务消费）
- 签收反馈（异步处理结果）

消息队列就是微服务的"物流中心"！

## 二、订单创建通知

```java
@Service
public class OrderService {
    
    @Autowired
    private OrderMapper orderMapper;
    
    @Autowired
    private RabbitTemplate rabbitTemplate;
    
    @Transactional
    public Order createOrder(Order order) {
        // 创建订单
        orderMapper.insert(order);
        
        // 发送消息通知
        OrderMessage message = new OrderMessage();
        message.setOrderId(order.getId());
        message.setUserId(order.getUserId());
        message.setAmount(order.getAmount());
        
        rabbitTemplate.convertAndSend(
            "order.exchange",
            "order.created",
            message
        );
        
        return order;
    }
}
```

## 三、库存扣减

```java
@Service
public class InventoryService {
    
    @Autowired
    private ProductMapper productMapper;
    
    @RabbitListener(queues = "inventory.queue")
    public void decreaseInventory(InventoryMessage message) {
        Long productId = message.getProductId();
        Integer quantity = message.getQuantity();
        
        Product product = productMapper.findById(productId);
        if (product.getStock() >= quantity) {
            productMapper.decreaseStock(productId, quantity);
            System.out.println("库存扣减成功：" + productId);
        } else {
            System.out.println("库存不足：" + productId);
        }
    }
}
```

## 四、异步发送邮件

```java
@Service
public class EmailService {
    
    @Autowired
    private RabbitTemplate rabbitTemplate;
    
    public void sendEmailAsync(String to, String subject, String content) {
        EmailMessage message = new EmailMessage();
        message.setTo(to);
        message.setSubject(subject);
        message.setContent(content);
        
        rabbitTemplate.convertAndSend("email.exchange", "email.routing", message);
    }
}

@Component
public class EmailConsumer {
    
    @Autowired
    private JavaMailSender mailSender;
    
    @RabbitListener(queues = "email.queue")
    public void sendEmail(EmailMessage message) {
        SimpleMailMessage mail = new SimpleMailMessage();
        mail.setTo(message.getTo());
        mail.setSubject(message.getSubject());
        mail.setText(message.getContent());
        
        mailSender.send(mail);
        System.out.println("邮件发送成功：" + message.getTo());
    }
}
```

## 五、日志收集（Kafka）

```java
@Service
public class LogService {
    
    @Autowired
    private KafkaTemplate<String, String> kafkaTemplate;
    
    public void log(String level, String message) {
        LogEntry log = new LogEntry();
        log.setLevel(level);
        log.setMessage(message);
        log.setTimestamp(System.currentTimeMillis());
        
        String json = JSON.toJSONString(log);
        kafkaTemplate.send("app-logs", json);
    }
}

@Component
public class LogConsumer {
    
    @KafkaListener(topics = "app-logs", groupId = "log-group")
    public void consume(String message) {
        LogEntry log = JSON.parseObject(message, LogEntry.class);
        // 存储到数据库或 Elasticsearch
        System.out.println("收到日志：" + log);
    }
}
```

## 六、常见疑问

**Q1：什么时候用 RabbitMQ，什么时候用 Kafka？**
A：RabbitMQ 适合复杂路由、事务消息；Kafka 适合高吞吐、日志收集、流处理。

**Q2：消息队列能保证顺序吗？**
A：RabbitMQ 单队列保证顺序；Kafka 单分区保证顺序。多分区/多队列不保证全局顺序。

## 七、小结

| 场景 | 方案 | 消息队列 |
|------|------|----------|
| 订单通知 | 发布订阅 | RabbitMQ |
| 库存扣减 | 异步处理 | RabbitMQ |
| 邮件发送 | 解耦异步 | RabbitMQ |
| 日志收集 | 高吞吐 | Kafka |
