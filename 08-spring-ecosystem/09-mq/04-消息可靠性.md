# 54.4 消息可靠性：不丢不重（生活类比：挂号信）

## 一、先问一个问题
**问题**：如何确保消息不丢失、不重复消费？

### 生活例子
挂号信：
- 寄件时回执（生产者确认）
- 投递时签收（消费者确认）
- 投递失败退回（死信队列）
- 重复投递检查（幂等性）

消息可靠性就是系统的"挂号信服务"！

## 二、消息重试

```yaml
spring:
  rabbitmq:
    listener:
      simple:
        retry:
          enabled: true
          max-attempts: 3
          initial-interval: 1000ms
          multiplier: 2.0
```

```java
@Component
public class RetryConsumer {
    
    @RabbitListener(queues = "retry.queue")
    public void handleMessage(String message) {
        // 处理失败会按配置重试
        if (shouldFail()) {
            throw new RuntimeException("处理失败");
        }
        System.out.println("处理成功：" + message);
    }
}
```

## 三、死信队列

```java
@Configuration
public class DLQConfig {
    
    // 死信队列
    @Bean
    public Queue dlq() {
        return new Queue("dlq", true);
    }
    
    // 普通队列，配置死信交换机
    @Bean
    public Queue normalQueue() {
        return QueueBuilder.durable("normal.queue")
            .withArgument("x-dead-letter-exchange", "dlx.exchange")
            .withArgument("x-dead-letter-routing-key", "dlq.routing")
            .build();
    }
}
```

## 四、幂等性

```java
@Service
public class IdempotentConsumer {
    
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    
    @RabbitListener(queues = "idempotent.queue")
    public void handleMessage(Message message) {
        String messageId = message.getMessageProperties().getMessageId();
        String key = "msg:processed:" + messageId;
        
        // 检查是否已处理
        Boolean processed = redisTemplate.opsForValue()
            .setIfAbsent(key, "1", 24, TimeUnit.HOURS);
        
        if (Boolean.TRUE.equals(processed)) {
            // 首次处理
            System.out.println("处理消息：" + messageId);
        } else {
            // 重复消息，忽略
            System.out.println("重复消息，忽略：" + messageId);
        }
    }
}
```

## 五、消息确认

### 5.1 生产者确认

```yaml
spring:
  rabbitmq:
    publisher-confirm-type: correlated
    publisher-returns: true
```

```java
@Configuration
public class ProducerConfig {
    
    @Bean
    public RabbitTemplate rabbitTemplate(ConnectionFactory factory) {
        RabbitTemplate template = new RabbitTemplate(factory);
        
        // 确认回调
        template.setConfirmCallback((correlation, ack, cause) -> {
            if (ack) {
                System.out.println("消息发送成功");
            } else {
                System.out.println("消息发送失败：" + cause);
            }
        });
        
        return template;
    }
}
```

## 六、常见疑问

**Q1：什么是幂等性？为什么重要？**
A：幂等性指同一消息多次消费结果相同。防止网络重试导致重复处理。

**Q2：死信队列有什么用？**
A：存储处理失败的消息，便于后续分析和手动处理，防止消息丢失。

## 七、小结

| 机制 | 作用 | 配置方式 |
|------|------|----------|
| 重试 | 自动重试失败消息 | retry.enabled |
| 死信队列 | 存储失败消息 | x-dead-letter-exchange |
| 幂等性 | 防止重复消费 | messageId + Redis |
| 生产者确认 | 确保消息到达 | publisher-confirm-type |
