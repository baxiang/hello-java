# 53.3 缓存注解：声明式缓存（生活类比：图书馆借阅）

## 一、先问一个问题

**问题**：每次查询都要写缓存逻辑，能不能更简洁？

### 生活例子：图书馆借阅

图书馆借阅流程：
- 第一次借书：去书库找（查数据库），同时复印一份放前台（写缓存）
- 第二次借书：前台直接拿（读缓存）
- 书更新后：前台旧版本作废，重新复印（更新缓存）

Spring Cache 注解就是"自动补书员"，让缓存操作变成声明式！

## 二、开启缓存

### 2.1 配置文件

```yaml
spring:
  cache:
    type: redis
    redis:
      time-to-live: 3600000  # 过期时间 1 小时
```

### 2.2 启用注解

```java
@SpringBootApplication
@EnableCaching
public class DemoApplication {
    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}
```

## 三、@Cacheable 查询缓存

**作用**：先查缓存，没有才执行方法。

```java
@Service
public class UserService {
    
    @Autowired
    private UserMapper userMapper;
    
    // 查询缓存
    @Cacheable(value = "user", key = "#id")
    public User findById(Long id) {
        System.out.println("查询数据库...");
        return userMapper.findById(id);
    }
    
    // 条件缓存
    @Cacheable(value = "user", key = "#id", condition = "#id > 0")
    public User findByIdConditional(Long id) {
        return userMapper.findById(id);
    }
}
```

## 四、@CachePut 更新缓存

**作用**：执行方法，同时更新缓存。

```java
@Service
public class UserService {
    
    @Autowired
    private UserMapper userMapper;
    
    // 更新缓存
    @CachePut(value = "user", key = "#user.id")
    public User update(User user) {
        userMapper.update(user);
        return user;
    }
}
```

## 五、@CacheEvict 删除缓存

**作用**：执行方法，同时删除缓存。

```java
@Service
public class UserService {
    
    @Autowired
    private UserMapper userMapper;
    
    // 删除单个缓存
    @CacheEvict(value = "user", key = "#id")
    public void delete(Long id) {
        userMapper.delete(id);
    }
    
    // 删除所有缓存
    @CacheEvict(value = "user", allEntries = true)
    public void deleteAll() {
        // 清空 user 缓存
    }
}
```

## 六、@Caching 组合注解

**作用**：组合多个缓存操作。

```java
@Service
public class UserService {
    
    @Caching(evict = {
        @CacheEvict(value = "user", key = "#id"),
        @CacheEvict(value = "user_list", allEntries = true)
    })
    public void delete(Long id) {
        userMapper.delete(id);
        // 删除单个用户缓存 + 用户列表缓存
    }
}
```

## 七、常见疑问

**Q1：@Cacheable 和 @CachePut 有什么区别？**

A：@Cacheable 先查缓存，有就直接返回；@CachePut 总是执行方法，然后更新缓存。

**Q2：key 怎么写？**

A：使用 SpEL 表达式。`#id` 表示方法参数 id，`#user.id` 表示 user 对象的 id 属性。

**Q3：value 和 key 的关系是什么？**

A：value 是缓存名称（类似 Redis 的 key 前缀），key 是缓存项的具体标识。完整 Redis key = value::key。

## 八、小结

| 注解 | 作用 | 执行方法？ | 缓存操作 |
|------|------|-----------|----------|
| @Cacheable | 查询缓存 | 缓存无时执行 | 读/写 |
| @CachePut | 更新缓存 | 总是执行 | 写 |
| @CacheEvict | 删除缓存 | 总是执行 | 删除 |
| @Caching | 组合操作 | 总是执行 | 多操作 |
