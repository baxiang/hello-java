# 26.4 线程池：ThreadPoolExecutor 详解

## 一、先问一个问题

**什么是线程池？为什么需要线程池？**

### 生活例子：出租车公司

```
没有线程池
├── 每次用车都买新车
├── 用完就扔
└── 浪费资源

有线程池
├── 公司有固定车辆
├── 用完归还
└── 重复利用
```

**线程池优势**：
- 复用线程，减少开销
- 控制并发数量
- 便于管理

## 二、ThreadPoolExecutor 详解

### 构造函数参数

```java
public ThreadPoolExecutor(
    int corePoolSize,              // 核心线程数
    int maximumPoolSize,           // 最大线程数
    long keepAliveTime,            // 空闲线程存活时间
    TimeUnit unit,                 // 时间单位
    BlockingQueue<Runnable> workQueue,  // 工作队列
    ThreadFactory threadFactory,   // 线程工厂
    RejectedExecutionHandler handler    // 拒绝策略
)
```

### 参数详解

```java
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    5,                    // 核心线程数：5 个
    10,                   // 最大线程数：10 个
    60,                   // 空闲线程存活：60 秒
    TimeUnit.SECONDS,     // 时间单位：秒
    new ArrayBlockingQueue<>(100),  // 工作队列：容量 100
    Executors.defaultThreadFactory(),  // 线程工厂：默认
    new ThreadPoolExecutor.CallerRunsPolicy()  // 拒绝策略：调用者运行
);
```

### 工作流程

```
提交任务
  ↓
核心线程数未满？
├── 是 → 创建核心线程执行
└── 否 → 加入工作队列
      ↓
      队列满？
      ├── 否 → 等待
      └── 是 → 最大线程数未满？
            ├── 是 → 创建非核心线程执行
            └── 否 → 执行拒绝策略
```

## 三、工作队列类型

### ArrayBlockingQueue（有界队列）

```java
// 固定容量的队列
BlockingQueue<Runnable> queue = new ArrayBlockingQueue<>(100);

// 特点：
// - 有界，防止内存溢出
// - 可能触发拒绝策略
// - 推荐使用
```

### LinkedBlockingQueue（无界队列）

```java
// 默认容量 Integer.MAX_VALUE
BlockingQueue<Runnable> queue = new LinkedBlockingQueue<>();

// 特点：
// - 无界，可能内存溢出
// - 不会触发拒绝策略
// - 谨慎使用
```

### SynchronousQueue（同步队列）

```java
// 不存储元素的队列
BlockingQueue<Runnable> queue = new SynchronousQueue<>();

// 特点：
// - 直接传递，不存储
// - 需要配合较大 maximumPoolSize
// - 适合短时任务
```

### PriorityBlockingQueue（优先队列）

```java
// 带优先级的队列
BlockingQueue<Runnable> queue = new PriorityBlockingQueue<>(
    100,
    (r1, r2) -> {
        // 比较优先级
        return Integer.compare(getPriority(r2), getPriority(r1));
    }
);

// 特点：
// - 按优先级执行
// - 无界
```

## 四、拒绝策略

### 四种内置策略

```java
// 1. CallerRunsPolicy：调用者运行
new ThreadPoolExecutor.CallerRunsPolicy();
// 特点：由提交任务的线程执行，不会丢失任务

// 2. AbortPolicy：抛异常（默认）
new ThreadPoolExecutor.AbortPolicy();
// 特点：抛 RejectedExecutionException

// 3. DiscardPolicy：直接丢弃
new ThreadPoolExecutor.DiscardPolicy();
// 特点：不抛异常，静默丢弃

// 4. DiscardOldestPolicy：丢弃最老任务
new ThreadPoolExecutor.DiscardOldestPolicy();
// 特点：丢弃队列中最老的任务，然后重试
```

### 自定义拒绝策略

```java
RejectedExecutionHandler customHandler = (r, executor) -> {
    // 记录日志
    System.out.println("任务被拒绝：" + r.toString());
    
    // 保存到数据库
    saveTaskToDatabase(r);
    
    // 或发送到消息队列
    sendToMQ(r);
};

ThreadPoolExecutor executor = new ThreadPoolExecutor(
    5, 10, 60, TimeUnit.SECONDS,
    new ArrayBlockingQueue<>(100),
    Executors.defaultThreadFactory(),
    customHandler
);
```

## 五、线程工厂

### 自定义线程工厂

```java
ThreadFactory customFactory = new ThreadFactory() {
    private final AtomicInteger count = new AtomicInteger(1);
    
    @Override
    public Thread newThread(Runnable r) {
        Thread thread = new Thread(r);
        thread.setName("Worker-" + count.getAndIncrement());
        thread.setDaemon(false);
        thread.setPriority(Thread.NORM_PRIORITY);
        return thread;
    }
};

ThreadPoolExecutor executor = new ThreadPoolExecutor(
    5, 10, 60, TimeUnit.SECONDS,
    new ArrayBlockingQueue<>(100),
    customFactory,
    new ThreadPoolExecutor.CallerRunsPolicy()
);
```

## 六、Executors 工具类

### 创建线程池

```java
// 固定大小线程池
ExecutorService fixed = Executors.newFixedThreadPool(10);

// 单线程池
ExecutorService single = Executors.newSingleThreadExecutor();

// 可缓存线程池
ExecutorService cached = Executors.newCachedThreadPool();

// 定时任务线程池
ScheduledExecutorService scheduled = Executors.newScheduledThreadPool(5);

// 定时任务单线程
ScheduledExecutorService singleScheduled = Executors.newSingleThreadScheduledExecutor();
```

### 不推荐使用 Executors

```java
// ❌ 不推荐：FixedThreadPool 和 SingleThreadPool
// 问题：队列无界，可能 OOM
ExecutorService executor = Executors.newFixedThreadPool(10);
// 内部使用 LinkedBlockingQueue（无界）

// ❌ 不推荐：CachedThreadPool
// 问题：线程数无界，可能 OOM
ExecutorService executor = Executors.newCachedThreadPool();
// 最大线程数 Integer.MAX_VALUE

// ✅ 推荐：使用 ThreadPoolExecutor
ExecutorService executor = new ThreadPoolExecutor(
    5, 10, 60, TimeUnit.SECONDS,
    new ArrayBlockingQueue<>(100)
);
```

## 七、线程池使用

### 提交任务

```java
ExecutorService executor = Executors.newFixedThreadPool(5);

// 提交 Runnable
executor.submit(() -> {
    System.out.println("执行任务");
});

// 提交 Callable
Future<Integer> future = executor.submit(() -> {
    int sum = 0;
    for (int i = 1; i <= 100; i++) {
        sum += i;
    }
    return sum;
});

// 获取结果
try {
    Integer result = future.get();  // 阻塞
    System.out.println("结果：" + result);
} catch (Exception e) {
    e.printStackTrace();
}
```

### 批量提交

```java
// 提交多个任务
List<Callable<Integer>> tasks = new ArrayList<>();
for (int i = 0; i < 10; i++) {
    int finalI = i;
    tasks.add(() -> finalI * finalI);
}

// 获取所有结果
List<Future<Integer>> futures = executor.invokeAll(tasks);
for (Future<Integer> future : futures) {
    System.out.println("结果：" + future.get());
}

// 获取第一个完成的结果
Integer result = executor.invokeAny(tasks);
System.out.println("第一个完成：" + result);
```

### 关闭线程池

```java
// 平滑关闭（等待任务完成）
executor.shutdown();

// 等待所有任务完成
if (!executor.awaitTermination(60, TimeUnit.SECONDS)) {
    // 超时后强制关闭
    List<Runnable> dropped = executor.shutdownNow();
    System.out.println("未完成任务数：" + dropped.size());
}

// 立即关闭
executor.shutdownNow();
```

## 八、线程池监控

### 监控方法

```java
ThreadPoolExecutor executor = (ThreadPoolExecutor) Executors.newFixedThreadPool(10);

// 获取线程池信息
System.out.println("核心线程数：" + executor.getCorePoolSize());
System.out.println("当前线程数：" + executor.getPoolSize());
System.out.println("最大线程数：" + executor.getMaximumPoolSize());
System.out.println("活跃线程数：" + executor.getActiveCount());
System.out.println("完成任务数：" + executor.getCompletedTaskCount());
System.out.println("队列大小：" + executor.getQueue().size());
System.out.println("队列容量：" + ((ArrayBlockingQueue<?>)executor.getQueue()).size());
```

### 自定义监控

```java
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    5, 10, 60, TimeUnit.SECONDS,
    new ArrayBlockingQueue<>(100)
) {
    @Override
    protected void beforeExecute(Thread t, Runnable r) {
        System.out.println("任务开始：" + t.getName());
    }
    
    @Override
    protected void afterExecute(Runnable r, Throwable t) {
        System.out.println("任务完成");
        if (t != null) {
            System.out.println("任务异常：" + t.getMessage());
        }
    }
    
    @Override
    protected void terminated() {
        System.out.println("线程池终止");
    }
};
```

## 九、线程池调优

### 参数配置建议

```
CPU 密集型任务
├── corePoolSize = CPU 核心数 + 1
├── maximumPoolSize = CPU 核心数 + 1
└── 队列容量 = 较小

IO 密集型任务
├── corePoolSize = CPU 核心数 * 2
├── maximumPoolSize = CPU 核心数 * 2
└── 队列容量 = 较大

混合型任务
├── 根据实际测试调整
└── 监控线程池状态
```

### 调优步骤

```
1. 监控
   └── 收集线程池指标

2. 分析
   └── 分析瓶颈（CPU/IO/队列）

3. 调整
   └── 调整核心参数

4. 验证
   └── 验证效果
```

## 十、小结

| 组件 | 作用 | 推荐 |
|------|------|------|
| corePoolSize | 核心线程数 | 根据任务类型 |
| maximumPoolSize | 最大线程数 | 根据任务类型 |
| workQueue | 工作队列 | ArrayBlockingQueue |
| handler | 拒绝策略 | CallerRunsPolicy |
| threadFactory | 线程工厂 | 自定义命名 |

**核心要点**：
- 使用 ThreadPoolExecutor
- 有界队列
- 自定义拒绝策略
- 监控和调优

## 十一、常见问题

### Q1：线程池大小如何设置？

```java
// CPU 密集型
int cores = Runtime.getRuntime().availableProcessors();
int poolSize = cores + 1;

// IO 密集型
int poolSize = cores * 2;

// 公式
// 线程数 = CPU 核心数 * (1 + 等待时间/计算时间)
```

### Q2：如何优雅关闭线程池？

```java
// 推荐做法
executor.shutdown();
try {
    if (!executor.awaitTermination(60, TimeUnit.SECONDS)) {
        executor.shutdownNow();
    }
} catch (InterruptedException e) {
    executor.shutdownNow();
    Thread.currentThread().interrupt();
}
```

### Q3：线程池满了怎么办？

```java
// 1. 增加队列容量
new ArrayBlockingQueue<>(200);

// 2. 增加最大线程数
maximumPoolSize = 20;

// 3. 使用 CallerRunsPolicy
new ThreadPoolExecutor.CallerRunsPolicy();

// 4. 自定义拒绝策略（保存到数据库/MQ）
```
