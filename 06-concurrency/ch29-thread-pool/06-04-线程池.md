# 6.4 线程池

## 一、为什么使用线程池

### 1.1 不使用线程池的问题

```java
// ❌ 每次创建新线程，开销大
for (int i = 0; i < 1000; i++) {
    new Thread(() -> {
        // 执行任务
    }).start();
}

// 问题：
// 1. 线程创建和销毁开销大
// 2. 无限制创建线程可能导致 OOM
// 3. 缺乏统一管理
```

### 1.2 线程池的好处

- **降低资源消耗**：复用已创建的线程
- **提高响应速度**：任务到达时无需等待线程创建
- **提高可管理性**：统一分配、调优和监控

## 二、ThreadPoolExecutor

### 2.1 构造方法参数

```java
new ThreadPoolExecutor(
    int corePoolSize,      // 核心线程数
    int maximumPoolSize,   // 最大线程数
    long keepAliveTime,    // 空闲线程存活时间
    TimeUnit unit,         // 时间单位
    BlockingQueue<Runnable> workQueue,  // 工作队列
    ThreadFactory threadFactory,        // 线程工厂
    RejectedExecutionHandler handler    // 拒绝策略
)
```

### 2.2 工作流程

```
1. 任务提交
   ↓
2. 核心线程数未满 → 创建核心线程执行
   ↓
3. 核心线程已满 → 加入工作队列
   ↓
4. 队列已满 → 创建非核心线程（直到最大线程数）
   ↓
5. 达到最大线程数 → 执行拒绝策略
```

### 2.3 基本使用

```java
import java.util.concurrent.*;

ThreadPoolExecutor executor = new ThreadPoolExecutor(
    5,                      // 核心线程数
    10,                     // 最大线程数
    60,                     // 空闲线程存活时间
    TimeUnit.SECONDS,       // 时间单位
    new ArrayBlockingQueue<>(100),  // 工作队列
    Executors.defaultThreadFactory(),  // 线程工厂
    new ThreadPoolExecutor.AbortPolicy()  // 拒绝策略
);

// 提交任务
executor.execute(() -> {
    System.out.println("执行任务：" + Thread.currentThread().getName());
});

// 提交有返回值的任务
Future<Integer> future = executor.submit(() -> {
    return 100;
});

// 获取返回值
Integer result = future.get();

// 关闭线程池
executor.shutdown();
```

## 三、Executors 工厂类

### 3.1 newFixedThreadPool（固定大小）

```java
// 固定大小的线程池
ExecutorService executor = Executors.newFixedThreadPool(5);

// 等价于
new ThreadPoolExecutor(
    5, 5,
    0L, TimeUnit.MILLISECONDS,
    new LinkedBlockingQueue<Runnable>()
);
```

### 3.2 newCachedThreadPool（可缓存）

```java
// 可缓存的线程池（适合大量短任务）
ExecutorService executor = Executors.newCachedThreadPool();

// 等价于
new ThreadPoolExecutor(
    0, Integer.MAX_VALUE,
    60L, TimeUnit.SECONDS,
    new SynchronousQueue<Runnable>()
);
```

### 3.3 newSingleThreadExecutor（单线程）

```java
// 单线程执行器（保证顺序执行）
ExecutorService executor = Executors.newSingleThreadExecutor();

// 等价于
new ThreadPoolExecutor(
    1, 1,
    0L, TimeUnit.MILLISECONDS,
    new LinkedBlockingQueue<Runnable>()
);
```

### 3.4 newScheduledThreadPool（定时任务）

```java
// 定时任务线程池
ScheduledExecutorService scheduler = Executors.newScheduledThreadPool(5);

// 延迟执行
scheduler.schedule(() -> {
    System.out.println("延迟 3 秒执行");
}, 3, TimeUnit.SECONDS);

// 定期执行
scheduler.scheduleAtFixedRate(() -> {
    System.out.println("每 2 秒执行一次");
}, 0, 2, TimeUnit.SECONDS);

// 定期执行（上一次完成后开始计时）
scheduler.scheduleWithFixedDelay(() -> {
    System.out.println("上一次完成后延迟 2 秒");
}, 0, 2, TimeUnit.SECONDS);
```

## 四、拒绝策略

### 4.1 内置策略

```java
// 1. AbortPolicy（默认）：抛异常
new ThreadPoolExecutor.AbortPolicy();

// 2. CallerRunsPolicy：由调用线程执行
new ThreadPoolExecutor.CallerRunsPolicy();

// 3. DiscardPolicy：直接丢弃
new ThreadPoolExecutor.DiscardPolicy();

// 4. DiscardOldestPolicy：丢弃最老的任务
new ThreadPoolExecutor.DiscardOldestPolicy();
```

### 4.2 自定义拒绝策略

```java
RejectedExecutionHandler customHandler = (r, executor) -> {
    System.out.println("任务被拒绝：" + r.toString());
    // 可以记录日志、保存到数据库等
};

ThreadPoolExecutor executor = new ThreadPoolExecutor(
    5, 10, 60, TimeUnit.SECONDS,
    new ArrayBlockingQueue<>(100),
    Executors.defaultThreadFactory(),
    customHandler
);
```

## 五、线程池配置

### 5.1 CPU 密集型任务

```java
// CPU 密集型：核心线程数 = CPU 核心数 + 1
int cpuCores = Runtime.getRuntime().availableProcessors();
ExecutorService executor = Executors.newFixedThreadPool(cpuCores + 1);
```

### 5.2 IO 密集型任务

```java
// IO 密集型：核心线程数 = CPU 核心数 * 2
int cpuCores = Runtime.getRuntime().availableProcessors();
ExecutorService executor = Executors.newFixedThreadPool(cpuCores * 2);

// 或根据等待/计算比率
// 线程数 = CPU 核心数 * (1 + 等待时间/计算时间)
```

### 5.3 混合型任务

```java
// 根据任务类型分离线程池
ExecutorService cpuExecutor = Executors.newFixedThreadPool(
    Runtime.getRuntime().availableProcessors() + 1
);

ExecutorService ioExecutor = Executors.newFixedThreadPool(
    Runtime.getRuntime().availableProcessors() * 2
);
```

## 六、线程池监控

### 6.1 获取线程池状态

```java
ThreadPoolExecutor executor = (ThreadPoolExecutor) Executors.newFixedThreadPool(5);

// 获取状态
int poolSize = executor.getPoolSize();           // 当前线程数
int corePoolSize = executor.getCorePoolSize();   // 核心线程数
int maxPoolSize = executor.getMaximumPoolSize(); // 最大线程数
long taskCount = executor.getTaskCount();        // 总任务数
long completedTaskCount = executor.getCompletedTaskCount();  // 已完成任务数
int queueSize = executor.getQueue().size();      // 队列中任务数
boolean isTerminated = executor.isTerminated();  // 是否已终止
```

### 6.2 自定义线程工厂

```java
class CustomThreadFactory implements ThreadFactory {
    private final AtomicInteger count = new AtomicInteger(1);
    
    @Override
    public Thread newThread(Runnable r) {
        Thread thread = new Thread(r);
        thread.setName("custom-thread-" + count.getAndIncrement());
        thread.setDaemon(false);
        thread.setPriority(Thread.NORM_PRIORITY);
        return thread;
    }
}

ThreadPoolExecutor executor = new ThreadPoolExecutor(
    5, 10, 60, TimeUnit.SECONDS,
    new ArrayBlockingQueue<>(100),
    new CustomThreadFactory()
);
```

## 七、综合示例

### 7.1 批量处理任务

```java
public class BatchProcessor {
    
    private final ExecutorService executor = Executors.newFixedThreadPool(10);
    
    public List<Result> processBatch(List<Data> dataList) throws Exception {
        List<Future<Result>> futures = new ArrayList<>();
        
        // 提交所有任务
        for (Data data : dataList) {
            Future<Result> future = executor.submit(() -> process(data));
            futures.add(future);
        }
        
        // 收集结果
        List<Result> results = new ArrayList<>();
        for (Future<Result> future : futures) {
            results.add(future.get());  // 阻塞等待结果
        }
        
        return results;
    }
    
    private Result process(Data data) {
        // 处理单个数据
        return new Result(data.getId(), "processed");
    }
    
    public void shutdown() {
        executor.shutdown();
    }
}
```

### 7.2 带超时的批量处理

```java
public class TimeoutBatchProcessor {
    
    private final ExecutorService executor = Executors.newFixedThreadPool(10);
    
    public List<Result> processWithTimeout(List<Data> dataList, long timeout, TimeUnit unit) 
            throws Exception {
        List<Future<Result>> futures = new ArrayList<>();
        
        for (Data data : dataList) {
            futures.add(executor.submit(() -> process(data)));
        }
        
        List<Result> results = new ArrayList<>();
        long deadline = System.nanoTime() + unit.toNanos(timeout);
        
        for (Future<Result> future : futures) {
            long remaining = deadline - System.nanoTime();
            if (remaining <= 0) {
                future.cancel(true);  // 取消未完成的任务
                throw new TimeoutException("处理超时");
            }
            results.add(future.get(remaining, TimeUnit.NANOSECONDS));
        }
        
        return results;
    }
    
    private Result process(Data data) {
        return new Result(data.getId(), "processed");
    }
}
```

### 7.3 线程池工具类

```java
public class ThreadPoolUtil {
    
    // CPU 密集型线程池
    private static final ExecutorService CPU_POOL = Executors.newFixedThreadPool(
        Runtime.getRuntime().availableProcessors() + 1
    );
    
    // IO 密集型线程池
    private static final ExecutorService IO_POOL = Executors.newFixedThreadPool(
        Runtime.getRuntime().availableProcessors() * 2
    );
    
    // 定时任务线程池
    private static final ScheduledExecutorService SCHEDULED_POOL = 
        Executors.newScheduledThreadPool(3);
    
    public static void executeCpu(Runnable task) {
        CPU_POOL.execute(task);
    }
    
    public static void executeIo(Runnable task) {
        IO_POOL.execute(task);
    }
    
    public static <T> Future<T> submitCpu(Callable<T> task) {
        return CPU_POOL.submit(task);
    }
    
    public static <T> Future<T> submitIo(Callable<T> task) {
        return IO_POOL.submit(task);
    }
    
    public static ScheduledFuture<?> schedule(Runnable task, long delay, TimeUnit unit) {
        return SCHEDULED_POOL.schedule(task, delay, unit);
    }
    
    public static ScheduledFuture<?> scheduleAtFixedRate(
            Runnable task, long initialDelay, long period, TimeUnit unit) {
        return SCHEDULED_POOL.scheduleAtFixedRate(task, initialDelay, period, unit);
    }
    
    public static void shutdown() {
        CPU_POOL.shutdown();
        IO_POOL.shutdown();
        SCHEDULED_POOL.shutdown();
    }
}
```

## 八、注意事项

### 8.1 避免使用 Executors 创建

```java
// ❌ 不推荐：可能导致 OOM
Executors.newFixedThreadPool(10);  // 队列无界
Executors.newCachedThreadPool();   // 最大线程数无界

// ✅ 推荐：使用 ThreadPoolExecutor
new ThreadPoolExecutor(
    5, 10, 60, TimeUnit.SECONDS,
    new ArrayBlockingQueue<>(100)  // 有界队列
);
```

### 8.2 正确关闭线程池

```java
// 优雅关闭
executor.shutdown();
try {
    if (!executor.awaitTermination(60, TimeUnit.SECONDS)) {
        executor.shutdownNow();  // 强制关闭
        if (!executor.awaitTermination(60, TimeUnit.SECONDS)) {
            System.out.println("线程池未正常关闭");
        }
    }
} catch (InterruptedException e) {
    executor.shutdownNow();
    Thread.currentThread().interrupt();
}
```

## 九、小结

1. **ThreadPoolExecutor**：核心类，7 个参数
2. **Executors 工厂**：快速创建，但要注意 OOM 风险
3. **拒绝策略**：AbortPolicy、CallerRunsPolicy、DiscardPolicy、DiscardOldestPolicy
4. **配置建议**：CPU 密集型、IO 密集型分别配置
5. **监控**：getPoolSize、getTaskCount、getQueue
6. **关闭**：先 shutdown，再 awaitTermination，最后 shutdownNow
