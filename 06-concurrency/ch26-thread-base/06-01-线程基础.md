# 26.1 线程基础：如何创建和管理线程？

## 一、先问一个问题

**什么是线程？为什么需要多线程？**

### 生活例子：餐厅厨房

```
单线程（一个厨师）
├── 切菜 → 炒菜 → 装盘
├── 再做下一道菜
└── 慢，资源利用率低

多线程（多个厨师）
├── 厨师 A 切菜
├── 厨师 B 炒菜
├── 厨师 C 装盘
└── 快，资源利用率高
```

**线程 vs 进程**：
- 进程：独立的程序（如整个餐厅）
- 线程：进程中的执行单元（如每个厨师）
- 一个进程可以有多个线程

## 二、创建线程的四种方式

### 方式一：继承 Thread 类

```java
/**
 * 继承 Thread 类创建线程
 */
class MyThread extends Thread {
    
    @Override
    public void run() {
        // 线程执行的代码
        for (int i = 0; i < 5; i++) {
            System.out.println(getName() + ": " + i);
        }
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        MyThread t1 = new MyThread();
        t1.setName("线程 A");
        t1.start();  // 启动线程（调用 run 方法）
        
        MyThread t2 = new MyThread();
        t2.setName("线程 B");
        t2.start();
        
        // 主线程继续执行
        System.out.println("主线程执行中...");
    }
}
```

**输出示例**：
```
主线程执行中...
线程 A: 0
线程 B: 0
线程 A: 1
线程 B: 1
...
```

**特点**：
- ✅ 简单直接
- ❌ 不能继承其他类（Java 单继承）
- ❌ 线程任务不能共享

### 方式二：实现 Runnable 接口

```java
/**
 * 实现 Runnable 接口
 */
class MyRunnable implements Runnable {
    
    @Override
    public void run() {
        for (int i = 0; i < 5; i++) {
            System.out.println(Thread.currentThread().getName() + ": " + i);
        }
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        // 创建 Runnable 实例
        MyRunnable task = new MyRunnable();
        
        // 创建线程
        Thread t1 = new Thread(task, "线程 A");
        Thread t2 = new Thread(task, "线程 B");
        
        // 启动线程
        t1.start();
        t2.start();
        
        // 多个线程可以共享同一个任务
    }
}
```

**特点**：
- ✅ 可以继承其他类
- ✅ 任务可以共享
- ❌ 不能返回结果

### 方式三：实现 Callable 接口（有返回值）

```java
import java.util.concurrent.*;

/**
 * 实现 Callable 接口（有返回值）
 */
class MyCallable implements Callable<Integer> {
    
    private final int n;
    
    public MyCallable(int n) {
        this.n = n;
    }
    
    @Override
    public Integer call() throws Exception {
        int sum = 0;
        for (int i = 1; i <= n; i++) {
            sum += i;
        }
        return sum;
    }
}

// 使用
public class Main {
    public static void main(String[] args) throws Exception {
        // 创建 Callable 任务
        MyCallable task1 = new MyCallable(100);
        MyCallable task2 = new MyCallable(200);
        
        // 创建 FutureTask（包装 Callable）
        FutureTask<Integer> future1 = new FutureTask<>(task1);
        FutureTask<Integer> future2 = new FutureTask<>(task2);
        
        // 创建线程
        Thread t1 = new Thread(future1, "计算线程 A");
        Thread t2 = new Thread(future2, "计算线程 B");
        
        // 启动线程
        t1.start();
        t2.start();
        
        // 获取返回值（阻塞等待）
        Integer result1 = future1.get();  // 5050
        Integer result2 = future2.get();  // 20100
        
        System.out.println("结果 1: " + result1);
        System.out.println("结果 2: " + result2);
    }
}
```

**特点**：
- ✅ 可以返回结果
- ✅ 可以抛出异常
- ❌ 需要 FutureTask 包装

### 方式四：使用线程池（推荐）

```java
import java.util.concurrent.*;

public class Main {
    public static void main(String[] args) {
        // 创建线程池
        ExecutorService executor = Executors.newFixedThreadPool(3);
        
        // 提交 Runnable 任务
        executor.submit(() -> {
            System.out.println("执行任务 1");
        });
        
        // 提交 Callable 任务
        Future<Integer> future = executor.submit(() -> {
            int sum = 0;
            for (int i = 1; i <= 100; i++) {
                sum += i;
            }
            return sum;
        });
        
        // 获取结果
        try {
            Integer result = future.get();
            System.out.println("结果：" + result);
        } catch (Exception e) {
            e.printStackTrace();
        }
        
        // 关闭线程池
        executor.shutdown();
    }
}
```

**特点**：
- ✅ 复用线程，减少开销
- ✅ 控制并发数量
- ✅ 推荐使用

## 三、线程状态详解

### 6 种线程状态

```java
public enum Thread.State {
    NEW,              // 新建
    RUNNABLE,         // 可运行/运行中
    BLOCKED,          // 阻塞（等待锁）
    WAITING,          // 无限期等待
    TIMED_WAITING,    // 限期等待
    TERMINATED        // 终止
}
```

### 状态转换图

```
NEW（新建）
  ↓ start()
RUNNABLE（可运行）
  ↓ 获取锁失败
BLOCKED（阻塞）
  ↓ 获取锁成功
RUNNABLE
  ↓ wait()/join()/LockSupport.park()
WAITING（无限期等待）
  ↓ notify()/notifyAll()/LockSupport.unpark()
RUNNABLE
  ↓ sleep(time)/wait(time)/join(time)
TIMED_WAITING（限期等待）
  ↓ 时间到
RUNNABLE
  ↓ run() 方法结束
TERMINATED（终止）
```

### 状态示例代码

```java
public class ThreadStateDemo {
    
    public static void main(String[] args) throws InterruptedException {
        Thread thread = new Thread(() -> {
            try {
                // RUNNABLE
                System.out.println("状态：" + Thread.currentThread().getState());
                
                // WAITING
                synchronized (this) {
                    wait();
                }
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        });
        
        // NEW
        System.out.println("NEW: " + thread.getState());
        
        // 启动后
        thread.start();
        Thread.sleep(100);
        
        // RUNNABLE 或 TIMED_WAITING
        System.out.println("启动后：" + thread.getState());
        
        // 唤醒线程
        synchronized (thread) {
            thread.notify();
        }
        
        thread.join();
        
        // TERMINATED
        System.out.println("终止：" + thread.getState());
    }
}
```

## 四、线程常用方法

### 基本方法

```java
Thread thread = new Thread(() -> {
    System.out.println("执行任务");
});

// 启动线程
thread.start();

// 获取线程名称
String name = thread.getName();

// 设置线程名称
thread.setName("Worker-1");

// 获取线程 ID
long id = thread.getId();

// 获取线程优先级（1-10）
int priority = thread.getPriority();

// 设置线程优先级
thread.setPriority(Thread.MAX_PRIORITY);  // 10

// 检查是否存活
boolean alive = thread.isAlive();

// 检查是否中断
boolean interrupted = thread.isInterrupted();

// 中断线程
thread.interrupt();

// 获取当前线程
Thread current = Thread.currentThread();
```

### 线程休眠

```java
try {
    // 休眠 1 秒
    Thread.sleep(1000);
    
    // 休眠 1 秒 500 毫秒
    Thread.sleep(1, 500);
} catch (InterruptedException e) {
    // 被中断时抛出
    Thread.currentThread().interrupt();  // 恢复中断状态
    e.printStackTrace();
}
```

### 线程加入（join）

```java
Thread worker = new Thread(() -> {
    // 耗时任务
    for (int i = 0; i < 5; i++) {
        System.out.println("工作：" + i);
        try {
            Thread.sleep(500);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
});

worker.start();

// 等待 worker 完成
worker.join();  // 无限期等待
// worker.join(1000);  // 最多等 1 秒

System.out.println("主线程继续执行");
```

### 线程礼让（yield）

```java
Thread t1 = new Thread(() -> {
    for (int i = 0; i < 5; i++) {
        System.out.println("线程 A: " + i);
        if (i == 2) {
            Thread.yield();  // 礼让其他线程
        }
    }
});

Thread t2 = new Thread(() -> {
    for (int i = 0; i < 5; i++) {
        System.out.println("线程 B: " + i);
    }
});

t1.start();
t2.start();
```

**注意**：yield() 不保证一定有效，取决于 JVM 实现。

### 守护线程

```java
Thread daemon = new Thread(() -> {
    while (true) {
        System.out.println("守护线程运行中...");
        try {
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
});

// 设置为守护线程
daemon.setDaemon(true);
daemon.start();

// 主线程运行
Thread.sleep(5000);

// 主线程结束后，守护线程也会结束
```

**特点**：
- 守护线程随主线程结束而结束
- 常用于后台任务（如 GC、监控）
- 必须在 start() 之前设置

## 五、线程优先级

### 优先级级别

```java
// 优先级常量
Thread.MIN_PRIORITY     // 1
Thread.NORM_PRIORITY    // 5（默认）
Thread.MAX_PRIORITY     // 10
```

### 设置优先级

```java
Thread t1 = new Thread(() -> {
    System.out.println("低优先级线程");
});
t1.setPriority(Thread.MIN_PRIORITY);

Thread t2 = new Thread(() -> {
    System.out.println("正常优先级线程");
});
t2.setPriority(Thread.NORM_PRIORITY);

Thread t3 = new Thread(() -> {
    System.out.println("高优先级线程");
});
t3.setPriority(Thread.MAX_PRIORITY);

t1.start();
t2.start();
t3.start();
```

**注意**：
- 优先级不保证执行顺序
- 取决于操作系统调度
- 不要过度依赖优先级

## 六、小结

| 创建方式 | 返回值 | 异常 | 共享 | 推荐度 |
|----------|--------|------|------|--------|
| 继承 Thread | ❌ | ❌ | ❌ | ⭐⭐ |
| 实现 Runnable | ❌ | ❌ | ✅ | ⭐⭐⭐⭐ |
| 实现 Callable | ✅ | ✅ | ✅ | ⭐⭐⭐⭐ |
| 线程池 | ✅ | ✅ | ✅ | ⭐⭐⭐⭐⭐ |

| 方法 | 作用 | 说明 |
|------|------|------|
| start() | 启动线程 | 调用 run 方法 |
| sleep() | 休眠 | 不释放锁 |
| join() | 等待 | 等待线程完成 |
| yield() | 礼让 | 不保证有效 |
| interrupt() | 中断 | 设置中断标志 |
| setDaemon() | 守护线程 | 必须在 start 前 |

**核心要点**：
- 推荐使用线程池
- 理解线程状态转换
- 掌握常用方法
- 注意线程安全

## 七、常见问题

### Q1：start() 和 run() 有什么区别？

```java
Thread thread = new Thread(() -> {
    System.out.println("执行任务");
});

// start()：启动新线程
thread.start();  // ✅ 新线程执行

// run()：普通方法调用
thread.run();  // ❌ 当前线程执行
```

### Q2：如何停止线程？

```java
// ❌ 不推荐：stop() 已废弃
thread.stop();

// ✅ 推荐：使用中断标志
thread.interrupt();

// 或在任务中检查标志
volatile boolean running = true;

public void stop() {
    running = false;
}

public void run() {
    while (running) {
        // 执行任务
    }
}
```

### Q3：线程池用完后为什么要 shutdown？

```java
ExecutorService executor = Executors.newFixedThreadPool(3);

// 使用线程池
executor.submit(task);

// 关闭线程池
executor.shutdown();  // 平滑关闭（等待任务完成）
// executor.shutdownNow();  // 立即关闭

// 不关闭会导致：
// 1. 线程不释放
// 2. 程序无法正常退出
// 3. 内存泄漏
```
