# 26.6 原子类：AtomicInteger、LongAdder

## 一、先问一个问题

**什么是原子操作？为什么需要原子类？**

### 生活例子：计数器

```
非原子操作（i++）
├── 读取 i 的值
├── 加 1
└── 写回 i
    └── 多线程可能覆盖

原子操作（AtomicInteger）
├── CAS 操作
└── 保证原子性
```

**原子操作**：
- 不可分割的操作
- 要么全部成功，要么全部失败
- 不会被线程调度打断

## 二、CAS（Compare And Swap）

### 什么是 CAS？

**生活例子：自动售货机**

```
CAS 操作
├── 比较：当前值是否等于预期值
├── 交换：如果是，更新为新值
└── 失败：重试或返回
```

**CAS 原理**：
```java
// 伪代码
boolean compareAndSet(int expected, int newValue) {
    if (currentValue == expected) {
        currentValue = newValue;
        return true;
    }
    return false;
}
```

### CAS 问题

```
ABA 问题
├── 线程 A 读取值 A
├── 线程 B 修改 A→B→A
├── 线程 A CAS 成功
└── 但值已经被修改过

解决：AtomicStampedReference（带版本号的 CAS）
```

## 三、AtomicInteger

### 基本使用

```java
import java.util.concurrent.atomic.*;

public class AtomicIntegerDemo {
    
    public static void main(String[] args) {
        AtomicInteger atomic = new AtomicInteger(0);
        
        // 自增
        atomic.incrementAndGet();  // 1（先加后返回）
        atomic.getAndIncrement();  // 1（先返回后加）
        
        // 自减
        atomic.decrementAndGet();  // 1
        atomic.getAndDecrement();  // 1
        
        // 加减
        atomic.addAndGet(5);  // 6
        atomic.getAndAdd(5);  // 6
        
        // 获取当前值
        int value = atomic.get();
        
        // 设置值
        atomic.set(10);
        
        // CAS 操作
        boolean success = atomic.compareAndSet(10, 20);  // true
        
        System.out.println("最终值：" + atomic.get());
    }
}
```

### 实际应用场景

### 场景 1：线程安全计数器

```java
public class Counter {
    private AtomicInteger count = new AtomicInteger(0);
    
    public void increment() {
        count.incrementAndGet();
    }
    
    public int getCount() {
        return count.get();
    }
}

// 使用
public class Main {
    public static void main(String[] args) throws InterruptedException {
        Counter counter = new Counter();
        
        List<Thread> threads = new ArrayList<>();
        for (int i = 0; i < 10; i++) {
            threads.add(new Thread(() -> {
                for (int j = 0; j < 1000; j++) {
                    counter.increment();
                }
            }));
        }
        
        threads.forEach(Thread::start);
        threads.forEach(t -> {
            try { t.join(); } catch (InterruptedException e) {}
        });
        
        System.out.println("最终计数：" + counter.getCount());  // 10000
    }
}
```

### 场景 2：序列号生成器

```java
public class IdGenerator {
    private static AtomicInteger idGenerator = new AtomicInteger(0);
    
    public static int generateId() {
        return idGenerator.incrementAndGet();
    }
}

// 使用
int id1 = IdGenerator.generateId();  // 1
int id2 = IdGenerator.generateId();  // 2
```

### 场景 3：限流器

```java
public class RateLimiter {
    private final AtomicInteger count = new AtomicInteger(0);
    private final int limit;
    
    public RateLimiter(int limit) {
        this.limit = limit;
    }
    
    public boolean tryAcquire() {
        int current = count.get();
        if (current >= limit) {
            return false;
        }
        
        // CAS 更新
        if (count.compareAndSet(current, current + 1)) {
            return true;
        }
        
        return false;
    }
    
    public void release() {
        count.decrementAndGet();
    }
}
```

## 四、其他原子类

### AtomicLong

```java
AtomicLong atomicLong = new AtomicLong(0);

atomicLong.incrementAndGet();
atomicLong.addAndGet(100);
long value = atomicLong.get();
```

### AtomicBoolean

```java
AtomicBoolean atomicBoolean = new AtomicBoolean(false);

// CAS 操作
boolean success = atomicBoolean.compareAndSet(false, true);

// 获取并设置
boolean oldValue = atomicBoolean.getAndSet(true);
```

### AtomicReference

```java
AtomicReference<User> atomicRef = new AtomicReference<>();

// 设置值
atomicRef.set(new User("张三", 18));

// CAS 操作
User expected = new User("张三", 18);
User newValue = new User("李四", 19);
boolean success = atomicRef.compareAndSet(expected, newValue);

// 获取值
User user = atomicRef.get();
```

### AtomicStampedReference（解决 ABA 问题）

```java
AtomicStampedReference<Integer> atomicRef = 
    new AtomicStampedReference<>(0, 0);

int[] stampHolder = new int[1];

// 获取当前值和版本号
Integer value = atomicRef.get(stampHolder);
int stamp = stampHolder[0];

// CAS 操作（带版本号）
boolean success = atomicRef.compareAndSet(
    value,           // 预期值
    newValue,        // 新值
    stamp,           // 预期版本号
    stamp + 1        // 新版本号
);
```

## 五、LongAdder（高性能计数器）

### 什么是 LongAdder？

**生活例子：多窗口计数**

```
AtomicLong（单窗口）
└── 所有线程竞争一个值 → 性能低

LongAdder（多窗口）
├── 多个 Cell（单元格）
├── 线程分散到不同 Cell
└── 最后求和 → 性能高
```

**LongAdder 特点**：
- 分段累加
- 减少竞争
- 适合高并发场景

### 基本使用

```java
import java.util.concurrent.atomic.*;

public class LongAdderDemo {
    
    public static void main(String[] args) throws InterruptedException {
        LongAdder adder = new LongAdder();
        
        // 自增
        adder.increment();
        
        // 自减
        adder.decrement();
        
        // 加法
        adder.add(10);
        
        // 获取值
        long sum = adder.sum();
        
        // 获取值并重置
        long sumThenReset = adder.sumThenReset();
        
        System.out.println("总和：" + sum);
    }
}
```

### LongAdder vs AtomicLong

```java
public class PerformanceTest {
    
    public static void main(String[] args) throws InterruptedException {
        int threadCount = 10;
        int operations = 100000;
        
        // AtomicLong 测试
        AtomicLong atomicLong = new AtomicLong(0);
        long start = System.currentTimeMillis();
        
        List<Thread> threads = new ArrayList<>();
        for (int i = 0; i < threadCount; i++) {
            threads.add(new Thread(() -> {
                for (int j = 0; j < operations; j++) {
                    atomicLong.incrementAndGet();
                }
            }));
        }
        
        threads.forEach(Thread::start);
        threads.forEach(t -> {
            try { t.join(); } catch (InterruptedException e) {}
        });
        
        long atomicTime = System.currentTimeMillis() - start;
        System.out.println("AtomicLong 耗时：" + atomicTime + "ms");
        
        // LongAdder 测试
        LongAdder longAdder = new LongAdder();
        start = System.currentTimeMillis();
        
        threads.clear();
        for (int i = 0; i < threadCount; i++) {
            threads.add(new Thread(() -> {
                for (int j = 0; j < operations; j++) {
                    longAdder.increment();
                }
            }));
        }
        
        threads.forEach(Thread::start);
        threads.forEach(t -> {
            try { t.join(); } catch (InterruptedException e) {}
        });
        
        long adderTime = System.currentTimeMillis() - start;
        System.out.println("LongAdder 耗时：" + adderTime + "ms");
        
        System.out.println("LongAdder 快 " + (atomicTime * 1.0 / adderTime) + " 倍");
    }
}
```

**结果**：
```
AtomicLong 耗时：500ms
LongAdder 耗时：100ms
LongAdder 快 5 倍
```

### 使用场景

```
AtomicLong 适合：
├── 低并发场景
├── 需要实时准确值
└── 频繁读取

LongAdder 适合：
├── 高并发场景
├── 累加操作多
└── 读取操作少
```

## 七、字段更新器

### AtomicIntegerFieldUpdater

```java
public class User {
    volatile int age;  // 必须是 volatile

        this.age = age;

// 使用
AtomicIntegerFieldUpdater<User> updater =
```

## 八、小结

| 原子类 | 作用 | 适用场景 |
|--------|------|----------|
| AtomicInteger | 原子整数 | 计数器、序列号 |
| AtomicLong | 原子长整数 | 大数值计数 |
| AtomicBoolean | 原子布尔 | 状态标志 |
| AtomicReference | 原子引用 | 对象引用 |
| LongAdder | 高性能计数 | 高并发累加 |
| AtomicStampedReference | 带版本号 CAS | 解决 ABA 问题 |

**核心要点**：
- CAS 保证原子性
- LongAdder 适合高并发
- 原子类避免锁竞争

## 九、常见问题

### Q1：AtomicInteger 和 synchronized 有什么区别？

```java
// AtomicInteger（CAS）
AtomicInteger count = new AtomicInteger(0);
count.incrementAndGet();

// synchronized（锁）
synchronized (this) {
    count++;
}

// 区别：
// - AtomicInteger：无锁，性能好
// - synchronized：有锁，性能低
```

### Q2：LongAdder 一定比 AtomicLong 快吗？

```
不一定：
- 低并发：AtomicLong 更快（无分段开销）
- 高并发：LongAdder 更快（减少竞争）
```

### Q3：volatile 和原子类有什么区别？

```java
// volatile（只保证可见性）
volatile int count = 0;
count++;  // 不是原子操作

// 原子类（保证原子性）
AtomicInteger count = new AtomicInteger(0);
count.incrementAndGet();  // 原子操作
```
