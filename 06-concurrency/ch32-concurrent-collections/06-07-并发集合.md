# 26.7 并发集合：ConcurrentHashMap、BlockingQueue

## 一、先问一个问题

**什么是并发集合？为什么需要并发集合？**

### 生活例子：共享文档

```
非并发集合（HashMap）
├── 多人同时编辑
└── 可能冲突、数据丢失

并发集合（ConcurrentHashMap）
├── 多人同时编辑
└── 自动协调、数据安全
```

**并发集合**：
- 线程安全
- 高并发性能
- 无需外部同步

## 二、ConcurrentHashMap

### 什么是 ConcurrentHashMap？

**生活例子：多格储物柜**

```
HashMap（单格储物柜）
├── 一个锁保护所有
└── 同时只能一人访问

ConcurrentHashMap（多格储物柜）
├── 分段锁（Java 7）
├── CAS + synchronized（Java 8）
└── 多人可同时访问不同格子
```

### 基本使用

```java
import java.util.concurrent.*;

public class ConcurrentHashMapDemo {
    
    public static void main(String[] args) {
        ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();
        
        // 添加
        map.put("A", 1);
        map.put("B", 2);
        
        // 获取
        Integer value = map.get("A");
        
        // 删除
        map.remove("A");
        
        // 判断
        boolean contains = map.containsKey("B");
        
        // 大小
        int size = map.size();
        
        // 遍历
        map.forEach((key, value) -> {
            System.out.println(key + " = " + value);
        });
    }
}
```

### 特有方法

```java
ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();

// putIfAbsent（不存在才添加）
map.putIfAbsent("A", 1);

// compute（计算并更新）
map.compute("A", (key, value) -> {
    return value == null ? 1 : value + 1;
});

// computeIfAbsent（不存在时计算）
map.computeIfAbsent("A", key -> 1);

// computeIfPresent（存在时计算）
map.computeIfPresent("A", (key, value) -> value + 1);

// merge（合并）
map.merge("A", 1, Integer::sum);

// replace（替换）
map.replace("A", 1, 2);  // CAS 替换

// 批量操作
map.forEach(10, (key, value) -> {
    System.out.println(key + " = " + value);
});

// 搜索
String result = map.search(10, (key, value) -> {
    return value > 5 ? key : null;
});

// 归约
Integer sum = map.reduce(10, (key, value) -> value, Integer::sum);
```

### 并发特性

```java
ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();

// 弱一致性迭代器
// 迭代时允许其他线程修改
// 不会抛 ConcurrentModificationException

// 分段锁（Java 7）
// 16 个 Segment，每个 Segment 一个锁

// CAS + synchronized（Java 8）
// 数组 + 链表 + 红黑树
// 锁粒度更细（桶级别）
```

### 实际应用场景

### 场景 1：缓存

```java
public class Cache {
    private final ConcurrentHashMap<String, Object> cache = new ConcurrentHashMap<>();
    
    public Object get(String key) {
        return cache.get(key);
    }
    
    public void put(String key, Object value) {
        cache.put(key, value);
    }
    
    public Object computeIfAbsent(String key, Function<String, Object> mapper) {
        return cache.computeIfAbsent(key, mapper);
    }
}
```

### 场景 2：计数器

```java
public class WordCounter {
    private final ConcurrentHashMap<String, LongAdder> counter = new ConcurrentHashMap<>();
    
    public void count(String word) {
        counter.computeIfAbsent(word, k -> new LongAdder()).increment();
    }
    
    public Map<String, Long> getCounts() {
        return counter.entrySet().stream()
            .collect(Collectors.toMap(
                Map.Entry::getKey,
                e -> e.getValue().sum()
            ));
    }
}
```

## 三、CopyOnWriteArrayList

### 什么是 CopyOnWriteArrayList？

**生活例子：拍照**

```
CopyOnWriteArrayList
├── 写时复制
├── 读操作无锁
└── 适合读多写少
```

**特点**：
- 写时复制数组
- 读操作无锁
- 迭代器不会抛异常
- 适合读多写少场景

### 基本使用

```java
import java.util.concurrent.*;

public class CopyOnWriteArrayListDemo {
    
    public static void main(String[] args) {
        CopyOnWriteArrayList<String> list = new CopyOnWriteArrayList<>();
        
        // 添加
        list.add("A");
        list.add("B");
        
        // 获取
        String value = list.get(0);
        
        // 删除
        list.remove("A");
        
        // 遍历（安全）
        for (String item : list) {
            System.out.println(item);
        }
    }
}
```

### 使用场景

```java
// ✅ 适合：读多写少
CopyOnWriteArrayList<String> listeners = new CopyOnWriteArrayList<>();

// 添加监听器（写操作少）
listeners.add(listener);

// 通知监听器（读操作多）
for (Listener listener : listeners) {
    listener.onEvent();
}

// ❌ 不适合：写多读少
// 每次写都复制数组，性能低
```

## 四、BlockingQueue（阻塞队列）

### 什么是 BlockingQueue？

**生活例子：排队取餐**

```
BlockingQueue
├── 队列满时，生产者等待
└── 队列空时，消费者等待
```

**特点**：
- 线程安全
- 自动阻塞
- 生产者消费者模式

### 实现类

```
BlockingQueue
├── ArrayBlockingQueue（有界，数组）
├── LinkedBlockingQueue（可选有界，链表）
├── PriorityBlockingQueue（优先队列）
├── SynchronousQueue（同步队列）
└── DelayQueue（延迟队列）
```

### ArrayBlockingQueue

```java
// 有界队列（容量 10）
BlockingQueue<Integer> queue = new ArrayBlockingQueue<>(10);

// 添加（队列满时抛异常）
queue.add(1);
queue.offer(1);  // 推荐

// 添加（队列满时阻塞）
queue.put(1);

// 添加（带超时）
queue.offer(1, 1, TimeUnit.SECONDS);

// 获取（队列空时抛异常）
Integer value = queue.poll();

// 获取（队列空时阻塞）
Integer value = queue.take();

// 获取（带超时）
Integer value = queue.poll(1, TimeUnit.SECONDS);

// 查看队首
Integer value = queue.peek();
```

### LinkedBlockingQueue

```java
// 无界队列（默认 Integer.MAX_VALUE）
BlockingQueue<Integer> queue = new LinkedBlockingQueue<>();

// 有界队列
BlockingQueue<Integer> queue = new LinkedBlockingQueue<>(100);

// 使用类似 ArrayBlockingQueue
```

### PriorityBlockingQueue

```java
// 优先队列
BlockingQueue<Integer> queue = new PriorityBlockingQueue<>(
    10,
    (a, b) -> b - a  // 降序
);

queue.offer(3);
queue.offer(1);
queue.offer(2);

// 取出顺序：3, 2, 1
```

### SynchronousQueue

```java
// 同步队列（不存储元素）
BlockingQueue<Integer> queue = new SynchronousQueue<>();

// 生产者必须等待消费者
new Thread(() -> {
    try {
        queue.put(1);  // 阻塞，等待消费者
    } catch (InterruptedException e) {}
}).start();

// 消费者
Integer value = queue.take();  // 接收
```

### 实际应用场景

### 场景 1：生产者消费者

```java
public class ProducerConsumer {
    private final BlockingQueue<Integer> queue = new ArrayBlockingQueue<>(10);
    
    // 生产者
    public void produce() throws InterruptedException {
        int value = 0;
        while (true) {
            queue.put(value++);
            System.out.println("生产：" + (value - 1));
            Thread.sleep(100);
        }
    }
    
    // 消费者
    public void consume() throws InterruptedException {
        while (true) {
            Integer value = queue.take();
            System.out.println("消费：" + value);
            Thread.sleep(200);
        }
    }
}
```

### 场景 2：线程池工作队列

```java
// 线程池使用阻塞队列
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    5,                      // 核心线程
    10,                     // 最大线程
    60, TimeUnit.SECONDS,   // 空闲时间
    new ArrayBlockingQueue<>(100)  // 工作队列
);
```

### 场景 3：数据管道

```java
public class DataPipeline {
    private final BlockingQueue<String> queue = new LinkedBlockingQueue<>();
    
    // 数据源
    public void source() {
        new Thread(() -> {
            try {
                for (int i = 0; i < 100; i++) {
                    queue.put("数据：" + i);
                }
            } catch (InterruptedException e) {}
        }).start();
    }
    
    // 数据处理
    public void processor() {
        new Thread(() -> {
            try {
                while (true) {
                    String data = queue.take();
                    System.out.println("处理：" + data);
                }
            } catch (InterruptedException e) {}
        }).start();
    }
}
```

## 六、小结

| 集合类 | 特点 | 适用场景 |
|--------|------|----------|
| ConcurrentHashMap | 分段锁、高并发 | 并发 Map |
| CopyOnWriteArrayList | 写时复制 | 读多写少 List |
| ArrayBlockingQueue | 有界阻塞队列 | 生产者消费者 |
| LinkedBlockingQueue | 可选有界 | 工作队列 |
| PriorityBlockingQueue | 优先队列 | 优先级任务 |
| SynchronousQueue | 同步队列 | 直接传递 |

**核心要点**：
- ConcurrentHashMap：并发 Map 首选
- CopyOnWriteArrayList：读多写少
- BlockingQueue：生产者消费者

## 七、常见问题

### Q1：ConcurrentHashMap 如何保证线程安全？

```
Java 7：分段锁（Segment）
├── 16 个 Segment
└── 每个 Segment 一个锁

Java 8：CAS + synchronized
├── 数组 + 链表 + 红黑树
└── 锁粒度更细（桶级别）
```

### Q2：ConcurrentHashMap 和 Hashtable 有什么区别？

```java
// Hashtable（已淘汰）
// - 全表锁
// - 性能低
// - 不推荐使用

// ConcurrentHashMap
// - 分段锁
// - 性能高
// - 推荐使用
```

### Q3：CopyOnWriteArrayList 为什么适合读多写少？

```
读操作：无锁，直接读
写操作：复制数组，修改，替换引用

读多：性能好
写多：频繁复制，性能差
```
