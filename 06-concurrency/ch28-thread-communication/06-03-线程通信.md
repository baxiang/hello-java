# 26.3 线程通信：wait、notify、Condition

## 一、先问一个问题

**线程之间如何通信？**

### 生活例子：餐厅叫号

```
厨师（生产者）
├── 做好菜
├── 叫号
└── 等待下一个订单

服务员（消费者）
├── 等待叫号
├── 听到叫号取菜
└── 通知厨师继续
```

**线程通信**：
- 线程之间协作
- 一个线程等待另一个线程
- 避免忙等待（浪费 CPU）

## 二、wait 和 notify

### wait() 方法

```java
// 当前线程等待，释放锁
synchronized (lock) {
    lock.wait();  // 无限期等待
}

// 带超时的等待
synchronized (lock) {
    lock.wait(1000);  // 最多等 1 秒
}
```

**特点**：
- 必须在 synchronized 块中
- 释放锁
- 被 interrupt 会抛 InterruptedException

### notify() 方法

```java
// 唤醒一个等待的线程
synchronized (lock) {
    lock.notify();
}

// 唤醒所有等待的线程
synchronized (lock) {
    lock.notifyAll();
}
```

**特点**：
- 必须在 synchronized 块中
- 不释放锁（当前线程执行完才释放）
- notify() 随机唤醒一个

### wait vs sleep

| 特性 | wait | sleep |
|------|------|-------|
| 所属类 | Object | Thread |
| 释放锁 | ✅ | ❌ |
| 同步块 | 必须 | 不必 |
| 唤醒 | notify/notifyAll | 时间到/interrupt |

## 三、生产者 - 消费者模式

### 使用 wait/notify

```java
import java.util.*;

public class ProducerConsumer {
    private final Queue<Integer> queue = new LinkedList<>();
    private final int MAX_SIZE = 10;
    
    // 生产者
    public void produce() throws InterruptedException {
        int value = 0;
        while (true) {
            synchronized (this) {
                // 队列满，等待
                while (queue.size() >= MAX_SIZE) {
                    wait();
                }
                
                queue.offer(value++);
                System.out.println("生产：" + (value - 1));
                
                // 通知消费者
                notify();
            }
            
            Thread.sleep(100);
        }
    }
    
    // 消费者
    public void consume() throws InterruptedException {
        while (true) {
            synchronized (this) {
                // 队列空，等待
                while (queue.isEmpty()) {
                    wait();
                }
                
                int value = queue.poll();
                System.out.println("消费：" + value);
                
                // 通知生产者
                notify();
            }
            
            Thread.sleep(200);
        }
    }
    
    public static void main(String[] args) {
        ProducerConsumer pc = new ProducerConsumer();
        
        new Thread(() -> {
            try {
                pc.produce();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }).start();
        
        new Thread(() -> {
            try {
                pc.consume();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }).start();
    }
}
```

### 为什么用 while 不用 if？

```java
// ❌ 错误：用 if
synchronized (lock) {
    if (queue.isEmpty()) {
        wait();
    }
    // 可能被虚假唤醒，队列还是空的
    queue.poll();  // NullPointerException
}

// ✅ 正确：用 while
synchronized (lock) {
    while (queue.isEmpty()) {
        wait();
    }
    // 确保队列不空
    queue.poll();
}
```

**原因**：
- 虚假唤醒（spurious wakeup）
- 条件可能再次改变

## 四、Condition

### 什么是 Condition？

**生活例子：多条件等待**

```
synchronized（单条件）
└── 一个休息室，所有人等同一个条件

Condition（多条件）
├── 多个休息室
├── 不同条件不同休息室
└── 精确唤醒
```

### Condition 使用

```java
import java.util.concurrent.locks.*;

public class ConditionDemo {
    private final Lock lock = new ReentrantLock();
    private final Condition notFull = lock.newCondition();
    private final Condition notEmpty = lock.newCondition();
    
    private final Queue<Integer> queue = new LinkedList<>();
    private final int MAX_SIZE = 10;
    
    // 生产者
    public void produce() throws InterruptedException {
        int value = 0;
        while (true) {
            lock.lock();
            try {
                // 队列满，等待
                while (queue.size() >= MAX_SIZE) {
                    notFull.await();
                }
                
                queue.offer(value++);
                System.out.println("生产：" + (value - 1));
                
                // 通知消费者
                notEmpty.signal();
            } finally {
                lock.unlock();
            }
            
            Thread.sleep(100);
        }
    }
    
    // 消费者
    public void consume() throws InterruptedException {
        while (true) {
            lock.lock();
            try {
                // 队列空，等待
                while (queue.isEmpty()) {
                    notEmpty.await();
                }
                
                int value = queue.poll();
                System.out.println("消费：" + value);
                
                // 通知生产者
                notFull.signal();
            } finally {
                lock.unlock();
            }
            
            Thread.sleep(200);
        }
    }
}
```

### Condition 方法

```java
Condition condition = lock.newCondition();

// 等待
condition.await();           // 无限期
condition.await(1, TimeUnit.SECONDS);  // 带超时

// 唤醒
condition.signal();          // 唤醒一个
condition.signalAll();       // 唤醒所有
```

### Condition vs wait/notify

| 特性 | wait/notify | Condition |
|------|-------------|-----------|
| 所属类 | Object | Lock |
| 锁类型 | synchronized | Lock |
| 条件数量 | 1 个 | 多个 |
| 等待方法 | wait | await |
| 唤醒方法 | notify/notifyAll | signal/signalAll |

## 五、管道通信

### PipedInputStream / PipedOutputStream

```java
import java.io.*;

public class PipeDemo {
    
    public static void main(String[] args) throws IOException {
        // 创建管道
        PipedInputStream pis = new PipedInputStream();
        PipedOutputStream pos = new PipedOutputStream(pis);
        
        // 生产者线程
        new Thread(() -> {
            try {
                for (int i = 0; i < 5; i++) {
                    pos.write(("消息：" + i).getBytes());
                    System.out.println("发送：消息" + i);
                }
                pos.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        }).start();
        
        // 消费者线程
        new Thread(() -> {
            try {
                byte[] buffer = new byte[1024];
                int len;
                while ((len = pis.read(buffer)) != -1) {
                    String message = new String(buffer, 0, len);
                    System.out.println("接收：" + message);
                }
                pis.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        }).start();
    }
}
```

### PipedReader / PipedWriter

```java
// 字符流版本
PipedReader reader = new PipedReader();
PipedWriter writer = new PipedWriter(reader);

// 使用类似字节流
```

## 六、小结

| 通信方式 | 适用场景 | 优点 | 缺点 |
|----------|----------|------|------|
| wait/notify | 简单通信 | 内置、简单 | 单条件 |
| Condition | 多条件等待 | 灵活、多条件 | 需配合 Lock |
| 管道 | 流式数据 | 方便 | 只能一对一 |

**核心要点**：
- wait/notify：基础通信
- Condition：多条件等待
- 管道：流式数据传输

## 七、常见问题

### Q1：为什么 wait 要在循环中调用？

```java
// ✅ 正确
while (condition) {
    wait();
}

// ❌ 错误
if (condition) {
    wait();
}

// 原因：
// 1. 虚假唤醒
// 2. 条件可能再次改变
```

### Q2：notify 和 notifyAll 有什么区别？

```java
// notify()：随机唤醒一个
// 适合：只有一个等待线程，或所有等待线程条件相同

// notifyAll()：唤醒所有
// 适合：多个等待线程，条件不同
// 更安全，推荐使用
```

### Q3：Condition 的 signal 和 signalAll 有什么区别？

```java
// signal()：唤醒一个等待该 Condition 的线程
// signalAll()：唤醒所有等待该 Condition 的线程

// 推荐：
// - 只有一个等待线程：signal()
// - 多个等待线程：signalAll()
```
