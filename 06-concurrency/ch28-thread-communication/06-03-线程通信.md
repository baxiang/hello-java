# 6.3 线程通信

## 一、wait 和 notify

### 1.1 基本概念

- **wait()**：当前线程等待，释放锁
- **notify()**：唤醒一个等待的线程
- **notifyAll()**：唤醒所有等待的线程

**注意**：
- 必须在 synchronized 块中使用
- wait() 会释放锁，notify() 不会

### 1.2 基本使用

```java
Object lock = new Object();

// 线程 1：等待
synchronized (lock) {
    lock.wait();  // 等待，释放锁
    // 被唤醒后继续执行
}

// 线程 2：通知
synchronized (lock) {
    lock.notify();   // 唤醒一个
    lock.notifyAll(); // 唤醒所有
}
```

### 1.3 wait 带超时

```java
synchronized (lock) {
    // 等待最多 1 秒
    lock.wait(1000);
    
    // 等待 1 秒 500 毫秒
    lock.wait(1, 500000);
}
```

## 二、生产者 - 消费者模式

### 2.1 wait/notify 实现

```java
class Buffer {
    private int data;
    private boolean empty = true;
    
    public synchronized void produce(int value) throws InterruptedException {
        while (!empty) {  // 如果不满，等待
            wait();
        }
        data = value;
        empty = false;
        System.out.println("生产：" + value);
        notifyAll();  // 通知消费者
    }
    
    public synchronized int consume() throws InterruptedException {
        while (empty) {  // 如果空，等待
            wait();
        }
        int value = data;
        empty = true;
        System.out.println("消费：" + value);
        notifyAll();  // 通知生产者
        return value;
    }
}

// 测试
Buffer buffer = new Buffer();

new Thread(() -> {
    for (int i = 0; i < 10; i++) {
        try { buffer.produce(i); } catch (Exception e) {}
    }
}, "生产者").start();

new Thread(() -> {
    for (int i = 0; i < 10; i++) {
        try { buffer.consume(); } catch (Exception e) {}
    }
}, "消费者").start();
```

### 2.2 为什么用 while 不用 if

```java
// ❌ 错误：使用 if
synchronized (lock) {
    if (condition) {
        wait();
    }
    // 可能被虚假唤醒
}

// ✅ 正确：使用 while
synchronized (lock) {
    while (condition) {
        wait();
    }
    // 确保条件真正满足
}
```

## 三、BlockingQueue

### 3.1 ArrayBlockingQueue

```java
import java.util.concurrent.*;

// 有界阻塞队列
BlockingQueue<Integer> queue = new ArrayBlockingQueue<>(10);

// 生产者
new Thread(() -> {
    for (int i = 0; i < 20; i++) {
        try {
            queue.put(i);  // 队列满时阻塞
            System.out.println("生产：" + i);
        } catch (InterruptedException e) {}
    }
}).start();

// 消费者
new Thread(() -> {
    while (true) {
        try {
            Integer item = queue.take();  // 队列空时阻塞
            System.out.println("消费：" + item);
        } catch (InterruptedException e) {}
    }
}).start();
```

### 3.2 LinkedBlockingQueue

```java
// 无界阻塞队列（默认容量 Integer.MAX_VALUE）
BlockingQueue<Integer> queue = new LinkedBlockingQueue<>();

// 有界
BlockingQueue<Integer> bounded = new LinkedBlockingQueue<>(100);
```

### 3.3 其他方法

```java
BlockingQueue<Integer> queue = new LinkedBlockingQueue<>(10);

// 添加（满时抛异常）
boolean added = queue.offer(1);

// 添加（满时阻塞）
queue.put(1);

// 添加（超时）
boolean added = queue.offer(1, 1, TimeUnit.SECONDS);

// 获取（空时抛异常）
Integer item = queue.poll();

// 获取（空时阻塞）
Integer item = queue.take();

// 获取（超时）
Integer item = queue.poll(1, TimeUnit.SECONDS);
```

## 四、Condition

### 4.1 基本使用

```java
import java.util.concurrent.locks.*;

class Buffer {
    private int data;
    private boolean empty = true;
    
    private final Lock lock = new ReentrantLock();
    private final Condition notEmpty = lock.newCondition();
    private final Condition notFull = lock.newCondition();
    
    public void produce(int value) throws InterruptedException {
        lock.lock();
        try {
            while (!empty) {
                notFull.await();  // 等待不满
            }
            data = value;
            empty = false;
            System.out.println("生产：" + value);
            notEmpty.signal();  // 通知不空
        } finally {
            lock.unlock();
        }
    }
    
    public int consume() throws InterruptedException {
        lock.lock();
        try {
            while (empty) {
                notEmpty.await();  // 等待不空
            }
            int value = data;
            empty = true;
            System.out.println("消费：" + value);
            notFull.signal();  // 通知不满
            return value;
        } finally {
            lock.unlock();
        }
    }
}
```

### 4.2 多 Condition

```java
class BoundedBuffer {
    private final Object[] items = new Object[10];
    private int putIndex, takeIndex, count;
    
    private final Lock lock = new ReentrantLock();
    private final Condition notFull = lock.newCondition();
    private final Condition notEmpty = lock.newCondition();
    
    public void put(Object item) throws InterruptedException {
        lock.lock();
        try {
            while (count == items.length) {
                notFull.await();
            }
            items[putIndex] = item;
            putIndex = (++putIndex == items.length) ? 0 : putIndex;
            count++;
            notEmpty.signal();
        } finally {
            lock.unlock();
        }
    }
    
    public Object take() throws InterruptedException {
        lock.lock();
        try {
            while (count == 0) {
                notEmpty.await();
            }
            Object item = items[takeIndex];
            items[takeIndex] = null;
            takeIndex = (++takeIndex == items.length) ? 0 : takeIndex;
            count--;
            notFull.signal();
            return item;
        } finally {
            lock.unlock();
        }
    }
}
```

## 五、Semaphore（信号量）

### 5.1 基本使用

```java
import java.util.concurrent.*;

// 信号量，允许 3 个并发
Semaphore semaphore = new Semaphore(3);

for (int i = 0; i < 10; i++) {
    new Thread(() -> {
        try {
            semaphore.acquire();  // 获取许可
            System.out.println(Thread.currentThread().getName() + " 开始执行");
            Thread.sleep(1000);
            System.out.println(Thread.currentThread().getName() + " 执行完成");
        } catch (InterruptedException e) {}
        finally {
            semaphore.release();  // 释放许可
        }
    }).start();
}
```

### 5.2 公平信号量

```java
// 公平信号量（按等待顺序分配）
Semaphore fairSemaphore = new Semaphore(3, true);

// 非公平信号量（默认）
Semaphore unfairSemaphore = new Semaphore(3);
```

## 六、CountDownLatch（倒计时门闩）

### 6.1 基本使用

```java
import java.util.concurrent.*;

// 等待 3 个任务完成
CountDownLatch latch = new CountDownLatch(3);

for (int i = 0; i < 3; i++) {
    new Thread(() -> {
        try {
            System.out.println(Thread.currentThread().getName() + " 开始");
            Thread.sleep(1000);
            System.out.println(Thread.currentThread().getName() + " 完成");
        } catch (InterruptedException e) {}
        finally {
            latch.countDown();  // 计数减 1
        }
    }).start();
}

latch.await();  // 等待计数为 0
System.out.println("所有任务完成");
```

### 6.2 应用场景

```java
// 等待多个服务启动
CountDownLatch startupLatch = new CountDownLatch(3);

new Thread(() -> { startDatabase(); startupLatch.countDown(); }).start();
new Thread(() -> { startCache(); startupLatch.countDown(); }).start();
new Thread(() -> { startMQ(); startupLatch.countDown(); }).start();

startupLatch.await();  // 等待所有服务启动
System.out.println("系统启动完成");
```

## 七、CyclicBarrier（循环屏障）

### 7.1 基本使用

```java
import java.util.concurrent.*;

// 等待 3 个线程到达屏障
CyclicBarrier barrier = new CyclicBarrier(3, () -> {
    System.out.println("所有线程到达，开始执行");
});

for (int i = 0; i < 3; i++) {
    new Thread(() -> {
        try {
            System.out.println(Thread.currentThread().getName() + " 到达屏障");
            barrier.await();  // 等待其他线程
            System.out.println(Thread.currentThread().getName() + " 继续执行");
        } catch (Exception e) {}
    }).start();
}
```

### 7.2 可重复使用

```java
// CyclicBarrier 可以重复使用
CyclicBarrier barrier = new CyclicBarrier(3);

// 第一轮
for (int i = 0; i < 3; i++) {
    new Thread(() -> {
        try {
            barrier.await();
        } catch (Exception e) {}
    }).start();
}

// 等待第一轮完成后，可以开始第二轮
```

## 八、综合示例

### 8.1 多线程数据处理

```java
public class DataProcessor {
    
    public static void main(String[] args) throws Exception {
        int threadCount = 4;
        CountDownLatch latch = new CountDownLatch(threadCount);
        List<Result> results = Collections.synchronizedList(new ArrayList<>());
        
        // 分割数据
        int dataSize = 1000;
        int chunkSize = dataSize / threadCount;
        
        for (int i = 0; i < threadCount; i++) {
            final int start = i * chunkSize;
            final int end = (i == threadCount - 1) ? dataSize : start + chunkSize;
            
            new Thread(() -> {
                try {
                    Result result = processChunk(start, end);
                    results.add(result);
                } finally {
                    latch.countDown();
                }
            }).start();
        }
        
        latch.await();  // 等待所有线程完成
        
        // 合并结果
        Result finalResult = mergeResults(results);
        System.out.println("最终结果：" + finalResult);
    }
    
    private static Result processChunk(int start, int end) {
        // 处理数据块
        return new Result(start, end);
    }
    
    private static Result mergeResults(List<Result> results) {
        // 合并结果
        return new Result(0, 1000);
    }
}
```

### 8.2 线程池 + 阻塞队列

```java
public class TaskExecutor {
    
    private final BlockingQueue<Runnable> queue = new LinkedBlockingQueue<>(100);
    private final ExecutorService executor = Executors.newFixedThreadPool(4);
    
    public void submit(Runnable task) {
        try {
            queue.put(task);  // 队列满时阻塞
            executor.execute(() -> {
                try {
                    Runnable t = queue.take();
                    t.run();
                } catch (InterruptedException e) {}
            });
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
    
    public void shutdown() {
        executor.shutdown();
    }
}
```

## 九、小结

1. **wait/notify**：Object 类的方法，必须在 synchronized 中使用
2. **BlockingQueue**：线程安全的队列，自动阻塞
3. **Condition**：Lock 的条件变量，更灵活
4. **Semaphore**：控制并发数量
5. **CountDownLatch**：等待多个任务完成
6. **CyclicBarrier**：等待多个线程到达屏障
