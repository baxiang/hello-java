# 6.5 并发编程进阶

## 一、Callable 和 Future

### 1.1 Callable 接口

```java
import java.util.concurrent.*;

// Callable 有返回值
class MyCallable implements Callable<Integer> {
    @Override
    public Integer call() throws Exception {
        int sum = 0;
        for (int i = 1; i <= 100; i++) {
            sum += i;
        }
        return sum;
    }
}

// 使用
ExecutorService executor = Executors.newFixedThreadPool(5);
Future<Integer> future = executor.submit(new MyCallable());

// 获取返回值（阻塞）
Integer result = future.get();
System.out.println("结果：" + result);  // 5050

executor.shutdown();
```

### 1.2 Future 方法

```java
Future<Integer> future = executor.submit(() -> {
    Thread.sleep(2000);
    return 100;
});

// 是否完成
boolean done = future.isDone();

// 取消任务
boolean cancelled = future.cancel(true);  // true=允许中断

// 是否取消
boolean isCancelled = future.isCancelled();

// 获取结果（带超时）
Integer result = future.get(1, TimeUnit.SECONDS);
```

## 二、CompletableFuture（Java 8+）

### 2.1 创建 CompletableFuture

```java
// 异步执行
CompletableFuture<Integer> future = CompletableFuture.supplyAsync(() -> {
    return 100;
});

// 获取结果
future.thenAccept(result -> {
    System.out.println("结果：" + result);
});
```

### 2.2 链式调用

```java
CompletableFuture.supplyAsync(() -> {
    return "Hello";
})
.thenApply(s -> s + " World")  // 转换结果
.thenApply(String::toUpperCase)  // 继续转换
.thenAccept(System.out::println);  // 消费结果
// 输出：HELLO WORLD
```

### 2.3 组合多个 Future

```java
CompletableFuture<Integer> future1 = CompletableFuture.supplyAsync(() -> 10);
CompletableFuture<Integer> future2 = CompletableFuture.supplyAsync(() -> 20);

// 等待两个都完成
CompletableFuture<Void> combined = CompletableFuture.allOf(future1, future2);

// 获取结果
combined.thenAccept(v -> {
    Integer result1 = future1.join();
    Integer result2 = future2.join();
    System.out.println(result1 + result2);  // 30
});
```

### 2.4 异常处理

```java
CompletableFuture.supplyAsync(() -> {
    if (true) throw new RuntimeException("错误");
    return 100;
})
.exceptionally(ex -> {
    System.out.println("异常：" + ex.getMessage());
    return 0;  // 返回默认值
})
.thenAccept(result -> {
    System.out.println("结果：" + result);  // 0
});
```

### 2.5 实际应用

```java
// 并行查询用户信息和订单信息
CompletableFuture<User> userFuture = CompletableFuture.supplyAsync(() -> 
    userService.getUser(userId)
);

CompletableFuture<List<Order>> orderFuture = CompletableFuture.supplyAsync(() -> 
    orderService.getOrders(userId)
);

// 组合结果
CompletableFuture<UserProfile> profileFuture = userFuture.thenCombine(orderFuture, 
    (user, orders) -> new UserProfile(user, orders)
);

// 处理结果
profileFuture.thenAccept(profile -> {
    System.out.println("用户：" + profile.getUser().getName());
    System.out.println("订单数：" + profile.getOrders().size());
});
```

## 三、CountDownLatch、CyclicBarrier、Semaphore

### 3.1 CountDownLatch 回顾

```java
// 等待 3 个任务完成
CountDownLatch latch = new CountDownLatch(3);

for (int i = 0; i < 3; i++) {
    new Thread(() -> {
        try {
            // 执行任务
            Thread.sleep(1000);
        } catch (InterruptedException e) {}
        finally {
            latch.countDown();
        }
    }).start();
}

latch.await();  // 等待
System.out.println("所有任务完成");
```

### 3.2 CyclicBarrier 回顾

```java
// 等待 3 个线程到达
CyclicBarrier barrier = new CyclicBarrier(3, () -> {
    System.out.println("所有线程到达，开始执行");
});

for (int i = 0; i < 3; i++) {
    new Thread(() -> {
        try {
            System.out.println("到达屏障");
            barrier.await();
            System.out.println("继续执行");
        } catch (Exception e) {}
    }).start();
}
```

### 3.3 Semaphore 回顾

```java
// 限制最多 3 个并发
Semaphore semaphore = new Semaphore(3);

for (int i = 0; i < 10; i++) {
    new Thread(() -> {
        try {
            semaphore.acquire();
            // 执行任务
            Thread.sleep(1000);
        } catch (InterruptedException e) {}
        finally {
            semaphore.release();
        }
    }).start();
}
```

## 四、ConcurrentHashMap

### 4.1 基本使用

```java
import java.util.concurrent.*;

ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();

// 基本操作
map.put("A", 1);
map.get("A");
map.remove("A");

// 原子操作
map.putIfAbsent("A", 1);  // 不存在才放入
map.replace("A", 1, 2);   // 只有当前值是 1 才替换
map.compute("A", (k, v) -> v == null ? 1 : v + 1);  // 计算新值
```

### 4.2 批量操作

```java
ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();
map.put("A", 1);
map.put("B", 2);
map.put("C", 3);

// 遍历
map.forEach(1, (k, v) -> System.out.println(k + "=" + v));

// 搜索
String result = map.search(1, (k, v) -> v > 1 ? k : null);

// 归约
Integer sum = map.reduce(1, (k, v) -> v, Integer::sum);
```

### 4.3 与 HashMap 对比

| 特性 | HashMap | ConcurrentHashMap |
|------|---------|-------------------|
| 线程安全 | 否 | 是 |
| 锁粒度 | - | 分段锁/CAS |
| null 键值 | 允许 | 不允许 |
| 性能 | 高 | 较高 |

## 五、ThreadLocal

### 5.1 基本使用

```java
// 线程本地变量
ThreadLocal<String> threadLocal = new ThreadLocal<>();

// 设置值
threadLocal.set("线程 A 的值");

// 获取值
String value = threadLocal.get();

// 删除值
threadLocal.remove();
```

### 5.2 实际应用

```java
// 用户上下文
public class UserContext {
    private static final ThreadLocal<User> userHolder = new ThreadLocal<>();
    
    public static void setUser(User user) {
        userHolder.set(user);
    }
    
    public static User getUser() {
        return userHolder.get();
    }
    
    public static void clear() {
        userHolder.remove();
    }
}

// 使用
public class UserController {
    public void handleRequest() {
        try {
            User user = login();
            UserContext.setUser(user);
            // 业务逻辑
        } finally {
            UserContext.clear();  // 必须清理，防止内存泄漏
        }
    }
}
```

### 5.3 内存泄漏问题

```java
// ❌ 可能导致内存泄漏
ThreadLocal<User> userHolder = new ThreadLocal<>();
userHolder.set(user);  // 如果没有 remove，线程复用时会泄漏

// ✅ 正确做法
try {
    userHolder.set(user);
    // 业务逻辑
} finally {
    userHolder.remove();  // 必须清理
}
```

## 六、原子类

### 6.1 基本原子类

```java
import java.util.concurrent.atomic.*;

// 原子整数
AtomicInteger atomicInt = new AtomicInteger(0);
atomicInt.incrementAndGet();  // ++i
atomicInt.getAndIncrement();  // i++
atomicInt.addAndGet(5);       // i += 5

// 原子长整型
AtomicLong atomicLong = new AtomicLong(0);

// 原子布尔
AtomicBoolean atomicBool = new AtomicBoolean(false);
atomicBool.compareAndSet(false, true);
```

### 6.2 数组原子类

```java
// 原子数组
AtomicIntegerArray atomicArray = new AtomicIntegerArray(5);
atomicArray.set(0, 10);
atomicArray.get(0);
atomicArray.incrementAndGet(0);
```

### 6.3 引用原子类

```java
// 原子引用
AtomicReference<User> userRef = new AtomicReference<>();
userRef.set(new User("张三"));
userRef.compareAndSet(oldUser, newUser);

// 带标记的引用
AtomicStampedReference<User> stampedRef = new AtomicStampedReference<>(user, 0);
int[] stampHolder = new int[1];
User user = stampedRef.get(stampHolder);
int stamp = stampedRef.getStamp();
```

### 6.4 LongAdder（高并发）

// 高并发场景下比 AtomicInteger 性能好

## 七、综合示例

### 7.1 并行处理

```java
public class ParallelProcessor {
    
    public List<Result> processParallel(List<Data> dataList) throws Exception {
        // 分割数据
        int size = dataList.size();
        int chunkSize = (size + 3) / 4;  // 分成 4 份
        
        List<CompletableFuture<List<Result>>> futures = new ArrayList<>();
        
        for (int i = 0; i < 4; i++) {
            final int start = i * chunkSize;
            final int end = Math.min(start + chunkSize, size);
            
            if (start >= size) break;
            
            CompletableFuture<List<Result>> future = CompletableFuture.supplyAsync(() -> {
                List<Result> results = new ArrayList<>();
                for (int j = start; j < end; j++) {
                    results.add(process(dataList.get(j)));
                }
                return results;
            });
            
            futures.add(future);
        }
        
        // 合并结果
        return CompletableFuture.allOf(
                futures.toArray(new CompletableFuture[0])
            )
            .thenApply(v -> 
                futures.stream()
                    .flatMap(f -> f.join().stream())
                    .collect(Collectors.toList())
            )
            .get();
    }
    
    private Result process(Data data) {
        // 处理单个数据
        return new Result(data.getId(), "processed");
    }
}
```

### 7.2 限流器

```java
public class RateLimiter {
    private final Semaphore semaphore;
    
    public RateLimiter(int permits) {
        semaphore = new Semaphore(permits);
    }
    
    public void acquire() throws InterruptedException {
        semaphore.acquire();
    }
    
    public void release() {
        semaphore.release();
    }
    
    public <T> T execute(Supplier<T> supplier) throws Exception {
        acquire();
        try {
            return supplier.get();
        } finally {
            release();
        }
    }
}

// 使用
RateLimiter limiter = new RateLimiter(10);  // 限制 10 个并发

for (int i = 0; i < 100; i++) {
    new Thread(() -> {
        try {
            limiter.execute(() -> {
                // 业务逻辑
                return null;
            });
        } catch (Exception e) {}
    }).start();
}
```

### 7.3 缓存实现

```java
public class Cache<K, V> {
    private final ConcurrentHashMap<K, Future<V>> cache = new ConcurrentHashMap<>();
    private final ExecutorService executor = Executors.newCachedThreadPool();
    
    public V get(K key, Callable<V> loader) throws Exception {
        Future<V> future = cache.get(key);
        
        if (future == null) {
            FutureTask<V> futureTask = new FutureTask<>(loader);
            future = cache.putIfAbsent(key, futureTask);
            
            if (future == null) {
                future = futureTask;
                executor.execute(futureTask);
            }
        }
        
        try {
            return future.get();
        } catch (ExecutionException e) {
            cache.remove(key);
            throw e;
        }
    }
    
    public void invalidate(K key) {
        cache.remove(key);
    }
    
    public void clear() {
        cache.clear();
    }
}
```

## 八、小结

1. **Callable/Future**：有返回值的异步任务
2. **CompletableFuture**：链式异步编程
3. **并发工具**：CountDownLatch、CyclicBarrier、Semaphore
4. **ConcurrentHashMap**：线程安全的 HashMap
5. **ThreadLocal**：线程本地变量，注意内存泄漏
6. **原子类**：AtomicInteger、AtomicReference、LongAdder
