# 26.5 并发工具类：CountDownLatch、CyclicBarrier、Semaphore

## 一、先问一个问题

**Java 有哪些常用的并发工具类？**

### 生活例子：团队协作

```
CountDownLatch（倒计时门闩）
├── 等待所有任务完成
└── 例子：等所有人到齐才开会

CyclicBarrier（循环屏障）
├── 等待所有人到达屏障
└── 例子：等所有人到齐才出发

Semaphore（信号量）
├── 控制并发数量
└── 例子：停车场限流
```

## 二、CountDownLatch（倒计时门闩）

### 什么是 CountDownLatch？

**生活例子：等人到齐**

```
CountDownLatch latch = new CountDownLatch(5);

5 个人
├── 第 1 个人到：latch.countDown() → 剩余 4
├── 第 2 个人到：latch.countDown() → 剩余 3
├── ...
└── 第 5 个人到：latch.countDown() → 剩余 0
    └── 等待的线程继续执行
```

**特点**：
- 一次性使用
- 一个或多个线程等待
- 多个线程执行

### 基本使用

```java
import java.util.concurrent.*;

public class CountDownLatchDemo {
    
    public static void main(String[] args) throws InterruptedException {
        // 创建 CountDownLatch（等待 5 个任务）
        CountDownLatch latch = new CountDownLatch(5);
        
        // 启动 5 个工作线程
        for (int i = 0; i < 5; i++) {
            new Thread(() -> {
                System.out.println(Thread.currentThread().getName() + " 开始工作");
                
                // 模拟工作
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                
                // 完成工作，计数减 1
                latch.countDown();
                System.out.println(Thread.currentThread().getName() + " 完成工作");
            }).start();
        }
        
        // 主线程等待所有工作完成
        System.out.println("等待所有任务完成...");
        latch.await();  // 阻塞等待
        
        System.out.println("所有任务完成，主线程继续执行");
    }
}
```

### 常用方法

```java
CountDownLatch latch = new CountDownLatch(5);

// 等待（阻塞）
latch.await();

// 等待（带超时）
latch.await(10, TimeUnit.SECONDS);

// 计数减 1
latch.countDown();

// 获取当前计数
long count = latch.getCount();
```

### 实际应用场景

### 场景 1：启动服务

```java
public class ServiceStartup {
    
    public static void main(String[] args) throws InterruptedException {
        CountDownLatch latch = new CountDownLatch(3);
        
        // 启动数据库服务
        new Thread(() -> {
            System.out.println("启动数据库服务...");
            try { Thread.sleep(2000); } catch (InterruptedException e) {}
            System.out.println("数据库服务启动完成");
            latch.countDown();
        }).start();
        
        // 启动缓存服务
        new Thread(() -> {
            System.out.println("启动缓存服务...");
            try { Thread.sleep(1500); } catch (InterruptedException e) {}
            System.out.println("缓存服务启动完成");
            latch.countDown();
        }).start();
        
        // 启动消息服务
        new Thread(() -> {
            System.out.println("启动消息服务...");
            try { Thread.sleep(1000); } catch (InterruptedException e) {}
            System.out.println("消息服务启动完成");
            latch.countDown();
        }).start();
        
        // 等待所有服务启动
        latch.await();
        System.out.println("所有服务启动完成，系统就绪！");
    }
}
```

### 场景 2：性能测试

```java
public class PerformanceTest {
    
    public static void main(String[] args) throws InterruptedException {
        int threadCount = 10;
        CountDownLatch startLatch = new CountDownLatch(1);
        CountDownLatch endLatch = new CountDownLatch(threadCount);
        
        // 启动 10 个线程
        for (int i = 0; i < threadCount; i++) {
            new Thread(() -> {
                try {
                    startLatch.await();  // 等待开始信号
                    
                    // 执行测试任务
                    long start = System.currentTimeMillis();
                    doWork();
                    long end = System.currentTimeMillis();
                    
                    System.out.println("任务完成，耗时：" + (end - start) + "ms");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } finally {
                    endLatch.countDown();
                }
            }).start();
        }
        
        Thread.sleep(1000);  // 确保所有线程准备就绪
        
        // 同时开始
        long testStart = System.currentTimeMillis();
        startLatch.countDown();
        
        // 等待所有任务完成
        endLatch.await();
        long testEnd = System.currentTimeMillis();
        
        System.out.println("总耗时：" + (testEnd - testStart) + "ms");
    }
    
    private static void doWork() {
        // 模拟工作
        try { Thread.sleep(100); } catch (InterruptedException e) {}
    }
}
```

## 三、CyclicBarrier（循环屏障）

### 什么是 CyclicBarrier？

**生活例子：等人集合**

```
CyclicBarrier barrier = new CyclicBarrier(5);

5 个人
├── 第 1 个人到：barrier.await() → 等待
├── 第 2 个人到：barrier.await() → 等待
├── ...
└── 第 5 个人到：barrier.await() → 所有人继续
    └── 屏障可重复使用
```

**特点**：
- 可重复使用
- 多个线程互相等待
- 到达屏障后一起继续

### 基本使用

```java
import java.util.concurrent.*;

public class CyclicBarrierDemo {
    
    public static void main(String[] args) {
        int threadCount = 5;
        
        // 创建 CyclicBarrier（5 个线程）
        CyclicBarrier barrier = new CyclicBarrier(threadCount, () -> {
            System.out.println("所有人都到齐了，开始出发！");
        });
        
        // 启动 5 个线程
        for (int i = 0; i < threadCount; i++) {
            new Thread(() -> {
                System.out.println(Thread.currentThread().getName() + " 出发");
                
                // 模拟走路
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                
                System.out.println(Thread.currentThread().getName() + " 到达集合点");
                
                try {
                    // 等待其他人
                    barrier.await();
                } catch (Exception e) {
                    e.printStackTrace();
                }
                
                System.out.println(Thread.currentThread().getName() + " 继续前进");
            }).start();
        }
    }
}
```

### 常用方法

```java
CyclicBarrier barrier = new CyclicBarrier(5);

// 等待
barrier.await();

// 等待（带超时）
barrier.await(10, TimeUnit.SECONDS);

// 获取等待的线程数
int parties = barrier.getParties();
int waiting = barrier.getNumberWaiting();

// 重置屏障
barrier.reset();

// 检查是否损坏
boolean broken = barrier.isBroken();
```

### CyclicBarrier vs CountDownLatch

| 特性 | CountDownLatch | CyclicBarrier |
|------|----------------|---------------|
| 可重复使用 | ❌ | ✅ |
| 等待方向 | 一个等多个 | 多个互相等 |
| 构造参数 | 计数 | 线程数 + 屏障动作 |
| 适用场景 | 等待任务完成 | 等待集合 |

## 四、Semaphore（信号量）

### 什么是 Semaphore？

**生活例子：停车场**

```
Semaphore semaphore = new Semaphore(10);  // 10 个车位

10 辆车
├── 第 1 辆车：semaphore.acquire() → 剩余 9
├── 第 2 辆车：semaphore.acquire() → 剩余 8
├── ...
└── 第 10 辆车：semaphore.acquire() → 剩余 0
    └── 第 11 辆车：等待
        └── 有车离开：semaphore.release() → 剩余 1
            └── 第 11 辆车进入
```

**特点**：
- 控制并发数量
- 可获取多个许可
- 可释放许可

### 基本使用

```java
import java.util.concurrent.*;

public class SemaphoreDemo {
    
    public static void main(String[] args) {
        // 创建信号量（3 个许可）
        Semaphore semaphore = new Semaphore(3);
        
        // 启动 10 个线程
        for (int i = 0; i < 10; i++) {
            new Thread(() -> {
                try {
                    // 获取许可
                    semaphore.acquire();
                    
                    System.out.println(Thread.currentThread().getName() + " 开始执行");
                    
                    // 模拟工作
                    Thread.sleep(2000);
                    
                    System.out.println(Thread.currentThread().getName() + " 完成执行");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } finally {
                    // 释放许可
                    semaphore.release();
                }
            }).start();
        }
    }
}
```

### 常用方法

```java
Semaphore semaphore = new Semaphore(10);

// 获取许可
semaphore.acquire();

// 获取多个许可
semaphore.acquire(3);

// 尝试获取许可（不阻塞）
if (semaphore.tryAcquire()) {
    // 获取成功
}

// 尝试获取许可（带超时）
if (semaphore.tryAcquire(1, TimeUnit.SECONDS)) {
    // 获取成功
}

// 释放许可
semaphore.release();

// 释放多个许可
semaphore.release(3);

// 获取可用许可数
int available = semaphore.availablePermits();

// 获取等待的线程数
int waiting = semaphore.getQueueLength();
```

### 公平锁 vs 非公平锁

```java
// 非公平锁（默认）
Semaphore semaphore = new Semaphore(10);

// 公平锁
Semaphore fairSemaphore = new Semaphore(10, true);

// 公平锁特点：
// - 按等待顺序获取许可
// - 避免线程饥饿
// - 性能略差
```

### 实际应用场景

### 场景 1：数据库连接池

```java
public class DatabaseConnectionPool {
    
    private final Semaphore semaphore;
    private final List<Connection> connections = new ArrayList<>();
    
    public DatabaseConnectionPool(int maxConnections) {
        this.semaphore = new Semaphore(maxConnections);
        
        // 初始化连接
        for (int i = 0; i < maxConnections; i++) {
            connections.add(createConnection());
        }
    }
    
    public Connection getConnection() throws InterruptedException {
        // 获取许可
        semaphore.acquire();
        
        // 获取连接
        synchronized (connections) {
            return connections.remove(0);
        }
    }
    
    public void returnConnection(Connection conn) {
        // 归还连接
        synchronized (connections) {
            connections.add(conn);
        }
        
        // 释放许可
        semaphore.release();
    }
    
    private Connection createConnection() {
        // 创建数据库连接
        return null;
    }
}
```

### 场景 2：限流器

```java
public class RateLimiter {
    
    private final Semaphore semaphore;
    
    public RateLimiter(int maxConcurrent) {
        this.semaphore = new Semaphore(maxConcurrent);
    }
    
    public boolean tryAcquire() {
        return semaphore.tryAcquire();
    }
    
    public void release() {
        semaphore.release();
    }
    
    public void execute(Runnable task) {
        if (tryAcquire()) {
            try {
                task.run();
            } finally {
                release();
            }
        } else {
            System.out.println("请求被限流");
        }
    }
}

// 使用
RateLimiter limiter = new RateLimiter(100);  // 最多 100 并发

limiter.execute(() -> {
    // 执行业务逻辑
});
```

## 五、其他并发工具

### Exchanger（交换器）

```java
Exchanger<String> exchanger = new Exchanger<>();

// 线程 A
new Thread(() -> {
    try {
        String data = "数据 A";
        String received = exchanger.exchange(data);
        System.out.println("线程 A 收到：" + received);
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
}).start();

// 线程 B
new Thread(() -> {
    try {
        String data = "数据 B";
        String received = exchanger.exchange(data);
        System.out.println("线程 B 收到：" + received);
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
}).start();
```

### Phaser（阶段器）

```java
Phaser phaser = new Phaser(3);  // 3 个线程

for (int i = 0; i < 3; i++) {
    new Thread(() -> {
        // 阶段 1
        doWork1();
        phaser.arriveAndAwaitAdvance();
        
        // 阶段 2
        doWork2();
        phaser.arriveAndAwaitAdvance();
        
        // 阶段 3
        doWork3();
        phaser.arriveAndDeregister();
    }).start();
}
```

## 六、小结

| 工具类 | 作用 | 可重复 | 适用场景 |
|--------|------|--------|----------|
| CountDownLatch | 等待任务完成 | ❌ | 等待所有任务完成 |
| CyclicBarrier | 等待集合 | ✅ | 等待所有线程到达 |
| Semaphore | 控制并发 | ✅ | 限流、资源池 |
| Exchanger | 数据交换 | ✅ | 线程间数据交换 |
| Phaser | 阶段同步 | ✅ | 多阶段任务 |

**核心要点**：
- CountDownLatch：一个等多个
- CyclicBarrier：多个互相等
- Semaphore：控制并发数

## 七、常见问题

### Q1：CountDownLatch 和 CyclicBarrier 有什么区别？

```java
// CountDownLatch
// - 一次性使用
// - 一个线程等待多个线程
CountDownLatch latch = new CountDownLatch(5);

// CyclicBarrier
// - 可重复使用
// - 多个线程互相等待
CyclicBarrier barrier = new CyclicBarrier(5);
```

### Q2：Semaphore 如何避免死锁？

```java
// ✅ 推荐：在 finally 中释放
semaphore.acquire();
try {
    // 执行任务
} finally {
    semaphore.release();
}

// ❌ 不推荐：不释放
semaphore.acquire();
// 执行任务
// 忘记 release → 死锁
```

### Q3：如何选择并发工具？

```
等待任务完成 → CountDownLatch
等待集合出发 → CyclicBarrier
控制并发数量 → Semaphore
线程数据交换 → Exchanger
多阶段同步 → Phaser
```
