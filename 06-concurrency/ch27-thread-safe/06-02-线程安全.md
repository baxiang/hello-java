# 26.2 线程安全：synchronized、Lock、volatile

## 一、先问一个问题

**什么是线程安全？为什么需要线程安全？**

### 生活例子：银行取款

```
不安全的情况
├── 账户余额：1000 元
├── ATM A：取款 800 元（读取余额 1000）
├── ATM B：取款 500 元（读取余额 1000）
├── ATM A：更新余额 200 元
├── ATM B：更新余额 500 元
└── 结果：余额 500 元（错误！应该是 -300 元或拒绝）

安全的情况
├── 账户余额：1000 元
├── ATM A：取款 800 元（锁定账户）
├── ATM B：等待...
├── ATM A：更新余额 200 元（解锁）
├── ATM B：取款 500 元（锁定账户）
└── 结果：余额不足，拒绝交易
```

**线程安全问题**：
- 多个线程同时修改共享数据
- 导致数据不一致
- 需要加锁保护

## 二、synchronized 关键字

### 什么是 synchronized？

**生活例子：厕所门锁**

```
单人间厕所
├── 有人使用时锁门
├── 其他人等待
└── 使用后开门
```

**synchronized**：
- Java 内置的锁机制
- 自动加锁和解锁
- 保证同一时间只有一个线程执行

### 用法一：同步实例方法

```java
public class Counter {
    private int count = 0;
    
    // 同步实例方法（锁 this）
    public synchronized void increment() {
        count++;
    }
    
    public synchronized int getCount() {
        return count;
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        Counter counter = new Counter();
        
        // 多个线程安全访问
        Thread t1 = new Thread(() -> {
            for (int i = 0; i < 1000; i++) {
                counter.increment();
            }
        });
        
        Thread t2 = new Thread(() -> {
            for (int i = 0; i < 1000; i++) {
                counter.increment();
            }
        });
        
        t1.start();
        t2.start();
        
        try {
            t1.join();
            t2.join();
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        
        System.out.println("最终计数：" + counter.getCount());  // 2000
    }
}
```

**锁对象**：当前实例（this）

### 用法二：同步静态方法

```java
public class StaticCounter {
    private static int count = 0;
    
    // 同步静态方法（锁 Class 对象）
    public static synchronized void increment() {
        count++;
    }
    
    public static synchronized int getCount() {
        return count;
    }
}
```

**锁对象**：类的 Class 对象（StaticCounter.class）

### 用法三：同步代码块

```java
public class BlockCounter {
    private int count = 0;
    private final Object lock = new Object();
    
    public void increment() {
        // 同步代码块（锁 lock 对象）
        synchronized (lock) {
            count++;
        }
    }
    
    // 也可以使用 this 作为锁
    public void increment2() {
        synchronized (this) {
            count++;
        }
    }
    
    // 使用 Class 对象作为锁
    public static void staticIncrement() {
        synchronized (BlockCounter.class) {
            // ...
        }
    }
}
```

**锁对象**：可以是任意对象

### synchronized 原理

```
对象头（Object Header）
├── Mark Word
│   ├── 锁标志位
│   ├── 偏向锁标记
│   └── 哈希码/锁指针
└── 类型指针

锁升级过程
无锁 → 偏向锁 → 轻量级锁 → 重量级锁
```

**锁升级**：
1. **无锁**：没有线程竞争
2. **偏向锁**：只有一个线程访问
3. **轻量级锁**：有竞争但线程交替执行
4. **重量级锁**：激烈竞争，阻塞线程

## 三、Lock 接口

### 什么是 Lock？

**生活例子：高级门锁**

```
synchronized（普通锁）
├── 自动上锁/解锁
└── 功能有限

Lock（高级锁）
├── 手动上锁/解锁
├── 可中断
├── 可超时
└── 可尝试获取
```

### ReentrantLock 使用

```java
import java.util.concurrent.locks.*;

public class LockCounter {
    private int count = 0;
    private final Lock lock = new ReentrantLock();
    
    public void increment() {
        // 手动加锁
        lock.lock();
        try {
            count++;
        } finally {
            // 必须手动解锁（通常在 finally 中）
            lock.unlock();
        }
    }
    
    public int getCount() {
        lock.lock();
        try {
            return count;
        } finally {
            lock.unlock();
        }
    }
}
```

**注意**：必须在 finally 中解锁，否则可能死锁。

### Lock vs synchronized

| 特性 | synchronized | Lock |
|------|--------------|------|
| 实现 | JVM 层面 | API 层面 |
| 加锁/解锁 | 自动 | 手动 |
| 可中断 | ❌ | ✅ |
| 可超时 | ❌ | ✅ |
| 公平锁 | ❌ | ✅ |
| 条件变量 | 一个（wait/notify） | 多个（Condition） |

### Lock 高级功能

```java
public class AdvancedLock {
    private final Lock lock = new ReentrantLock();
    
    // 尝试获取锁（不阻塞）
    public void tryLock() {
        if (lock.tryLock()) {
            try {
                // 执行任务
            } finally {
                lock.unlock();
            }
        } else {
            System.out.println("获取锁失败");
        }
    }
    
    // 尝试获取锁（带超时）
    public void tryLockWithTimeout() {
        try {
            if (lock.tryLock(1, TimeUnit.SECONDS)) {
                try {
                    // 执行任务
                } finally {
                    lock.unlock();
                }
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
    
    // 可中断的锁
    public void interruptibleLock() {
        try {
            lock.lockInterruptibly();
            try {
                // 执行任务
            } finally {
                lock.unlock();
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
}
```

### 公平锁 vs 非公平锁

```java
// 非公平锁（默认）
Lock lock = new ReentrantLock();

// 公平锁
Lock fairLock = new ReentrantLock(true);

// 公平锁特点：
// - 按等待顺序获取锁
// - 避免线程饥饿
// - 性能略差
```

## 四、volatile 关键字

### 什么是 volatile？

**生活例子：公告板**

```
没有 volatile
├── 线程 A 修改数据（只改自己缓存）
├── 线程 B 读取数据（读到旧值）
└── 数据不一致

有 volatile
├── 线程 A 修改数据（立即写入主内存）
├── 线程 B 读取数据（从主内存读取）
└── 数据一致
```

**volatile 作用**：
- 保证可见性（一个线程修改，其他线程立即可见）
- 禁止指令重排序
- 不保证原子性

### volatile 使用

```java
public class VolatileExample {
    // volatile 变量
    private volatile boolean flag = false;
    
    // 线程 A
    public void changeFlag() {
        flag = true;  // 修改立即对其他线程可见
    }
    
    // 线程 B
    public void checkFlag() {
        while (!flag) {
            // 等待 flag 变为 true
        }
        System.out.println("flag 已改变");
    }
}
```

### volatile vs synchronized

| 特性 | volatile | synchronized |
|------|----------|--------------|
| 可见性 | ✅ | ✅ |
| 原子性 | ❌ | ✅ |
| 有序性 | ✅ | ✅ |
| 阻塞 | ❌ | ✅ |
| 性能 | 高 | 低 |

### volatile 适用场景

```java
// ✅ 适合：状态标志
private volatile boolean running = true;

public void stop() {
    running = false;
}

public void run() {
    while (running) {
        // 执行任务
    }
}

// ✅ 适合：单例模式的双重检查
public class Singleton {
    private static volatile Singleton instance;
    
    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}

// ❌ 不适合：计数器（需要原子性）
private volatile int count = 0;

public void increment() {
    count++;  // 不安全！count++ 不是原子操作
}
```

## 五、CAS（Compare And Swap）

### 什么是 CAS？

**生活例子：自动售货机**

```
CAS 操作
├── 比较：当前值是否等于预期值
├── 交换：如果是，更新为新值
└── 失败：重试或返回
```

**CAS 特点**：
- 无锁并发
- 硬件级别支持
- 可能有 ABA 问题

### AtomicInteger 使用

```java
import java.util.concurrent.atomic.*;

public class AtomicCounter {
    // 原子整数
    private AtomicInteger count = new AtomicInteger(0);
    
    public void increment() {
        // CAS 操作，原子性
        count.incrementAndGet();
        // 或 count.getAndIncrement();
    }
    
    public int getCount() {
        return count.get();
    }
    
    // CAS 操作
    public boolean compareAndSet(int expected, int newValue) {
        return count.compareAndSet(expected, newValue);
    }
}
```

### CAS vs synchronized

| 特性 | CAS | synchronized |
|------|-----|--------------|
| 实现 | 硬件指令 | JVM 锁 |
| 阻塞 | ❌ | ✅ |
| 性能（低竞争） | 高 | 低 |
| 性能（高竞争） | 低 | 中 |
| ABA 问题 | ✅ | ❌ |

## 六、死锁

### 什么是死锁？

**生活例子：互相等待**

```
线程 A
├── 持有资源 1
└── 等待资源 2

线程 B
├── 持有资源 2
└── 等待资源 1

结果：互相等待，永远无法继续
```

### 死锁示例

```java
public class DeadlockDemo {
    private final Object lock1 = new Object();
    private final Object lock2 = new Object();
    
    public void method1() {
        synchronized (lock1) {
            System.out.println("方法 1：持有 lock1");
            try { Thread.sleep(100); } catch (InterruptedException e) {}
            
            synchronized (lock2) {
                System.out.println("方法 1：持有 lock2");
            }
        }
    }
    
    public void method2() {
        synchronized (lock2) {
            System.out.println("方法 2：持有 lock2");
            try { Thread.sleep(100); } catch (InterruptedException e) {}
            
            synchronized (lock1) {
                System.out.println("方法 2：持有 lock1");
            }
        }
    }
    
    public static void main(String[] args) {
        DeadlockDemo demo = new DeadlockDemo();
        
        new Thread(demo::method1).start();
        new Thread(demo::method2).start();
        
        // 结果：死锁
    }
}
```

### 避免死锁

```java
// 方法 1：按顺序获取锁
public void safeMethod1() {
    synchronized (lock1) {
        synchronized (lock2) {
            // ...
        }
    }
}

public void safeMethod2() {
    synchronized (lock1) {  // 也先获取 lock1
        synchronized (lock2) {
            // ...
        }
    }
}

// 方法 2：使用 tryLock
public void tryLockMethod() {
    if (lock1.tryLock()) {
        try {
            if (lock2.tryLock()) {
                try {
                    // ...
                } finally {
                    lock2.unlock();
                }
            }
        } finally {
            lock1.unlock();
        }
    }
}
```

## 七、小结

| 机制 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| synchronized | 简单同步 | 简单、自动 | 功能有限 |
| Lock | 复杂同步 | 灵活、可中断 | 需手动解锁 |
| volatile | 状态标志 | 轻量、可见性 | 不保证原子性 |
| CAS | 计数器 | 无锁、高性能 | ABA 问题 |

**核心要点**：
- synchronized：简单场景首选
- Lock：需要高级功能
- volatile：状态标志
- CAS：原子操作

## 八、常见问题

### Q1：synchronized 锁的是什么？

```java
// 实例方法：锁 this
public synchronized void method() { }

// 静态方法：锁 Class 对象
public static synchronized void method() { }

// 代码块：锁指定对象
synchronized (lockObject) { }
```

### Q2：Lock 为什么要在 finally 中解锁？

```java
lock.lock();
try {
    // 执行任务
} finally {
    lock.unlock();  // 确保解锁
}

// 如果不解锁：
// 1. 其他线程无法获取锁
// 2. 可能导致死锁
```

### Q3：volatile 能保证原子性吗？

```java
// ❌ 不能
private volatile int count = 0;
count++;  // 不是原子操作（读取 - 修改 - 写入）

// ✅ 使用 AtomicInteger
private AtomicInteger count = new AtomicInteger(0);
count.incrementAndGet();  // 原子操作
```
