# 6.2 线程安全

## 一、线程安全问题

### 1.1 什么是线程安全

当多个线程同时访问共享数据时，如果程序的行为始终正确，则称该程序是线程安全的。

### 1.2 线程安全问题示例

```java
// ❌ 线程不安全
class TicketWindow {
    private int tickets = 100;
    
    public void sell() {
        if (tickets > 0) {
            try { Thread.sleep(10); } catch (Exception e) {}
            tickets--;
            System.out.println(Thread.currentThread().getName() + " 卖出第 " + tickets + " 张票");
        }
    }
}

// 测试
TicketWindow window = new TicketWindow();
new Thread(() -> {
    for (int i = 0; i < 50; i++) window.sell();
}, "窗口 1").start();

new Thread(() -> {
    for (int i = 0; i < 50; i++) window.sell();
}, "窗口 2").start();

// 问题：可能卖出负数票、重复票
```

## 二、synchronized 关键字

### 2.1 synchronized 方法

```java
// ✅ 线程安全
class TicketWindow {
    private int tickets = 100;
    
    public synchronized void sell() {
        if (tickets > 0) {
            try { Thread.sleep(10); } catch (Exception e) {}
            tickets--;
            System.out.println(Thread.currentThread().getName() + " 卖出第 " + tickets + " 张票");
        }
    }
}
```

### 2.2 synchronized 代码块

```java
class TicketWindow {
    private int tickets = 100;
    private final Object lock = new Object();
    
    public void sell() {
        synchronized (lock) {
            if (tickets > 0) {
                try { Thread.sleep(10); } catch (Exception e) {}
                tickets--;
                System.out.println(Thread.currentThread().getName() + " 卖出第 " + tickets + " 张票");
            }
        }
    }
}
```

### 2.3 锁对象

```java
// 同步方法锁的是 this
public synchronized void method() {}
// 等价于
synchronized (this) { }

// 静态同步方法锁的是 Class 对象
public static synchronized void staticMethod() {}
// 等价于
synchronized (MyClass.class) { }
```

## 三、Lock 接口

### 3.1 ReentrantLock

```java
import java.util.concurrent.locks.Lock;
import java.util.concurrent.locks.ReentrantLock;

class TicketWindow {
    private int tickets = 100;
    private final Lock lock = new ReentrantLock();
    
    public void sell() {
        lock.lock();  // 加锁
        try {
            if (tickets > 0) {
                try { Thread.sleep(10); } catch (Exception e) {}
                tickets--;
                System.out.println(Thread.currentThread().getName() + " 卖出第 " + tickets + " 张票");
            }
        } finally {
            lock.unlock();  // 解锁（必须在 finally 中）
        }
    }
}
```

### 3.2 尝试获取锁

```java
Lock lock = new ReentrantLock();

// 尝试获取锁
if (lock.tryLock()) {
    try {
        // 执行临界区代码
    } finally {
        lock.unlock();
    }
}

// 带超时的尝试
if (lock.tryLock(100, TimeUnit.MILLISECONDS)) {
    try {
        // 执行临界区代码
    } finally {
        lock.unlock();
    }
}
```

### 3.3 synchronized vs Lock

| 特性 | synchronized | Lock |
|------|--------------|------|
| 实现层面 | JVM 层面 | API 层面 |
| 锁释放 | 自动释放 | 手动释放 |
| 尝试获取 | 不支持 | 支持 tryLock |
| 可中断 | 不支持 | 支持 |
| 公平锁 | 不支持 | 支持 |
| 条件变量 | 单一 | 多个 Condition |

## 四、volatile 关键字

### 4.1 可见性

```java
// ❌ 线程不可见
class MyThread implements Runnable {
    private boolean running = true;
    
    public void run() {
        while (running) {
            // 可能永远无法停止
        }
    }
    
    public void stop() {
        running = false;
    }
}

// ✅ 使用 volatile 保证可见性
class MyThread implements Runnable {
    private volatile boolean running = true;
    
    public void run() {
        while (running) {
            // 能及时看到 running 的变化
        }
    }
    
    public void stop() {
        running = false;
    }
}
```

### 4.2 不保证原子性

```java
// ❌ volatile 不保证原子性
class Counter {
    private volatile int count = 0;
    
    public void increment() {
        count++;  // 不是原子操作（读 - 改 - 写）
    }
}

// ✅ 使用 synchronized 或 AtomicInteger
class Counter {
    private int count = 0;
    
    public synchronized void increment() {
        count++;
    }
}
```

### 4.3 使用场景

- 状态标记（如 running、stopped）
- 双重检查锁定（DCL）单例
- 不依赖原子性的场景

## 五、线程安全的集合

### 5.1 同步包装

```java
// 使用 Collections 工具类
List<String> list = Collections.synchronizedList(new ArrayList<>());
Map<String, Integer> map = Collections.synchronizedMap(new HashMap<>());
Set<String> set = Collections.synchronizedSet(new HashSet<>());

// 遍历时需要手动同步
synchronized (list) {
    for (String s : list) {
        System.out.println(s);
    }
}
```

### 5.2 并发集合

```java
import java.util.concurrent.*;

// ConcurrentHashMap
ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();

// CopyOnWriteArrayList（适合读多写少）
CopyOnWriteArrayList<String> list = new CopyOnWriteArrayList<>();

// ConcurrentLinkedQueue（无界队列）
ConcurrentLinkedQueue<String> queue = new ConcurrentLinkedQueue<>();

// BlockingQueue（阻塞队列）
BlockingQueue<String> blockingQueue = new LinkedBlockingQueue<>();
```

## 六、原子类

### 6.1 AtomicInteger

```java
import java.util.concurrent.atomic.AtomicInteger;

AtomicInteger count = new AtomicInteger(0);

// 原子自增
count.incrementAndGet();  // ++count
count.getAndIncrement();  // count++

// 原子加减
count.addAndGet(5);  // count += 5
count.getAndAdd(5);

// 原子比较交换（CAS）
boolean success = count.compareAndSet(0, 1);  // 如果当前值是 0，则设为 1
```

### 6.2 AtomicReference

```java
AtomicReference<String> ref = new AtomicReference<>("A");

// CAS 操作
boolean success = ref.compareAndSet("A", "B");
String value = ref.get();
```

### 6.3 LongAdder（高并发场景）

```java
import java.util.concurrent.atomic.LongAdder;

LongAdder adder = new LongAdder();

// 原子自增
adder.increment();
adder.add(5);

// 获取值
long sum = adder.sum();

// 性能比 AtomicInteger 好（分段累加）
```

## 七、综合示例

### 7.1 线程安全的单例

```java
// 双重检查锁定（DCL）
class Singleton {
    private static volatile Singleton instance;
    
    private Singleton() {}
    
    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}

// 静态内部类（推荐）
class Singleton {
    private Singleton() {}
    
    private static class Holder {
        private static final Singleton INSTANCE = new Singleton();
    }
    
    public static Singleton getInstance() {
        return Holder.INSTANCE;
    }
}

// 枚举（最简洁）
enum Singleton {
    INSTANCE;
    
    public void doSomething() {}
}
```

### 7.2 线程安全的计数器

```java
class SafeCounter {
    private final AtomicInteger count = new AtomicInteger(0);
    
    public void increment() {
        count.incrementAndGet();
    }
    
    public int getCount() {
        return count.get();
    }
}

// 多线程测试
SafeCounter counter = new SafeCounter();
List<Thread> threads = new ArrayList<>();

for (int i = 0; i < 10; i++) {
    Thread t = new Thread(() -> {
        for (int j = 0; j < 1000; j++) {
            counter.increment();
        }
    });
    threads.add(t);
    t.start();
}

for (Thread t : threads) {
    try { t.join(); } catch (Exception e) {}
}

System.out.println("最终计数：" + counter.getCount());  // 10000
```

### 7.3 生产者 - 消费者（线程安全）

```java
class SafeBuffer {
    private final Queue<Integer> queue = new LinkedList<>();
    private final int capacity;
    private final Lock lock = new ReentrantLock();
    private final Condition notFull = lock.newCondition();
    private final Condition notEmpty = lock.newCondition();
    
    public SafeBuffer(int capacity) {
        this.capacity = capacity;
    }
    
    public void produce(int item) throws InterruptedException {
        lock.lock();
        try {
            while (queue.size() >= capacity) {
                notFull.await();  // 等待不满
            }
            queue.offer(item);
            System.out.println("生产：" + item);
            notEmpty.signal();  // 通知不空
        } finally {
            lock.unlock();
        }
    }
    
    public int consume() throws InterruptedException {
        lock.lock();
        try {
            while (queue.isEmpty()) {
                notEmpty.await();  // 等待不空
            }
            int item = queue.poll();
            System.out.println("消费：" + item);
            notFull.signal();  // 通知不满
            return item;
        } finally {
            lock.unlock();
        }
    }
}
```

## 八、小结

1. **线程安全问题**：多个线程访问共享数据
2. **synchronized**：内置锁，自动释放
3. **Lock**：API 锁，手动释放，功能更强
4. **volatile**：保证可见性，不保证原子性
5. **线程安全集合**：同步包装、并发集合
6. **原子类**：AtomicInteger、LongAdder
