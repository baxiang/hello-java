# 3.5 枚举与注解

## 一、枚举（Enum）

### 1.1 什么是枚举

枚举是一种特殊的数据类型，用于定义一组固定的常量。

**使用场景**：
- 星期、月份、季节
- 订单状态、支付状态
- 用户角色、权限级别

### 1.2 定义枚举

```java
// 基本枚举
public enum Season {
    SPRING, SUMMER, AUTUMN, WINTER
}

// 使用
Season season = Season.SPRING;
```

### 1.3 枚举的常用方法

```java
public enum Season {
    SPRING, SUMMER, AUTUMN, WINTER
}

// 使用
Season season = Season.SPRING;

// name()：获取枚举名
String name = season.name();  // "SPRING"

// ordinal()：获取序号（从 0 开始）
int ordinal = season.ordinal();  // 0

// values()：获取所有枚举值
Season[] seasons = Season.values();

// valueOf()：字符串转枚举
Season s = Season.valueOf("SPRING");

// 比较
boolean isEqual = season == Season.SPRING;  // true
```

### 1.4 带属性和方法的枚举

```java
public enum Season {
    SPRING("春天", "春暖花开"),
    SUMMER("夏天", "烈日炎炎"),
    AUTUMN("秋天", "秋高气爽"),
    WINTER("冬天", "白雪皑皑");
    
    // 属性
    private final String chineseName;
    private final String description;
    
    // 构造方法（必须是 private）
    Season(String chineseName, String description) {
        this.chineseName = chineseName;
        this.description = description;
    }
    
    // Getter 方法
    public String getChineseName() {
        return chineseName;
    }
    
    public String getDescription() {
        return description;
    }
    
    // 普通方法
    public void printInfo() {
        System.out.println(chineseName + "：" + description);
    }
}

// 使用
Season season = Season.SPRING;
System.out.println(season.getChineseName());  // 春天
System.out.println(season.getDescription());  // 春暖花开
season.printInfo();  // 春天：春暖花开
```

### 1.5 枚举实现接口

```java
// 接口
interface Operatable {
    void operate();
}

// 枚举实现接口
public enum Role implements Operatable {
    ADMIN {
        @Override
        public void operate() {
            System.out.println("管理员操作");
        }
    },
    USER {
        @Override
        public void operate() {
            System.out.println("用户操作");
        }
    };
    
    @Override
    public abstract void operate();
}

// 使用
Role admin = Role.ADMIN;
admin.operate();  // 管理员操作
```

### 1.6 枚举与 switch

```java
public enum OrderStatus {
    PENDING, PAID, SHIPPED, COMPLETED, CANCELLED

public void processOrder(OrderStatus status) {
    switch (status) {
```

## 二、注解（Annotation）

### 2.1 什么是注解

注解是一种用于提供元数据的标记，可以添加到类、方法、字段等元素上。

**用途**：
- 编译时检查（@Override）
- 编译时处理（注解处理器）
- 运行时处理（反射）

### 2.2 内置注解

```java
// @Override：检查是否是重写
class Parent {
    public void show() {}
}

class Child extends Parent {
    @Override  // 如果方法签名不对，编译会报错
    public void show() {}
}

// @Deprecated：标记已过时
/**
 * @deprecated 请使用 newMethod()
 */
@Deprecated
public void oldMethod() {}

// @SuppressWarnings：抑制警告
@SuppressWarnings("unchecked")
public void method() {
    List list = new ArrayList();
}

// @FunctionalInterface：函数式接口
@FunctionalInterface
interface MyFunction {
    void apply();
}
```

### 2.3 元注解

元注解是用于注解其他注解的注解。

```java
// @Target：指定注解可以应用的位置
@Target(ElementType.METHOD)  // 只能用于方法
@Target({ElementType.TYPE, ElementType.METHOD})  // 可用于类和方法

// ElementType 常用值：
// TYPE - 类、接口、枚举
// FIELD - 字段
// METHOD - 方法
// PARAMETER - 参数
// CONSTRUCTOR - 构造方法

// @Retention：指定注解保留策略
@Retention(RetentionPolicy.RUNTIME)  // 运行时可见（反射可用）
@Retention(RetentionPolicy.CLASS)    // 编译时保留，运行时不可见
@Retention(RetentionPolicy.SOURCE)   // 源码中保留，编译时丢弃

// @Documented：是否包含在 JavaDoc 中
@Documented

// @Inherited：是否可被子类继承
@Inherited
```

### 2.4 自定义注解

```java
// 定义注解
@Target({ElementType.METHOD, ElementType.TYPE})
@Retention(RetentionPolicy.RUNTIME)
@Documented
public @interface MyAnnotation {
    // 属性
    String value() default "";
    String name();
    int age() default 0;
    String[] tags() default {};
}

// 使用注解
@MyAnnotation(name = "测试", age = 10, tags = {"a", "b"})
public class MyClass {
    
    @MyAnnotation(value = "hello", name = "method")
    public void myMethod() {
    }
}
```

### 2.5 运行时读取注解

```java
import java.lang.reflect.*;

// 定义注解
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
@interface Test {
    String value() default "";
}

// 使用注解
public class TestClass {
    @Test("test1")
    public void method1() {}
    
    @Test("test2")
    public void method2() {}
    
    public void method3() {}  // 没有注解
}

// 读取注解
public class AnnotationReader {
    public static void main(String[] args) throws Exception {
        Class<?> clazz = TestClass.class;
        
        // 获取所有方法
        Method[] methods = clazz.getDeclaredMethods();
        
        for (Method method : methods) {
            // 检查是否有注解
            if (method.isAnnotationPresent(Test.class)) {
                // 获取注解
                Test test = method.getAnnotation(Test.class);
                System.out.println("方法：" + method.getName());
                System.out.println("注解值：" + test.value());
            }
        }
    }
}
```

## 三、综合示例

### 3.1 订单状态枚举

```java
public enum OrderStatus {
    PENDING(0, "待付款"),
    PAID(1, "已付款"),
    SHIPPED(2, "已发货"),
    COMPLETED(3, "已完成"),
    CANCELLED(4, "已取消");
    
    private final int code;
    private final String description;
    
    OrderStatus(int code, String description) {
        this.code = code;
        this.description = description;
    }
    
    public int getCode() {
        return code;
    }
    
    public String getDescription() {
        return description;
    }
    
    // 根据 code 获取枚举
    public static OrderStatus fromCode(int code) {
        for (OrderStatus status : values()) {
            if (status.code == code) {
                return status;
            }
        }
        throw new IllegalArgumentException("无效的状态码：" + code);
    }
}

// 使用
OrderStatus status = OrderStatus.PENDING;
System.out.println(status.getDescription());  // 待付款
OrderStatus s2 = OrderStatus.fromCode(1);     // PAID
```

### 3.2 日志注解

```java
// 定义日志注解
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
@interface Log {
    String value() default "";
    String level() default "INFO";
}

// 使用注解
public class UserService {
    
    @Log(value = "创建用户", level = "INFO")
    public void createUser(String username) {
        System.out.println("创建用户：" + username);
    }
    
    @Log(value = "删除用户", level = "WARN")
    public void deleteUser(Long userId) {
        System.out.println("删除用户：" + userId);
    }
}

// 注解处理器（模拟 AOP）
public class LogProcessor {
    public static void process(Object obj) throws Exception {
        Class<?> clazz = obj.getClass();
        
        for (Method method : clazz.getDeclaredMethods()) {
            if (method.isAnnotationPresent(Log.class)) {
                Log log = method.getAnnotation(Log.class);
                
                System.out.println("=== 日志开始 ===");
                System.out.println("方法：" + method.getName());
                System.out.println("描述：" + log.value());
                System.out.println("级别：" + log.level());
                
                // 执行方法
                method.setAccessible(true);
                method.invoke(obj, "testUser");
                
                System.out.println("=== 日志结束 ===\n");
            }
        }
    }
}

// 测试
public class Main {
    public static void main(String[] args) throws Exception {
        LogProcessor.process(new UserService());
    }
}
```

### 3.3 权限枚举与注解

```java
// 权限枚举
public enum Permission {
    USER_READ, USER_WRITE, 
    ORDER_READ, ORDER_WRITE,
    ADMIN
}

// 权限注解
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
@Repeatable(Permissions.class)  // 支持重复注解
@interface RequirePermission {
    Permission value();
}

// 权限容器注解
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
@interface Permissions {
    RequirePermission[] value();
}

// 使用
public class OrderController {
    
    @RequirePermission(Permission.ORDER_READ)
    public Order getOrder(Long id) {
        return new Order();
    }
    
    @RequirePermission(Permission.ORDER_WRITE)
    @RequirePermission(Permission.USER_READ)
    public void createOrder(Order order) {
        // ...
    }
}
```

## 四、小结

1. **枚举**：定义固定常量，可带属性和方法
2. **内置注解**：@Override、@Deprecated、@SuppressWarnings
3. **元注解**：@Target、@Retention、@Documented、@Inherited
4. **自定义注解**：定义、使用、运行时读取
5. **应用**：状态管理、日志、权限控制
