# 31.4 Java 11 新特性：有哪些实用改进？

## 一、先问一个问题

**Java 11 有什么新特性？**

### 生活例子：手机系统升级

```
iOS 12 → iOS 13
├── 新功能
├── 性能优化
└── Bug 修复

Java 8 → Java 11
├── 新语法
├── 新 API
└── 性能提升
```

**Java 11**：
- 2018 年 9 月发布
- LTS（长期支持）版本
- 实用改进为主

## 二、局部变量类型推断（var）

### Java 10 引入，Java 11 改进

**之前**：
```java
// 重复啰嗦
Map<String, List<Integer>> map = new HashMap<String, List<Integer>>();
String name = "张三";
```

**现在**：
```java
// 简洁明了
var map = new HashMap<String, List<Integer>>();
var name = "张三";
```

### var 的使用场景

```java
// ✅ 好的使用
var list = new ArrayList<String>();
var map = new HashMap<String, Integer>();
var stream = list.stream();
var result = stream.filter(s -> s.length() > 2).toList();

// ❌ 坏的使用
var x = 10;  // 类型不明确
var obj = getObject();  // 不知道是什么类型
var data = processData();  // 不知道返回什么
```

### var 的限制

```java
// ❌ 不能使用 var 的情况
var nullVar = null;  // 错误：无法推断类型
var lambda = () -> {};  // 错误：Lambda 类型不明确
var methodRef = String::length;  // 错误：方法引用类型不明确

// ✅ 正确使用
String str = null;
Runnable lambda = () -> {};
ToIntFunction<String> methodRef = String::length;
```

## 三、字符串新方法

### 1. isBlank()

```java
String str1 = "   ";
String str2 = "";

// Java 8
if (str1.trim().isEmpty()) {  // true
    System.out.println("空白");
}

// Java 11
if (str1.isBlank()) {  // true
    System.out.println("空白");
}

if (str2.isEmpty()) {  // true
    System.out.println("空");
}
```

**区别**：
- `isEmpty()`：长度为 0
- `isBlank()`：长度为 0 或 只包含空白字符

### 2. lines()

```java
String text = "第一行\n第二行\n第三行";

// Java 11
text.lines().forEach(line -> {
    System.out.println("行：" + line);
});

// 输出：
// 行：第一行
// 行：第二行
// 行：第三行
```

### 3. strip()

```java
String str = "  Hello  ";

// Java 8
str.trim();  // "Hello"

// Java 11
str.strip();  // "Hello"（支持 Unicode 空白字符）
str.stripLeading();  // "Hello  "（去除前导空白）
str.stripTrailing();  // "  Hello"（去除尾部空白）
```

**strip() vs trim()**：
- `trim()`：只处理 ASCII 空白字符（\u0020, \n, \t 等）
- `strip()`：处理所有 Unicode 空白字符

### 4. repeat()

```java
// Java 11
String str = "Ha";
String repeated = str.repeat(3);  // "HaHaHa"

// 实用示例
String separator = "-".repeat(50);  // "--------------------------------------------------"
String indent = "    ".repeat(2);   // "        "（8 个空格）
```

## 四、集合转数组新方法

### toArray(IntFunction)

```java
List<String> list = Arrays.asList("A", "B", "C");

// Java 8
String[] array = list.toArray(new String[0]);

// Java 11（更简洁）
String[] array = list.toArray(String[]::new);
```

**优势**：
- 不需要创建空数组
- 性能更好
- 代码更简洁

## 五、HTTP Client（标准化）

### 什么是 HTTP Client？

**生活例子：快递员**

```
传统方式（HttpURLConnection）
├── 用法复杂
├── 代码啰嗦
└── 不支持异步

HTTP Client
├── 用法简单
├── 代码简洁
└── 支持异步
```

### 发送 GET 请求

```java
import java.net.http.*;
import java.net.URI;

// 创建客户端
var client = HttpClient.newHttpClient();

// 创建请求
var request = HttpRequest.newBuilder()
    .uri(URI.create("https://api.example.com/users"))
    .GET()
    .build();

// 发送请求（同步）
HttpResponse<String> response = client.send(
    request,
    HttpResponse.BodyHandlers.ofString()
);

// 获取响应
int statusCode = response.statusCode();
String body = response.body();

System.out.println("状态码：" + statusCode);
System.out.println("响应：" + body);
```

### 发送 POST 请求

```java
// 创建请求
var request = HttpRequest.newBuilder()
    .uri(URI.create("https://api.example.com/users"))
    .POST(HttpRequest.BodyPublishers.ofString("{\"name\":\"张三\"}"))
    .header("Content-Type", "application/json")
    .build();

// 发送请求
HttpResponse<String> response = client.send(
    request,
    HttpResponse.BodyHandlers.ofString()
);
```

### 异步请求

```java
// 异步发送（不阻塞）
client.sendAsync(request, HttpResponse.BodyHandlers.ofString())
    .thenApply(HttpResponse::body)
    .thenAccept(System.out::println)
    .join();  // 等待完成
```

### HTTP Client vs RestTemplate

| 特性 | HTTP Client | RestTemplate |
|------|-------------|--------------|
| Java 版本 | Java 11+ | Spring |
| 异步支持 | ✅ | ❌ |
| 响应式 | ✅ | ❌ |
| 依赖 | 无 | Spring |

## 六、其他改进

### 1. Optional 新增 isEmpty()

```java
Optional<String> opt = Optional.empty();

// Java 8
if (!opt.isPresent()) {  // 双重否定
    System.out.println("空");
}

// Java 11
if (opt.isEmpty()) {  // 更直观
    System.out.println("空");
}
```

### 2. Predicate 新增 not()

```java
List<String> list = Arrays.asList("A", "", "B", null, "C");

// Java 8
list.stream()
    .filter(s -> s != null && !s.isEmpty())
    .forEach(System.out::println);

// Java 11
list.stream()
    .filter(Predicate.not(Objects::isNull))
    .filter(Predicate.not(String::isEmpty))
    .forEach(System.out::println);
```

### 3. Files 新增方法

```java
// 读取字符串
String content = Files.readString(Path.of("file.txt"));

// 写入字符串
Files.writeString(Path.of("file.txt"), "Hello World");

// 判断是否相同
boolean same = Files.isSameFile(path1, path2);
```


## 八、小结

| 特性 | 说明 | 示例 |
|------|------|------|
| var | 局部变量类型推断 | var list = new ArrayList<>() |
| isBlank | 空白字符串判断 | str.isBlank() |
| lines | 行流 | str.lines() |
| strip | 去除空白 | str.strip() |
| repeat | 重复字符串 | str.repeat(3) |
| toArray | 集合转数组 | list.toArray(String[]::new) |
| HTTP Client | HTTP 客户端 | HttpClient.newHttpClient() |
| isEmpty | Optional 判断 | opt.isEmpty() |
| not | Predicate 取反 | Predicate.not() |

**核心要点**：
- var 简化代码
- 字符串新方法实用
- HTTP Client 标准化
- 集合 API 改进

## 九、常见问题

### Q1：var 会影响性能吗？

**不会**：
```java
// 编译后完全一样
var list = new ArrayList<String>();
ArrayList<String> list = new ArrayList<String>();
```

### Q2：什么时候用 var？

**推荐**：
```java
// ✅ 类型明显
var map = new HashMap<String, Integer>();
var stream = list.stream();

// ❌ 类型不明显
var result = processData();  // 不知道是什么
var x = 10;  // 用 int 更清晰
```

### Q3：HTTP Client 和 RestTemplate 选哪个？

**建议**：
- 新项目：HTTP Client（标准库）
- Spring 项目：RestTemplate/WebClient（集成好）
