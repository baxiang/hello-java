# 31.6 Java 21 新特性：虚拟线程和 Record 模式

## 一、先问一个问题

**Java 21 有什么重磅特性？**

### 生活例子：高速公路

```
传统线程（平台线程）
├── 每条车道一辆车
├── 车道有限（线程池）
└── 堵车（线程阻塞）

虚拟线程
├── 一辆车多条虚拟车道
├── 车道无限（按需创建）
└── 不堵车（阻塞时切换）
```

**Java 21**：
- 2023 年 9 月发布
- LTS（长期支持）版本
- 虚拟线程是重磅特性

## 二、虚拟线程（Virtual Threads）

### 什么是虚拟线程？

**传统线程（平台线程）**：
```java
// 一个线程对应一个操作系统线程
Thread thread = Thread.startVirtualThread(() -> {
    System.out.println("Hello");
});

// 问题：
// 1. 操作系统线程有限（约 1000-2000 个）
// 2. 创建开销大
// 3. 阻塞时浪费资源
```

**虚拟线程**：
```java
// 多个虚拟线程共享操作系统线程
Thread virtualThread = Thread.startVirtualThread(() -> {
    System.out.println("Hello from virtual thread");
});

// 优势：
// 1. 可以创建百万级虚拟线程
// 2. 创建开销小
// 3. 阻塞时自动切换
```

### 创建虚拟线程

**方式一：startVirtualThread**
```java
// 最简单的方式
Thread thread = Thread.startVirtualThread(() -> {
    System.out.println("执行任务");
});
```

**方式二：newThread + start**
```java
Thread virtualThread = Thread.ofVirtual()
    .name("worker", 0)  // 设置名称
    .start(() -> {
        System.out.println("执行任务");
    });
```

**方式三：ExecutorService（推荐）**
```java
// 使用线程池
try (ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor()) {
    for (int i = 0; i < 10000; i++) {
        executor.submit(() -> {
            // 处理任务
            doWork();
        });
    }
}
// 自动关闭
```

### 虚拟线程 vs 平台线程

| 特性 | 平台线程 | 虚拟线程 |
|------|----------|----------|
| 映射 | 1:1（一个 Java 线程=一个 OS 线程） | M:N（多个 Java 线程共享 OS 线程） |
| 数量 | 有限（约 1000-2000） | 几乎无限（百万级） |
| 创建开销 | 大 | 小 |
| 阻塞 | 阻塞 OS 线程 | 自动切换 |
| 适用场景 | CPU 密集型 | IO 密集型 |

### 实际应用场景

**传统方式（线程池）**：
```java
// 需要配置线程池大小
ExecutorService executor = Executors.newFixedThreadPool(100);

for (int i = 0; i < 10000; i++) {
    executor.submit(() -> {
        // 处理 HTTP 请求
        HttpClient.newHttpClient()
            .send(request, BodyHandlers.ofString());
    });
}
```

**虚拟线程方式**：
```java
// 不需要线程池，每个任务一个虚拟线程
try (ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor()) {
    for (int i = 0; i < 10000; i++) {
        executor.submit(() -> {
            // 处理 HTTP 请求
            HttpClient.newHttpClient()
                .send(request, BodyHandlers.ofString());
        });
    }
}
```

**优势**：
- 代码更简单
- 性能更好（高并发场景）
- 不需要调优线程池大小

### 注意事项

```java
// ✅ 适合：IO 密集型任务
executor.submit(() -> {
    // HTTP 请求
    httpClient.send(request, BodyHandlers.ofString());
    
    // 数据库查询
    jdbcTemplate.query("SELECT * FROM users");
    
    // 文件读写
    Files.readAllBytes(path);
});

// ❌ 不适合：CPU 密集型任务
executor.submit(() -> {
    // 复杂计算
    for (int i = 0; i < 1000000; i++) {
        result += Math.sqrt(i);
    }
    // 用平台线程更好
});
```

## 三、Record 模式匹配（Pattern Matching for Record）

### 什么是 Record 模式匹配？

**生活例子：快递分拣**

```
传统方式
├── 检查包裹类型
├── 转换为具体类型
└── 提取数据

Record 模式匹配
└── 一步完成（检查 + 转换 + 提取）
```

### Switch 模式匹配

```java
// 定义 Record
public record Point(int x, int y) { }
public record Circle(Point center, int radius) { }
public record Rectangle(Point topLeft, Point bottomRight) { }

// 使用 Switch 模式匹配
public String format(Shape shape) {
    return switch (shape) {
        case Point(int x, int y) -> 
            "点：(" + x + ", " + y + ")";
        
        case Circle(Point center, int radius) -> 
            "圆：中心=" + center + ", 半径=" + radius;
        
        case Rectangle(Point topLeft, Point bottomRight) -> 
            "矩形：左上=" + topLeft + ", 右下=" + bottomRight;
        
        case null -> "空形状";
        
        default -> "未知形状";
    };
}
```

### If 语句模式匹配

```java
// 传统方式
if (obj instanceof Circle circle) {
    // 自动转换为 Circle 类型
    System.out.println("半径：" + circle.radius());
}

// 嵌套 Record
if (shape instanceof Circle(Point center, int radius)) {
    System.out.println("中心：" + center + ", 半径：" + radius);
}
```

### 守卫条件（Guard）

```java
// Java 21 支持守卫条件
public String describe(Shape shape) {
    return switch (shape) {
        case Circle c when c.radius() > 10 -> "大圆";
        case Circle c -> "小圆";
        case Rectangle r when r.width() > r.height() -> "横向矩形";
        case Rectangle r -> "纵向矩形";
        default -> "未知";
    };
}
```

## 四、Switch 模式匹配增强

### 类型模式匹配

```java
// 根据类型匹配
public String format(Object obj) {
    return switch (obj) {
        case Integer i -> "整数：" + i;
        case Long l -> "长整数：" + l;
        case String s -> "字符串：" + s;
        case List<?> list -> "列表：" + list.size() + "个元素";
        case null -> "空值";
        default -> "其他：" + obj.toString();
    };
}
```

### 细化类型匹配

```java
// 细化类型（Type Refinement）
public void process(Number num) {
    switch (num) {
        case Integer i -> {
            // num 被细化为 Integer
            int value = i.intValue();
            System.out.println("整数：" + value);
        }
        case Double d -> {
            // num 被细化为 Double
            double value = d.doubleValue();
            System.out.println("小数：" + value);
        }
    }
}
```

## 五、文本块（Text Blocks）

### 什么是文本块？

**传统方式**：
```java
// 需要转义和拼接
String json = "{\n" +
    "  \"name\": \"张三\",\n" +
    "  \"age\": 18,\n" +
    "  \"email\": \"zhang@example.com\"\n" +
    "}";
```

**文本块（Java 15+，Java 21 正式）**：
```java
// 保持原始格式
String json = """
    {
      "name": "张三",
      "age": 18,
      "email": "zhang@example.com"
    }
    """;
```

### 文本块语法

```java
// SQL 语句
String sql = """
    SELECT id, name, email
    FROM users
    WHERE age > ?
    ORDER BY name
    """;

// HTML 内容
String html = """
    <html>
      <head><title>标题</title></head>
      <body>
        <h1>欢迎</h1>
      </body>
    </html>
    """;

// 多行字符串
String multiLine = """
    第一行
    第二行
    第三行
    """;
```

### 文本块特性

```java
// 自动去除前导空白
String str = """
    第一行
    第二行
    """;

// 保留相对缩进
String code = """
    public class Hello {
        public static void main(String[] args) {
            System.out.println("Hello");
        }
    }
    """;

// 使用 \ 避免换行
String singleLine = """
    这是一行\
    连续的\
    字符串
    """;  // 实际是"这是一行连续的字符串"
```

## 八、常见问题

### Q1：虚拟线程会完全取代平台线程吗？

**不会**：
- 虚拟线程：适合 IO 密集型
- 平台线程：适合 CPU 密集型
- 两者共存

### Q2：虚拟线程性能如何？

**IO 密集型场景**：
- 虚拟线程性能更好
- 可以处理百万级并发
- 不需要线程池调优

**CPU 密集型场景**：
- 平台线程更好
- 虚拟线程有额外开销

### Q3：什么时候用 Java 21？

**建议**：
- 新项目：直接用 Java 21
- 高并发项目：强烈推荐（虚拟线程）
- 老项目：评估依赖兼容性
