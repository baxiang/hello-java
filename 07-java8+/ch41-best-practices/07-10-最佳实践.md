## 二、Lambda 最佳实践

### ✅ 好的做法

```java
// 1. 使用方法引用
list.stream()
    .filter(Objects::nonNull)
    .map(String::toUpperCase)
    .forEach(System.out::println);

// 2. Lambda 参数类型推断
list.forEach(item -> process(item));  // 不需要显式类型

// 3. 简洁的 Lambda
list.sort((a, b) -> a.compareTo(b));
```

### ❌ 坏的做法

```java
// 1. 过度使用 Lambda
// 复杂逻辑应该提取为方法
list.stream()
    .filter(item -> {
        // 50 行复杂逻辑...
    });

// 2. 显式类型（不需要时）
list.forEach((String item) -> process(item));  // 冗余

// 3. Lambda 变量捕获问题
int count = 0;
list.forEach(item -> count++);  // 编译错误
```

## 三、Stream 最佳实践

### ✅ 好的做法

```java
// 1. 链式调用
List<String> result = list.stream()
    .filter(Objects::nonNull)
    .map(String::toLowerCase)
    .filter(s -> s.length() > 2)
    .collect(Collectors.toList());

// 2. 使用 Collectors
Map<String, List<User>> byCity = users.stream()
    .collect(Collectors.groupingBy(User::getCity));

// 3. 并行流（适合大数据量）
list.parallelStream()
    .filter(this::isValid)
    .forEach(this::process);
```

### ❌ 坏的做法

```java
// 1. Stream 过度使用
// 简单循环用 for 更好
for (Item item : list) {
    process(item);
}

// 2. 并行流滥用
// 小数据量用并行流反而慢
list.parallelStream().forEach(System.out::println);

// 3. Stream 重复使用
Stream<String> stream = list.stream();
stream.count();  // OK
stream.forEach(System.out::println);  // 错误：Stream 已关闭
```

## 四、Optional 最佳实践

### ✅ 好的做法

```java
// 1. 作为返回值
public Optional<User> findById(Long id) {
    return Optional.ofNullable(userMapper.selectById(id));
}

// 2. 链式调用
Optional<String> email = user.flatMap(User::getEmail)
    .filter(email -> email.contains("@"))
    .map(String::toLowerCase);

// 3. 提供默认值
String name = user.map(User::getName)
    .orElse("Unknown");
```

### ❌ 坏的做法

```java
// 1. 作为字段
public class User {
    private Optional<String> name;  // 错误
}

// 2. 作为参数
public void process(Optional<User> user) {  // 不推荐
}

// 3. 调用 get() 前不检查
Optional<User> opt = getUser();
User user = opt.get();  // 可能抛异常

// 正确做法
User user = opt.orElse(null);
```

## 五、日期时间 API 最佳实践

### ✅ 好的做法

```java
// 1. 使用不可变类
LocalDate date = LocalDate.now();
LocalDate tomorrow = date.plusDays(1);  // 创建新对象

// 2. 明确时区
ZonedDateTime meeting = ZonedDateTime.of(
    2024, 1, 15, 14, 30, 0, 0,
    ZoneId.of("Asia/Shanghai")
);

// 3. 使用 DateTimeFormatter
DateTimeFormatter formatter = DateTimeFormatter.ofPattern("yyyy-MM-dd");
String formatted = LocalDate.now().format(formatter);
```

### ❌ 坏的做法

```java
// 1. 使用旧 API
Date date = new Date();  // 不推荐
Calendar calendar = Calendar.getInstance();  // 不推荐

// 2. 忽略时区
LocalDateTime meeting = LocalDateTime.of(2024, 1, 15, 14, 30);
// 没有时区信息，容易混淆

// 3. 线程安全问题
// SimpleDateFormat 不是线程安全的
SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd");
// 多线程共享会出问题
```

## 六、Record 最佳实践

### ✅ 好的做法

```java
// 1. 数据传输对象（DTO）
public record UserDTO(Long id, String username, String email) { }

// 2. 多返回值
public record Result<T>(int code, String message, T data) { }

// 3. 添加验证
public record Email(String address) {
    public Email {
        if (!address.matches("^[\\w-]+(\\.[\\w-]+)*@[\\w-]+(\\.[\\w-]+)+$")) {
            throw new IllegalArgumentException("无效邮箱");
        }
    }
}
```

### ❌ 坏的做法

```java
// 1. 需要继承时
public record User() { }
// 不能继承其他类

// 2. 需要可变状态
public record Counter(int count) { }
// 字段是 final，不能修改

// 3. 复杂业务逻辑
public record Order() {
    public void process() {
        // 50 行复杂逻辑...
    }
}
// 应该用普通类
```

## 七、Switch 表达式最佳实践

### ✅ 好的做法

```java
// 1. 使用箭头语法
String result = switch (day) {
    case MONDAY, TUESDAY -> "工作日";
    case SATURDAY, SUNDAY -> "周末";
    default -> "未知";
};

// 2. 模式匹配
String format(Object obj) {
    return switch (obj) {
        case Integer i -> "整数：" + i;
        case String s -> "字符串：" + s;
        case null -> "空值";
        default -> "其他";
    };
}
```

### ❌ 坏的做法

```java
// 1. 忘记 break（传统 switch）
switch (day) {
    case MONDAY:
        result = "周一";
        // 忘记 break
    case TUESDAY:
        result = "周二";
}

// 2. 过度使用 switch
// 复杂逻辑应该用策略模式
```

## 八、虚拟线程最佳实践

### ✅ 好的做法

```java
// 1. IO 密集型任务
try (ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor()) {
    for (int i = 0; i < 10000; i++) {
        executor.submit(() -> {
            // HTTP 请求、数据库查询等
            httpClient.send(request, BodyHandlers.ofString());
        });
    }
}

// 2. 每个请求一个虚拟线程
@appGetMapping("/data")
public String getData() {
    return dataService.fetchData();
}
// Spring Boot 3.2+ 自动使用虚拟线程
```

### ❌ 坏的做法

```java
// 1. CPU 密集型任务
try (ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor()) {
    executor.submit(() -> {
        // 复杂计算
        for (int i = 0; i < 1000000; i++) {
            result += Math.sqrt(i);
        }
    });
}
// 应该用平台线程

// 2. 线程池 + 虚拟线程
ExecutorService executor = Executors.newFixedThreadPool(100);
// 虚拟线程不需要线程池
```

## 九、代码质量

### 命名规范

```java
// ✅ 好的命名
public record User(Long id, String username, String email) { }
public interface UserService { }
public class UserServiceImpl implements UserService { }

// ❌ 坏命名
public record Data(Long a, String b, String c) { }
```

### 注释规范

```java
// ✅ 好的注释
/**
 * 根据 ID 查询用户
 * 
 * @param id 用户 ID
 * @return 用户信息，不存在返回空
 */
public Optional<User> findById(Long id);

// ❌ 坏注释
// 获取用户
public User getUser(Long id);  // 方法名已经说明了
```

### 异常处理

```java
// ✅ 好的做法
try {
    process(data);
} catch (BusinessException e) {
    log.warn("业务异常：{}", e.getMessage());
    throw e;
} catch (Exception e) {
    log.error("系统异常", e);
    throw new SystemException("处理失败", e);
}

// ❌ 坏的做法
try {
    process(data);
} catch (Exception e) {
    // 吞掉异常
}

// 或
try {
    process(data);
} catch (Exception e) {
    e.printStackTrace();  // 生产环境不用
}
```

## 十、性能优化

### 集合操作

```java
// ✅ 好的做法
// 预分配容量
List<String> list = new ArrayList<>(1000);

// 使用 Stream 优化
Map<String, User> userMap = users.stream()
    .collect(Collectors.toMap(User::getId, Function.identity()));

// ❌ 坏的做法
// 循环中扩容
List<String> list = new ArrayList<>();
for (int i = 0; i < 1000; i++) {
    list.add(item);  // 多次扩容
}
```

### 字符串操作

```java
// ✅ 好的做法
String result = list.stream()
    .map(String::valueOf)
    .collect(Collectors.joining(","));

// ❌ 坏的做法
String result = "";
for (String s : list) {
    result += s + ",";  // 每次创建新对象
}
```

## 十一、小结

| 特性 | 推荐做法 | 避免做法 |
|------|----------|----------|
| Lambda | 方法引用 | 复杂逻辑 |
| Stream | 链式调用 | 滥用并行流 |
| Optional | 返回值 | 字段/参数 |
| 日期时间 | Java 8 API | Date/Calendar |
| Record | DTO | 复杂业务 |
| Switch | 箭头语法 | 忘记 break |
| 虚拟线程 | IO 密集型 | CPU 密集型 |

**核心要点**：
- 遵循最佳实践
- 避免常见陷阱
- 保持代码质量
- 持续学习改进
