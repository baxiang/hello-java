# 87.1 AI 智能客服实战：从零搭建智能问答系统（生活类比：培训新客服）

## 一、先问一个问题

**问题**：如何用 Spring AI 构建一个真正可用的智能客服，既能回答产品问题，又能查询订单？

### 生活例子：培训新客服

培训一个新客服需要三步：
1. **给资料** → 让他学习产品知识（RAG）
2. **教工具** → 教他使用订单系统（Function Calling）
3. **定规矩** → 规定回答方式和边界（System Prompt）

Spring AI 智能客服也是这三步！

## 二、项目架构

```
用户提问
  ↓
ChatClient（统一入口）
  ├── System Prompt（角色定义 + 回答规范）
  ├── RAG（产品知识库检索）
  ├── Function Calling（订单查询、库存查询）
  └── 对话记忆（多轮上下文）
  ↓
结构化回答
```

## 三、核心实现

### 3.1 知识库初始化

```java
@Component
public class KnowledgeBaseInit {

    private final VectorStore vectorStore;

    @EventListener(ApplicationReadyEvent.class)
    public void init() {
        List<Document> docs = new TikaDocumentReader(
            new ClassPathResource("knowledge/product-faq.pdf")
        ).get();

        List<Document> chunks = new TokenTextSplitter(800, 200, 5, 10000, true)
            .apply(docs);

        vectorStore.add(chunks);
    }
}
```

### 3.2 工具函数

```java
@Configuration
public class CustomerServiceTools {

    @Bean
    @Description("查询订单状态，参数为订单号")
    public Function<OrderQuery, OrderInfo> orderStatus(OrderService service) {
        return q -> service.getOrder(q.orderId());
    }

    @Bean
    @Description("查询商品库存和价格，参数为商品名称")
    public Function<ProductQuery, ProductInfo> productInfo(ProductService service) {
        return q -> service.getProduct(q.name());
    }
}
```

### 3.3 客服服务

```java
@Service
public class CustomerServiceBot {

    private final ChatClient chatClient;

    public CustomerServiceBot(ChatClient.Builder builder, VectorStore vectorStore) {
        this.chatClient = builder
            .defaultSystem("""
                你是一个专业的客服代表。
                规则：
                1. 基于提供的知识库回答产品问题
                2. 订单相关问题使用工具查询
                3. 不确定时诚实说明，不编造信息
                4. 保持礼貌、简洁、专业
                """)
            .defaultFunctions("orderStatus", "productInfo")
            .defaultAdvisors(
                new QuestionAnswerAdvisor(vectorStore, SearchRequest.builder().topK(3).build())
            )
            .build();
    }

    public String chat(String sessionId, String message) {
        return chatClient.prompt()
            .user(message)
            .advisors(a -> a.param("chat_memory_conversation_id", sessionId))
            .call()
            .content();
    }
}
```

### 3.4 REST 接口

```java
@RestController
@RequestMapping("/api/chat")
public class ChatController {

    private final CustomerServiceBot bot;

    @PostMapping
    public Map<String, String> chat(@RequestBody Map<String, String> req) {
        String reply = bot.chat(req.get("sessionId"), req.get("message"));
        return Map.of("reply", reply);
    }
}
```

## 四、常见疑问

**Q1：如何防止 AI 编造信息？**
A：在 System Prompt 中明确要求"只基于提供的知识库回答，不确定时说明"，并设置较低的 temperature（如 0.3）。

**Q2：多轮对话如何实现？**
A：Spring AI 提供 ChatMemory 机制，通过 conversationId 关联历史消息，自动注入上下文。

**Q3：生产环境需要注意什么？**
A：API 限流、Token 用量监控、敏感信息过滤、人工转接兜底、响应缓存。

## 五、小结

| 组件 | 作用 | 关键代码 |
|------|------|----------|
| System Prompt | 定义角色和规则 | .defaultSystem("...") |
| RAG (QuestionAnswerAdvisor) | 知识库检索 | new QuestionAnswerAdvisor(vectorStore, ...) |
| Function Calling | 调用业务系统 | .defaultFunctions("orderStatus") |
| ChatMemory | 多轮对话记忆 | chat_memory_conversation_id |
| ChatClient | 统一入口 | builder.build() |
