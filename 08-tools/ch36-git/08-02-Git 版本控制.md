# 8.2 Git 版本控制

## 一、Git 概述

Git 是一个分布式版本控制系统，用于跟踪代码变更。

**特点**：
- 分布式：每个开发者都有完整仓库
- 分支管理：轻量级分支
- 离线工作：大部分操作可离线进行

## 二、Git 安装配置

### 2.1 安装

```bash
# macOS
brew install git

# Windows
# 下载 https://git-scm.com/

# Linux
sudo apt install git  # Ubuntu/Debian
sudo yum install git  # CentOS/RHEL
```

### 2.2 配置

```bash
# 用户信息
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# 查看配置
git config --list

# 查看单个配置
git config user.name
```

### 2.3 行尾配置

```bash
# Windows（CRLF）
git config --global core.autocrlf true

# macOS/Linux（LF）
git config --global core.autocrlf input
```

## 三、基本操作

### 3.1 初始化仓库

```bash
# 初始化新仓库
git init

# 克隆远程仓库
git clone https://github.com/user/repo.git
git clone git@github.com:user/repo.git  # SSH
```

### 3.2 查看状态

```bash
# 查看状态
git status

# 查看简短状态
git status -s
```

### 3.3 添加文件

```bash
# 添加单个文件
git add file.txt

# 添加所有文件
git add .
git add -A

# 添加所有修改
git add -u
```

### 3.4 提交

```bash
# 提交
git commit -m "提交信息"

# 提交所有修改
git commit -am "提交信息"

# 修改上次提交
git commit --amend -m "新的提交信息"
```

### 3.5 查看历史

```bash
# 查看提交历史
git log
git log --oneline  # 简洁模式
git log --graph    # 图形化

# 查看指定文件历史
git log file.txt

# 查看指定数量
git log -n 5
```

## 四、分支管理

### 4.1 分支操作

```bash
# 查看分支
git branch

# 查看包含远程分支
git branch -a

# 创建分支
git branch feature-1

# 切换分支
git checkout feature-1
git switch feature-1  # Git 2.23+

# 创建并切换
git checkout -b feature-1
git switch -c feature-1  # Git 2.23+

# 删除分支
git branch -d feature-1  # 安全删除
git branch -D feature-1  # 强制删除
```

### 4.2 分支合并

```bash
# 切换到主分支
git checkout main

# 合并 feature-1 到 main
git merge feature-1

# 查看合并历史
git log --graph --oneline
```

### 4.3 分支策略

```bash
# Git Flow 工作流
main/master      # 生产分支
develop          # 开发分支
feature/*        # 功能分支
release/*        # 发布分支
hotfix/*         # 热修复分支

# GitHub Flow
main             # 主分支
feature/*        # 功能分支（PR 后合并）
```

## 五、远程操作

### 5.1 远程仓库

```bash
# 查看远程
git remote -v

# 添加远程
git remote add origin https://github.com/user/repo.git

# 重命名远程
git remote rename origin upstream

# 删除远程
git remote remove origin
```

### 5.2 推送

```bash
# 推送到远程
git push origin main

# 首次推送（设置上游）
git push -u origin main

# 推送所有分支
git push --all origin

# 强制推送（危险）
git push -f origin main
```

### 5.3 拉取

```bash
# 拉取并合并
git pull origin main

# 只下载不合并
git fetch origin

# 拉取并变基
git pull --rebase origin main
```

## 六、撤销操作

### 6.1 撤销工作区修改

```bash
# 撤销单个文件
git checkout file.txt
git restore file.txt  # Git 2.23+

# 撤销所有修改
git checkout .
git restore .
```

### 6.2 撤销暂存

```bash
# 撤销暂存
git reset HEAD file.txt
git restore --staged file.txt  # Git 2.23+
```

### 6.3 撤销提交

```bash
# 软重置（保留修改）
git reset --soft HEAD~1

# 混合重置（默认）
git reset HEAD~1

# 硬重置（丢弃修改）
git reset --hard HEAD~1
```

### 6.4 回退到指定版本

```bash
# 查看历史
git log --oneline

# 回退到指定 commit
git reset --hard abc123

# 创建新提交回退
git revert abc123
```

## 七、标签管理

```bash
# 查看标签
git tag

# 创建标签
git tag v1.0.0

# 创建带注释标签
git tag -a v1.0.0 -m "版本 1.0.0"

# 推送标签
git push origin v1.0.0
git push origin --tags  # 推送所有标签

# 删除标签
git tag -d v1.0.0
git push origin --delete v1.0.0
```

## 八、暂存操作

```bash
# 暂存当前工作
git stash

# 查看暂存列表
git stash list

# 恢复最近暂存
git stash pop

# 恢复指定暂存
git stash apply stash@{1}

# 删除暂存
git stash drop stash@{0}

# 清空所有暂存
git stash clear
```

## 九、查看差异

```bash
# 工作区与暂存区差异
git diff

# 暂存区与最新提交差异
git diff --cached

# 两个提交差异
git diff abc123 def456

# 两个分支差异
git diff main feature-1

# 查看文件差异
git diff file.txt
```

## 十、忽略文件

### 10.1 .gitignore

```bash
# .gitignore 文件

# 忽略所有.class 文件
*.class

# 忽略 target 目录
target/

# 忽略.idea 目录
.idea/

# 忽略.env 文件
.env

# 但保留.env.example
!.env.example

# 忽略所有.log 文件
*.log

# 忽略 node_modules
node_modules/

# 忽略系统文件
.DS_Store
Thumbs.db
```

### 10.2 全局忽略

```bash
# 创建全局忽略文件
git config --global core.excludesfile ~/.gitignore_global

# 编辑全局忽略文件
echo "*.log" >> ~/.gitignore_global
```

## 十一、综合示例

### 11.1 开始新功能

```bash
# 1. 切换到主分支
git checkout main

# 2. 拉取最新代码
git pull origin main

# 3. 创建功能分支
git checkout -b feature/user-login

# 4. 开发并提交
git add .
git commit -m "feat: 添加用户登录功能"

# 5. 推送到远程
git push -u origin feature/user-login

# 6. 创建 Pull Request（GitHub/GitLab）

# 7. 合并后删除分支
git checkout main
git pull origin main
git branch -d feature/user-login
```

### 11.2 解决冲突

```bash
# 1. 拉取远程代码
git pull origin main

# 2. 出现冲突，编辑冲突文件
# <<<<<<< HEAD
# 当前分支的代码
# =======
# 要合并的代码
# >>>>>>> feature-1

# 3. 解决冲突后
git add file.txt
git commit -m "fix: 解决合并冲突"

# 4. 完成合并
git merge --continue
```

### 11.3 回滚错误提交

```bash
# 1. 查看历史
git log --oneline

# 2. 回滚到上一个版本
git reset --hard HEAD~1

# 或者创建回滚提交
git revert abc123

# 3. 如果已经推送，强制推送（谨慎使用）
git push -f origin main
```

## 十二、最佳实践

### 12.1 提交信息规范

```bash
# 格式：<type>(<scope>): <subject>

# type 类型
feat:     新功能
fix:      修复 bug
docs:     文档更新
style:    代码格式
refactor: 重构
test:     测试
chore:    构建/工具

# 示例
feat(auth): 添加用户登录功能
fix(api): 修复订单查询接口 bug
docs(readme): 更新安装说明
```

### 12.2 分支命名

```bash
# 功能分支
feature/user-login
feature/order-management

# 修复分支
fix/login-bug
fix/order-issue

# 发布分支
release/v1.0.0
release/v1.1.0

# 热修复分支
hotfix/critical-bug
```

### 12.3 提交频率

```bash
# ✅ 好的做法
- 每个功能点一个提交
- 提交信息清晰
- 提交前测试

# ❌ 不好的做法
- 一次性提交所有修改
- 提交信息模糊（"update"、"fix"）
- 提交未测试的代码
```

## 十三、小结

1. **基本操作**：init、add、commit、status、log
2. **分支管理**：branch、checkout、merge
3. **远程操作**：remote、push、pull、fetch
4. **撤销操作**：reset、revert、checkout
5. **暂存**：stash
6. **标签**：tag
7. **最佳实践**：提交规范、分支命名
