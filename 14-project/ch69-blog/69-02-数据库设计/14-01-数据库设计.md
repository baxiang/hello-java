# 69.2 数据库设计：如何设计表结构？

## 一、先问一个问题

**设计数据库表，需要考虑什么？**

### 生活例子：整理房间

```
整理房间
├── 衣服放衣柜
├── 书籍放书架
├── 餐具放厨房
└── 建立关联（衣柜钥匙在哪）
```

**数据库设计**：
- 数据分类存放（分表）
- 建立表关系（外键）
- 考虑查询效率（索引）

## 二、核心表设计

### 用户表（user）

```sql
CREATE TABLE `user` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '用户 ID',
    `username` VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
    `password` VARCHAR(100) NOT NULL COMMENT '密码（加密）',
    `email` VARCHAR(100) COMMENT '邮箱',
    `avatar` VARCHAR(255) COMMENT '头像',
    `role` VARCHAR(20) DEFAULT 'USER' COMMENT '角色（USER/ADMIN）',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX `idx_username` (`username`),
    INDEX `idx_email` (`email`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

**设计要点**：
- `username` 唯一索引（不能重复）
- `password` 加密存储（BCrypt）
- `role` 区分用户和管理员
- 时间字段自动维护

### 文章表（article）

```sql
CREATE TABLE `article` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '文章 ID',
    `title` VARCHAR(200) NOT NULL COMMENT '标题',
    `summary` VARCHAR(500) COMMENT '摘要',
    `content` TEXT NOT NULL COMMENT '内容（Markdown）',
    `author_id` BIGINT NOT NULL COMMENT '作者 ID',
    `category_id` BIGINT COMMENT '分类 ID',
    `view_count` INT DEFAULT 0 COMMENT '阅读量',
    `comment_count` INT DEFAULT 0 COMMENT '评论数',
    `status` TINYINT DEFAULT 1 COMMENT '状态（0 草稿/1 发布）',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
    `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX `idx_author` (`author_id`),
    INDEX `idx_category` (`category_id`),
    INDEX `idx_status` (`status`),
    INDEX `idx_create_time` (`create_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='文章表';
```

**设计要点**：
- `author_id` 外键关联用户表
- `category_id` 外键关联分类表
- 统计字段（阅读量、评论数）
- 状态字段（草稿/发布）

### 分类表（category）

```sql
CREATE TABLE `category` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '分类 ID',
    `name` VARCHAR(50) NOT NULL UNIQUE COMMENT '分类名',
    `description` VARCHAR(200) COMMENT '描述',
    `parent_id` BIGINT DEFAULT 0 COMMENT '父分类 ID（0 表示一级）',
    `sort` INT DEFAULT 0 COMMENT '排序',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX `idx_parent` (`parent_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='分类表';
```

**设计要点**：
- `parent_id` 支持多级分类
- `sort` 字段控制显示顺序

### 标签表（tag）

```sql
CREATE TABLE `tag` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '标签 ID',
    `name` VARCHAR(50) NOT NULL UNIQUE COMMENT '标签名',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='标签表';
```

### 文章标签关联表（article_tag）

```sql
CREATE TABLE `article_tag` (
    `article_id` BIGINT NOT NULL COMMENT '文章 ID',
    `tag_id` BIGINT NOT NULL COMMENT '标签 ID',
    PRIMARY KEY (`article_id`, `tag_id`),
    INDEX `idx_tag` (`tag_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='文章标签关联表';
```

**设计要点**：
- 多对多关系
- 联合主键

### 评论表（comment）

```sql
CREATE TABLE `comment` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '评论 ID',
    `article_id` BIGINT NOT NULL COMMENT '文章 ID',
    `user_id` BIGINT NOT NULL COMMENT '用户 ID',
    `parent_id` BIGINT DEFAULT 0 COMMENT '父评论 ID（0 表示一级评论）',
    `content` TEXT NOT NULL COMMENT '评论内容',
    `like_count` INT DEFAULT 0 COMMENT '点赞数',
    `status` TINYINT DEFAULT 1 COMMENT '状态（0 审核/1 通过）',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX `idx_article` (`article_id`),
    INDEX `idx_user` (`user_id`),
    INDEX `idx_parent` (`parent_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='评论表';
```

**设计要点**：
- `parent_id` 支持回复（嵌套评论）
- `status` 支持评论审核

## 四、索引设计

### 为什么需要索引？

**生活例子**：字典目录

```
查字典
├── 没有目录 → 一页页翻（慢）
└── 有目录 → 直接翻到（快）
```

**索引的作用**：
- 加快查询速度
- 但会降低写入速度

### 索引设计原则

```
需要索引的字段
├── 主键（自动创建）
├── 外键
├── 经常查询的字段
└── 经常作为条件的字段

不需要索引的字段
├── 很少查询
├── 重复值多（如性别）
└── 文本内容（如文章内容）
```

## 六、常见疑问

### Q1：为什么用 BIGINT 不用 INT？

**BIGINT**：
- 范围更大（2^63-1）
- 适合数据量大的场景
- 未来扩展性好

**INT**：
- 范围较小（2^31-1）
- 适合数据量小的场景

### Q2：为什么密码要加密？

**不加密的风险**：
- 数据库泄露 → 密码全部暴露
- 用户损失 → 账号被盗

**加密方式**：
```java
// BCrypt 加密
String encoded = passwordEncoder.encode(password);

// 验证密码
boolean matches = passwordEncoder.matches(password, encoded);
```

### Q3：为什么要设计文章标签关联表？

**多对多关系**：
- 一篇文章 → 多个标签
- 一个标签 → 多篇文章

**需要中间表**：
```
文章 1 ←→ 标签 1
文章 1 ←→ 标签 2
文章 2 ←→ 标签 1
```

## 七、小结

| 表名 | 作用 | 核心字段 |
|------|------|----------|
| user | 用户 | username, password, role |
| article | 文章 | title, content, author_id |
| category | 分类 | name, parent_id |
| tag | 标签 | name |
| article_tag | 文章标签关联 | article_id, tag_id |
| comment | 评论 | article_id, user_id, parent_id |

**设计要点**：
- 主键用 BIGINT 自增
- 外键建立索引
- 时间字段自动维护
- 状态字段控制业务
