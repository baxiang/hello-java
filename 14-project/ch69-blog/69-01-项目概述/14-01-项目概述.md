# 69.1 项目概述：为什么要做博客系统？

## 一、先问一个问题

**为什么学习要从博客系统开始？**

### 生活例子：学做菜

```
学做菜的过程
├── 炒青菜（简单）
├── 鱼香肉丝（中等）
└── 佛跳墙（复杂）
```

**博客系统就是"鱼香肉丝"**：
- 不太难，适合练手
- 涵盖核心技术
- 可以扩展功能

## 二、项目功能

### 核心功能

```
博客系统核心功能
├── 写文章（发布、编辑、删除）
├── 看文章（列表、详情、搜索）
├── 评文章（评论、回复）
└── 管文章（分类、标签）
```

### 用户视角

```
读者
├── 浏览文章
├── 搜索内容
└── 发表评论

作者
├── 写文章
├── 管理分类
├── 回复评论
└── 查看统计
```

## 三、技术选型

### 后端技术

| 技术 | 作用 | 为什么选它 |
|------|------|------------|
| Spring Boot | Web 框架 | 简单、开箱即用 |
| MyBatis-Plus | ORM 框架 | 简化 SQL |
| MySQL | 数据库 | 成熟、稳定 |
| Redis | 缓存 | 高性能 |
| JWT | 认证 | 无状态、跨域 |

### 前端技术

| 技术 | 作用 | 为什么选它 |
|------|------|------------|
| Vue.js 3 | 前端框架 | 易学、生态好 |
| Element Plus | UI 组件库 | 美观、组件全 |
| Axios | HTTP 客户端 | 简单、好用 |

## 四、项目架构

### 分层架构

```
用户
  ↓
前端（Vue.js）
  ↓
后端（Spring Boot）
  ↓
数据库（MySQL）
```

### 目录结构

```
blog/
├── blog-common/      # 公共模块
│   ├── entity/       # 实体类
│   ├── result/       # 统一响应
│   └── utils/        # 工具类
├── blog-dao/         # 数据访问层
│   └── mapper/       # Mapper 接口
├── blog-service/     # 业务逻辑层
│   ├── service/      # Service 接口
│   └── impl/         # Service 实现
└── blog-web/         # Web 层
    ├── controller/   # Controller
    └── config/       # 配置类
```

## 八、小结

| 方面 | 说明 |
|------|------|
| 项目定位 | 练手项目、涵盖核心技术 |
| 技术栈 | Spring Boot + Vue.js |
| 核心功能 | 文章管理、评论、分类标签 |
| 学习收获 | 全栈开发经验 |

**博客系统的特点**：
- 功能适中，适合学习
- 技术全面，涵盖核心
- 可以扩展，持续优化
