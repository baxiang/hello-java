# 69.6 部署上线：如何部署到服务器？

## 一、先问一个问题

**部署的本质是什么？**

### 生活例子：开店

```
开店流程
├── 找店面（服务器）
├── 装修（安装环境）
├── 进货（部署应用）
└── 开业（启动服务）
```

**部署就是**：
- 准备服务器
- 安装运行环境
- 部署应用
- 启动服务

## 三、环境安装

### 安装 Java

```bash
# Ubuntu
sudo apt update
sudo apt install openjdk-17-jdk

# CentOS
sudo yum install java-17-openjdk

# 验证
java -version
```

### 安装 MySQL

```bash
# Ubuntu
sudo apt install mysql-server

# 配置
sudo mysql_secure_installation

# 创建数据库
mysql -u root -p
CREATE DATABASE blog DEFAULT CHARSET utf8mb4;
```

### 安装 Redis

```bash
# Ubuntu
sudo apt install redis-server

# 启动
sudo systemctl start redis

# 验证
redis-cli ping  # 输出 PONG
```

### 安装 Nginx

```bash
# Ubuntu
sudo apt install nginx

# 启动
sudo systemctl start nginx

# 验证
# 访问 http://服务器 IP，看到 Nginx 欢迎页面
```

## 四、部署应用

### 打包应用

```bash
# Maven 打包
mvn clean package -DskipTests

# 生成 jar 包
# target/blog-web-1.0.0.jar
```

### 上传到服务器

```bash
# 使用 scp
scp target/blog-web-1.0.0.jar user@server_ip:/opt/blog/

# 或使用 FTP 工具（FileZilla、Xftp）
```

### 配置 application.yml

```yaml
# 生产环境配置
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/blog?useSSL=false
    username: blog_user
    password: your_password
  redis:
    host: localhost
    port: 6379

server:
  port: 8080

logging:
  file:
    name: /var/log/blog/app.log
```

## 五、启动服务

### 后台运行

```bash
# 使用 nohup
nohup java -jar blog-web-1.0.0.jar \
  --spring.profiles.active=prod \
  > /var/log/blog/app.log 2>&1 &

# 查看进程
ps aux | grep blog

# 查看日志
tail -f /var/log/blog/app.log

# 停止服务
kill $(ps aux | grep blog | grep -v grep | awk '{print $2}')
```

### 使用 systemd

```bash
# 创建服务文件
sudo vim /etc/systemd/system/blog.service

# 内容
[Unit]
Description=Blog Application
After=syslog.target network.target

[Service]
User=root
WorkingDirectory=/opt/blog
ExecStart=/usr/bin/java -jar /opt/blog/blog-web-1.0.0.jar
SuccessExitStatus=143
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target

# 启动服务
sudo systemctl enable blog
sudo systemctl start blog
sudo systemctl status blog

# 查看日志
sudo journalctl -u blog -f
```

## 六、Docker 部署

### Dockerfile

```dockerfile
FROM openjdk:17-jdk-slim

WORKDIR /app

COPY blog-web-1.0.0.jar app.jar

EXPOSE 8080

ENTRYPOINT ["java", "-jar", "app.jar"]
```

### 构建镜像

```bash
# 构建
docker build -t blog:1.0.0 .

# 运行
docker run -d \
  -p 8080:8080 \
  -v /var/log/blog:/var/log/blog \
  --name blog \
  blog:1.0.0
```

### Docker Compose

```yaml
version: '3.8'

services:
  blog:
    image: blog:1.0.0
    ports:
      - "8080:8080"
    depends_on:
      - mysql
      - redis
    environment:
      - SPRING_DATASOURCE_URL=jdbc:mysql://mysql:3306/blog
      - SPRING_REDIS_HOST=redis
  
  mysql:
    image: mysql:8.0
    environment:
      - MYSQL_ROOT_PASSWORD=root
      - MYSQL_DATABASE=blog
  
  redis:
    image: redis:alpine
```

## 七、Nginx 配置

### 反向代理

```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    
    location /static/ {
        alias /opt/blog/static/;
    }
}
```

### HTTPS 配置

```bash
# 使用 Let's Encrypt 免费证书
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

## 八、CI/CD

### GitHub Actions

```yaml
name: Deploy

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      
      - name: Set up JDK 17
        uses: actions/setup-java@v2
        with:
          java-version: '17'
      
      - name: Build with Maven
        run: mvn clean package -DskipTests
      
      - name: Deploy to server
        uses: easingthemes/ssh-deploy@v2
        with:
          SSH_PRIVATE_KEY: ${{ secrets.SSH_KEY }}
          REMOTE_HOST: ${{ secrets.SERVER_IP }}
          REMOTE_USER: root
          SOURCE: "target/blog-web-1.0.0.jar"
          TARGET: "/opt/blog/"
      
      - name: Restart service
        uses: appleboy/ssh-action@master
        with:
          host: ${{ secrets.SERVER_IP }}
          username: root
          key: ${{ secrets.SSH_KEY }}
          script: |
            systemctl restart blog
```

## 九、监控和日志

### 日志配置

```yaml
logging:
  level:
    root: INFO
    com.example.blog: DEBUG
  file:
    name: /var/log/blog/app.log
  pattern:
    file: "%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger{36} - %msg%n"
```

### 日志查看

```bash
# 实时查看
tail -f /var/log/blog/app.log

# 搜索错误
grep ERROR /var/log/blog/app.log

# 按日期查看
cat /var/log/blog/app.log.2024-01-01
```

## 十、小结

| 步骤 | 关键命令 | 说明 |
|------|----------|------|
| 环境安装 | apt install | Java、MySQL、Redis |
| 打包 | mvn package | 生成 jar 包 |
| 上传 | scp | 上传到服务器 |
| 启动 | nohup/systemd | 后台运行 |
| 反向代理 | Nginx | 域名访问 |
| Docker | docker build/run | 容器化部署 |

**部署要点**：
- 生产环境配置分离
- 日志记录完整
- 服务自动重启
- 监控告警
