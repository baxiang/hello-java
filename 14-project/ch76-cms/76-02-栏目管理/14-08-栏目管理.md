# 76.2 栏目管理：如何管理树形结构？

## 一、先问一个问题

**如何存储树形结构？**

### 生活例子：公司组织架构

```
总公司
├── 北京分公司
│   ├── 技术部
│   └── 销售部
└── 上海分公司
    ├── 技术部
    └── 市场部
```

**树形存储**：
- 父级 ID
- 层级路径
- 排序号

## 二、数据库设计

### 栏目表

```sql
CREATE TABLE `category` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
    `name` VARCHAR(100) NOT NULL COMMENT '栏目名称',
    `parent_id` BIGINT DEFAULT 0 COMMENT '父栏目 ID',
    `level` INT DEFAULT 0 COMMENT '层级',
    `path` VARCHAR(500) COMMENT '层级路径',
    `sort` INT DEFAULT 0 COMMENT '排序',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP
) COMMENT='栏目表';
```

## 三、树形查询

### 查询所有栏目（树形）

```java
public List<CategoryTreeVO> getTree() {
    // 1. 查询所有栏目
    List<Category> categories = categoryMapper.selectAll();
    
    // 2. 构建树形结构
    return buildTree(categories, 0L);
}

private List<CategoryTreeVO> buildTree(List<Category> categories, Long parentId) {
    List<CategoryTreeVO> tree = new ArrayList<>();
    
    for (Category category : categories) {
        if (category.getParentId().equals(parentId)) {
            CategoryTreeVO vo = new CategoryTreeVO();
            BeanUtils.copyProperties(category, vo);
            
            // 递归查询子栏目
            vo.setChildren(buildTree(categories, category.getId()));
            
            tree.add(vo);
        }
    }
    
    return tree;
}
```

## 四、小结

| 字段 | 作用 | 说明 |
|------|------|------|
| parent_id | 父栏目 ID | 构建树形 |
| level | 层级 | 快速筛选 |
| path | 层级路径 | 祖先查询 |
| sort | 排序 | 显示顺序 |

**核心要点**：
- 父级 ID 构建树
- 递归查询子栏目
- 层级路径优化
