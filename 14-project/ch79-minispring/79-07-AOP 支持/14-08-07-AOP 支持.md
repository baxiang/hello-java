# 79.7 AOP 支持：什么是代理？

## 一、先问一个问题

**什么是 AOP？为什么需要它？**

### 生活例子：明星与经纪人

```
明星（业务逻辑）
├── 拍戏（核心业务）
├── 唱歌（核心业务）
└── 跳舞（核心业务）

经纪人（AOP）
├── 安排行程（前置处理）
├── 收钱记账（后置处理）
├── 处理绯闻（异常处理）
└── 安排采访（环绕处理）
```

**AOP 的作用**：
- 明星只管拍戏（核心业务）
- 经纪人处理杂事（横切关注点）
- 各司其职，互不干扰

## 二、什么是代理

### 生活例子：代购

```
你想买国外商品
  ↓
找代购（代理）
  ↓
代购去国外买
  ↓
商品寄给你
```

**代理的好处**：
- 你不用出国
- 代购处理海关、物流
- 你只管收货

### 代码中的代理

```java
// 目标对象（真正的服务）
UserService target = new UserService();

// 代理对象
UserService proxy = new UserServiceProxy(target);

// 通过代理调用
proxy.sayHello();
```

**代理做了什么**：
```java
class UserServiceProxy implements UserService {
    private UserService target;
    
    public void sayHello() {
        // 前置处理
        System.out.println("准备打招呼...");
        
        // 调用目标方法
        target.sayHello();
        
        // 后置处理
        System.out.println("打招呼结束了");
    }
}
```

## 三、为什么需要 AOP

### 问题：重复代码

```java
// 用户服务
public class UserService {
    public void createUser() {
        // 开启事务
        System.out.println("开启事务");
        
        // 业务逻辑
        System.out.println("创建用户");
        
        // 提交事务
        System.out.println("提交事务");
    }
}

// 订单服务
public class OrderService {
    public void createOrder() {
        // 开启事务
        System.out.println("开启事务");
        
        // 业务逻辑
        System.out.println("创建订单");
        
        // 提交事务
        System.out.println("提交事务");
    }
}
```

**问题**：
- "开启事务"、"提交事务"重复出现
- 每个方法都要写
- 修改麻烦

### AOP 的解决方案

```java
// 业务逻辑（只关心核心业务）
public class UserService {
    public void createUser() {
        System.out.println("创建用户");
    }
}

// 事务切面（处理事务）
@Aspect
public class TransactionAspect {
    
    @Before("execution(* com.example.service.*.*(..))")
    public void before() {
        System.out.println("开启事务");
    }
    
    @After("execution(* com.example.service.*.*(..))")
    public void after() {
        System.out.println("提交事务");
    }
}
```

**效果**：
- 业务代码清爽
- 事务逻辑集中管理
- 修改方便

## 四、Spring AOP 的实现

### 动态代理

**Spring 有两种代理方式**：

```
JDK 动态代理（基于接口）
└── 目标类实现接口 → 用 JDK 代理

CGLIB 代理（基于继承）
└── 目标类没接口 → 用 CGLIB 代理
```

### JDK 动态代理

```java
// 目标接口
public interface UserService {
    void sayHello();
}

// 目标实现
public class UserServiceImpl implements UserService {
    public void sayHello() {
        System.out.println("Hello!");
    }
}

// 创建代理
UserService target = new UserServiceImpl();

UserService proxy = (UserService) Proxy.newProxyInstance(
    target.getClass().getClassLoader(),
    target.getClass().getInterfaces(),
    (proxyObj, method, args) -> {
        // 前置处理
        System.out.println("准备打招呼...");
        
        // 调用目标方法
        Object result = method.invoke(target, args);
        
        // 后置处理
        System.out.println("打招呼结束了");
        
        return result;
    }
);

// 通过代理调用
proxy.sayHello();
```

**输出**：
```
准备打招呼...
Hello!
打招呼结束了
```

### CGLIB 代理

**如果目标类没有接口**：

```java
// 没有接口
public class UserService {
    public void sayHello() {
        System.out.println("Hello!");
    }
}

// CGLIB 代理
Enhancer enhancer = new Enhancer();
enhancer.setSuperclass(UserService.class);
enhancer.setCallback((MethodInterceptor) (obj, method, args, proxy) -> {
    // 前置处理
    System.out.println("准备打招呼...");
    
    // 调用目标方法
    Object result = proxy.invokeSuper(obj, args);
    
    // 后置处理
    System.out.println("打招呼结束了");
    
    return result;
});

UserService proxy = (UserService) enhancer.create();
proxy.sayHello();
```

## 五、通知类型

### 生活例子：看电影

```
电影开始前 → 前置通知（Before）
  ↓
电影播放   → 环绕通知（Around）
  ↓
电影结束后 → 后置通知（After）
  ↓
正常结束   → 返回通知（AfterReturning）
  ↓
出意外了   → 异常通知（AfterThrowing）
```

### 代码示例

```java
@Aspect
public class LogAspect {
    
    // 前置通知
    @Before("execution(* com.example.service.*.*(..))")
    public void before() {
        System.out.println("方法执行前");
    }
    
    // 后置通知
    @After("execution(* com.example.service.*.*(..))")
    public void after() {
        System.out.println("方法执行后");
    }
    
    // 返回通知
    @AfterReturning("execution(* com.example.service.*.*(..))")
    public void afterReturning() {
        System.out.println("方法正常返回");
    }
    
    // 异常通知
    @AfterThrowing("execution(* com.example.service.*.*(..))")
    public void afterThrowing() {
        System.out.println("方法抛出异常");
    }
    
    // 环绕通知
    @Around("execution(* com.example.service.*.*(..))")
    public Object around(ProceedingJoinPoint pjp) throws Throwable {
        System.out.println("环绕前");
        Object result = pjp.proceed();  // 执行目标方法
        System.out.println("环绕后");
        return result;
    }
}
```

## 六、切点表达式

### 什么是切点？

**生活例子**：钓鱼

```
切点 = 鱼竿
  ↓
指定在哪里钓鱼
  ↓
哪些方法需要拦截
```

### 切点表达式

```java
// 匹配所有方法
@Pointcut("execution(* *(..))")

// 匹配指定类的所有方法
@Pointcut("execution(* UserService.*(..))")

// 匹配指定包下的所有类
@Pointcut("execution(* com.example.service.*.*(..))")

// 匹配指定方法名
@Pointcut("execution(* save*(..))")

// 匹配带@Transaction 注解的方法
@Pointcut("@annotation(Transaction)")
```

### 表达式解析

```
execution(* com.example.service.*.*(..))
         ↑  ↑            ↑    ↑ ↑
         |  |            |    | └── 任意参数
         |  |            |    └──── 方法名（任意）
         |  |            └───────── 类名（service 包下的任意类）
         |  └────────────────────── 包名
         └───────────────────────── 返回类型（任意）
```

## 七、Mini-Spring 的 AOP 实现

### 简单实现

```java
public class ProxyFactory {
    
    private Object target;
    private Advice advice;
    
    public ProxyFactory(Object target) {
        this.target = target;
    }
    
    public void addAdvice(Advice advice) {
        this.advice = advice;
    }
    
    public Object getProxy() {
        return Proxy.newProxyInstance(
            target.getClass().getClassLoader(),
            target.getClass().getInterfaces(),
            (proxy, method, args) -> {
                // 执行通知
                if (advice instanceof MethodInterceptor) {
                    return ((MethodInterceptor) advice).invoke(
                        new MethodInvocation(target, method, args)
                    );
                }
                return method.invoke(target, args);
            }
        );
    }
}
```

### 使用示例

```java
// 目标对象
UserService target = new UserServiceImpl();

// 创建代理工厂
ProxyFactory factory = new ProxyFactory(target);

// 添加环绕通知
factory.addAdvice(new MethodInterceptor() {
    @Override
    public Object invoke(MethodInvocation invocation) throws Throwable {
        System.out.println("[通知] 方法执行前");
        Object result = invocation.proceed();
        System.out.println("[通知] 方法执行后");
        return result;
    }
});

// 获取代理对象
UserService proxy = (UserService) factory.getProxy();

// 通过代理调用
proxy.sayHello();
```

**输出**：
```
[通知] 方法执行前
Hello!
[通知] 方法执行后
```

## 八、常见疑问

### Q1：AOP 会影响性能吗？

**会，但影响很小**：
- 代理调用多一层
- 现代 JVM 优化很好
- 业务时间远大于代理时间

### Q2：所有方法都能代理吗？

**不是**：
- private 方法不能代理
- final 方法不能代理
- 静态方法不能代理

### Q3：什么时候用 JDK 代理，什么时候用 CGLIB？

**Spring 默认规则**：
- 有接口 → JDK 代理
- 没接口 → CGLIB 代理

**Spring Boot 2.x 后**：
- 默认都用 CGLIB
- 更灵活

## 九、小结

| 概念 | 生活类比 | 作用 |
|------|----------|------|
| AOP | 明星经纪人 | 处理横切关注点 |
| 代理 | 代购 | 代理调用 |
| 切点 | 鱼竿 | 指定拦截位置 |
| 通知 | 电影前后 | 拦截后做什么 |

**通知类型**：
- Before：方法执行前
- After：方法执行后
- AfterReturning：正常返回后
- AfterThrowing：抛出异常后
- Around：环绕（最强大）
