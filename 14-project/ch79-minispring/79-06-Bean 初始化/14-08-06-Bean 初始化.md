# 79.6 Bean 初始化：Bean 的生命周期

## 一、先问一个问题

**什么是 Bean 的生命周期？**

### 生活例子：人的一生

```
出生 → 成长 → 工作 → 退休 → 去世
  ↓      ↓      ↓      ↓      ↓
实例化  填充   初始化  使用   销毁
```

**Bean 的生命周期**：
1. 出生（实例化）：创建对象
2. 成长（属性填充）：注入依赖
3. 工作（初始化）：准备就绪
4. 使用：被调用
5. 去世（销毁）：释放资源

## 二、初始化的作用

### 问题：为什么需要初始化？

**生活例子**：开餐厅

```
1. 租店面（实例化）
2. 装修（属性填充）
3. 开业准备（初始化）← 这里
   ├── 办理营业执照
   ├── 招聘员工
   └── 准备食材
4. 正式营业（使用）
```

**没有初始化会怎样**：
- 营业执照没办 → 不能合法经营
- 员工没招聘 → 没人服务
- 食材没准备 → 没法做菜

### 代码例子

```java
@Component
public class DatabaseConnection {
    
    private String url;
    private String username;
    private String password;
    
    // 连接对象
    private Connection connection;
    
    // 属性填充后，需要初始化连接
    public void init() {
        try {
            connection = DriverManager.getConnection(url, username, password);
            System.out.println("数据库连接已建立");
        } catch (SQLException e) {
            throw new RuntimeException("连接失败", e);
        }
    }
}
```

**没有初始化**：
- connection 是 null
- 无法操作数据库

**有初始化**：
- connection 已建立
- 可以操作数据库

## 三、初始化方式

### 方式一：@PostConstruct 注解

```java
@Component
public class UserService {
    
    @PostConstruct  // 初始化方法
    public void init() {
        System.out.println("UserService 初始化完成");
    }
}
```

**特点**：
- Java 标准注解
- 最简单
- 推荐优先使用

### 方式二：InitializingBean 接口

```java
@Component
public class UserService implements InitializingBean {
    
    @Override
    public void afterPropertiesSet() {
        System.out.println("UserService 初始化完成");
    }
}
```

**特点**：
- Spring 特有接口
- 代码侵入性强
- 不推荐优先使用

### 方式三：自定义 init-method

```java
@Component
public class UserService {
    
    public void init() {
        System.out.println("UserService 初始化完成");
    }
}
```

**配置方式**：
```java
BeanDefinition bd = new BeanDefinition(UserService.class);
bd.setInitMethodName("init");  // 指定初始化方法名
```

**特点**：
- 灵活
- 需要配置
- 适合第三方类

## 四、初始化时机

### 在属性填充之后

```
1. 实例化（创建对象）
   ↓
2. 属性填充（注入依赖）
   ↓
3. 初始化（调用 init 方法）← 这里
   ↓
4. 使用
```

**为什么在属性填充之后**？

**简单道理**：
- 初始化方法可能需要用到注入的属性
- 得先注入，再初始化

```java
@Component
public class UserService {
    
    @Autowired
    private UserDao userDao;
    
    @PostConstruct
    public void init() {
        // 这里可以使用 userDao
        userDao.insert();  // ✅ 安全
    }
}
```

### 完整流程

```java
protected void initializeBean(String beanName, Object bean, BeanDefinition bd) {
    // 1. 调用 InitializingBean 的 afterPropertiesSet
    if (bean instanceof InitializingBean) {
        ((InitializingBean) bean).afterPropertiesSet();
    }
    
    // 2. 调用自定义 init-method
    String initMethodName = bd.getInitMethodName();
    if (initMethodName != null && !initMethodName.isEmpty()) {
        try {
            java.lang.reflect.Method method = bean.getClass()
                .getDeclaredMethod(initMethodName);
            method.setAccessible(true);
            method.invoke(bean);
        } catch (Exception e) {
            throw new BeansException("初始化失败", e);
        }
    }
}
```

## 五、销毁方法

### 有始有终

```
出生（实例化）
  ↓
初始化（@PostConstruct）
  ↓
使用
  ↓
销毁（@PreDestroy）← 这里
```

### 销毁方式

**@PreDestroy 注解**：
```java
@Component
public class DatabaseConnection {
    
    @PreDestroy  // 销毁方法
    public void close() {
        if (connection != null) {
            try {
                connection.close();
                System.out.println("数据库连接已关闭");
            } catch (SQLException e) {}
        }
    }
}
```

**DisposableBean 接口**：
```java
@Component
public class DatabaseConnection implements DisposableBean {
    
    @Override
    public void destroy() {
        if (connection != null) {
            connection.close();
        }
    }
}
```

### 销毁时机

**容器关闭时**：
```java
AnnotationConfigApplicationContext context = 
    new AnnotationConfigApplicationContext();

// ... 使用 Bean

context.close();  // 关闭容器，触发销毁方法
```

**注册关机钩子**：
```java
context.registerShutdownHook();

// JVM 关闭时自动关闭容器
// 比如：Ctrl+C 停止程序
```

## 六、完整的生命周期

### 完整流程

```
1. 实例化（构造方法）
   ↓
2. BeanPostProcessor 前置处理
   ↓
3. 属性填充（@Autowired）
   ↓
4. @PostConstruct 初始化
   ↓
5. InitializingBean.afterPropertiesSet
   ↓
6. 自定义 init-method
   ↓
7. BeanPostProcessor 后置处理
   ↓
8. Bean 就绪，可以使用
   ↓
9. 容器关闭
   ↓
10. @PreDestroy 销毁
    ↓
11. DisposableBean.destroy
    ↓
12. 自定义 destroy-method
```

### 代码测试

```java
@Component
public class LifecycleBean implements InitializingBean, DisposableBean {
    
    public LifecycleBean() {
        System.out.println("1. 构造方法");
    }
    
    @PostConstruct
    public void postConstruct() {
        System.out.println("2. @PostConstruct");
    }
    
    @Override
    public void afterPropertiesSet() {
        System.out.println("3. afterPropertiesSet");
    }
    
    public void init() {
        System.out.println("4. 自定义 init");
    }
    
    public void doWork() {
        System.out.println("5. 执行业务");
    }
    
    @PreDestroy
    public void preDestroy() {
        System.out.println("6. @PreDestroy");
    }
    
    @Override
    public void destroy() {
        System.out.println("7. destroy");
    }
}

// 测试
AnnotationConfigApplicationContext context = 
    new AnnotationConfigApplicationContext();
context.register(LifecycleBean.class);
context.refresh();

LifecycleBean bean = context.getBean(LifecycleBean.class);
bean.doWork();

context.close();
```

**输出**：
```
1. 构造方法
2. @PostConstruct
3. afterPropertiesSet
4. 自定义 init
5. 执行业务
6. @PreDestroy
7. destroy
```

## 七、常见疑问

### Q1：多种初始化方式，执行顺序是什么？

**执行顺序**：
```
1. @PostConstruct
2. InitializingBean.afterPropertiesSet
3. 自定义 init-method
```

**推荐**：
- 优先使用@PostConstruct
- 简单清晰
- Java 标准

### Q2：初始化方法可以抛异常吗？

**可以**：
```java
@PostConstruct
public void init() throws Exception {
    // 如果初始化失败，抛异常
    if (!checkDatabase()) {
        throw new Exception("数据库连接失败");
    }
}
```

**Spring 会**：
- 捕获异常
- 停止创建 Bean
- 抛出 BeansException

### Q3：销毁方法一定会执行吗？

**不一定**：
- 正常关闭容器 → 执行
- JVM 崩溃 → 不执行
- kill -9 → 不执行

**建议**：
- 重要资源及时关闭
- 不要完全依赖销毁方法

## 八、小结

| 阶段 | 注解/接口 | 执行时机 |
|------|----------|----------|
| 初始化 | @PostConstruct | 属性填充后 |
| 初始化 | InitializingBean | @PostConstruct 后 |
| 初始化 | init-method | 最后 |
| 销毁 | @PreDestroy | 容器关闭时 |
| 销毁 | DisposableBean | @PreDestroy 后 |
| 销毁 | destroy-method | 最后 |

**Bean 生命周期**：
```
构造 → 填充 → 初始化 → 使用 → 销毁
```

1. 创建一个 Bean，实现完整的生命周期

3. 尝试多种初始化方式

```java
public class MyBean {

    }

    }

    public void destroy() {

    }
}
```
