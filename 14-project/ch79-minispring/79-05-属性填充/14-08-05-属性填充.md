# 79.5 属性填充：依赖注入是如何实现的？

## 一、先问一个问题

**什么是依赖注入？为什么需要它？**

### 生活例子：组装电脑

```
你自己组装电脑
├── 需要 CPU
├── 需要内存
├── 需要硬盘
└── 需要显卡

问题：每个配件都要自己买、自己装
```

**依赖注入**：
```
电脑城组装好
├── CPU 已安装
├── 内存已安装
├── 硬盘已安装
└── 显卡已安装

你直接拿整机用就行
```

### 代码对比

**没有依赖注入**：
```java
public class Computer {
    private CPU cpu;
    private Memory memory;
    private Disk disk;
    
    public Computer() {
        this.cpu = new CPU();
        this.memory = new Memory();
        this.disk = new Disk();
    }
}
```

**有依赖注入**：
```java
public class Computer {
    @Autowired
    private CPU cpu;
    
    @Autowired
    private Memory memory;
    
    @Autowired
    private Disk disk;
}
```

**区别**：
- 之前：自己创建依赖
- 现在：别人给你注入

## 二、属性填充的时机

### Bean 创建流程回顾

```
1. 实例化（创建对象）
   ↓
2. 属性填充（注入依赖）← 这里
   ↓
3. 初始化（调用初始化方法）
```

### 为什么在实例化之后？

**简单道理**：
- 得先有对象，才能给对象设置属性
- 就像：得先有电脑，才能装配件

```java
// 1. 实例化：创建 Computer 对象
Computer computer = new Computer();

// 2. 属性填充：注入 CPU、内存等
computer.cpu = new CPU();
computer.memory = new Memory();
computer.disk = new Disk();
```

## 三、属性填充的实现

### 简单实现

```java
protected void populateBean(String beanName, BeanDefinition bd, Object bean) {
    // 1. 获取定义的属性值
    PropertyValues pvs = bd.getPropertyValues();
    
    // 2. 遍历所有属性
    for (PropertyValue pv : pvs.getPropertyValues()) {
        String propertyName = pv.getName();
        Object value = pv.getValue();
        
        // 3. 设置属性
        try {
            java.lang.reflect.Field field = bean.getClass().getDeclaredField(propertyName);
            field.setAccessible(true);  // 私有属性也能访问
            field.set(bean, value);     // 设置值
        } catch (Exception e) {
            throw new BeansException("设置属性失败：" + propertyName, e);
        }
    }
}
```

### 代码解释

**PropertyValues**：
```java
// 存储所有属性值
PropertyValues pvs = new PropertyValues();
pvs.addPropertyValue(new PropertyValue("cpu", new CPU()));
pvs.addPropertyValue(new PropertyValue("memory", new Memory()));
```

**PropertyValue**：
```java
// 单个属性值
public class PropertyValue {
    private String name;   // 属性名
    private Object value;  // 属性值
}
```

## 四、@Autowired 注解

### 什么是@Autowired？

**生活例子**：外卖订单

```
你点外卖
├── 备注：多放辣椒
└── 骑手看到备注，照做

@Autowired
└── 告诉 Spring：这个字段需要注入
```

### @Autowired 的工作原理

```java
@Component
public class UserService {
    
    @Autowired  // 告诉 Spring：请注入 UserDao
    private UserDao userDao;
}
```

**Spring 看到@Autowired 后**：
1. 扫描所有字段
2. 找到带@Autowired 的字段
3. 从容器获取对应类型的 Bean
4. 设置到字段中

### 实现代码

```java
public class AutowiredAnnotationBeanPostProcessor implements BeanPostProcessor {
    
    private BeanFactory beanFactory;
    
    @Override
    public Object postProcessBeforeInitialization(Object bean, String beanName) {
        // 扫描所有字段
        for (Field field : bean.getClass().getDeclaredFields()) {
            // 检查是否有@Autowired 注解
            if (field.isAnnotationPresent(Autowired.class)) {
                field.setAccessible(true);
                
                // 从容器获取 Bean
                String fieldName = field.getName();
                Object fieldValue = beanFactory.getBean(fieldName);
                
                // 设置到字段
                try {
                    field.set(bean, fieldValue);
                } catch (Exception e) {
                    throw new BeansException("注入失败", e);
                }
            }
        }
        return bean;
    }
}
```

## 五、BeanPostProcessor

### 什么是 BeanPostProcessor？

**生活例子**：质检员

```
工厂生产线
├── 产品组装（Bean 创建）
├── 质检员检查（BeanPostProcessor）
└── 包装出厂（Bean 就绪）
```

**BeanPostProcessor 的作用**：
- 在 Bean 创建前后执行自定义逻辑
- 可以修改 Bean
- 可以添加额外功能

### 接口定义

```java
public interface BeanPostProcessor {
    
    /**
     * 在 Bean 初始化之前执行
     */
    Object postProcessBeforeInitialization(Object bean, String beanName);
    
    /**
     * 在 Bean 初始化之后执行
     */
    Object postProcessAfterInitialization(Object bean, String beanName);
}
```

### 执行时机

```
1. 实例化 Bean
   ↓
2. postProcessBeforeInitialization ← 前置处理
   ↓
3. 属性填充
   ↓
4. 初始化
   ↓
5. postProcessAfterInitialization ← 后置处理
```

## 六、完整的依赖注入流程

### 流程图

```
getBean("userService")
   ↓
从缓存检查（没有）
   ↓
创建 Bean
├── 1. 实例化（new UserService()）
│   ↓
├── 2. 前置处理（BeanPostProcessor）
│   ↓
├── 3. 属性填充
│   ├── 扫描@Autowired 字段
│   ├── 从容器获取依赖
│   └── 设置到字段
│   ↓
├── 4. 初始化（afterPropertiesSet）
│   ↓
└── 5. 后置处理（BeanPostProcessor）
   ↓
存入缓存
   ↓
返回 Bean
```

### 代码实现

```java
public abstract class AbstractAutowireCapableBeanFactory {
    
    private final List<BeanPostProcessor> beanPostProcessors = new ArrayList<>();
    
    protected Object createBean(String beanName, BeanDefinition bd) {
        // 1. 实例化
        Object bean = instantiateBean(beanName, bd);
        
        // 2. 前置处理
        for (BeanPostProcessor bpp : beanPostProcessors) {
            bean = bpp.postProcessBeforeInitialization(bean, beanName);
        }
        
        // 3. 属性填充
        populateBean(beanName, bd, bean);
        
        // 4. 初始化
        initializeBean(beanName, bean, bd);
        
        // 5. 后置处理
        for (BeanPostProcessor bpp : beanPostProcessors) {
            bean = bpp.postProcessAfterInitialization(bean, beanName);
        }
        
        return bean;
    }
}
```

## 七、常见疑问

### Q1：@Autowired 和@Resource 有什么区别？

**@Autowired**（Spring 的）：
```java
@Autowired  // 按类型注入
private UserDao userDao;
```
- 默认按类型
- 可以配合@Qualifier 按名称

**@Resource**（Java 标准的）：
```java
@Resource(name = "userDao")  // 按名称注入
private UserDao userDao;
```
- 默认按名称
- 找不到再按类型

### Q2：如果找不到依赖怎么办？

**默认行为**：
```java
@Autowired  // 必需
private UserDao userDao;

// 如果容器没有 UserDao，抛异常
```

**可选依赖**：
```java
@Autowired(required = false)
private UserDao userDao;

// 如果容器没有 UserDao，设为 null
```

### Q3：循环依赖怎么解决？

**参考 79.3 单例注册表**：
- 二级缓存存放早期引用
- A 创建时暴露早期引用给 B
- B 完成后再给 A

## 八、小结

| 概念 | 生活类比 | 作用 |
|------|----------|------|
| 依赖注入 | 电脑城组装电脑 | 自动注入依赖 |
| @Autowired | 外卖备注 | 告诉 Spring 需要注入 |
| BeanPostProcessor | 质检员 | 创建前后执行逻辑 |
| 属性填充 | 安装配件 | 设置属性值 |

**依赖注入流程**：
1. 扫描@Autowired 字段
2. 从容器获取依赖
3. 设置到字段

## 九、动手练习

1. 创建两个 Bean，一个依赖另一个
2. 使用@Autowired 注入
3. 观察注入是否成功

```java
@Component
class UserDao {
    public void insert() {
        System.out.println("插入数据");
    }
}

@Component
class UserService {
    @Autowired
    private UserDao userDao;
    
    public void createUser() {
        userDao.insert();
    }
}

// 测试
UserService userService = context.getBean(UserService.class);
userService.createUser();  // 能成功调用
```
