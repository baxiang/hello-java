# 79.8 应用上下文：Spring 容器是什么？

## 一、先问一个问题

**什么是 ApplicationContext？和 BeanFactory 有什么区别？**

### 生活例子：手机 vs 功能机

```
BeanFactory（功能机）
├── 打电话（获取 Bean）
├── 发短信（基础功能）
└── 只能做基本事情

ApplicationContext（智能机）
├── 打电话（获取 Bean）
├── 上网（更多功能）
├── 拍照（事件机制）
├── 安装 APP（扩展功能）
└── 能做更多事情
```

**区别**：
- BeanFactory：基础容器，只能获取 Bean
- ApplicationContext：完整容器，更多功能

## 二、ApplicationContext 的功能

### 1. 获取 Bean（基础功能）

```java
ApplicationContext context = new AnnotationConfigApplicationContext();

// 获取 Bean
UserService userService = context.getBean(UserService.class);
```

### 2. 国际化支持

```java
// 根据不同语言返回不同消息
String msg = context.getMessage("welcome", null, Locale.CHINA);
// 中文："欢迎"
// 英文："Welcome"
```

### 3. 事件机制

```java
// 发布事件
context.publishEvent(new OrderCreatedEvent(this, order));

// 监听事件
@Component
class OrderListener {
    @EventListener
    public void handleOrder(OrderCreatedEvent event) {
        System.out.println("订单已创建");
    }
}
```

### 4. 资源加载

```java
// 加载文件
Resource resource = context.getResource("classpath:config.properties");
InputStream is = resource.getInputStream();
```

### 5. 生命周期管理

```java
// 注册关机钩子
context.registerShutdownHook();

// 关闭容器
context.close();
```

## 三、容器的启动流程

### 生活例子：开餐厅

```
1. 租店面（创建容器）
   ↓
2. 办执照（准备刷新）
   ↓
3. 装修（获取 BeanFactory）
   ↓
4. 招聘（注册 Bean）
   ↓
5. 准备食材（实例化 Bean）
   ↓
6. 开业（刷新完成）
```

### 代码流程

```java
public void refresh() {
    // 1. 准备刷新
    prepareRefresh();
    
    // 2. 获取 BeanFactory
    ConfigurableListableBeanFactory beanFactory = obtainFreshBeanFactory();
    
    // 3. 准备 BeanFactory
    prepareBeanFactory(beanFactory);
    
    // 4. 后置处理 BeanFactory
    postProcessBeanFactory(beanFactory);
    
    // 5. 调用 BeanFactory 后置处理器
    invokeBeanFactoryPostProcessors(beanFactory);
    
    // 6. 注册 Bean 后置处理器
    registerBeanPostProcessors(beanFactory);
    
    // 7. 初始化消息源
    initMessageSource();
    
    // 8. 初始化事件多播器
    initApplicationEventMulticaster();
    
    // 9. 留给子类的模板方法
    onRefresh();
    
    // 10. 注册事件监听器
    registerListeners();
    
    // 11. 实例化剩余的单例 Bean
    finishBeanFactoryInitialization(beanFactory);
    
    // 12. 完成刷新
    finishRefresh();
}
```

## 四、AnnotationConfigApplicationContext

### 最常用的实现类

```java
// 创建容器
ApplicationContext context = new AnnotationConfigApplicationContext(AppConfig.class);

// 获取 Bean
UserService userService = context.getBean(UserService.class);
```

### 配置类

```java
@Configuration  // 告诉 Spring：这是一个配置类
@ComponentScan("com.example")  // 扫描包
public class AppConfig {
    
    @Bean  // 定义 Bean
    public DataSource dataSource() {
        return new HikariDataSource();
    }
}
```

### 扫描的注解

```java
@ComponentScan("com.example")
// 会扫描以下注解：

@Component   // 通用组件
@Service     // 服务层
@Repository  // 数据层
@Controller  // 控制层
@RestController // REST 控制器
```

## 五、Bean 的作用域

### singleton（单例）

```java
@Component
@Scope("singleton")  // 默认
class UserService {
}

// 整个容器只有一个实例
UserService s1 = context.getBean(UserService.class);
UserService s2 = context.getBean(UserService.class);
System.out.println(s1 == s2);  // true
```

**就像**：小区的垃圾桶（只有一个）

### prototype（原型）

```java
@Component
@Scope("prototype")
class UserService {
}

// 每次获取都创建新实例
UserService p1 = context.getBean(UserService.class);
UserService p2 = context.getBean(UserService.class);
System.out.println(p1 == p2);  // false
```

**就像**：一次性筷子（每次用新的）

### request（请求作用域）

```java
@Component
@Scope(value = "request", proxyMode = ScopedProxyMode.TARGET_CLASS)
class RequestBean {
}

// 每个 HTTP 请求一个实例
// Web 环境专用
```

**就像**：餐厅菜单（每桌一份）

### session（会话作用域）

```java
@Component
@Scope(value = "session", proxyMode = ScopedProxyMode.TARGET_CLASS)
class SessionBean {
}

// 每个 HTTP Session 一个实例
// Web 环境专用
```

**就像**：酒店房卡（每个房间一张）

## 六、容器的关闭

### 为什么要关闭？

**释放资源**：
- 数据库连接
- 文件句柄
- 网络 socket

### 关闭方式

**手动关闭**：
```java
AnnotationConfigApplicationContext context = 
    new AnnotationConfigApplicationContext();

// ... 使用

context.close();  // 关闭容器
```

**注册钩子**：
```java
context.registerShutdownHook();

// JVM 关闭时自动关闭
// 比如：Ctrl+C、kill 命令
```

### 销毁方法

```java
@Component
class MyBean implements DisposableBean {
    
    @PreDestroy
    public void destroy() {
        System.out.println("Bean 被销毁了");
    }
}
```

## 七、完整示例

### 配置类

```java
@Configuration
@ComponentScan("com.example")
public class AppConfig {
    
    @Bean
    public DataSource dataSource() {
        HikariDataSource ds = new HikariDataSource();
        ds.setJdbcUrl("jdbc:mysql://localhost:3306/demo");
        ds.setUsername("root");
        ds.setPassword("123456");
        return ds;
    }
}
```

### Bean 定义

```java
@Service
class UserService {
    
    @Autowired
    private UserDao userDao;
    
    @PostConstruct
    public void init() {
        System.out.println("UserService 初始化完成");
    }
    
    public void createUser() {
        userDao.insert();
    }
}

@Repository
class UserDao {
    public void insert() {
        System.out.println("插入用户");
    }
}
```

### 启动应用

```java
public class Main {
    public static void main(String[] args) {
        // 1. 创建容器
        ApplicationContext context = 
            new AnnotationConfigApplicationContext(AppConfig.class);
        
        // 2. 获取 Bean
        UserService userService = context.getBean(UserService.class);
        
        // 3. 使用 Bean
        userService.createUser();
        
        // 4. 关闭容器
        ((AnnotationConfigApplicationContext) context).close();
    }
}
```

**输出**：
```
UserService 初始化完成
插入用户
Bean 被销毁了
```

## 八、常见疑问

### Q1：ApplicationContext 和 BeanFactory 用哪个？

**推荐 ApplicationContext**：
- 功能更多
- 是 BeanFactory 的超集
- Spring Boot 默认用 ApplicationContext

### Q2：可以创建多个容器吗？

**可以，但不推荐**：
```java
ApplicationContext parent = new AnnotationConfigApplicationContext(ParentConfig.class);
ApplicationContext child = new AnnotationConfigApplicationContext(ChildConfig.class, parent);

// 子容器可以访问父容器的 Bean
// 父容器不能访问子容器的 Bean
```

**问题**：
- 管理复杂
- 容易混乱
- 除非必要，不要这样做

### Q3：容器启动慢怎么办？

**可能原因**：
- Bean 太多
- 初始化逻辑复杂
- 数据库连接慢

**优化方法**：
- 懒加载：@Lazy
- 异步初始化
- 优化数据库连接

## 九、小结

| 概念 | 生活类比 | 作用 |
|------|----------|------|
| ApplicationContext | 智能机 | 完整容器 |
| BeanFactory | 功能机 | 基础容器 |
| singleton | 小区垃圾桶 | 单例 |
| prototype | 一次性筷子 | 多例 |
| refresh | 开餐厅 | 启动容器 |
| close | 关餐厅 | 关闭容器 |

**ApplicationContext 功能**：
1. 获取 Bean
2. 国际化
3. 事件机制
4. 资源加载
5. 生命周期管理
