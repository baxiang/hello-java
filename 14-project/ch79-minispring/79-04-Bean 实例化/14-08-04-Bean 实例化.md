# 79.4 Bean 实例化：Spring 是如何创建对象的？

## 一、先问一个问题

**Spring 创建对象和我们 new 对象有什么区别？**

### 我们自己 new 对象

```java
UserService userService = new UserService();
```

**简单直接**，但有问题：
- 如果 UserService 需要 UserDao 怎么办？
  ```java
  UserService userService = new UserService(new UserDao());
  ```
- 如果 UserDao 需要 DataSource 呢？
  ```java
  UserService userService = new UserService(
      new UserDao(new DataSource("jdbc:mysql://...", "root", "123456"))
  );
  ```

**问题**：依赖关系太复杂，创建对象变得很麻烦。

### Spring 创建对象

```java
// 告诉 Spring 创建 Bean
@Autowired
private UserService userService;
```

**Spring 内部做了什么**：
1. 查看 BeanDefinition（图纸）
2. 找到 UserService 类
3. 调用构造方法创建对象
4. 发现需要 UserDao
5. 创建或获取 UserDao
6. 注入到 UserService
7. 返回完整的 Bean

## 二、实例化策略

### 生活例子：做菜的方式

```
简单做菜（SimpleInstantiationStrategy）
└── 直接炒（无参构造）

复杂做菜（其他策略）
├── 需要配料（构造注入）
├── 需要调味（属性注入）
└── 需要特殊工具（工厂方法）
```

### Spring 的默认策略

**最简单的方式：调用无参构造**

```java
public class UserService {
    public UserService() {
        // Spring 调用这里
    }
}
```

**Spring 内部代码**：
```java
public class SimpleInstantiationStrategy {
    
    public Object instantiate(BeanDefinition beanDefinition) {
        Class<?> beanClass = beanDefinition.getBeanClass();
        
        // 找到无参构造方法
        Constructor<?> constructor = beanClass.getDeclaredConstructor();
        
        // 调用构造方法
        return constructor.newInstance();
    }
}
```

## 三、完整的 Bean 创建流程

### 生活例子：餐厅点餐

```
1. 点餐（getBean）
   ↓
2. 查看菜谱（BeanDefinition）
   ↓
3. 准备食材（实例化）
   ↓
4. 加入配料（属性填充）
   ↓
5. 装盘装饰（初始化）
   ↓
6. 上菜（返回 Bean）
```

### 代码流程

```java
public abstract class AbstractAutowireCapableBeanFactory {
    
    protected Object createBean(String beanName, BeanDefinition bd) {
        // 1. 实例化：创建对象
        Object bean = instantiateBean(beanName, bd);
        
        // 2. 属性填充：注入依赖
        populateBean(beanName, bd, bean);
        
        // 3. 初始化：调用初始化方法
        initializeBean(beanName, bean, bd);
        
        return bean;
    }
}
```

## 四、实例化 Bean

### 简单实现

```java
public class SimpleInstantiationStrategy implements InstantiationStrategy {
    
    @Override
    public Object instantiate(BeanDefinition beanDefinition) {
        Class<?> beanClass = beanDefinition.getBeanClass();
        
        try {
            // 调用无参构造方法
            return beanClass.getDeclaredConstructor().newInstance();
        } catch (Exception e) {
            throw new BeansException("创建 Bean 失败：" + beanClass.getName(), e);
        }
    }
}
```

### 测试

```java
@Component
public class UserService {
    public UserService() {
        System.out.println("UserService 被创建了");
    }
}

// 测试
DefaultListableBeanFactory factory = new DefaultListableBeanFactory();
factory.registerBeanDefinition("userService", 
    new BeanDefinition(UserService.class));

UserService userService = (UserService) factory.getBean("userService");
// 输出：UserService 被创建了
```

## 五、为什么需要 InstantiationStrategy？

### 问题：如果 Bean 没有无参构造怎么办？

```java
@Component
public class UserService {
    private UserDao userDao;
    
    // 只有有参构造
    public UserService(UserDao userDao) {
        this.userDao = userDao;
    }
}
```

### Spring 的解决方案

**策略模式**：不同的 Bean 用不同的实例化策略

```java
public interface InstantiationStrategy {
    Object instantiate(BeanDefinition beanDefinition);
}

// 简单策略：无参构造
public class SimpleInstantiationStrategy implements InstantiationStrategy {
    // ...
}

// 复杂策略：有参构造
public class ConstructorInstantiationStrategy implements InstantiationStrategy {
    // ...
}
```

## 六、常见疑问

### Q1：为什么 Spring 默认用无参构造？

**简单**：
- 无参构造最简单
- 不需要处理参数
- 大多数 Bean 可以用无参构造

### Q2：构造注入和属性注入有什么区别？

**构造注入**：
```java
@Component
public class UserService {
    private final UserDao userDao;
    
    @Autowired  // 构造注入
    public UserService(UserDao userDao) {
        this.userDao = userDao;
    }
}
```
**优点**：
- 依赖关系明确
- 创建后一定可用
- 适合必需依赖

**属性注入**：
```java
@Component
public class UserService {
    @Autowired  // 属性注入
    private UserDao userDao;
}
```
**优点**：
- 代码简洁
- 适合可选依赖

### Q3：推荐用哪种注入方式？

**推荐构造注入**：
- 依赖关系清晰
- 便于测试
- 避免空指针

## 七、小结

| 概念 | 生活类比 | 作用 |
|------|----------|------|
| 实例化策略 | 做菜方式 | 决定如何创建对象 |
| 简单策略 | 直接炒 | 调用无参构造 |
| 复杂策略 | 配料炒 | 调用有参构造 |

**Bean 创建流程**：
1. 实例化（创建对象）
2. 属性填充（注入依赖）
3. 初始化（调用初始化方法）

## 八、动手练习

1. 创建一个带无参构造的 Bean
2. 创建一个带有参构造的 Bean
3. 观察 Spring 如何创建它们

```java
@Component
class NoArgBean {
    public NoArgBean() {
        System.out.println("无参构造被调用");
    }
}

@Component
class ArgBean {
    private NoArgBean noArgBean;
    
    @Autowired
    public ArgBean(NoArgBean noArgBean) {
        System.out.println("有参构造被调用");
        this.noArgBean = noArgBean;
    }
}
```
