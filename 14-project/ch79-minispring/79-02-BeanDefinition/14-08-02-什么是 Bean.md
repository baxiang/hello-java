# 79.2 什么是 Bean？什么是 BeanDefinition？

## 一、先问一个问题

**什么是 Bean？**

用生活中的例子来理解：

### 例子：汽车工厂

```
汽车设计图纸  →  生产出  →  一辆真实的汽车
    ↓                        ↓
BeanDefinition          Bean 实例
   (图纸)                (真实对象)
```

**BeanDefinition（图纸）**：
- 描述了这个 Bean 是什么类型
- 是单例还是多例
- 有哪些属性
- 如何创建它

**Bean（真实的汽车）**：
- 是可以根据图纸创建出来的真实对象
- 可以被使用（开车）
- 有状态（颜色、里程数）

## 二、Spring 中的 Bean

### 你以前这样写代码

```java
UserService userService = new UserService();
userService.sayHello();
```

**问题**：
- `UserService` 里如果需要 `UserDao` 怎么办？
  ```java
  UserService userService = new UserService(new UserDao());
  ```
- 如果 `UserDao` 又需要 `DataSource` 呢？
  ```java
  UserService userService = new UserService(
      new UserDao(new DataSource())
  );
  ```
- 依赖关系越来越复杂...

### Spring 的解决方案

**把创建对象的权利交给 Spring**：

```java
// 告诉 Spring：我需要这个 Bean
@Autowired
private UserService userService;

// Spring 自动创建并注入
userService.sayHello();
```

**核心思想**：
- 你只管用，不管创建
- Spring 负责创建和管理所有对象
- 对象之间的依赖关系由 Spring 处理

## 三、BeanDefinition 详解

### 生活例子：菜谱

```
菜谱（BeanDefinition）
├── 菜名：鱼香肉丝
├── 主料：猪肉、胡萝卜、木耳
├── 调料：盐、糖、醋
├── 做法：炒
└── 份量：1 人份（单例）或 多人份（原型）
```

### 代码中的 BeanDefinition

```java
public class BeanDefinition {
    
    // 1. 这个 Bean 是什么类型？
    private Class<?> beanClass;  // 比如：UserService.class
    
    // 2. 是单例还是多例？
    private String scope = "singleton";  
    // singleton: 单例，整个容器只有一个实例
    // prototype: 原型，每次获取都创建新实例
    
    // 3. 依赖哪些其他 Bean？
    private PropertyValues propertyValues;  
    // 比如：UserService 依赖 UserDao
}
```

### 单例 vs 原型

**单例（singleton）**：
```
整个 Spring 容器中只有一个实例

就像：小区的垃圾桶
- 整个小区只有一个
- 所有人都用同一个
- 节省资源
```

**原型（prototype）**：
```
每次获取都创建新实例

就像：一次性筷子
- 每次用都拿新的
- 用完就丢
- 适合有状态的对象
```

## 四、Spring 容器如何工作

### 例子：餐厅点餐

```
你（开发者）
  ↓
点餐（@Autowired）
  ↓
服务员（Spring 容器）
  ↓
查看菜谱（BeanDefinition）
  ↓
厨房做菜（创建 Bean）
  ↓
上菜（注入 Bean）
```

### 完整流程

```
1. 启动 Spring 容器
   ↓
2. 扫描所有带注解的类（@Component、@Service 等）
   ↓
3. 为每个类创建 BeanDefinition（菜谱）
   ↓
4. 根据 BeanDefinition 创建 Bean（做菜）
   ↓
5. 填充属性（注入依赖）
   ↓
6. 调用初始化方法
   ↓
7. Bean 就绪，可以使用
```

## 五、代码实践

### 第一步：定义 Bean

```java
@Component  // 告诉 Spring：这是一个 Bean，请管理它
public class UserService {
    
    public void sayHello() {
        System.out.println("Hello!");
    }
}
```

**@Component 的作用**：
- 相当于告诉 Spring："请管理这个类"
- Spring 会为它创建 BeanDefinition
- 在需要的时候创建实例

### 第二步：注册到容器

```java
// 创建 Spring 容器
AnnotationConfigApplicationContext context = 
    new AnnotationConfigApplicationContext();

// 注册 Bean
context.register(UserService.class);

// 或者扫描包
context.scan("com.example");

// 启动容器
context.refresh();
```

**容器启动后做了什么**：
1. 扫描指定包
2. 找到所有带 @Component 的类
3. 为每个类创建 BeanDefinition
4. 创建 Bean 实例
5. 存入容器

### 第三步：从容器获取 Bean

```java
// 从容器获取 Bean
UserService userService = context.getBean(UserService.class);

// 使用 Bean
userService.sayHello();
```

**为什么从容器获取**：
- 容器里的 Bean 是完整的（依赖已注入）
- 容器管理 Bean 的生命周期
- 保证单例（如果是单例）

## 六、常见疑问

### Q1：为什么不直接 new，要用 Spring 管理？

**直接 new 的问题**：
```java
// 依赖关系复杂时
UserService userService = new UserService(
    new UserDao(
        new DataSource(
            "jdbc:mysql://...",
            "root",
            "password"
        )
    )
);
```

**Spring 管理后**：
```java
@Autowired
private UserService userService;
// Spring 自动处理所有依赖
```

### Q2：BeanDefinition 什么时候用？

**大多数情况你不需要直接操作 BeanDefinition**。

Spring 会自动为你创建：
- 看到 @Component → 自动创建 BeanDefinition
- 看到 @Autowired → 自动注入依赖

**但了解 BeanDefinition 有助于理解 Spring 原理**。

### Q3：单例 Bean 有线程安全问题吗？

**有！需要注意**：

```java
@Component
public class UserService {
    // ❌ 危险：成员变量不是线程安全的
    private int count = 0;
    
    public void increment() {
        count++;  // 多线程下会出问题
    }
}

// ✅ 安全：使用局部变量
public void safeMethod() {
    int count = 0;  // 局部变量，线程安全
    count++;
}
```

## 七、小结

| 概念 | 生活类比 | 作用 |
|------|----------|------|
| Bean | 真实的汽车 | Spring 管理的对象 |
| BeanDefinition | 汽车设计图纸 | Bean 的元数据定义 |
| 单例 | 小区垃圾桶 | 整个容器只有一个实例 |
| 原型 | 一次性筷子 | 每次获取都创建新实例 |
| 容器 | 餐厅厨房 | 创建和管理 Bean |

## 八、动手练习

1. 创建一个带 @Component 的类
2. 启动 Spring 容器
3. 从容器获取 Bean
4. 观察 Bean 是否是单例

```java
@Component
public class MyBean {
    public MyBean() {
        System.out.println("MyBean 被创建了");
    }
}

// 测试
AnnotationConfigApplicationContext context = 
    new AnnotationConfigApplicationContext();
context.register(MyBean.class);
context.refresh();

MyBean bean1 = context.getBean(MyBean.class);
MyBean bean2 = context.getBean(MyBean.class);

System.out.println("是否单例：" + (bean1 == bean2));
// 输出：true
```
