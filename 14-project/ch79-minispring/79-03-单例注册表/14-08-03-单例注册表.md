# 79.3 单例注册表与三级缓存

## 一、先问一个问题

**什么是缓存？为什么需要缓存？**

### 生活例子：超市货架

```
仓库（数据库/创建 Bean 的过程）
  ↓ 取货慢
货架（缓存）
  ↓ 拿取快
顾客（使用 Bean）
```

**缓存的作用**：
- 仓库取货慢 → 货架拿取快
- 创建 Bean 慢 → 从缓存拿快
- 避免重复创建

## 二、Spring 中的缓存

### 问题场景

```java
// 第一次获取 Bean
UserService userService1 = context.getBean(UserService.class);
// Spring 创建 Bean，耗时 100ms

// 第二次获取 Bean
UserService userService2 = context.getBean(UserService.class);
// 如果重新创建，又耗时 100ms

// 如果从缓存拿，耗时 1ms
```

**Spring 的做法**：
- 创建过的 Bean 存起来
- 下次直接拿，不再创建
- 这就是**单例注册表**

## 三、什么是单例注册表

### 简单理解：一个 Map

```java
// Spring 内部大概是这样
Map<String, Object> singletonObjects = new HashMap<>();

// 创建 Bean 后存起来
singletonObjects.put("userService", new UserService());

// 获取 Bean 时
Object bean = singletonObjects.get("userService");
```

**这就是单例注册表的核心**：
- 存：创建后的 Bean
- 取：根据名字获取 Bean
- 保证：每个 Bean 只有一个实例

## 四、为什么需要三级缓存

### 一级缓存：完整的 Bean

```
就像：成品仓库

存放：已经完全创建好的 Bean
- 实例创建了
- 属性填充了
- 初始化完成了
```

```java
// 一级缓存
Map<String, Object> singletonObjects = new HashMap<>();

// 存放完整的 Bean
singletonObjects.put("userService", userService);
```

### 问题：循环依赖

```java
@Component
class A {
    @Autowired
    private B b;  // A 需要 B
}

@Component
class B {
    @Autowired
    private A a;  // B 需要 A
}
```

**创建过程**：
```
创建 A
  ↓
发现需要 B
  ↓
创建 B
  ↓
发现需要 A
  ↓
A 还没创建好！死循环！
```

### 二级缓存：早期引用

```
就像：半成品仓库

存放：创建了一半的 Bean
- 实例创建了
- 属性还没填充
- 初始化还没做
```

```java
// 二级缓存
Map<String, Object> earlySingletonObjects = new HashMap<>();

// 存放早期引用
earlySingletonObjects.put("a", a);  // A 刚实例化，还没填充属性
```

### 解决循环依赖的流程

```
1. 创建 A 的实例
   ↓
2. 把 A 的早期引用放入二级缓存
   ↓
3. 填充 A 的属性，发现需要 B
   ↓
4. 创建 B 的实例
   ↓
5. 填充 B 的属性，发现需要 A
   ↓
6. 从二级缓存拿到 A 的早期引用
   ↓
7. B 创建完成，放入一级缓存
   ↓
8. A 拿到 B，完成创建，放入一级缓存
```

**关键点**：
- A 刚实例化就暴露早期引用
- B 可以拿到这个早期引用
- 虽然 A 还没完全创建好，但 B 可以先用

### 三级缓存：Bean 工厂

```
就像：定制工厂

存放：可以创建 Bean 的工厂
- 用于创建 AOP 代理
- 延迟创建
```

```java
// 三级缓存
Map<String, ObjectFactory<?>> singletonFactories = new HashMap<>();

// 存放 Bean 工厂
singletonFactories.put("a", () -> {
    // 这里可以创建 AOP 代理
    return createProxy(a);
});
```

### 为什么需要三级缓存？

**因为 AOP 代理**：

```java
@Component
class UserService {
    
    @Transactional  // 需要 AOP 代理
    public void createUser() {
        // ...
    }
}
```

**问题**：
- UserService 需要代理对象
- 但代理对象要在初始化后才能创建
- 如果 B 依赖 UserService，应该给原始对象还是代理对象？

**三级缓存的解决方案**：
```
1. A 实例化后，把工厂放入三级缓存
   ↓
2. B 需要 A 时，从三级缓存拿工厂
   ↓
3. 工厂创建代理对象（如果需要）
   ↓
4. 代理对象放入二级缓存
   ↓
5. B 拿到代理对象
```

## 五、代码实现

### 单例注册表实现

```java
public class DefaultSingletonBeanRegistry {
    
    // 一级缓存：完整的单例 Bean
    private final Map<String, Object> singletonObjects = new ConcurrentHashMap<>();
    
    // 二级缓存：早期 Bean 引用
    private final Map<String, Object> earlySingletonObjects = new ConcurrentHashMap<>();
    
    // 三级缓存：Bean 工厂
    private final Map<String, ObjectFactory<?>> singletonFactories = new ConcurrentHashMap<>();
    
    /**
     * 获取单例 Bean
     */
    public Object getSingleton(String beanName) {
        // 1. 先从一级缓存拿
        Object bean = singletonObjects.get(beanName);
        if (bean != null) {
            return bean;  // 拿到了完整的 Bean
        }
        
        // 2. 从二级缓存拿
        bean = earlySingletonObjects.get(beanName);
        if (bean != null) {
            return bean;  // 拿到了早期引用
        }
        
        // 3. 从三级缓存拿工厂，创建 Bean
        ObjectFactory<?> factory = singletonFactories.get(beanName);
        if (factory != null) {
            bean = factory.getObject();  // 创建代理对象
            earlySingletonObjects.put(beanName, bean);  // 放入二级缓存
            singletonFactories.remove(beanName);  // 移除工厂
            return bean;
        }
        
        return null;  // 没找到
    }
    
    /**
     * 注册完整的 Bean
     */
    public void addSingleton(String beanName, Object bean) {
        singletonObjects.put(beanName, bean);  // 放入一级缓存
        earlySingletonObjects.remove(beanName);  // 清理二级缓存
        singletonFactories.remove(beanName);  // 清理三级缓存
    }
    
    /**
     * 添加早期 Bean 引用
     */
    public void addEarlyBean(String beanName, Object bean) {
        earlySingletonObjects.put(beanName, bean);
    }
    
    /**
     * 添加 Bean 工厂
     */
    public void addSingletonFactory(String beanName, ObjectFactory<?> factory) {
        singletonFactories.put(beanName, factory);
    }
}
```

## 六、生活例子理解三级缓存

### 例子：餐厅做菜

```
一级缓存：成品菜
- 菜已经做好，可以上桌
- 完整的 Bean

二级缓存：半成品菜
- 菜切好了，还没炒
- 早期引用

三级缓存：菜谱 + 厨师
- 可以根据菜谱做菜
- 可以定制（加辣、少盐）
- Bean 工厂
```

### 场景：循环依赖

```
A 菜需要 B 菜的汤汁
B 菜需要 A 菜的食材

解决：
1. A 菜先把食材准备好（实例化）
2. 把食材给 B 菜（早期引用）
3. B 菜做好（完成创建）
4. A 菜拿到 B 菜的汤汁，完成
```

## 七、常见疑问

### Q1：为什么需要这么复杂？

**简单场景不需要**：
- 如果没有循环依赖，一级缓存就够了
- 但 Spring 要处理所有场景

### Q2：所有 Bean 都有循环依赖吗？

**不是**：
- 大部分 Bean 没有循环依赖
- 三级缓存是为了处理特殊情况

### Q3：循环依赖一定是好的吗？

**不一定**：
- 循环依赖通常说明设计有问题
- 考虑重构：提取公共接口、拆分模块

## 八、小结

| 缓存 | 生活类比 | 存放内容 |
|------|----------|----------|
| 一级缓存 | 成品仓库 | 完整的 Bean |
| 二级缓存 | 半成品仓库 | 早期引用 |
| 三级缓存 | 定制工厂 | Bean 工厂 |

**核心思想**：
- 缓存避免重复创建
- 二级缓存解决循环依赖
- 三级缓存支持 AOP 代理

1. 创建两个互相依赖的 Bean
2. 观察 Spring 如何解决循环依赖
3. 尝试去掉二级缓存，看会发生什么

```java
@Component
class A {
    @Autowired
    private B b;
}

@Component
class B {
    @Autowired
    private A a;
}

// 测试
context.register(A.class, B.class);
context.refresh();

A a = context.getBean(A.class);  // 能成功获取
B b = context.getBean(B.class);  // 能成功获取
```
