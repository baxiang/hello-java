# 79.3 单例 Bean 注册表（三级缓存）

## 一、为什么需要三级缓存

Spring 使用三级缓存解决**循环依赖**问题。

### 循环依赖场景

```java
@Component
class A {
    @Autowired
    private B b;
}

@Component
class B {
    @Autowired
    private A a;
}
```

**问题**：创建 A 需要 B，创建 B 需要 A，形成死循环。

### 三级缓存解决方案

```
一级缓存：singletonObjects → 完整的单例 Bean
二级缓存：earlySingletonObjects → 早期 Bean 引用（未填充属性）
三级缓存：singletonFactories → Bean 工厂（用于创建 AOP 代理）
```

## 二、单例 Bean 注册表实现

```java
package com.example.minispring.beans.factory.support;

import java.util.Map;
import java.util.concurrent.ConcurrentHashMap;

/**
 * 单例 Bean 注册表：三级缓存
 */
public class DefaultSingletonBeanRegistry {
    
    /** 一级缓存：完整的单例 Bean */
    private final Map<String, Object> singletonObjects = new ConcurrentHashMap<>();
    
    /** 二级缓存：早期 Bean 引用（未填充属性） */
    private final Map<String, Object> earlySingletonObjects = new ConcurrentHashMap<>();
    
    /** 三级缓存：Bean 工厂（用于创建 AOP 代理） */
    private final Map<String, ObjectFactory<?>> singletonFactories = new ConcurrentHashMap<>();
    
    /**
     * 获取单例 Bean
     */
    public Object getSingleton(String beanName) {
        // 1. 从一级缓存获取
        Object singletonObject = singletonObjects.get(beanName);
        
        // 2. 一级没有，从二级缓存获取
        if (singletonObject == null) {
            singletonObject = earlySingletonObjects.get(beanName);
        }
        
        return singletonObject;
    }
    
    /**
     * 注册单例 Bean
     */
    public void addSingleton(String beanName, Object singletonObject) {
        // 放入一级缓存
        singletonObjects.put(beanName, singletonObject);
        // 清理二级和三级缓存
        earlySingletonObjects.remove(beanName);
        singletonFactories.remove(beanName);
    }
    
    /**
     * 添加早期 Bean 引用
     */
    public void addEarlyBean(String beanName, Object singletonObject) {
        earlySingletonObjects.put(beanName, singletonObject);
    }
    
    /**
     * 添加 Bean 工厂（三级缓存）
     */
    public void addSingletonFactory(String beanName, ObjectFactory<?> factory) {
        singletonFactories.put(beanName, factory);
    }
    
    /**
     * 从三级缓存获取 Bean
     */
    public Object getSingletonFromFactory(String beanName) {
        ObjectFactory<?> factory = singletonFactories.get(beanName);
        if (factory != null) {
            return factory.getObject();
        }
        return null;
    }
}

/**
 * Bean 工厂接口
 */
@FunctionalInterface
interface ObjectFactory<T> {
    T getObject();
}
```

## 三、三级缓存工作流程

```
创建 Bean A
    ↓
1. 实例化 A（创建对象）
    ↓
2. 将 A 的工厂放入三级缓存
    ↓
3. 填充 A 的属性，发现需要 B
    ↓
4. 创建 B，实例化 B
    ↓
5. 填充 B 的属性，发现需要 A
    ↓
6. 从三级缓存获取 A 的工厂，创建 A 的早期引用
    ↓
7. 将 A 的早期引用放入二级缓存
    ↓
8. B 完成创建，放入一级缓存
    ↓
9. A 获取 B，完成创建，放入一级缓存
```

## 四、测试循环依赖

```java
package com.example.minispring.test;

import com.example.minispring.beans.config.BeanDefinition;
import com.example.minispring.beans.factory.support.DefaultListableBeanFactory;

public class CircularDependencyTest {
    
    public static void main(String[] args) {
        DefaultListableBeanFactory factory = new DefaultListableBeanFactory();
        
        // 注册 Bean A
        BeanDefinition bdA = new BeanDefinition(ServiceA.class);
        factory.registerBeanDefinition("serviceA", bdA);
        
        // 注册 Bean B
        BeanDefinition bdB = new BeanDefinition(ServiceB.class);
        factory.registerBeanDefinition("serviceB", bdB);
        
        // 获取 Bean（会触发循环依赖解决）
        ServiceA serviceA = (ServiceA) factory.getBean("serviceA");
        ServiceB serviceB = (ServiceB) factory.getBean("serviceB");
        
        // 验证
        System.out.println("serviceA 的 serviceB: " + (serviceA.getServiceB() != null));
        System.out.println("serviceB 的 serviceA: " + (serviceB.getServiceA() != null));
    }
}

class ServiceA {
    private ServiceB serviceB;
    
    public ServiceB getServiceB() {
        return serviceB;
    }
    
    public void setServiceB(ServiceB serviceB) {
        this.serviceB = serviceB;
    }
}

class ServiceB {
    private ServiceA serviceA;
    
    public ServiceA getServiceA() {
        return serviceA;
    }
    
    public void setServiceA(ServiceA serviceA) {
        this.serviceA = serviceA;
    }
}
```

## 五、小结

1. **一级缓存**：完整的单例 Bean
2. **二级缓存**：早期 Bean 引用（未填充属性）
3. **三级缓存**：Bean 工厂（用于创建 AOP 代理）
4. **循环依赖解决**：提前暴露早期引用
