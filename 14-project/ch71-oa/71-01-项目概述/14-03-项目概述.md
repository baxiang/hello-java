# 71.1 项目概述：OA 系统是什么？

## 一、先问一个问题

**什么是 OA 系统？**

### 生活例子：公司办公室

```
OA 系统 = 线上办公室
├── 员工档案（人事管理）
├── 打卡机（考勤管理）
├── 请假条（审批流程）
├── 公告栏（公告管理）
└── 文件柜（文档管理）
```

**OA 系统作用**：
- 提高办公效率
- 规范工作流程
- 信息透明共享

## 二、功能模块

### 核心功能

```
企业 OA 系统
├── 权限管理
│   ├── 用户管理
│   ├── 角色管理
│   └── 权限分配
├── 考勤管理
│   ├── 打卡签到
│   ├── 请假申请
│   └── 考勤统计
├── 审批流程
│   ├── 流程定义
│   ├── 流程审批
│   └── 流程监控
├── 公告管理
│   ├── 公告发布
│   └── 公告查看
└── 数据权限
    ├── 部门隔离
    └── 数据范围
```

## 三、技术栈

| 技术 | 作用 | 为什么选它 |
|------|------|------------|
| Spring Boot | Web 框架 | 快速开发 |
| Spring Security | 安全框架 | 认证授权 |
| MyBatis-Plus | ORM | 简化 SQL |
| MySQL | 数据库 | 数据存储 |
| Redis | 缓存 | 会话存储 |
| JWT | Token | 无状态认证 |

## 四、RBAC 权限模型

### 什么是 RBAC？

```
RBAC（Role-Based Access Control）
└── 基于角色的访问控制

核心思想：
└── 用户 → 角色 → 权限
```

### 关系图

```
用户（User）
  ↓
角色（Role）
├── 管理员
├── 部门经理
├── 普通员工
└── ...
  ↓
权限（Permission）
├── 用户管理
├── 审批流程
├── 查看报表
└── ...
```

### 数据库设计

```sql
-- 用户表
user (id, username, password, dept_id)

-- 角色表
role (id, name, code)

-- 权限表
permission (id, name, code, type)

-- 用户角色关联
user_role (user_id, role_id)

-- 角色权限关联
role_permission (role_id, permission_id)
```

## 五、项目架构

### 单体架构

```
用户
  ↓
Nginx
  ↓
Spring Boot 应用
  ↓
MySQL + Redis
```

### 功能分层

```
Controller 层（接收请求）
  ↓
Service 层（业务逻辑）
  ↓
Mapper 层（数据访问）
  ↓
Database（数据存储）
```

## 六、核心流程

### 登录流程

```
用户登录
  ↓
验证用户名密码
  ↓
生成 JWT Token
  ↓
返回 Token 和用户信息
  ↓
后续请求携带 Token
```

### 权限验证流程

```
请求接口
  ↓
解析 Token 获取用户
  ↓
查询用户角色
  ↓
查询角色权限
  ↓
验证是否有权限
  ↓
有权限→放行/无权限→拒绝
```

### 请假审批流程

```
员工提交请假
  ↓
部门经理审批
  ↓
人事备案
  ↓
更新考勤记录
```

## 七、小结

| 方面 | 说明 |
|------|------|
| 项目定位 | 企业办公自动化系统 |
| 核心特性 | RBAC 权限、工作流 |
| 技术栈 | Spring Boot + Vue.js |
| 学习收获 | 企业级权限管理 |

**OA 系统的特点**：
- 权限管理复杂
- 流程审批严格
- 数据安全性高
- 可扩展性强
