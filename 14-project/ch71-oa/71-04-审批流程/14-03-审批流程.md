# 71.4 审批流程：工作流如何实现？

## 一、先问一个问题

**审批流程的本质是什么？**

### 生活例子：公司审批

```
员工提交申请
  ↓
部门经理审批
  ↓
人事审批
  ↓
总经理审批（大额）
  ↓
审批完成
```

**审批流程**：
- 定义流程节点
- 按顺序审批
- 记录审批意见

## 二、数据库设计

### 流程定义表

```sql
CREATE TABLE `process_definition` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
    `name` VARCHAR(100) NOT NULL COMMENT '流程名称',
    `code` VARCHAR(50) NOT NULL UNIQUE COMMENT '流程编码',
    `type` VARCHAR(50) COMMENT '流程类型',
    `status` TINYINT DEFAULT 1,
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP
) COMMENT='流程定义表';
```

### 流程节点表

```sql
CREATE TABLE `process_node` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
    `process_id` BIGINT NOT NULL COMMENT '流程 ID',
    `name` VARCHAR(100) NOT NULL COMMENT '节点名称',
    `order` INT NOT NULL COMMENT '顺序',
    `approver_role` BIGINT COMMENT '审批人角色',
    `condition` VARCHAR(500) COMMENT '条件表达式'
) COMMENT='流程节点表';
```

### 流程实例表

```sql
CREATE TABLE `process_instance` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
    `process_id` BIGINT NOT NULL,
    `business_id` BIGINT COMMENT '业务 ID',
    `status` TINYINT DEFAULT 0 COMMENT '状态',
    `current_node` BIGINT COMMENT '当前节点',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP
) COMMENT='流程实例表';
```

### 审批记录表

```sql
CREATE TABLE `approval_record` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
    `instance_id` BIGINT NOT NULL,
    `node_id` BIGINT NOT NULL,
    `approver_id` BIGINT NOT NULL,
    `action` TINYINT COMMENT '操作',
    `comment` VARCHAR(500) COMMENT '意见',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP
) COMMENT='审批记录表';

-- 操作：1 通过/2 拒绝/3 转交
```

## 三、流程引擎

### 启动流程

```java
@Service
public class ProcessService {
    
    @Autowired
    private ProcessInstanceMapper instanceMapper;
    
    @Transactional
    public ProcessInstance startProcess(Long userId, String processCode, 
                                        Long businessId) {
        // 1. 查询流程定义
        ProcessDefinition process = processMapper.selectByCode(processCode);
        
        // 2. 创建流程实例
        ProcessInstance instance = new ProcessInstance();
        instance.setProcessId(process.getId());
        instance.setBusinessId(businessId);
        instance.setStatus(0);  // 进行中
        
        // 3. 获取第一个节点
        ProcessNode firstNode = nodeMapper.selectFirstByProcessId(process.getId());
        instance.setCurrentNode(firstNode.getId());
        
        instanceMapper.insert(instance);
        
        // 4. 通知第一个审批人
        notifyApprover(instance, firstNode);
        
        return instance;
    }
}
```

### 审批操作

```java
@Transactional
public void approve(Long instanceId, Long userId, int action, String comment) {
    // 1. 查询流程实例
    ProcessInstance instance = instanceMapper.selectById(instanceId);
    
    // 2. 查询当前节点
    ProcessNode currentNode = nodeMapper.selectById(instance.getCurrentNode());
    
    // 3. 记录审批意见
    ApprovalRecord record = new ApprovalRecord();
    record.setInstanceId(instanceId);
    record.setNodeId(currentNode.getId());
    record.setApproverId(userId);
    record.setAction(action);
    record.setComment(comment);
    recordMapper.insert(record);
    
    // 4. 判断审批结果
    if (action == 2) {  // 拒绝
        instance.setStatus(2);  // 已拒绝
        instanceMapper.updateById(instance);
        return;
    }
    
    // 5. 获取下一个节点
    ProcessNode nextNode = nodeMapper.selectNextNode(
        currentNode.getProcessId(), 
        currentNode.getOrder()
    );
    
    if (nextNode == null) {
        // 流程结束
        instance.setStatus(1);  // 已完成
        instanceMapper.updateById(instance);
        
        // 通知申请人
        notifyApplicant(instance);
    } else {
        // 流转到下一节点
        instance.setCurrentNode(nextNode.getId());
        instanceMapper.updateById(instance);
        
        // 通知下一审批人
        notifyApprover(instance, nextNode);
    }
}
```

```java
public ProcessNode getNextNode(ProcessInstance instance, Map<String, Object> variables) {
    ProcessNode currentNode = nodeMapper.selectById(instance.getCurrentNode());
    
    // 查询所有下一节点
    List<ProcessNode> nextNodes = nodeMapper.selectNextNodes(
        currentNode.getProcessId(), 
        currentNode.getOrder()
    );
    
    if (nextNodes.size() == 1) {
        // 只有一个下一节点，直接返回
        return nextNodes.get(0);
    }
    
    // 多个下一节点，根据条件判断
    for (ProcessNode node : nextNodes) {
        String condition = node.getCondition();
        if (evaluateCondition(condition, variables)) {
```

### 示例：请假流程

```
请假流程
├── 节点 1：部门经理审批
├── 节点 2：人事审批
└── 节点 3：总经理审批（请假>3 天）

条件：
- 请假<=3 天：节点 1 → 节点 2 → 完成
- 请假>3 天：节点 1 → 节点 2 → 节点 3 → 完成
```

## 六、小结

| 表名 | 作用 | 核心字段 |
|------|------|----------|
| process_definition | 流程定义 | name, code |
| process_node | 流程节点 | process_id, order |
| process_instance | 流程实例 | process_id, status |
| approval_record | 审批记录 | instance_id, approver_id |

**核心要点**：
- 流程定义和实例分离
- 节点顺序控制
- 条件分支处理
