# 70.1 项目概述：电商系统如何设计？

## 一、先问一个问题

**电商系统的核心是什么？**

### 生活例子：开超市

```
开超市
├── 进货（商品管理）
├── 顾客挑选（购物车）
├── 结账（订单）
└── 送货（物流）
```

**电商系统核心流程**：
1. 商品展示
2. 加入购物车
3. 下单支付
4. 发货配送

## 二、功能模块

### 前台功能（用户端）

```
用户端
├── 商品浏览
│   ├── 搜索商品
│   ├── 分类筛选
│   └── 商品详情
├── 购物车
│   ├── 添加商品
│   ├── 修改数量
│   └── 结算
├── 订单
│   ├── 提交订单
│   ├── 支付
│   └── 查看订单
└── 个人中心
    ├── 个人信息
    └── 订单管理
```

### 后台功能（管理端）

```
管理端
├── 商品管理
│   ├── 添加商品
│   ├── 库存管理
│   └── 上下架
├── 订单管理
│   ├── 订单列表
│   ├── 发货
│   └── 退款
├── 用户管理
└── 数据统计
```

## 三、技术栈

### 后端技术

| 技术 | 作用 | 为什么选它 |
|------|------|------------|
| Spring Boot | Web 框架 | 快速开发 |
| Spring Cloud Alibaba | 微服务 | 服务治理 |
| MyBatis-Plus | ORM | 简化 SQL |
| MySQL | 数据库 | 数据存储 |
| Redis | 缓存 | 高性能 |
| Elasticsearch | 搜索 | 全文检索 |
| RabbitMQ | 消息队列 | 异步解耦 |

### 前端技术

| 技术 | 作用 | 为什么选它 |
|------|------|------------|
| Vue.js 3 | 前端框架 | 易学、生态好 |
| Element Plus | UI 组件 | 美观、组件全 |
| Nuxt.js | SSR | SEO 友好 |

## 四、项目架构

### 单体架构（初期）

```
用户
  ↓
Nginx（负载均衡）
  ↓
Spring Boot 应用
  ↓
MySQL + Redis
```

### 微服务架构（后期）

```
用户
  ↓
API Gateway
  ↓
┌─────────┬─────────┬─────────┐
│用户服务  │商品服务  │订单服务  │
└─────────┴─────────┴─────────┘
  ↓
MySQL + Redis + Elasticsearch
```

## 五、核心流程

### 购物流程

```
1. 浏览商品
   ↓
2. 加入购物车
   ↓
3. 结算
   ↓
4. 提交订单
   ↓
5. 支付
   ↓
6. 发货
   ↓
7. 确认收货
```

### 订单状态流转

```
待付款 → 已付款 → 已发货 → 已完成
   ↓        ↓        ↓
  取消     退款     售后
```

## 六、数据库设计

### 核心表

| 表名 | 作用 | 核心字段 |
|------|------|----------|
| user | 用户 | username, password |
| product | 商品 | name, price, stock |
| sku | 库存单元 | product_id, specs, stock |
| cart | 购物车 | user_id, sku_id, quantity |
| order | 订单 | order_no, user_id, amount |
| order_item | 订单项 | order_id, sku_id, quantity |

## 七、小结

| 方面 | 说明 |
|------|------|
| 项目定位 | 电商平台、涵盖完整购物流程 |
| 技术栈 | Spring Boot + Vue.js |
| 核心功能 | 商品、购物车、订单 |
| 学习收获 | 电商系统开发经验 |

**电商系统的特点**：
- 业务流程复杂
- 并发量大
- 数据一致性要求高
- 可扩展性强
