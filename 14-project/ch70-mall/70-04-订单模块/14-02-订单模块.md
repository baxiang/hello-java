# 70.4 订单模块：订单状态如何流转？

## 一、先问一个问题

**订单的本质是什么？**

### 生活例子：餐厅点餐

```
点餐（提交订单）
  ↓
付款（已付款）
  ↓
厨房做菜（已发货）
  ↓
上菜（已完成）
```

**订单状态流转**：
- 待付款 → 已付款 → 已发货 → 已完成

## 二、数据库设计

### 订单表（order）

```sql
CREATE TABLE `order` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
    `order_no` VARCHAR(50) NOT NULL UNIQUE COMMENT '订单号',
    `user_id` BIGINT NOT NULL COMMENT '用户 ID',
    `total_amount` DECIMAL(10,2) COMMENT '订单金额',
    `pay_amount` DECIMAL(10,2) COMMENT '实付金额',
    `freight_amount` DECIMAL(10,2) COMMENT '运费',
    `status` TINYINT DEFAULT 0 COMMENT '状态',
    `payment_time` DATETIME COMMENT '支付时间',
    `delivery_time` DATETIME COMMENT '发货时间',
    `receive_time` DATETIME COMMENT '收货时间',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX `idx_user` (`user_id`),
    INDEX `idx_order_no` (`order_no`),
    INDEX `idx_status` (`status`)
) COMMENT='订单表';
```

### 订单项表（order_item）

```sql
CREATE TABLE `order_item` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
    `order_id` BIGINT NOT NULL COMMENT '订单 ID',
    `sku_id` BIGINT NOT NULL COMMENT 'SKU ID',
    `sku_name` VARCHAR(200) COMMENT '商品名称',
    `price` DECIMAL(10,2) COMMENT '单价',
    `quantity` INT COMMENT '数量',
    INDEX `idx_order` (`order_id`)
) COMMENT='订单项表';
```

## 三、订单状态

### 状态定义

```java
public enum OrderStatus {
    PENDING(0, "待付款"),
    PAID(1, "已付款"),
    SHIPPED(2, "已发货"),
    COMPLETED(3, "已完成"),
    CANCELLED(4, "已取消"),
    REFUNDED(5, "已退款");
}
```

### 状态流转图

```
待付款 (0)
├── 支付 → 已付款 (1)
└── 超时 → 已取消 (4)

已付款 (1)
└── 发货 → 已发货 (2)

已发货 (2)
└── 确认收货 → 已完成 (3)

已完成 (3)
└── 售后 → 已退款 (5)
```

## 四、创建订单

### 订单流程

```
1. 验证库存
   ↓
2. 扣减库存
   ↓
3. 生成订单号
   ↓
4. 创建订单
   ↓
5. 清空购物车
```

### 代码实现

```java
@Service
public class OrderService {
    
    @Autowired
    private SkuService skuService;
    
    @Transactional
    public Order createOrder(Long userId, List<OrderItemDTO> items) {
        // 1. 验证库存
        for (OrderItemDTO item : items) {
            Sku sku = skuMapper.selectById(item.getSkuId());
            if (sku.getStock() < item.getQuantity()) {
                throw new BusinessException(sku.getName() + " 库存不足");
            }
        }
        
        // 2. 生成订单号
        String orderNo = generateOrderNo();
        
        // 3. 创建订单
        Order order = new Order();
        order.setOrderNo(orderNo);
        order.setUserId(userId);
        order.setStatus(OrderStatus.PENDING.getCode());
        
        // 4. 计算金额
        BigDecimal totalAmount = BigDecimal.ZERO;
        for (OrderItemDTO item : items) {
            Sku sku = skuMapper.selectById(item.getSkuId());
            totalAmount = totalAmount.add(sku.getPrice().multiply(new BigDecimal(item.getQuantity())));
        }
        order.setTotalAmount(totalAmount);
        
        orderMapper.insert(order);
        
        // 5. 创建订单项
        for (OrderItemDTO item : items) {
            OrderItem orderItem = new OrderItem();
            orderItem.setOrderId(order.getId());
            orderItem.setSkuId(item.getSkuId());
            orderItem.setQuantity(item.getQuantity());
            orderItemMapper.insert(orderItem);
        }
        
        // 6. 扣减库存
        for (OrderItemDTO item : items) {
            skuService.decreaseStock(item.getSkuId(), item.getQuantity());
        }
        
        // 7. 清空购物车
        cartService.clearCart(userId);
        
        return order;
    }
    
    private String generateOrderNo() {
        return "ORD" + System.currentTimeMillis() + 
               ThreadLocalRandom.current().nextInt(1000, 9999);
    }
}
```

## 五、支付回调

```java
@Service
public class PaymentService {
    
    @Autowired
    private OrderService orderService;
    
    @Transactional
    public void payCallback(String orderNo, String transactionId) {
        // 1. 查询订单
        Order order = orderMapper.selectByOrderNo(orderNo);
        if (order == null) {
            throw new BusinessException("订单不存在");
        }
        
        // 2. 更新订单状态
        order.setStatus(OrderStatus.PAID.getCode());
        order.setPaymentTime(new Date());
        orderMapper.updateById(order);
        
        // 3. 记录支付信息
        Payment payment = new Payment();
        payment.setOrderNo(orderNo);
        payment.setTransactionId(transactionId);
        paymentMapper.insert(payment);
    }
}
```

## 六、订单超时取消

### 延迟队列

```java
@Component
public class OrderTimeoutListener {
    
    @Autowired
    private OrderService orderService;
    
    @RabbitListener(queues = "order.timeout.queue")
    public void handleTimeout(String orderNo) {
        Order order = orderMapper.selectByOrderNo(orderNo);
        
        // 检查订单状态
        if (order.getStatus() == OrderStatus.PENDING.getCode()) {
            // 取消订单
            orderService.cancelOrder(order.getId());
            
            // 恢复库存
            orderService.restoreStock(order.getId());
        }
    }
}
```

### 发送延迟消息

```java
public void sendTimeoutMessage(String orderNo) {
    rabbitTemplate.convertAndSend(
        "order.exchange",
        "order.timeout",
        orderNo,
        message -> {
            message.getMessageProperties().setDelay(30 * 60 * 1000); // 30 分钟
            return message;
        }
    );
}
```

## 七、小结

| 状态 | 代码 | 说明 |
|------|------|------|
| 待付款 | 0 | 订单创建，等待支付 |
| 已付款 | 1 | 支付成功 |
| 已发货 | 2 | 商家发货 |
| 已完成 | 3 | 确认收货 |
| 已取消 | 4 | 订单取消 |
| 已退款 | 5 | 退款完成 |

**核心要点**：
- 订单号唯一
- 状态流转控制
- 超时自动取消
- 库存扣减和恢复
