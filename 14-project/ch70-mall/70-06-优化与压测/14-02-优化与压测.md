## 二、缓存优化

### 多级缓存

```
用户请求
  ↓
浏览器缓存（静态资源）
  ↓
CDN 缓存（图片、CSS、JS）
  ↓
Nginx 缓存（页面缓存）
  ↓
Redis 缓存（热点数据）
  ↓
数据库（持久化）
```

### Redis 缓存策略

```java
@Service
public class ProductService {
    
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    
    // 缓存商品详情
    public ProductDetailVO getProductDetail(Long id) {
        String key = "product:detail:" + id;
        
        // 1. 查缓存
        ProductDetailVO vo = (ProductDetailVO) redisTemplate.opsForValue().get(key);
        if (vo != null) {
            return vo;
        }
        
        // 2. 查数据库
        vo = loadFromDatabase(id);
        
        // 3. 写缓存（随机过期时间，防止缓存雪崩）
        long ttl = 3600 + ThreadLocalRandom.current().nextInt(1800);
        redisTemplate.opsForValue().set(key, vo, ttl, TimeUnit.SECONDS);
        
        return vo;
    }
    
    // 缓存预热
    public void warmUpCache() {
        List<Long> hotProductIds = productMapper.selectHotProductIds();
        for (Long id : hotProductIds) {
            getProductDetail(id);  // 触发缓存
        }
    }
}
```

### 缓存更新策略

```java
// 更新商品时
@Transactional
public void updateProduct(Product product) {
    // 1. 更新数据库
    productMapper.updateById(product);
    
    // 2. 删除缓存（Cache Aside Pattern）
    String key = "product:detail:" + product.getId();
    redisTemplate.delete(key);
}
```


## 五、压测

### JMeter 压测

```
压测场景
├── 商品列表页（QPS 目标：1000）
├── 商品详情页（QPS 目标：2000）
├── 加入购物车（QPS 目标：500）
└── 提交订单（QPS 目标：200）
```

### 压测脚本

```java
@SpringBootTest
public class PerformanceTest {
    
    @Autowired
    private TestRestTemplate restTemplate;
    
    @Test
    public void testProductList() {
        // 并发 100，执行 1000 次
        for (int i = 0; i < 1000; i++) {
            restTemplate.getForObject("/api/products?page=1&size=10", String.class);
        }
    }
}
```

### 压测指标

| 指标 | 说明 | 目标 |
|------|------|------|
| QPS | 每秒请求数 | > 1000 |
| RT | 响应时间 | < 200ms |
| 成功率 | 成功请求比例 | > 99.9% |
| 错误率 | 失败请求比例 | < 0.1% |

## 六、CDN 加速

### 静态资源 CDN

```
用户请求
  ↓
CDN 节点（就近访问）
  ↓
源站（CDN 未命中）
```

### Nginx 配置

```nginx
# 静态资源缓存
location /static/ {
    expires 30d;
    add_header Cache-Control "public, immutable";
}

# 图片缓存
location ~* \.(jpg|jpeg|png|gif)$ {
    expires 7d;
    add_header Cache-Control "public";
}
```

## 七、小结

| 优化方式 | 效果 | 实施难度 |
|----------|------|----------|
| 缓存 | 高 | 低 |
| 索引 | 高 | 低 |
| 异步 | 中 | 中 |
| 读写分离 | 中 | 中 |
| 分库分表 | 高 | 高 |
| CDN | 高 | 低 |

1. 先优化缓存
2. 再优化数据库
3. 最后考虑架构调整
