# 73.6 分布式事务：Seata 如何使用？

## 一、分布式事务问题

```
本地事务
└── 单个数据库 → 没问题

分布式事务
├── 订单服务（数据库 A）
└── 库存服务（数据库 B）
    └── 如何保证一致性？
```

## 二、Seata AT 模式

```java
@GlobalTransactional
public Order createOrder(OrderDTO dto) {
    // 1. 创建订单
    orderMapper.insert(order);
    
    // 2. 扣减库存
    inventoryClient.decreaseStock(dto);
    
    // 3. 扣减余额
    paymentClient.pay(dto);
}
```

## 三、小结

**核心要点**：
- Seata AT 模式
- 全局事务注解
- 最终一致性
