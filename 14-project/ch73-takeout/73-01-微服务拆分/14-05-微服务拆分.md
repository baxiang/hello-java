# 73.1 微服务拆分：如何划分服务？

## 一、先问一个问题

**为什么要微服务拆分？**

### 生活例子：公司部门

```
小公司（单体）
└── 一人多职

大公司（微服务）
├── 销售部
├── 技术部
├── 财务部
└── 人力资源部
```

**微服务优势**：
- 独立开发部署
- 故障隔离
- 弹性扩展

## 二、服务划分

### 外卖平台服务

```
外卖平台
├── 用户服务（user-service）
├── 商家服务（merchant-service）
├── 订单服务（order-service）
├── 支付服务（payment-service）
└── 配送服务（delivery-service）
```

### 拆分原则

```
按业务领域拆分
├── 高内聚（相关功能在一起）
└── 低耦合（服务间依赖少）
```

## 三、服务注册与发现

### Nacos 配置

```yaml
spring:
  cloud:
    nacos:
      discovery:
        server-addr: localhost:8848
        service: ${spring.application.name}
```

### 服务注册

```java
@SpringBootApplication
@EnableDiscoveryClient
public class UserServiceApplication {
    public static void main(String[] args) {
        SpringApplication.run(UserServiceApplication.class, args);
    }
}
```

## 四、小结

| 服务 | 职责 | 端口 |
|------|------|------|
| 用户服务 | 登录认证 | 8081 |
| 商家服务 | 店铺菜品 | 8082 |
| 订单服务 | 下单支付 | 8083 |
| 支付服务 | 支付处理 | 8084 |
| 配送服务 | 配送管理 | 8085 |

**核心要点**：
- 按业务领域拆分
- 服务注册到 Nacos
- 独立部署
