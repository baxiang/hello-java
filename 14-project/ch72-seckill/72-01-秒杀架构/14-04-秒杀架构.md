# 72.1 秒杀架构：如何应对高并发？

## 一、先问一个问题

**秒杀的核心挑战是什么？**

### 生活例子：春运抢票

```
平时买票
└── 100 人抢 100 张票（轻松）

春运抢票
└── 10000 人抢 100 张票（难）
```

**秒杀特点**：
- 瞬时大流量
- 库存有限
- 防止超卖
- 防止作弊

## 二、核心问题

### 问题 1：数据库扛不住

```
正常情况
└── 100 QPS → MySQL 轻松应对

秒杀情况
└── 10000 QPS → MySQL 直接挂掉
```

**解决方案**：
- Redis 预减库存
- 消息队列异步

### 问题 2：超卖

```
库存 100 个
├── 用户 A 查询库存：100
├── 用户 B 查询库存：100
├── 用户 A 下单扣减：100-1=99
└── 用户 B 下单扣减：99-1=98

如果并发：
├── 用户 A 查询：100
├── 用户 B 查询：100
├── 用户 A 扣减：100-1=99
└── 用户 B 扣减：100-1=99（超卖！）
```

**解决方案**：
- 分布式锁
- 乐观锁

### 问题 3：黄牛刷单

```
黄牛用脚本
└── 1 秒发起 1000 次请求

正常用户
└── 1 秒发起 1 次请求
```

**解决方案**：
- 限流
- 验证码
- IP 限制

## 三、架构设计

### 秒杀架构

```
用户请求
  ↓
Nginx 限流
  ↓
Sentinel 限流
  ↓
Redis 预减库存
  ↓
RabbitMQ 异步下单
  ↓
MySQL 最终扣减
```

### 核心流程

```
1. 秒杀开始前
   └── 预热：库存加载到 Redis

2. 秒杀进行中
   ├── Redis 预减库存（高性能）
   ├── 成功后发送 MQ 消息
   └── 立即返回排队中

3. 异步处理
   ├── 消费者接收消息
   ├── 创建订单
   └── MySQL 扣减库存
```

## 四、数据库设计

### 秒杀商品表

```sql
CREATE TABLE `seckill_product` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
    `product_id` BIGINT NOT NULL,
    `seckill_price` DECIMAL(10,2) NOT NULL,
    `stock` INT NOT NULL,
    `start_time` DATETIME NOT NULL,
    `end_time` DATETIME NOT NULL,
    `version` INT DEFAULT 0 COMMENT '乐观锁版本',
    INDEX `idx_time` (`start_time`, `end_time`)
) COMMENT='秒杀商品表';
```

### 秒杀订单表

```sql
CREATE TABLE `seckill_order` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
    `user_id` BIGINT NOT NULL,
    `order_no` VARCHAR(50) NOT NULL,
    `product_id` BIGINT NOT NULL,
    `status` TINYINT DEFAULT 0,
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY `uk_user_product` (`user_id`, `product_id`)
) COMMENT='秒杀订单表';
```

## 五、接口设计

### 秒杀接口

```java
@RestController
@RequestMapping("/api/seckill")
public class SeckillController {
    
    @Autowired
    private SeckillService seckillService;
    
    @PostMapping("/execute/{productId}")
    public Result<String> seckill(
        @PathVariable Long productId,
        @RequestParam String token
    ) {
        String orderNo = seckillService.seckill(productId, token);
        return Result.success(orderNo);
    }
    
    @GetMapping("/result/{productId}")
    public Result<SeckillResult> getResult(
        @PathVariable Long productId
    ) {
        SeckillResult result = seckillService.getResult(productId);
        return Result.success(result);
    }
}
```

## 六、小结

| 问题 | 解决方案 | 技术 |
|------|----------|------|
| 数据库扛不住 | Redis+MQ | 缓存 + 异步 |
| 超卖 | 分布式锁 | Redis 锁 |
| 黄牛刷单 | 限流 | Sentinel |

**核心思想**：
- 能缓存的都缓存
- 能异步的都异步
- 限流保护系统
