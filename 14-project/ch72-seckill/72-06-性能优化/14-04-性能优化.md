# 72.6 性能优化：如何压测调优？

## 一、先问一个问题

**如何找到性能瓶颈？**

### 生活例子：赛车调优

```
不调优
└── 随便开 → 性能一般

调优
├── 测速（压测）
├── 找问题（瓶颈）
├── 调引擎（优化）
└── 再测速（验证）
```

**性能优化流程**：
1. 压测
2. 找瓶颈
3. 优化
4. 验证

## 二、压测方案

### JMeter 压测

```
压测场景
├── 秒杀接口（QPS 目标：5000）
├── Redis 预减（QPS 目标：10000）
└── MQ 异步（QPS 目标：2000）
```

### 压测脚本

```java
@SpringBootTest
public class SeckillPressureTest {
    
    @Autowired
    private TestRestTemplate restTemplate;
    
    @Test
    public void testSeckill() {
        // 并发 1000，执行 10000 次
        for (int i = 0; i < 10000; i++) {
            restTemplate.postForObject(
                "/api/seckill/execute/1?token=xxx",
                null,
                String.class
            );
        }
    }
}
```

## 三、优化方案

### 缓存优化

```java
// 商品详情缓存
@Cacheable(value = "seckill:product", key = "#productId")
public SeckillProduct getProduct(Long productId) {
    return productMapper.selectById(productId);
}
```

### 数据库优化

```sql
-- 添加索引
CREATE INDEX idx_time ON seckill_product(start_time, end_time);

-- 分表
seckill_order_0, seckill_order_1, ... seckill_order_9
```

### 连接池优化

```yaml
spring:
  datasource:
    hikari:
      maximum-pool-size: 50
      minimum-idle: 10
      connection-timeout: 30000
```

## 四、小结

| 优化项 | 目标 | 手段 |
|--------|------|------|
| 缓存 | 减少 DB 查询 | Redis |
| 异步 | 快速响应 | MQ |
| 限流 | 保护系统 | Sentinel |
| 锁 | 防超卖 | Redis 锁 |

**核心要点**：
- 压测找到瓶颈
- 针对性优化
- 持续验证
