# 77.1 代码规范：如何写出好代码？

## 一、先问一个问题

**什么是好代码？**

### 生活例子：整洁的房间

```
好代码
├── 命名清晰（物品标签）
├── 结构合理（物品摆放）
└── 注释适当（使用说明）
```

**好代码标准**：
- 易读
- 易维护
- 易扩展

## 二、命名规范

### 命名规则

```java
// 类名：大驼峰
public class UserService { }

// 方法名：小驼峰
public void getUserById() { }

// 变量名：小驼峰
private String userName;

// 常量名：全大写
public static final int MAX_COUNT = 100;

// 包名：全小写
package com.example.user.service;
```

### 命名建议

```java
// ✅ 好的命名
public class OrderService { }
public List<User> findActiveUsers() { }
private int maxRetryCount;

// ❌ 坏的命名
public class OService { }  // 太短
public List<User> list() { }  // 不明确
private int n;  // 无意义
```

## 三、代码结构

### 类结构顺序

```java
public class UserService {
    
    // 1. 常量
    private static final int MAX_RETRY = 3;
    
    // 2. 静态变量
    private static Logger logger = LoggerFactory.getLogger(UserService.class);
    
    // 3. 成员变量
    @Autowired
    private UserMapper userMapper;
    
    // 4. 构造方法
    public UserService() { }
    
    // 5. 公有方法
    public User getUserById(Long id) { }
    
    // 6. 私有方法
    private void validateUser(User user) { }
}
```

### 方法长度

```java
// ✅ 好：方法短小
public User getUser(Long id) {
    validateId(id);
    return userMapper.selectById(id);
}

// ❌ 坏：方法过长
public User getUser(Long id) {
    // 100 行代码...
    // 应该拆分成多个小方法
}
```

## 四、注释规范

### 类注释

```java
/**
 * 用户服务类
 * 负责用户相关的业务逻辑
 * 
 * @author张三
 * @version 1.0
 */
public class UserService { }
```

### 方法注释

```java
/**
 * 根据 ID 查询用户
 * 
 * @param id 用户 ID
 * @return 用户对象
 * @throws BusinessException 用户不存在时抛出
 */
public User getUserById(Long id) { }
```

## 五、小结

| 方面 | 规范 | 示例 |
|------|------|------|
| 类名 | 大驼峰 | UserService |
| 方法名 | 小驼峰 | getUserById |
| 变量名 | 小驼峰 | userName |
| 常量名 | 全大写 | MAX_COUNT |
| 包名 | 全小写 | com.example |

**核心要点**：
- 命名清晰
- 方法短小
- 注释适当
