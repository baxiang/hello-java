# 77.2 Git 工作流：如何管理分支？

## 一、先问一个问题

**如何管理代码版本？**

### 生活例子：文档版本

```
V1.0（初稿）
  ↓
V1.1（修改）
  ↓
V2.0（正式版）
```

**Git 工作流**：
- 分支管理
- 版本控制
- 协作开发

## 二、Git Flow

### 分支定义

```
main/master    → 生产分支（稳定）
develop        → 开发分支（日常开发）
feature/*      → 功能分支（新功能）
release/*      → 发布分支（预发布）
hotfix/*       → 热修复分支（紧急修复）
```

### 工作流程

```
1. 从 develop 创建 feature 分支
   ↓
2. 开发功能
   ↓
3. 合并到 develop
   ↓
4. 创建 release 分支
   ↓
5. 测试修复
   ↓
6. 合并到 main 和 develop
   ↓
7. 打标签
```

## 三、常用命令

### 创建功能分支

```bash
# 从 develop 创建功能分支
git checkout develop
git checkout -b feature/user-login

# 开发完成后合并
git checkout develop
git merge feature/user-login
git branch -d feature/user-login
```

### 发布流程

```bash
# 创建发布分支
git checkout -b release/v1.0 develop

# 测试修复后
git checkout main
git merge release/v1.0
git tag v1.0

git checkout develop
git merge release/v1.0
git branch -d release/v1.0
```

### 紧急修复

```bash
# 从 main 创建热修复分支
git checkout -b hotfix/bug-fix main

# 修复后
git checkout main
git merge hotfix/bug-fix
git tag v1.0.1

git checkout develop
git merge hotfix/bug-fix
git branch -d hotfix/bug-fix
```

## 四、提交规范

### Commit Message 格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### 类型定义

```
feat:     新功能
fix:      修复 bug
docs:     文档更新
style:    代码格式
refactor: 重构
test:     测试
chore:    构建/工具
```

### 示例

```bash
git commit -m "feat(user): 添加用户登录功能

- 实现登录接口
- 添加 JWT 认证
- 编写单元测试

Closes #123"
```

## 五、小结

| 分支 | 用途 | 来源 |
|------|------|------|
| main | 生产环境 | - |
| develop | 日常开发 | main |
| feature | 新功能 | develop |
| release | 预发布 | develop |
| hotfix | 紧急修复 | main |

**核心要点**：
- 分支职责明确
- 提交规范
- 版本标签
