# 77.4 单元测试：如何编写测试用例？

## 一、先问一个问题

**为什么要写单元测试？**

### 生活例子：产品质量检测

```
不测试
└── 直接发货 → 可能有质量问题

测试
├── 出厂检测
└── 质量保证
```

**单元测试作用**：
- 保证质量
- 快速发现问题
- 方便重构

## 二、JUnit 基础

### 引入依赖

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>
```

### 基本测试

```java
@SpringBootTest
public class UserServiceTest {
    
    @Autowired
    private UserService userService;
    
    @Test
    public void testGetUserById() {
        // 准备数据
        Long userId = 1L;
        
        // 执行测试
        User user = userService.getUserById(userId);
        
        // 断言验证
        assertNotNull(user);
        assertEquals("张三", user.getUsername());
    }
}
```

### 常用注解

```java
@SpringBootTest          // Spring Boot 测试
@Test                    // 测试方法
@BeforeEach              // 每个测试前执行
@AfterEach               // 每个测试后执行
@BeforeAll               // 所有测试前执行
@AfterAll                // 所有测试后执行
@DisplayName("测试显示名称")
```

## 三、Mockito Mock

### Mock 对象

```java
@SpringBootTest
public class OrderServiceTest {
    
    @Mock
    private OrderMapper orderMapper;
    
    @InjectMocks
    private OrderService orderService;
    
    @Test
    public void testCreateOrder() {
        // 准备 Mock 数据
        Order mockOrder = new Order();
        mockOrder.setId(1L);
        
        when(orderMapper.insert(any())).thenReturn(1);
        
        // 执行测试
        Order order = new Order();
        orderService.createOrder(order);
        
        // 验证
        verify(orderMapper, times(1)).insert(any());
    }
}
```

### 常用方法

```java
// Mock 返回值
when(service.getById(1L)).thenReturn(new User());

// Mock 异常
when(service.getById(1L)).thenThrow(new BusinessException());

// 验证调用
verify(service, times(1)).getById(1L);
verify(service, never()).delete(1L);
```

## 四、测试覆盖率

### JaCoCo 配置

```xml
<plugin>
    <groupId>org.jacoco</groupId>
    <artifactId>jacoco-maven-plugin</artifactId>
    <version>0.8.8</version>
    <executions>
        <execution>
            <goals>
                <goal>prepare-agent</goal>
            </goals>
        </execution>
        <execution>
            <id>report</id>
            <phase>test</phase>
            <goals>
                <goal>report</goal>
            </goals>
        </execution>
    </executions>
</plugin>
```

### 生成报告

```bash
mvn test
mvn jacoco:report

# 查看报告
# target/site/jacoco/index.html
```

## 五、小结

| 注解 | 作用 | 示例 |
|------|------|------|
| @Test | 测试方法 | @Test |
| @BeforeEach | 每个测试前 | @BeforeEach |
| @Mock | Mock 对象 | @Mock |
| @InjectMocks | 注入 Mock | @InjectMocks |

**核心要点**：
- 测试用例编写
- Mock 对象使用
- 测试覆盖率
