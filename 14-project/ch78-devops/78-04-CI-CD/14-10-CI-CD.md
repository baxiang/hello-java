# 78.4 CI/CD：如何实现持续集成？

## 一、先问一个问题

**什么是 CI/CD？**

### 生活例子：工厂流水线

```
传统部署
└── 手动打包 → 手动部署 → 慢

CI/CD
├── 自动构建
├── 自动测试
└── 自动部署 → 快
```

**CI/CD 优势**：
- 自动化
- 快速迭代
- 减少人为错误

## 二、GitHub Actions

### 工作流配置

```yaml
# .github/workflows/deploy.yml
name: Deploy

on:
  push:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v2
      
      - name: Set up JDK 17
        uses: actions/setup-java@v2
        with:
          java-version: '17'
      
      - name: Build with Maven
        run: mvn clean package -DskipTests
      
      - name: Upload artifact
        uses: actions/upload-artifact@v2
        with:
          name: app
          path: target/myapp.jar
```

### 部署到服务器

```yaml
  deploy:
    needs: build
    runs-on: ubuntu-latest
    
    steps:
      - name: Deploy to server
        uses: easingthemes/ssh-deploy@v2
        with:
          SSH_PRIVATE_KEY: ${{ secrets.SSH_KEY }}
          REMOTE_HOST: ${{ secrets.SERVER_IP }}
          REMOTE_USER: root
          SOURCE: "target/myapp.jar"
          TARGET: "/opt/app/"
      
      - name: Restart service
        uses: appleboy/ssh-action@master
        with:
          host: ${{ secrets.SERVER_IP }}
          username: root
          key: ${{ secrets.SSH_KEY }}
          script: |
            systemctl restart myapp
```

## 三、Jenkins

### 流水线配置

```groovy
pipeline {
    agent any
    
    stages {
        stage('Checkout') {
            steps {
                git 'https://github.com/user/repo.git'
            }
        }
        
        stage('Build') {
            steps {
                sh 'mvn clean package -DskipTests'
            }
        }
        
        stage('Deploy') {
            steps {
                sh 'scp target/myapp.jar user@server:/opt/app/'
                sh 'ssh user@server systemctl restart myapp'
            }
        }
    }
}
```

## 四、小结

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| GitHub Actions | 集成 GitHub | GitHub 项目 |
| Jenkins | 功能强大 | 企业项目 |
| GitLab CI | 集成 GitLab | GitLab 项目 |

**核心要点**：
- 工作流配置
- 自动构建
- 自动部署
