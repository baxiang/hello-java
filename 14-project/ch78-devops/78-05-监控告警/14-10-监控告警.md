# 78.5 监控告警：如何监控系统？

## 一、先问一个问题

**为什么要监控？**

### 生活例子：汽车仪表盘

```
没有监控
└── 出问题不知道

有监控
├── 实时状态
├── 异常告警
└── 及时处理
```

**监控作用**：
- 实时状态
- 异常告警
- 性能分析

## 二、Prometheus + Grafana

### 架构

```
应用 → Prometheus（采集） → Grafana（展示）
                              ↓
                          告警通知
```

### Prometheus 配置

```yaml
# prometheus.yml
global:
  scrape_interval: 15s

scrape_configs:
  - job_name: 'spring-boot'
    static_configs:
      - targets: ['localhost:8080']
    metrics_path: '/actuator/prometheus'
```

### Spring Boot 集成

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>
<dependency>
    <groupId>io.micrometer</groupId>
    <artifactId>micrometer-registry-prometheus</artifactId>
</dependency>
```

```yaml
management:
  endpoints:
    web:
      exposure:
        include: prometheus,health,metrics
```

## 三、Grafana 配置

### 添加数据源

```
1. 登录 Grafana
2. Configuration → Data Sources
3. Add data source → Prometheus
4. URL: http://localhost:9090
5. Save & Test
```

### 导入仪表盘

```
1. Create → Dashboard
2. Import
3. 输入模板 ID（如：10280）
4. 选择数据源
5. Import
```

## 四、告警配置

### Prometheus 告警规则

```yaml
# alerting.yml
groups:
  - name: example
    rules:
      - alert: HighCPU
        expr: process_cpu_usage > 0.8
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "CPU 使用率过高"
          description: "CPU 使用率 {{ $value }}"
```

### 告警通知

```yaml
# alertmanager.yml
route:
  receiver: 'email'

receivers:
  - name: 'email'
    email_configs:
      - to: 'admin@example.com'
        from: 'alert@example.com'
        smarthost: 'smtp.example.com:587'
```

## 五、小结

| 组件 | 作用 | 说明 |
|------|------|------|
| Prometheus | 数据采集 | 采集指标 |
| Grafana | 数据展示 | 仪表盘 |
| Alertmanager | 告警通知 | 发送邮件/短信 |

**核心要点**：
- Prometheus 配置
- Grafana 仪表盘
- 告警规则
