# 22.2 通配符与类型擦除（生活类比：万能接收器与隐形标签）

## 一、先问一个问题

**问题**：`List<Integer>` 不是 `List<Number>` 的子类型，那如何编写通用的集合方法？

### 生活例子：万能接收器

想象快递站：
- "只收苹果箱"（`List<Apple>`）→ 拒收橘子箱
- "收任何水果箱"（`List<? extends Fruit>`）→ 苹果箱、橘子箱都行
- "往这个箱子里放水果"（`List<? super Apple>`）→ 能放苹果的水果箱都行

通配符就是 Java 泛型的"万能接收器"。

## 二、类型通配符

### 2.1 无界通配符 `<?>`

```java
// 可以接受任何类型的 List
public void printList(List<?> list) {
    for (Object obj : list) {
        System.out.println(obj);
    }
}

printList(new ArrayList<String>());
printList(new ArrayList<Integer>());

// 注意：不能添加元素（除了 null）
// list.add("A");  // 编译错误
```

### 2.2 上界通配符 `<? extends T>`

```java
// 可以接受 Number 或其子类的 List
public double sum(List<? extends Number> list) {
    double total = 0;
    for (Number num : list) {
        total += num.doubleValue();  // 可以安全读取为 Number
    }
    return total;
}

List<Integer> ints = Arrays.asList(1, 2, 3);
List<Double> doubles = Arrays.asList(1.1, 2.2, 3.3);
System.out.println(sum(ints));     // 6.0
System.out.println(sum(doubles));  // 6.6

// 注意：不能添加元素（编译器无法确定具体类型）
// list.add(1);  // 编译错误
```

### 2.3 下界通配符 `<? super T>`

```java
// 可以接受 Integer 或其父类的 List
public void addNumbers(List<? super Integer> list) {
    list.add(1);
    list.add(2);
    list.add(3);  // 可以安全添加 Integer
}

List<Integer> ints = new ArrayList<>();
List<Number> numbers = new ArrayList<>();
List<Object> objects = new ArrayList<>();

addNumbers(ints);     // OK
addNumbers(numbers);  // OK
addNumbers(objects);  // OK

// 注意：读取只能得到 Object
Object obj = list.get(0);
```

## 三、PECS 原则

**Producer Extends, Consumer Super**

- 如果只需**读取**（生产者）→ 用 `<? extends T>`
- 如果只需**写入**（消费者）→ 用 `<? super T>`
- 既要读又要写 → 不用通配符，用精确类型 `<T>`

```java
// 经典示例：拷贝列表
public static <T> void copy(List<? super T> dest, List<? extends T> src) {
    for (int i = 0; i < src.size(); i++) {
        dest.set(i, src.get(i));
    }
}

List<Number> dest = new ArrayList<>(Collections.nCopies(3, 0));
List<Integer> src = Arrays.asList(1, 2, 3);
copy(dest, src);  // src 是生产者（extends），dest 是消费者（super）
```

## 四、类型擦除

### 4.1 什么是类型擦除

泛型只在编译时存在，运行时泛型信息被擦除：

```java
Box<String> stringBox = new Box<>();
Box<Integer> intBox = new Box<>();

// 运行时类型相同
System.out.println(stringBox.getClass() == intBox.getClass());  // true

// 擦除后 Box<T> 变成 Box<Object>（无边界）或 Box<Number>（有边界 extends Number）
```

### 4.2 类型擦除的限制

```java
// ❌ 不能创建泛型数组
// T[] array = new T[10];

// ✅ 用 Object 数组 + 强转
public class GenericArray<T> {
    private Object[] array;

    public GenericArray(int size) { array = new Object[size]; }

    @SuppressWarnings("unchecked")
    public T get(int index) { return (T) array[index]; }
}

// ❌ 不能用 instanceof 检查泛型类型
// if (obj instanceof List<String>) {}  // 编译错误

// ✅ 可以用无界通配符
if (obj instanceof List<?>) {}

// ❌ 不能实例化类型参数
// new T();  // 编译错误

// ✅ 用 Class<T> 反射创建
public <T> T create(Class<T> clazz) throws Exception {
    return clazz.getDeclaredConstructor().newInstance();
}
```

### 4.3 桥接方法

编译器自动生成桥接方法以保持多态：

```java
class Box<T> {
    public void set(T value) {}
}

class StringBox extends Box<String> {
    @Override
    public void set(String value) {  // 你写的方法
        System.out.println("StringBox.set");
    }

    // 编译器自动生成的桥接方法
    // public void set(Object value) { set((String) value); }
}
```

## 五、常见疑问

**Q1：`List<Object>` 和 `List<?>` 有什么区别？**
A：`List<Object>` 只能接受 Object 类型的元素；`List<?>` 可以接受任何类型的 List，但不能添加元素。

**Q2：为什么 `List<Integer>` 不是 `List<Number>` 的子类型？**
A：如果成立，就可以向 `List<Integer>` 中添加 Double，破坏类型安全。这就是泛型不变性。

**Q3：如何绕过类型擦除获取泛型类型？**
A：通过反射获取父类的泛型参数（`getGenericSuperclass()`），这是 Gson/Jackson 等库的原理。

## 六、小结

| 通配符 | 方向 | 能读 | 能写 | 场景 |
|--------|------|------|------|------|
| `<?>` | 无界 | Object | null | 通用打印/展示 |
| `<? extends T>` | 上界 | T | null | 只读（生产者） |
| `<? super T>` | 下界 | Object | T | 只写（消费者） |

| 擦除限制 | 替代方案 |
|----------|----------|
| 不能 new T() | 传入 Class<T> 反射 |
| 不能 new T[] | 用 Object[] + 强转 |
| 不能 instanceof List<String> | 用 instanceof List<?> |
