# 22.1 泛型基础：类型安全容器（生活类比：带标签的收纳盒）

## 一、先问一个问题

**问题**：如何让编译器帮你检查类型，避免运行时的 ClassCastException？

### 生活例子：带标签的收纳盒

- 没有标签的盒子里什么都装，拿出时不确定是什么类型（Object）
- 贴了标签的盒子只能装指定类型，拿出一定是那种类型（泛型）

泛型就是给集合"贴标签"——编译时检查类型，运行时自动转型。

## 二、泛型类

```java
public class Box<T> {
    private T content;

    public void set(T content) { this.content = content; }
    public T get() { return content; }
}

// 使用
Box<String> strBox = new Box<>();
strBox.set("Hello");
String value = strBox.get();  // 无需强转

Box<Integer> intBox = new Box<>();
intBox.set(100);
Integer num = intBox.get();
```

### 多个类型参数

```java
public class Pair<K, V> {
    private K key;
    private V value;

    public Pair(K key, V value) {
        this.key = key;
        this.value = value;
    }

    public K getKey() { return key; }
    public V getValue() { return value; }
}

Pair<String, Integer> pair = new Pair<>("age", 18);
```

## 三、泛型方法

泛型方法有自己的类型参数，不依赖类的泛型：

```java
public class Utils {
    // 泛型声明在返回值前
    public static <T> T getFirst(T[] array) {
        if (array == null || array.length == 0) return null;
        return array[0];
    }

    // 多类型参数
    public static <T, U> List<U> map(List<T> list, Function<T, U> func) {
        List<U> result = new ArrayList<>();
        for (T item : list) {
            result.add(func.apply(item));
        }
        return result;
    }
}

// 使用
String first = Utils.getFirst(new String[]{"A", "B", "C"});
List<Integer> lengths = Utils.map(List.of("Hi", "Hello"), String::length);
```

## 四、泛型边界

### 4.1 单边界

```java
// T 必须是 Number 或其子类
public class NumberBox<T extends Number> {
    private T value;

    public double getDoubleValue() {
        return value.doubleValue();  // 可以调用 Number 的方法
    }
}

NumberBox<Integer> intBox = new NumberBox<>();
// NumberBox<String> strBox = new NumberBox<>();  // 编译错误
```

### 4.2 多边界

```java
// T 必须同时实现 Comparable 和 Serializable
// 类只能放第一个，接口放后面
public class SortedBox<T extends Comparable<T> & Serializable> {
    private T value;

    public int compareTo(T other) {
        return value.compareTo(other);
    }
}
```

## 五、实战示例

### 5.1 通用 Result 类

```java
public class Result<T> {
    private Integer code;
    private String message;
    private T data;

    public static <T> Result<T> success(T data) {
        return new Result<>(200, "success", data);
    }

    public static <T> Result<T> error(String message) {
        return new Result<>(500, message, null);
    }
}

Result<User> userResult = Result.success(new User("张三", 18));
Result<Void> errorResult = Result.error("操作失败");
```

### 5.2 泛型缓存

```java
public class Cache<K, V> {
    private final Map<K, V> map = new HashMap<>();

    public void put(K key, V value) { map.put(key, value); }
    public V get(K key) { return map.get(key); }
    public V getOrDefault(K key, V defaultValue) {
        return map.getOrDefault(key, defaultValue);
    }
}
```

## 六、常见疑问

**Q1：泛型类和泛型方法有什么区别？**
A：泛型类的 T 在整个类中有效；泛型方法的 T 只在该方法中有效，可以与类的 T 不同。

**Q2：为什么 `new T()` 不行？**
A：类型擦除导致运行时 T 的信息丢失，无法实例化。可用 `Class<T>` 传入反射创建。

**Q3：泛型能用基本类型吗？**
A：不能。`List<int>` 编译错误，必须用 `List<Integer>`，自动装箱/拆箱。

## 七、小结

| 概念 | 语法 | 说明 |
|------|------|------|
| 泛型类 | `class Box<T>` | 整个类使用 T |
| 泛型方法 | `<T> T method()` | 方法级别类型参数 |
| 单边界 | `<T extends Number>` | 限制 T 的上界 |
| 多边界 | `<T extends A & B>` | 多个约束 |
