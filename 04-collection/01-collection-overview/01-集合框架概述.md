# 17.1 集合框架概述：数据收纳的艺术（生活类比：收纳箱体系）

## 一、先问一个问题

**问题**：数组已经能存储多个数据了，为什么还需要集合？

### 生活例子：收纳箱体系

想象你在家整理物品：
- **数组**像一个固定大小的收纳箱——装满了就不能再加，空着也浪费空间
- **集合**像一套可扩展的收纳箱——List 是有序箱（贴标签）、Set 是去重箱（不重复）、Map 是标签箱（键值配对）

集合框架就是 Java 的"收纳箱体系"！

## 二、集合 vs 数组

| 特性 | 数组 | 集合 |
|------|------|------|
| 长度 | 固定 | 动态 |
| 类型 | 可存基本类型 | 只存引用类型 |
| 功能 | 简单存取 | 排序、查找、去重等 |
| 内存 | 连续 | 不一定连续 |

```java
String[] array = new String[3];     // 固定长度
List<String> list = new ArrayList<>();  // 动态扩容
list.add("A");
list.add("B");
list.add("C");
list.add("D");  // 自动扩容，无需手动处理
```

## 三、集合体系结构

```
Collection（单列集合）
├── List（有序、可重复）
│   ├── ArrayList    — 动态数组，查询快
│   ├── LinkedList   — 双向链表，增删快
│   └── Vector       — 线程安全的动态数组（少用）
└── Set（无序、不可重复）
    ├── HashSet      — 哈希表，最快
    ├── LinkedHashSet— 保持插入顺序
    └── TreeSet      — 红黑树，自动排序

Map（双列集合，键值对）
├── HashMap         — 哈希表，最常用
├── LinkedHashMap   — 保持插入/访问顺序
├── TreeMap         — 按 key 排序
└── Hashtable       — 线程安全（少用）
```

## 四、Collection 接口通用方法

```java
Collection<String> col = new ArrayList<>();

// 添加
col.add("A");
col.addAll(List.of("B", "C"));

// 删除
col.remove("A");
col.removeAll(List.of("B"));
col.clear();

// 判断
boolean empty = col.isEmpty();
boolean has = col.contains("A");
int size = col.size();

// 遍历
for (String s : col) {
    System.out.println(s);
}
col.forEach(System.out::println);
```

## 五、泛型基础

泛型让集合"只能装一种类型"，避免运行时类型错误：

```java
// ❌ 不用泛型（可能出错）
List list = new ArrayList();
list.add("A");
list.add(123);           // 编译通过，运行时出错
String s = (String) list.get(1);  // ClassCastException

// ✅ 使用泛型（编译时检查）
List<String> list = new ArrayList<>();
list.add("A");
// list.add(123);        // 编译错误，提前发现
String s = list.get(0);  // 无需强转
```

## 六、常见疑问

**Q1：集合只能存引用类型，基本类型怎么办？**
A：使用包装类。`List<Integer>` 代替 `List<int>`，自动装箱/拆箱。

**Q2：什么时候用 List，什么时候用 Set？**
A：需要保留重复元素和顺序用 List，需要去重用 Set，键值映射用 Map。

**Q3：为什么 Vector 和 Hashtable 不推荐使用？**
A：它们用 synchronized 方法实现线程安全，性能差。多线程场景推荐 ConcurrentHashMap 和 CopyOnWriteArrayList。

## 七、小结

| 概念 | 说明 | 特点 |
|------|------|------|
| List | 有序可重复 | ArrayList 查询快，LinkedList 增删快 |
| Set | 无序不重复 | HashSet 最快，TreeSet 可排序 |
| Map | 键值对 | HashMap 最常用，TreeMap 按 key 排序 |
| 泛型 | 类型约束 | 编译时检查，避免运行时错误 |
