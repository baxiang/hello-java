# 21.2 不可变集合与同步集合（生活类比：博物馆展柜 vs 联合办公区）

## 一、先问一个问题

**问题**：如何创建只读集合？如何在多线程环境下安全使用集合？

### 生活例子：博物馆展柜 vs 联合办公区

- **不可变集合**像博物馆展柜——只能看，不能碰（增删改会抛异常）
- **同步集合**像联合办公区——有门禁，同一时间只允许一人操作

## 二、不可变集合

### 2.1 Collections 包装方法

```java
// 包装为不可变 List
List<String> list = new ArrayList<>(Arrays.asList("A", "B", "C"));
List<String> unmodifiable = Collections.unmodifiableList(list);
// unmodifiable.add("D");  // UnsupportedOperationException

// 不可变 Map
Map<String, Integer> map = new HashMap<>();
map.put("A", 1);
Map<String, Integer> unmodMap = Collections.unmodifiableMap(map);

// 不可变 Set
Set<String> set = Collections.unmodifiableSet(new HashSet<>(Arrays.asList("A", "B")));
```

> **注意**：原始集合的修改会影响不可变视图！如需真正独立，应先拷贝再包装。

### 2.2 Java 9+ 便捷方法

```java
// List.of（最多 10 个元素，或传入数组）
List<String> list = List.of("A", "B", "C");

// Set.of（不允许重复元素和 null）
Set<Integer> set = Set.of(1, 2, 3);

// Map.of（最多 10 个键值对）
Map<String, Integer> map = Map.of("A", 1, "B", 2, "C", 3);

// Map.ofEntries（无限制）
Map<String, Integer> bigMap = Map.ofEntries(
    Map.entry("A", 1),
    Map.entry("B", 2)
);
```

### 2.3 不可变集合的限制

```java
List<String> list = List.of("A", "B", "C");

// 以下操作都会抛 UnsupportedOperationException
// list.add("D");
// list.remove(0);
// list.set(0, "X");
// list.add(null);  // 也不允许 null
```

### 2.4 工厂方法

```java
// 空集合（复用实例，节省内存）
List<String> empty = Collections.emptyList();
Map<String, Integer> emptyMap = Collections.emptyMap();
Set<String> emptySet = Collections.emptySet();

// 单元素集合
List<String> single = Collections.singletonList("A");
Map<String, Integer> singleMap = Collections.singletonMap("key", 1);
Set<String> singleSet = Collections.singleton("A");

// 重复元素列表
List<String> copies = Collections.nCopies(5, "A");  // [A, A, A, A, A]
```

## 三、同步集合

### 3.1 Collections.synchronizedXxx

```java
// 同步 List
List<String> syncList = Collections.synchronizedList(new ArrayList<>());

// 同步 Map
Map<String, Integer> syncMap = Collections.synchronizedMap(new HashMap<>());

// 同步 Set
Set<String> syncSet = Collections.synchronizedSet(new HashSet<>());
```

### 3.2 遍历必须手动同步

```java
List<String> list = Collections.synchronizedList(new ArrayList<>());
list.add("A");
list.add("B");

// ❌ 不安全的遍历
// for (String s : list) { ... }  // 可能 ConcurrentModificationException

// ✅ 安全的遍历
synchronized (list) {
    for (String s : list) {
        System.out.println(s);
    }
}
```

### 3.3 更好的选择：ConcurrentHashMap

```java
// 推荐：ConcurrentHashMap（分段锁，性能更好）
Map<String, Integer> map = new ConcurrentHashMap<>();
map.put("A", 1);
map.forEach((k, v) -> System.out.println(k + "=" + v));  // 无需手动同步
```

| 方案 | 性能 | 推荐度 |
|------|------|--------|
| synchronizedXxx | 低（全锁） | 不推荐 |
| ConcurrentHashMap | 高（分段锁） | 推荐 |
| CopyOnWriteArrayList | 写慢读快 | 读多写少时推荐 |

## 四、常见疑问

**Q1：List.of 和 Arrays.asList 有什么区别？**
A：`List.of` 创建完全不可变列表，不允许 null；`Arrays.asList` 允许 set 但不允许 add/remove，允许 null。

**Q2：不可变集合真的不可变吗？**
A：`Collections.unmodifiableXxx` 只是视图，原始集合修改会影响它。`List.of` 等才是真正不可变。

**Q3：什么场景用不可变集合？**
A：配置项、常量池、方法返回值（防止调用方修改内部数据）。

## 五、小结

| 类型 | 创建方式 | 特点 |
|------|----------|------|
| 不可变 List | List.of() / unmodifiableList() | 不可增删改 |
| 不可变 Set | Set.of() / unmodifiableSet() | 不可增删改 |
| 不可变 Map | Map.of() / unmodifiableMap() | 不可增删改 |
| 同步集合 | synchronizedXxx() | 遍历需手动同步 |
| 并发 Map | ConcurrentHashMap | 推荐替代方案 |
