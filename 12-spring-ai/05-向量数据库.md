# 85.1 向量数据库：AI 的语义记忆库（生活类比：图书馆索书号）

## 一、先问一个问题

**问题**：如何让计算机理解"开心"和"快乐"是同一个意思，而"苹果手机"和"苹果水果"不是？

### 生活例子：图书馆索书号

图书馆用索书号把相似主题的书放在相邻位置：
- "Java 编程"和"Python 编程" → 放在同一排（语义相近）
- "Java 咖啡" → 放在饮食区（语义不同）

向量数据库就是 AI 世界的"索书号系统"——把文本变成数字，语义相近的文本数字也相近。

## 二、向量嵌入原理

```
文本："Java 是一门编程语言"
  ↓ Embedding 模型
向量：[0.12, -0.34, 0.56, ..., 0.78]  （1536 维）
  ↓ 存入向量数据库
相似搜索时计算余弦距离
```

| 概念 | 说明 | 类比 |
|------|------|------|
| Embedding | 将文本转为向量 | 给书编索书号 |
| 余弦相似度 | 衡量两个向量的相似程度 | 两本书在书架上的距离 |
| TopK 检索 | 返回最相似的 K 个结果 | 找最近的 K 本相关书籍 |

## 三、Spring AI 支持的向量数据库

| 数据库 | 依赖 Starter | 特点 |
|--------|--------------|------|
| PgVector | spring-ai-pgvector-store-spring-boot-starter | 基于 PostgreSQL，运维简单 |
| Chroma | spring-ai-chroma-store-spring-boot-starter | 轻量级，适合开发测试 |
| Milvus | spring-ai-milvus-store-spring-boot-starter | 高性能，适合大规模生产 |
| Redis | spring-ai-redis-store-spring-boot-starter | 复用现有 Redis 基础设施 |

## 四、实战：PgVector 集成

### 4.1 添加依赖与配置

```xml
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-pgvector-store-spring-boot-starter</artifactId>
</dependency>
```

```yaml
spring:
  ai:
    vectorstore:
      pgvector:
        index-type: HNSW
        dimension: 1536
        distance-type: COSINE
```

### 4.2 使用 VectorStore

```java
@Service
public class VectorSearchService {

    private final VectorStore vectorStore;

    public VectorSearchService(VectorStore vectorStore) {
        this.vectorStore = vectorStore;
    }

    // 存入文档
    public void index(String text, Map<String, Object> metadata) {
        Document doc = new Document(text, metadata);
        vectorStore.add(List.of(doc));
    }

    // 语义搜索
    public List<Document> search(String query, int topK) {
        return vectorStore.similaritySearch(
            SearchRequest.builder()
                .query(query)
                .topK(topK)
                .build()
        );
    }
}
```

### 4.3 带元数据过滤

```java
public List<Document> searchWithFilter(String query, String category) {
    return vectorStore.similaritySearch(
        SearchRequest.builder()
            .query(query)
            .topK(5)
            .filterExpression("category == '" + category + "'")
            .build()
    );
}
```

## 五、常见疑问

**Q1：为什么不用 MySQL 的 LIKE 搜索？**
A：LIKE 只能做关键词匹配，无法理解语义。"汽车"搜不到"轿车"，但向量搜索可以。

**Q2：Embedding 维度越高越好吗？**
A：不一定。维度越高表达能力越强，但存储和计算成本也越高。常见 768-1536 维。

**Q3：生产环境如何选择向量数据库？**
A：小规模用 PgVector（复用 PostgreSQL），中大规模用 Milvus，快速原型用 Chroma。

## 六、小结

| 概念 | 说明 | 关键代码 |
|------|------|----------|
| Embedding | 文本→向量 | embeddingModel.embed(text) |
| VectorStore | 向量存储与检索 | vectorStore.add() / similaritySearch() |
| 余弦相似度 | 衡量语义距离 | COSINE 距离类型 |
| 元数据过滤 | 缩小检索范围 | filterExpression("key == 'value'") |
