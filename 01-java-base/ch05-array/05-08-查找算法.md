# 5.8 查找算法：快速定位（生活类比：查字典）

## 一、先问一个问题
**问题**：如何在数组中快速找到目标元素？

### 生活例子：查字典

想象你要在字典中找一个单词"Java"：

- **线性查找**：从第一页开始一页页翻，直到找到为止
- **二分查找**：从中间翻开，看"Java"在字典的前半还是后半，然后只查那一半，再从中间翻开，重复这个过程

两种方式差距巨大，第一种可能要翻几百页，第二种只需翻几次就能定位。

## 二、线性查找

**原理**：从头到尾逐个比较，找到为止。
就像查字典时从第一页开始一页页翻，不管要找的单词在哪里，都要按顺序看过去。

```java
public class LinearSearch {
    public static void main(String[] args) {
        int[] nums = {12, 45, 67, 23, 89, 34};
        int target = 23;
        
        int index = linearSearch(nums, target);
        
        if (index != -1) {
            System.out.println("找到，索引：" + index);
        } else {
            System.out.println("未找到");
        }
    }
    
    public static int linearSearch(int[] arr, int target) {
        for (int i = 0; i < arr.length; i++) {
            if (arr[i] == target) {
                return i;
            }
        }
        return -1;
    }
}
```

**特点**：
- 时间复杂度：O(n)
- 适用于无序数组
- 简单直接

## 三、二分查找（迭代版）

**原理**：每次比较中间元素，缩小一半范围。
就像查字典时先翻到中间，如果要找的单词在字典的前半部分，就只查前半；如果在后半，就只查后半。
每次排除一半内容，查找速度远超线性查找。

**前提**：数组必须有序！

```java
public class BinarySearch {
    public static void main(String[] args) {
        int[] nums = {1, 3, 5, 7, 9, 11, 13};
        int target = 7;
        
        int index = binarySearch(nums, target);
        System.out.println("索引：" + index);  // 3
    }
    
    public static int binarySearch(int[] arr, int target) {
        int left = 0;
        int right = arr.length - 1;
        
        while (left <= right) {
            int mid = left + (right - left) / 2;  // 防止溢出
            
            if (arr[mid] == target) {
                return mid;
            } else if (arr[mid] < target) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
        
        return -1;
    }
}
```

**特点**：
- 时间复杂度：O(log n)
- 适用于有序数组
- 效率极高

## 四、二分查找（递归版）

```java
public class RecursiveBinarySearch {
    public static void main(String[] args) {
        int[] nums = {1, 3, 5, 7, 9, 11, 13};
        int target = 7;
        
        int index = binarySearch(nums, 0, nums.length - 1, target);
        System.out.println("索引：" + index);  // 3
    }
    
    public static int binarySearch(int[] arr, int left, int right, int target) {
        if (left > right) {
            return -1;
        }
        
        int mid = left + (right - left) / 2;
        
        if (arr[mid] == target) {
            return mid;
        } else if (arr[mid] < target) {
            return binarySearch(arr, mid + 1, right, target);
        } else {
            return binarySearch(arr, left, mid - 1, target);
        }
    }
}
```

## 五、算法对比

| 算法 | 时间复杂度 | 前提条件 | 适用场景 |
|------|-----------|----------|----------|
| 线性查找 | O(n) | 无 | 无序数组 |
| 二分查找 | O(log n) | 有序 | 有序数组 |

## 六、常见疑问

**Q1：为什么 mid = left + (right - left) / 2 而不是 (left + right) / 2？**
A：防止 left + right 溢出。当数组很大时，两者相加可能超过 int 最大值。

**Q2：二分查找为什么要求数组有序？**
A：因为二分查找依赖中间值判断目标在左半还是右半，无序数组无法判断。

## 七、小结

| 算法 | 核心思想 | 关键代码 |
|------|----------|----------|
| 线性查找 | 逐个比较 | `for` 循环遍历 |
| 二分查找 | 折半缩小范围 | `mid = left + (right-left)/2` |
| 递归二分 | 函数调用自身 | 递归边界 `left > right` |
