# 5.7 排序算法：让数据有序（生活类比：整理扑克牌）

### 一、先问一个问题

**问题**：如果给你一堆乱序的数字，你会怎么排序？

#### 生活例子：整理扑克牌

想象你手里有一副洗乱的扑克牌，需要按从小到大排列：

- **冒泡排序**：像气泡一样，两两比较相邻的牌，大的往后挪，最大的牌像气泡一样"浮"到最右边
- **选择排序**：每次从剩下的牌中挑出最小的一张，放到已排好序的那堆末尾
- **插入排序**：摸牌时最常用的方式——拿起一张新牌，从右往左找到合适的位置插进去

这三种算法虽然时间复杂度都是 O(n²)，但思想和适用场景各不相同。

### 二、冒泡排序

**原理**：相邻两两比较，大的往后移，像气泡一样浮到水面。每一轮都会把当前最大的元素"冒"到最后。

**执行过程演示**（以 `[5, 3, 8, 1, 9]` 为例）：

```
第1轮：[3, 5, 1, 8, 9]  ← 9 冒到最后
第2轮：[3, 1, 5, 8, 9]  ← 8 到位
第3轮：[1, 3, 5, 8, 9]  ← 5 到位
第4轮：[1, 3, 5, 8, 9]  ← 完成
```

```java
public class BubbleSort {
    public static void main(String[] args) {
        int[] nums = {5, 3, 8, 1, 9};
        
        // 外层控制轮数，内层进行相邻比较
        for (int i = 0; i < nums.length - 1; i++) {
            for (int j = 0; j < nums.length - 1 - i; j++) {
                if (nums[j] > nums[j + 1]) {
                    // 交换相邻元素
                    int temp = nums[j];
                    nums[j] = nums[j + 1];
                    nums[j + 1] = temp;
                }
            }
        }
        
        System.out.println(java.util.Arrays.toString(nums));  // [1, 3, 5, 8, 9]
    }
}
```

**特点**：
- 时间复杂度：O(n²)
- 稳定排序（相等元素不会交换位置）
- 代码简单，适合教学演示

### 三、选择排序

**原理**：每次从未排序部分选出最小的，放到已排序部分末尾。就像每次从剩下的牌中挑最小的放在最前面。

**执行过程演示**：

```
初始：[5, 3, 8, 1, 9]
第1轮：[1, 3, 8, 5, 9]  ← 选出 1 放到第0位
第2轮：[1, 3, 8, 5, 9]  ← 3 已经在正确位置
第3轮：[1, 3, 5, 8, 9]  ← 选出 5 放到第2位
第4轮：[1, 3, 5, 8, 9]  ← 完成
```

```java
public class SelectionSort {
    public static void main(String[] args) {
        int[] nums = {5, 3, 8, 1, 9};
        
        for (int i = 0; i < nums.length - 1; i++) {
            int minIndex = i;
            
            // 在未排序部分找到最小值的索引
            for (int j = i + 1; j < nums.length; j++) {
                if (nums[j] < nums[minIndex]) {
                    minIndex = j;
                }
            }
            
            // 将最小值交换到已排序部分末尾
            if (minIndex != i) {
                int temp = nums[i];
                nums[i] = nums[minIndex];
                nums[minIndex] = temp;
            }
        }
        
        System.out.println(java.util.Arrays.toString(nums));  // [1, 3, 5, 8, 9]
    }
}
```

**特点**：
- 时间复杂度：O(n²)
- 不稳定排序（交换可能打乱相等元素的相对顺序）
- 交换次数最少（最多 n-1 次）

### 四、插入排序

**原理**：将未排序元素逐个插入到已排序部分的正确位置。就像打牌时摸一张新牌，从右往左找到合适的位置插进去。

**执行过程演示**：

```
初始：[5, 3, 8, 1, 9]
i=1：[3, 5, 8, 1, 9]  ← 3 插入到 5 前面
i=2：[3, 5, 8, 1, 9]  ← 8 已经在正确位置
i=3：[1, 3, 5, 8, 9]  ← 1 插入到最前面
i=4：[1, 3, 5, 8, 9]  ← 完成
```

```java
public class InsertionSort {
    public static void main(String[] args) {
        int[] nums = {5, 3, 8, 1, 9};
        
        for (int i = 1; i < nums.length; i++) {
            int key = nums[i];   // 当前要插入的元素
            int j = i - 1;
            
            // 将比 key 大的元素向后移动，腾出插入位置
            while (j >= 0 && nums[j] > key) {
                nums[j + 1] = nums[j];
                j--;
            }
            
            nums[j + 1] = key;  // 插入到正确位置
        }
        
        System.out.println(java.util.Arrays.toString(nums));  // [1, 3, 5, 8, 9]
    }
}
```

**特点**：
- 时间复杂度：O(n²)
- 稳定排序
- 适合基本有序的数据（最好情况 O(n)）

### 五、算法对比

| 算法 | 时间复杂度 | 稳定性 | 交换次数 | 适用场景 |
|------|-----------|--------|----------|----------|
| 冒泡排序 | O(n²) | 稳定 | 多 | 教学演示 |
| 选择排序 | O(n²) | 不稳定 | 最少 | 交换成本高时 |
| 插入排序 | O(n²) | 稳定 | 中等 | 基本有序数据 |

### 六、常见疑问

**Q1：为什么这些算法都是 O(n²)？**
A：因为它们都使用了两层嵌套循环，比较次数约为 n²/2。数据量翻倍时，耗时约变为 4 倍。

**Q2：实际开发中用什么排序？**
A：使用 `Arrays.sort()`，底层针对基本类型使用双轴快速排序，针对对象使用 TimSort，时间复杂度 O(n log n)，远优于 O(n²)。

**Q3：什么时候插入排序比快速排序快？**
A：当数据量很小（n < 50）或数据基本有序时，插入排序的常数因子更小，实际运行可能更快。

### 七、小结

| 算法 | 核心思想 | 关键代码 |
|------|----------|----------|
| 冒泡 | 相邻比较，大的往后冒 | `if(nums[j] > nums[j+1])` |
| 选择 | 选最小放前面 | `minIndex` 记录最小索引 |
| 插入 | 逐个插入正确位置 | `while(nums[j] > key)` |

# 5.8 查找算法：快速定位（生活类比：查字典）

### 一、先问一个问题
**问题**：如何在数组中快速找到目标元素？

#### 生活例子：查字典

想象你要在字典中找一个单词"Java"：

- **线性查找**：从第一页开始一页页翻，直到找到为止
- **二分查找**：从中间翻开，看"Java"在字典的前半还是后半，然后只查那一半，再从中间翻开，重复这个过程

两种方式差距巨大，第一种可能要翻几百页，第二种只需翻几次就能定位。

### 二、线性查找

**原理**：从头到尾逐个比较，找到为止。
就像查字典时从第一页开始一页页翻，不管要找的单词在哪里，都要按顺序看过去。

```java
public class LinearSearch {
    public static void main(String[] args) {
        int[] nums = {12, 45, 67, 23, 89, 34};
        int target = 23;
        
        int index = linearSearch(nums, target);
        
        if (index != -1) {
            System.out.println("找到，索引：" + index);
        } else {
            System.out.println("未找到");
        }
    }
    
    public static int linearSearch(int[] arr, int target) {
        for (int i = 0; i < arr.length; i++) {
            if (arr[i] == target) {
                return i;
            }
        }
        return -1;
    }
}
```

**特点**：
- 时间复杂度：O(n)
- 适用于无序数组
- 简单直接

### 三、二分查找（迭代版）

**原理**：每次比较中间元素，缩小一半范围。
就像查字典时先翻到中间，如果要找的单词在字典的前半部分，就只查前半；如果在后半，就只查后半。
每次排除一半内容，查找速度远超线性查找。

**前提**：数组必须有序！

```java
public class BinarySearch {
    public static void main(String[] args) {
        int[] nums = {1, 3, 5, 7, 9, 11, 13};
        int target = 7;
        
        int index = binarySearch(nums, target);
        System.out.println("索引：" + index);  // 3
    }
    
    public static int binarySearch(int[] arr, int target) {
        int left = 0;
        int right = arr.length - 1;
        
        while (left <= right) {
            int mid = left + (right - left) / 2;  // 防止溢出
            
            if (arr[mid] == target) {
                return mid;
            } else if (arr[mid] < target) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
        
        return -1;
    }
}
```

**特点**：
- 时间复杂度：O(log n)
- 适用于有序数组
- 效率极高

### 四、二分查找（递归版）

```java
public class RecursiveBinarySearch {
    public static void main(String[] args) {
        int[] nums = {1, 3, 5, 7, 9, 11, 13};
        int target = 7;
        
        int index = binarySearch(nums, 0, nums.length - 1, target);
        System.out.println("索引：" + index);  // 3
    }
    
    public static int binarySearch(int[] arr, int left, int right, int target) {
        if (left > right) {
            return -1;
        }
        
        int mid = left + (right - left) / 2;
        
        if (arr[mid] == target) {
            return mid;
        } else if (arr[mid] < target) {
            return binarySearch(arr, mid + 1, right, target);
        } else {
            return binarySearch(arr, left, mid - 1, target);
        }
    }
}
```

### 五、算法对比

| 算法 | 时间复杂度 | 前提条件 | 适用场景 |
|------|-----------|----------|----------|
| 线性查找 | O(n) | 无 | 无序数组 |
| 二分查找 | O(log n) | 有序 | 有序数组 |

### 六、常见疑问

**Q1：为什么 mid = left + (right - left) / 2 而不是 (left + right) / 2？**
A：防止 left + right 溢出。当数组很大时，两者相加可能超过 int 最大值。

**Q2：二分查找为什么要求数组有序？**
A：因为二分查找依赖中间值判断目标在左半还是右半，无序数组无法判断。

### 七、小结

| 算法 | 核心思想 | 关键代码 |
|------|----------|----------|
| 线性查找 | 逐个比较 | `for` 循环遍历 |
| 二分查找 | 折半缩小范围 | `mid = left + (right-left)/2` |
| 递归二分 | 函数调用自身 | 递归边界 `left > right` |

# 5.9 数组操作算法：数据处理（生活类比：整理书架）

### 一、先问一个问题
**问题**：如何高效地反转、去重、合并数组？

#### 生活例子
整理书架：
- 反转：把书从后往前重新摆放
- 去重：去掉重复的书籍
- 合并：两个书架的书合并到一个

在编程中，数组是最常用的数据结构之一。我们经常需要对数组进行各种操作，比如反转顺序、去除重复元素、合并多个数组等。这些操作看似简单，但不同的实现方式会导致截然不同的性能表现。

本节通过生活类比，帮助你理解这些数组操作的核心思想，并掌握最优的实现方式。

#### 核心概念
- **双指针法**：使用两个指针从不同方向遍历数组，减少不必要的循环
- **原地操作**：直接在原数组上修改，避免额外的内存开销
- **有序数组的特性**：有序数组中相同元素必然相邻，可以利用这一特性简化操作

### 二、反转数组

**原理**：双指针法，首尾交换向中间靠拢。

想象你要把一排书从左到右的顺序变成从右到左。最简单的方法是：拿起最左边和最右边的书交换位置，然后向内移动，继续交换，直到两本书相遇。

**代码实现**：

```java
public class ReverseArray {
    public static void main(String[] args) {
        int[] nums = {1, 2, 3, 4, 5};
        
        // 双指针法
        for (int i = 0, j = nums.length - 1; i < j; i++, j--) {
            int temp = nums[i];
            nums[i] = nums[j];
            nums[j] = temp;
        }
        
        System.out.println(java.util.Arrays.toString(nums));  // [5, 4, 3, 2, 1]
    }
}
```

### 三、数组去重

**原理**：假设数组已排序，比较相邻元素，保留不重复的。

生活中，如果你有重复的书籍，你会一本一本地检查，发现重复的就拿走。对于有序数组，重复的书必然挨在一起，所以只需要比较相邻的两本书即可。

**复杂度分析**：
- 时间复杂度：O(n)，只需遍历一次数组
- 空间复杂度：O(n)，需要额外数组存储结果

**代码实现**：

```java
public class RemoveDuplicates {
    public static void main(String[] args) {
        int[] nums = {1, 2, 2, 3, 3, 3, 4, 5};
        
        // 临时数组存储结果
        int[] temp = new int[nums.length];
        int count = 0;
        
        temp[count++] = nums[0];
        
        for (int i = 1; i < nums.length; i++) {
            if (nums[i] != nums[i - 1]) {
                temp[count++] = nums[i];
            }
        }
        
        // 创建结果数组
        int[] result = java.util.Arrays.copyOf(temp, count);
        
        System.out.println(java.util.Arrays.toString(result));  // [1, 2, 3, 4, 5]
    }
}
```

### 四、数组合并

**原理**：创建新数组，依次拷贝两个数组的元素。

把两个书架的书合并到一个大书架上，最简单的方法是：先把第一个书架的书全部搬过去，再把第二个书架的书接着搬过去。

**代码实现**：

```java
public class MergeArrays {
    public static void main(String[] args) {
        int[] arr1 = {1, 3, 5};
        int[] arr2 = {2, 4, 6};
        
        int[] result = new int[arr1.length + arr2.length];
        
        // 拷贝第一个数组
        System.arraycopy(arr1, 0, result, 0, arr1.length);
        
        // 拷贝第二个数组
        System.arraycopy(arr2, 0, result, arr1.length, arr2.length);
        
        System.out.println(java.util.Arrays.toString(result));  // [1, 3, 5, 2, 4, 6]
    }
}
```

### 五、有序数组合并

**原理**：双指针法，比较两个数组当前元素，小的放入结果。

如果两个书架上的书都已经按大小排好序，合并时不需要先全部搬过去再排序。你可以同时看两个书架最左边的书，把较小的那本先放到新书架上，然后继续比较。

**复杂度分析**：
- 时间复杂度：O(m+n)，m 和 n 分别为两个数组的长度
- 空间复杂度：O(m+n)，需要额外数组存储结果

**代码实现**：

```java
public class MergeSortedArrays {
    public static void main(String[] args) {
        int[] arr1 = {1, 3, 5, 7};
        int[] arr2 = {2, 4, 6, 8};
        
        int[] result = new int[arr1.length + arr2.length];
        int i = 0, j = 0, k = 0;
        
        // 合并两个有序数组
        while (i < arr1.length && j < arr2.length) {
            if (arr1[i] <= arr2[j]) {
                result[k++] = arr1[i++];
            } else {
                result[k++] = arr2[j++];
            }
        }
        
        // 处理剩余元素
        while (i < arr1.length) {
            result[k++] = arr1[i++];
        }
        while (j < arr2.length) {
            result[k++] = arr2[j++];
        }
        
        System.out.println(java.util.Arrays.toString(result));  // [1, 2, 3, 4, 5, 6, 7, 8]
    }
}
```

### 六、常见疑问

**Q1：为什么去重要求数组有序？**
A：有序数组中重复元素必然相邻，只需比较相邻元素。无序数组需要使用 HashSet 等数据结构。

**Q2：有序数组合并的时间复杂度是多少？**
A：O(m+n)，只需遍历两个数组各一次，比先合并后排序 O((m+n)log(m+n)) 更高效。

### 七、小结

| 操作 | 核心思想 | 关键代码 |
|------|----------|----------|
| 反转 | 双指针首尾交换 | `i++, j--` 向中间靠拢 |
| 去重 | 比较相邻元素 | `if(nums[i] != nums[i-1])` |
| 合并 | 依次拷贝 | `System.arraycopy()` |
| 有序合并 | 双指针比较 | `while(i<len1 && j<len2)` |
