# 排序与查找算法

## 一、问题引入

给你一堆乱序的成绩，如何快速排好？如何在一万个数据中找到目标？

```java
// ❌ 每次从头找，1 万条数据最多比较 1 万次
for (int i = 0; i < nums.length; i++) {
    if (nums[i] == target) { /* 找到了 */ }
}

// ✅ 二分查找，1 万条数据最多比较 14 次
Arrays.binarySearch(nums, target);
```

> 生活类比：整理扑克牌——冒泡是两两比较把大的往后挪，插入是摸一张新牌插到合适位置，二分查找是翻字典时从中间劈开。

## 二、核心概念

### 排序算法对比

| 算法 | 核心思想 | 时间复杂度 | 稳定性 |
|------|----------|-----------|--------|
| 冒泡排序 | 相邻比较，大的往后冒 | O(n²) | 稳定 |
| 选择排序 | 每轮选最小放前面 | O(n²) | 不稳定 |
| 插入排序 | 逐个插入正确位置 | O(n²) | 稳定 |

### 查找算法对比

| 算法 | 前提 | 时间复杂度 |
|------|------|-----------|
| 线性查找 | 无 | O(n) |
| 二分查找 | 有序 | O(log n) |

## 三、代码对比

### 冒泡排序

```java
int[] nums = {5, 3, 8, 1, 9};

// ❌ 常见错误：内层边界写错
for (int i = 0; i < nums.length - 1; i++) {
    for (int j = 0; j < nums.length; j++) {  // 越界！应减 i 和 1
        if (nums[j] > nums[j + 1]) { /* 交换 */ }
    }
}

// ✅ 正确写法
for (int i = 0; i < nums.length - 1; i++) {
    for (int j = 0; j < nums.length - 1 - i; j++) {  // 每轮少比较一次
        if (nums[j] > nums[j + 1]) {
            int temp = nums[j];
            nums[j] = nums[j + 1];
            nums[j + 1] = temp;
        }
    }
}
// 输出: [1, 3, 5, 8, 9]
```

### 二分查找

```java
int[] nums = {1, 3, 5, 7, 9, 11, 13};

// ❌ 常见错误：数组未排序就二分查找
int[] unsorted = {5, 1, 9, 3};
// binarySearch 要求有序！否则结果不可预测

// ✅ 先排序再查找，或确认已排序
Arrays.sort(nums);
int index = Arrays.binarySearch(nums, 7);  // 3
```

## 四、实现方式

### 冒泡排序（手动实现）

```
执行过程：[5, 3, 8, 1, 9]
第1轮：[3, 5, 1, 8, 9]  ← 9 冒到最后
第2轮：[3, 1, 5, 8, 9]  ← 8 到位
第3轮：[1, 3, 5, 8, 9]  ← 完成
```

```java
for (int i = 0; i < nums.length - 1; i++) {      // 外层：轮数
    for (int j = 0; j < nums.length - 1 - i; j++) { // 内层：相邻比较
        if (nums[j] > nums[j + 1]) {
            int temp = nums[j];
            nums[j] = nums[j + 1];
            nums[j + 1] = temp;
        }
    }
}
```

### 二分查找（手动实现）

```
在 [1, 3, 5, 7, 9] 中查找 7：
第1次：mid=2(值5) → 7>5 → 右半 [7, 9]
第2次：mid=3(值7) → 找到！
```

```java
int[] nums = {1, 3, 5, 7, 9, 11, 13};
int target = 7;
int left = 0, right = nums.length - 1;
int result = -1;

while (left <= right) {
    int mid = (left + right) / 2;  // 中间索引
    if (nums[mid] == target) {
        result = mid;
        break;
    } else if (nums[mid] < target) {
        left = mid + 1;   // 目标在右半部分
    } else {
        right = mid - 1;  // 目标在左半部分
    }
}
// result = 3
```

### 实际开发：用 Arrays 即可

```java
int[] nums = {5, 3, 8, 1, 9};

Arrays.sort(nums);                          // 排序
int index = Arrays.binarySearch(nums, 8);   // 查找
// 底层是双轴快排 + 二分查找，性能远优于手写 O(n²)
```

## 五、Q&A

**Q：为什么手写排序都是 O(n²)，实际开发还用 Arrays.sort？**
A：`Arrays.sort` 底层用双轴快排（基本类型）或 TimSort（对象），都是 O(n log n)。1 万条数据，O(n²) 约 1 亿次操作，O(n log n) 约 13 万次，差 700 倍。

**Q：什么场景手写排序有意义？**
A：面试考察算法思维，或数据量极小且基本有序时插入排序可能更快（n < 50 时常数因子更优）。

**Q：二分查找为什么要求有序？**
A：二分查找依赖"左半部分都小、右半部分都大"的前提来缩小范围。无序数组无法保证这一点，结果不可预测。

**Q：冒泡排序能优化吗？**
A：可以加一个 `swapped` 标志位，某轮没有交换则说明已有序，提前退出。

## 六、小结表格

| 算法 | 核心思想 | 时间复杂度 | 关键代码 |
|------|----------|-----------|----------|
| 冒泡排序 | 相邻比较，大的往后冒 | O(n²) | `if(nums[j] > nums[j+1])` 交换 |
| 二分查找 | 折半缩小范围 | O(log n) | `mid = (left+right)/2` |
| Arrays.sort | 双轴快排/TimSort | O(n log n) | `Arrays.sort(nums)` |
| Arrays.binarySearch | 内置二分查找 | O(log n) | `Arrays.binarySearch(nums, key)` |
