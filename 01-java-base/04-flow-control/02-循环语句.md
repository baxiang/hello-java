# 循环语句：for、while 与 do-while

## 一、问题引入

打印 1 到 100 的数字，手写 100 行 `System.out.println`？计算 1 到 1000 的累加和，手写 1000 行加法？**重复操作是编程最常见的场景，如何让代码自动重复执行？**

生活类比：for 像**知道圈数的跑步**——跑 10 圈就停；while 像**不知道何时停的跑步**——累了才停；do-while 像**先跑一圈再决定**——至少跑一圈。

## 二、核心概念

```
循环结构
├── for        → 确定次数：知道循环几轮
├── while      → 不确定次数，可能 0 次：先判断后执行
└── do-while   → 不确定次数，至少 1 次：先执行后判断
```

**通用要素**：初始化 → 条件判断 → 循环体 → 更新变量。缺少更新变量会导致死循环。

## 三、代码对比

### 3.1 忘记更新变量 → 死循环

```java
// ❌ while 忘记 i++，死循环
int i = 0;
while (i < 10) {
    System.out.println(i);
}

// ✅ 循环体内更新变量
int i = 0;
while (i < 10) {
    System.out.println(i);
    i++;
}
```

### 3.2 while 后面加分号

```java
int i = 0;

// ❌ 分号使 while 变成空循环体，后面的代码块不属于循环
// while (i < 10); {
//     System.out.println(i);
//     i++;
// }

// ✅ 正确写法
while (i < 10) {
    System.out.println(i);
    i++;
}
```

### 3.3 差一错误

```java
// ❌ 打印 1-10，实际只到 9
for (int i = 1; i < 10; i++) { }

// ✅ 正确：i <= 10
for (int i = 1; i <= 10; i++) { }
```

### 3.4 变量作用域

```java
// ❌ i 的作用域仅在 for 内
for (int i = 0; i < 5; i++) { }
// System.out.println(i);  // 编译错误

// ✅ 需要在循环外使用时，在外部声明
int i;
for (i = 0; i < 5; i++) { }
System.out.println(i);  // 输出: 5
```

## 四、实现方式

### 4.1 for 循环：确定次数

```java
// 累加 1 到 100
int sum = 0;
for (int i = 1; i <= 100; i++) {
    sum += i;
}
System.out.println(sum);  // 输出: 5050
```

**执行流程**：初始化（1次）→ 判断条件 → 执行循环体 → 更新变量 → 回到判断。

**for 的灵活写法**：

```java
// 多变量
for (int i = 0, j = 10; i < j; i++, j--) { }

// 倒序
for (int i = 10; i >= 1; i--) { }

// 步长
for (int i = 0; i <= 100; i += 10) { }

// 死循环
for (;;) { }
```

### 4.2 while 循环：不确定次数，可能 0 次

```java
// 数字反转：12345 → 54321
int num = 12345;
int reversed = 0;

while (num != 0) {
    int digit = num % 10;           // 取末位
    reversed = reversed * 10 + digit; // 追加
    num /= 10;                      // 去末位
}
System.out.println(reversed);  // 输出: 54321
```

**典型场景**：读取输入直到满足条件、处理不知道长度的数据。

### 4.3 do-while 循环：至少执行 1 次

```java
// 密码验证：至少输入一次
String password;
int tries = 0;

do {
    password = input();    // 至少执行一次
    tries++;
} while (!password.equals("123456") && tries < 3);
```

**while vs do-while 区别**：

```java
int x = 11;

// while：条件不满足，不执行
while (x <= 10) {
    System.out.println("while");  // 无输出
}

// do-while：先执行一次，再判断
do {
    System.out.println("do-while");  // 输出: do-while
} while (x <= 10);
```

### 4.4 循环选择指南

| 场景 | 推荐 | 示例 |
|------|------|------|
| 已知循环次数 | `for` | 遍历数组、累加求和 |
| 不知次数，可能 0 次 | `while` | 读取数据、条件等待 |
| 不知次数，至少 1 次 | `do-while` | 输入验证、菜单选择 |

### 4.5 死循环的应用

死循环不是 bug，服务器、游戏主循环都是 `while(true)`，配合 break 跳出。

```java
// 服务器监听
while (true) {
    Socket client = server.accept();
    handle(client);
}

// 带退出条件
while (true) {
    String cmd = readCommand();
    if ("quit".equals(cmd)) break;
    execute(cmd);
}
```

## 五、Q&A

**Q：for 和 while 本质有什么区别？**

A：功能上完全等价，任何 for 都能改写为 while。区别在于**语义**：for 把"初始化-条件-更新"集中在一行，循环结构一目了然；while 更适合条件不由简单计数器控制的场景。

**Q：什么时候用 do-while？**

A：当你需要**先做一次再判断**时。最常见的场景是输入验证——必须先让用户输入一次，才能判断是否有效。实际开发中 do-while 使用频率远低于 for 和 while。

**Q：for 的三个表达式都能省略吗？**

A：都能省略，但分号不能省。`for(;;)` 等价于 `while(true)`。省略条件即默认 true（死循环），省略初始化和更新需要手动处理。

**Q：嵌套循环太多层会有什么问题？**

A：时间复杂度指数增长——两层 O(n²)、三层 O(n³)。嵌套超过 3 层通常意味着需要重构：提取方法、使用集合操作、或选择更优算法。

## 六、小结表格

| 循环 | 语法 | 判断时机 | 最少执行 | 适用场景 |
|------|------|----------|----------|----------|
| `for` | `for(初始化; 条件; 更新)` | 先判断 | 0 次 | 已知次数 |
| `while` | `while(条件)` | 先判断 | 0 次 | 不知次数 |
| `do-while` | `do{}while(条件);` | 后判断 | 1 次 | 至少执行一次 |

| 常见陷阱 | 正确做法 |
|----------|----------|
| 忘记更新循环变量 | 确保 `i++` 或等效更新 |
| `while(条件);` 加了分号 | 条件后直接跟 `{` |
| 差一错误（`<` vs `<=`） | 明确边界：含不含端点 |
| `for` 内变量外部访问 | 在外部声明，或重新设计 |
