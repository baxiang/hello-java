# 流程控制：break、continue 与嵌套循环

## 一、问题引入

循环找到了目标，后面的数还要逐个检查吗？遍历数据时，偶数不需要处理，能否跳过只处理奇数？两层循环找数据，内层找到了如何直接跳出外层？**需要更精细地控制循环的执行流程——提前退出、跳过本轮、跳出多层。**

生活类比：break 像**紧急出口**——直接离开整栋楼；continue 像**跳过这一站**——公交不停车继续走；标签 break 像**消防通道**——从地下室直接到一楼外面。

## 二、核心概念

```
流程控制
├── break           → 跳出整个循环 / switch
├── continue        → 跳过本次，进入下一次循环
├── 标签 break      → 跳出指定层级的循环
└── 标签 continue   → 跳到指定层级循环的下一次
```

**嵌套循环**：外层控制行，内层控制列。执行总次数 = 各层循环次数之积。

## 三、代码对比

### 3.1 break vs continue

```java
// break：遇到 5 就停止整个循环
for (int i = 1; i <= 10; i++) {
    if (i == 5) break;
    System.out.print(i + " ");
}
// 输出: 1 2 3 4

// continue：跳过 5，继续后面的循环
for (int i = 1; i <= 10; i++) {
    if (i == 5) continue;
    System.out.print(i + " ");
}
// 输出: 1 2 3 4 6 7 8 9 10
```

### 3.2 while 中 continue 导致死循环

```java
int i = 0;

// ❌ i == 0 时 continue，i 永远不加 1，死循环
while (i < 10) {
    if (i % 2 == 0) continue;
    i++;
}

// ✅ 先更新变量，再判断是否跳过
int i = 0;
while (i < 10) {
    i++;                   // 先更新
    if (i % 2 == 0) continue;
    System.out.print(i + " ");  // 输出: 1 3 5 7 9
}
```

### 3.3 普通 break 只能跳出内层

```java
// ❌ 需求：找到第一个 7 的倍数就停止所有循环
// 普通 break 只能跳出内层 for，外层继续
for (int i = 0; i < 5; i++) {
    for (int j = 0; j < 5; j++) {
        if (i * 10 + j == 7) break;  // 只跳出内层
    }
    // 外层还在继续
}
```

## 四、实现方式

### 4.1 break：提前退出

```java
// 找到第一个能被 7 整除的数
for (int i = 1; i <= 100; i++) {
    if (i % 7 == 0) {
        System.out.println(i);  // 输出: 7
        break;                  // 找到就停
    }
}
```

### 4.2 continue：跳过本轮

```java
// 只打印奇数
for (int i = 1; i <= 10; i++) {
    if (i % 2 == 0) continue;   // 跳过偶数
    System.out.print(i + " ");  // 输出: 1 3 5 7 9
}
```

### 4.3 标签 break：跳出多层循环

```java
// 在二维数据中查找目标，找到即停
outer:
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (i == 1 && j == 2) {
            System.out.println("找到: [" + i + "][" + j + "]");
            break outer;        // 直接跳出外层
        }
    }
}
// 输出: 找到: [1][2]
```

### 4.4 标签 continue：跳到外层下一次

```java
// 打印 2-50 的素数
outer:
for (int i = 2; i <= 50; i++) {
    for (int j = 2; j <= Math.sqrt(i); j++) {
        if (i % j == 0) continue outer;  // 不是素数，跳到下一个 i
    }
    System.out.print(i + " ");  // 素数才输出
}
```

### 4.5 嵌套循环：九九乘法表

```java
for (int i = 1; i <= 9; i++) {
    for (int j = 1; j <= i; j++) {
        System.out.print(j + "×" + i + "=" + (i * j) + "\t");
    }
    System.out.println();
}
```

### 4.6 嵌套循环：冒泡排序

```java
int[] arr = {5, 3, 8, 1, 9};

for (int i = 0; i < arr.length - 1; i++) {         // 外层：轮数
    for (int j = 0; j < arr.length - 1 - i; j++) {  // 内层：相邻比较
        if (arr[j] > arr[j + 1]) {
            int temp = arr[j];
            arr[j] = arr[j + 1];
            arr[j + 1] = temp;
        }
    }
}
// arr = {1, 3, 5, 8, 9}
```

## 五、Q&A

**Q：标签名可以随便取吗？**

A：可以，是合法的 Java 标识符。常用 `outer`、`loop` 等语义化名称。标签只在紧随其后的循环语句上有效，不能跨方法使用。

**Q：break 和 return 有什么区别？**

A：`break` 跳出当前循环，继续执行循环后面的代码；`return` 直接退出整个方法。如果只想结束循环而不是整个方法，用 break。

**Q：嵌套循环性能很差吗？**

A：两层 O(n²)、三层 O(n³)，数据量大时确实慢。优化思路：① 提前 break 减少无效迭代；② 用 HashMap 替代内层查找（O(1) 查找替代 O(n) 遍历）；③ 选择更优算法。

**Q：continue 在 for 和 while 中行为一样吗？**

A：不完全一样。`for` 中 continue 后会执行更新表达式（`i++`）；`while` 中 continue 直接回到条件判断，跳过了更新语句——容易死循环。

## 六、小结表格

| 关键字 | 作用 | 影响范围 |
|--------|------|----------|
| `break` | 跳出当前循环 | 当前一层循环 |
| `continue` | 跳过本次，继续下一轮 | 当前一层循环 |
| `break 标签` | 跳出指定层循环 | 标签标记的那层 |
| `continue 标签` | 跳到指定层下一轮 | 标签标记的那层 |

| 嵌套循环要点 | 说明 |
|-------------|------|
| 执行次数 | 各层循环次数相乘 |
| 外层控制行，内层控制列 | 打印图形的经典模式 |
| 性能优化 | 提前 break、用 Map 替代内层查找 |
| while + continue | 必须在 continue 前更新变量 |
