# 条件判断：if-else 与 switch

## 一、问题引入

程序默认从上到下逐行执行，但现实世界充满选择：用户登录成功才能进入系统，成绩不同等级不同，月份不同季节不同。**如何让程序具备判断能力，根据条件走不同的路？**

生活类比：if-else 像**路口选择**——向左走还是向右走；switch 像**菜单点菜**——选哪道就上哪道。

## 二、核心概念

```
条件判断
├── if 系列
│   ├── if         → 单分支：满足就做
│   ├── if-else    → 双分支：二选一
│   └── if-else if → 多分支：多选一
└── switch         → 等值匹配：多选一（适合离散值）
```

**关键区别**：if-else 适合**范围判断**（score >= 90），switch 适合**等值匹配**（day == 1）。

## 三、代码对比

### 3.1 条件表达式必须是布尔值

```java
int x = 5;

// ❌ C/C++ 习惯，Java 不允许
// if (x) { }

// ✅ 必须写完整的布尔表达式
if (x > 0) { }
```

### 3.2 大括号不能省略

```java
int age = 18;

// ❌ 省略大括号，后续维护容易出错
if (age >= 18)
    System.out.println("成年");

// ✅ 始终使用大括号
if (age >= 18) {
    System.out.println("成年");
}
```

### 3.3 if 后面不能加分号

```java
int x = 5;

// ❌ 分号使 if 变成空语句，代码块无条件执行
// if (x > 0); {
//     System.out.println("正数");
// }

// ✅ 正确写法
if (x > 0) {
    System.out.println("正数");
}
```

### 3.4 switch 忘记 break 导致穿透

```java
int day = 1;

// ❌ 忘记 break，case 穿透
switch (day) {
    case 1: System.out.println("一"); // 没有 break
    case 2: System.out.println("二"); break;
}
// 输出: 一 二

// ✅ 每个 case 都加 break
switch (day) {
    case 1: System.out.println("一"); break;
    case 2: System.out.println("二"); break;
}
// 输出: 一
```

### 3.5 switch 不支持的类型

```java
// ❌ long、float、double、boolean 不支持
// switch (3.14) { }  // 编译错误

// ✅ 支持：byte, short, int, char, String, 枚举
switch ("MONDAY") { }
```

## 四、实现方式

### 4.1 if-else：范围判断

```java
int score = 85;

if (score >= 90) {
    System.out.println("优秀");       // >= 90
} else if (score >= 80) {
    System.out.println("良好");       // 80-89
} else if (score >= 60) {
    System.out.println("及格");       // 60-79
} else {
    System.out.println("不及格");     // < 60
}
// 输出: 良好
```

### 4.2 switch：等值匹配

```java
int month = 5;

// 利用穿透性合并 case
switch (month) {
    case 3: case 4: case 5:
        System.out.println("春季"); break;
    case 6: case 7: case 8:
        System.out.println("夏季"); break;
    case 9: case 10: case 11:
        System.out.println("秋季"); break;
    case 12: case 1: case 2:
        System.out.println("冬季"); break;
    default:
        System.out.println("无效月份");
}
// 输出: 春季
```

### 4.3 Java 14+ switch 表达式

```java
int month = 5;

// 箭头语法 + 多值匹配，不再需要 break
String season = switch (month) {
    case 3, 4, 5   -> "春季";
    case 6, 7, 8   -> "夏季";
    case 9, 10, 11 -> "秋季";
    case 12, 1, 2  -> "冬季";
    default         -> "无效月份";
};
// 输出: 春季

// 需要多行逻辑时用 yield 返回值
int days = switch (month) {
    case 1, 3, 5, 7, 8, 10, 12 -> 31;
    case 4, 6, 9, 11 -> 30;
    case 2 -> {
        boolean isLeap = (2026 % 4 == 0 && 2026 % 100 != 0) || (2026 % 400 == 0);
        yield isLeap ? 29 : 28;   // 多行用 yield 返回
    }
    default -> 0;
};
```

### 4.4 if-else 与三元运算符互换

```java
int age = 20;

// if-else 版本
String status;
if (age >= 18) {
    status = "成年";
} else {
    status = "未成年";
}

// 三元运算符（简单的二选一更简洁）
String status2 = age >= 18 ? "成年" : "未成年";
```

## 五、Q&A

**Q：什么时候用 if-else，什么时候用 switch？**

A：范围判断用 if-else（如 score >= 90），等值匹配用 switch（如 day == 1）。switch 在离散值多时更清晰，Java 14+ 的箭头语法让 switch 更简洁安全。

**Q：switch 的 case 穿透是 bug 还是 feature？**

A：两者都是。忘记写 break 是常见 bug，但**故意利用穿透**合并多个 case 是合法技巧（如季节判断）。Java 14+ 箭头语法 `->` 彻底避免了穿透问题。

**Q：嵌套 if 太深怎么办？**

A：超过 3 层嵌套时考虑：① 用 `return` 提前退出（卫语句）；② 用 `else if` 扁平化；③ 抽取方法。嵌套越深，可读性越差。

**Q：switch 能用 String 吗？**

A：Java 7+ 支持。底层用 `hashCode()` + `equals()` 实现，case 值写常量即可，null 会抛 NullPointerException。

## 六、小结表格

| 语句 | 适用场景 | 关键要点 |
|------|----------|----------|
| `if` | 单条件判断 | 条件必须是布尔值 |
| `if-else` | 二选一 | 始终使用大括号 |
| `if-else if` | 多条件范围判断 | 从上到下匹配，首个满足即执行 |
| `switch` | 多个离散值匹配 | 别忘 break；支持穿透合并 case |
| `switch` 表达式 | Java 14+ 等值赋值 | 箭头 `->` 无穿透；`yield` 返回值 |
| 三元运算符 | 简单二选一赋值 | `条件 ? 值1 : 值2`，嵌套难读 |
