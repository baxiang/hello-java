# 9.3 接口与抽象类的区别

## 一、核心区别

| 特性 | 接口 (Interface) | 抽象类 (Abstract Class) |
|------|-----------------|------------------------|
| 设计目的 | 定义规范/能力 | 代码复用/模板 |
| 关键字 | interface | abstract class |
| 实现关键字 | implements | extends |
| 多继承 | 支持 | 不支持 |
| 构造方法 | 无 | 有 |
| 成员变量 | 只能有常量 | 可以有各种变量 |
| 方法类型 | 默认抽象方法 | 可以有普通方法 |

## 二、使用场景

### 2.1 何时使用接口

```java
// 1. 定义能力/行为
interface Flyable {
    void fly();
}

interface Swimmable {
    void swim();
}

// 2. 定义规范
interface Comparable<T> {
    int compareTo(T o);
}

// 3. 实现回调
interface OnClickListener {
    void onClick(View v);
}

// 4. 实现策略模式
interface PaymentStrategy {
    void pay(double amount);
}
```

### 2.2 何时使用抽象类

```java
// 1. 提取共性代码
abstract class Animal {
    protected String name;
    
    // 共性方法
    public void eat() {
        System.out.println("吃东西");
    }
    
    // 抽象方法
    public abstract void makeSound();
}

// 2. 定义模板
abstract class Database {
    public final void connect() {
        open();
        authenticate();
        establish();
    }
    
    protected abstract void open();
    protected abstract void authenticate();
    protected abstract void establish();
}

// 3. 有共同属性和行为
abstract class Employee {
    protected String name;
    protected double salary;
    
    public double getSalary() {
        return salary;
    }
}
```

## 三、代码对比

### 3.1 定义方式

```java
// 接口
interface Flyable {
    // 常量
    int MAX_SPEED = 1000;
    
    // 抽象方法
    void fly();
    
    // 默认方法
    default void land() {
        System.out.println("降落");
    }
}

// 抽象类
abstract class Animal {
    // 成员变量
    protected String name;
    
    // 构造方法
    public Animal(String name) {
        this.name = name;
    }
    
    // 普通方法
    public void eat() {
        System.out.println("吃东西");
    }
    
    // 抽象方法
    public abstract void makeSound();
}
```

### 3.2 实现/继承

```java
// 实现接口
class Bird implements Flyable {
    @Override
    public void fly() {
        System.out.println("鸟在飞");
    }
}

// 继承抽象类
class Dog extends Animal {
    @Override
    public void makeSound() {
        System.out.println("汪汪叫");
    }
}

// 同时使用
class Duck extends Animal implements Flyable, Swimmable {
    public Duck(String name) {
        super(name);
    }
    
    @Override
    public void makeSound() {
        System.out.println("嘎嘎叫");
    }
    
    @Override
    public void fly() {
        System.out.println("鸭子飞");
    }
    
    @Override
    public void swim() {
        System.out.println("鸭子游泳");
    }
}
```

## 四、设计思想

### 4.1 "Has-a" vs "Is-a"

```java
// 接口：Has-a（有能力）
// 鸟有飞行的能力
class Bird implements Flyable { }

// 抽象类：Is-a（是一种）
// 狗是一种动物
class Dog extends Animal { }
```

### 4.2 组合使用

```java
// 抽象类定义共性
abstract class Vehicle {
    protected String brand;
    
    public void start() {
        System.out.println("启动");
    }
}

// 接口定义能力
interface ElectricVehicle {
    void charge();
}

interface AutonomousVehicle {
    void autoPilot();
}

// 组合使用
class Tesla extends Vehicle implements ElectricVehicle, AutonomousVehicle {
    @Override
    public void charge() {
        System.out.println("充电");
    }
    
    @Override
    public void autoPilot() {
        System.out.println("自动驾驶");
    }
}
```

## 五、Java 8+ 的变化

### 5.1 默认方法

```java
interface Animal {
    // 抽象方法
    void makeSound();
    
    // 默认方法（Java 8+）
    default void sleep() {
        System.out.println("睡觉");
    }
}
```

### 5.2 静态方法

```java
interface MathUtils {
    // 静态方法（Java 8+）
    static int max(int a, int b) {
        return a > b ? a : b;
    }
}

// 使用
int result = MathUtils.max(10, 20);
```

### 5.3 私有方法（Java 9+）

```java
interface Calculator {
    // 私有方法（Java 9+）
    private int add(int a, int b) {
        return a + b;
    }
    
    // 私有静态方法（Java 9+）
    private static int multiply(int a, int b) {
        return a * b;
    }
    
    default int calculate(int a, int b) {
        return add(a, b) + multiply(a, b);
    }
}
```

## 六、综合示例

```java
// ==================== 接口定义能力 ====================

// 飞行能力
interface Flyable {
    void fly();
    
    default void land() {
        System.out.println("降落");
    }
}

// 游泳能力
interface Swimmable {
    void swim();
}

// 攻击能力
interface Attackable {
    void attack();
}

// ==================== 抽象类定义共性 ====================

// 生物
abstract class LivingThing {
    protected String name;
    protected int age;
    
    public LivingThing(String name, int age) {
        this.name = name;
        this.age = age;
    }
    
    // 共性方法
    public void breathe() {
        System.out.println(name + " 在呼吸");
    }
    
    public void grow() {
        age++;
        System.out.println(name + " 长大了，现在" + age + "岁");
    }
    
    // 抽象方法
    public abstract void move();
}

// ==================== 具体实现 ====================

// 鸟类：继承生物 + 实现飞行能力
class Bird extends LivingThing implements Flyable {
    private double wingspan;
    
    public Bird(String name, int age, double wingspan) {
        super(name, age);
        this.wingspan = wingspan;
    }
    
    @Override
    public void move() {
        System.out.println(name + " 用翅膀移动");
    }
    
    @Override
    public void fly() {
        System.out.println(name + " 在天空飞翔，翼展" + wingspan + "米");
    }
}

// 鱼类：继承生物 + 实现游泳能力
class Fish extends LivingThing implements Swimmable {
    private String color;
    
    public Fish(String name, int age, String color) {
        super(name, age);
        this.color = color;
    }
    
    @Override
    public void move() {
        System.out.println(name + " 用鳍移动");
    }
    
    @Override
    public void swim() {
        System.out.println(name + " 在水中游动，颜色是" + color);
    }
}

// 龙：继承生物 + 实现多种能力
class Dragon extends LivingThing implements Flyable, Swimmable, Attackable {
    private int power;
    
    public Dragon(String name, int age, int power) {
        super(name, age);
        this.power = power;
    }
    
    @Override
    public void move() {
        System.out.println(name + " 快速移动");
    }
    
    @Override
    public void fly() {
        System.out.println(name + " 在空中飞翔");
    }
    
    @Override
    public void swim() {
        System.out.println(name + " 在水中游泳");
    }
    
    @Override
    public void attack() {
        System.out.println(name + " 发动攻击，威力" + power);
    }
}

// ==================== 使用 ====================

public class World {
    public static void main(String[] args) {
        LivingThing[] creatures = {
            new Bird("老鹰", 3, 1.5),
            new Fish("金鱼", 1, "金色"),
            new Dragon("巨龙", 100, 9999)
        };
        
        for (LivingThing creature : creatures) {
            creature.breathe();
            creature.move();
            creature.grow();
            
            if (creature instanceof Flyable) {
                ((Flyable) creature).fly();
            }
            if (creature instanceof Swimmable) {
                ((Swimmable) creature).swim();
            }
            if (creature instanceof Attackable) {
                ((Attackable) creature).attack();
            }
            System.out.println();
        }
    }
}
```

## 七、选择指南

```
选择接口还是抽象类？

问自己以下问题：

1. 是否需要定义一组规范？
   → 是：使用接口

2. 是否需要支持多继承？
   → 是：使用接口

3. 是否有共同的属性和行为需要复用？
   → 是：使用抽象类

4. 是否需要定义构造方法？
   → 是：使用抽象类

5. 是否需要定义非 public 成员？
   → 是：使用抽象类

6. 是否需要添加新方法而不破坏现有代码？
   → 是：使用接口（默认方法）
```

## 八、小结

本节要点：
1. **设计目的**：接口定义规范，抽象类代码复用
2. **继承方式**：接口多继承，抽象类单继承
3. **成员特点**：接口只有常量，抽象类可以有各种成员
4. **使用场景**：能力用接口，共性用抽象类
5. **组合使用**：抽象类 + 接口实现灵活设计
