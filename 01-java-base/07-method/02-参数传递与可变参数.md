# 参数传递与可变参数

## 一、问题引入

面试官问："Java 是值传递还是引用传递？"你脱口而出"基本类型值传递，引用类型引用传递"——错了。Java **只有值传递**，没有引用传递。

那为什么方法内修改数组元素，外面的数组也变了？为什么修改对象属性，外面的对象也变了？理解这个看似矛盾的现象，是掌握 Java 参数传递的关键。

> 生活类比：值传递像**复印文件**——你改复印件，原件不变。引用类型传递像**共享文档链接**——你把链接（地址值）复制给别人，别人通过链接修改了文档内容，你再看原文档内容也变了。但你改不了别人的链接本身。

## 二、核心概念

### Java 只有值传递

- **基本类型**：传递值的副本（复印文件）
- **引用类型**：传递地址值的副本（共享文档链接）

两者本质相同——都是复制一份给形参。区别在于：复制的是"值"还是"地址值"。

| 类型 | 传递内容 | 修改形参本身 | 通过形参修改内容 |
|------|----------|:---:|:---:|
| 基本类型 | 值的副本 | ✗ 外部不变 | 不适用 |
| 引用类型 | 地址值的副本 | ✗ 外部不变 | ✓ 外部同步变 |
| String | 地址值的副本 | ✗ 外部不变 | ✗ 不可变类 |

## 三、代码对比

### 3.1 基本类型：改副本不影响原件

```java
int x = 10;
change(x);
System.out.println(x);  // 输出: 10，不变

// ❌ 误区：以为方法内改了 n，外面的 x 也会变
public static void change(int n) {
    n = 100;  // 改的是副本，原件不受影响
}
```

### 3.2 引用类型：改内容 vs 改引用

```java
int[] arr = {1, 2, 3};

// ✅ 修改内容：通过地址操作同一对象，外部同步变化
modifyElement(arr);
System.out.println(arr[0]);  // 输出: 100

// ❌ 修改引用：让形参指向新对象，不影响外部引用
modifyRef(arr);
System.out.println(arr[0]);  // 输出: 100（不变）

public static void modifyElement(int[] a) {
    a[0] = 100;                  // 通过地址修改堆内存中的对象
}
public static void modifyRef(int[] a) {
    a = new int[]{4, 5, 6};      // 形参指向新对象，与外部无关
}
```

### 3.3 String：引用类型但不可变

```java
String str = "Hello";
change(str);
System.out.println(str);  // 输出: Hello，不变

public static void change(String s) {
    s = s + " World";  // 关键：创建新字符串对象，形参指向新对象
}
```

String 是引用类型，但 `s = s + " World"` 不是修改原对象，而是创建了新对象——等同于"改引用"，所以外部不变。

### 3.4 StringBuilder vs String

```java
// StringBuilder 可变，append 修改的是原对象内容
StringBuilder sb = new StringBuilder("Hello");
change(sb);
System.out.println(sb);  // 输出: Hello World，变了！

public static void change(StringBuilder s) {
    s.append(" World");  // 修改原对象内容，不是创建新对象
}
```

## 四、实现方式

### 4.1 参数传递执行流程

```
基本类型传递：
  main: x = 10
        ↓ 复制值
  change: n = 10 → n = 100  （改的是副本）
        ↓ 返回
  main: x = 10（不变）

引用类型传递：
  main: arr → [1,2,3]（地址 0x123）
        ↓ 复制地址值
  change: a → 0x123 → a[0] = 100  （通过同一地址修改堆内存）
        ↓ 返回
  main: arr → [100,2,3]（内容变了）
```

### 4.2 可变参数

可变参数（Varargs）是数组的语法糖，允许方法接受任意数量的同类型参数：

```java
public static int sum(int... nums) {  // nums 本质是数组
    int total = 0;
    for (int n : nums) total += n;
    return total;
}

sum(1, 2, 3);       // 输出: 6
sum(1, 2, 3, 4, 5); // 输出: 15
sum();              // 输出: 0（空数组）
```

以下两种调用方式等价：

```java
sum(1, 2, 3);                // 编译器自动封装为数组
sum(new int[]{1, 2, 3});     // 显式传数组
```

### 4.3 可变参数的规则

```java
// ✅ 可变参数必须在参数列表最后
void print(String sep, String... items) { ... }

// ❌ 编译错误：可变参数不在最后
// void print(String... items, String sep) { }

// ❌ 编译错误：只能有一个可变参数
// void method(int... a, String... b) { }
```

### 4.4 可变参数与重载

```java
public static void method(int a)       { System.out.println("int"); }
public static void method(int a, int b) { System.out.println("int,int"); }
public static void method(int... a)    { System.out.println("varargs"); }

method(10);      // 输出: int      （精确匹配优先）
method(10, 20);  // 输出: int,int  （精确匹配优先）
method(10, 20, 30); // 输出: varargs（前两个不匹配，才走可变参数）
```

## 五、Q&A

**Q：Java 到底是值传递还是引用传递？**
A：只有值传递。引用类型传递的是"地址值的副本"，不是引用本身。方法内无法让外部的引用指向新对象。

**Q：为什么 swap 方法交换不了两个基本类型？**
A：因为形参是实参的副本，交换副本不影响原件。Java 没有办法在方法内交换两个基本类型的值。

**Q：String 是引用类型，为什么方法内改不了？**
A：String 是不可变类。`s = s + " World"` 创建了新对象，等同于让形参指向新地址，不影响外部。

**Q：可变参数传 null 会怎样？**
A：可变参数可以传 null，但使用前要判空，否则 `nums.length` 会抛 NullPointerException。

**Q：可变参数和数组参数有什么区别？**
A：调用时，可变参数可以直接传多个值 `sum(1,2,3)`，数组参数必须传数组 `sum(new int[]{1,2,3})`。方法内部完全一样。

## 六、小结表格

| 要点 | 内容 |
|------|------|
| 传递方式 | Java 只有值传递 |
| 基本类型 | 传递值的副本，方法内修改不影响外部 |
| 引用类型 | 传递地址值的副本，通过地址可修改对象内容 |
| 修改引用 | 形参指向新对象，不影响外部引用 |
| String 特殊 | 不可变类，看似值传递，实则是"改引用" |
| 可变参数语法 | `类型... 参数名`，本质是数组 |
| 可变参数规则 | 必须在最后，只能有一个 |
| 匹配优先级 | 精确匹配 > 自动类型提升 > 可变参数 |
