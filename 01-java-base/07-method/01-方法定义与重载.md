# 方法定义与重载

## 一、问题引入

你写了一个计算两数之和的方法 `sum(int, int)`，后来又需要三数之和，于是写了 `sum3(int, int, int)`，再后来要算小数之和……方法名越来越乱，调用者根本记不住。

重复代码是万恶之源，而方法就是消灭重复的第一道武器。但如果方法命名混乱，反而制造新的混乱——重载正是解决这个问题的优雅方案。

> 生活类比：方法像菜谱，参数像食材，返回值像做好的菜。同一道"红烧肉"菜谱，你可以放土豆、放豆腐——菜谱名不变，食材不同，这就是重载。

## 二、核心概念

### 方法定义五要素

```java
修饰符 返回值类型 方法名(参数列表) {
    方法体;
    return 返回值;
}
```

| 要素 | 说明 | 示例 |
|------|------|------|
| 修饰符 | 控制访问权限 | `public`, `static` |
| 返回值类型 | 返回数据类型，无返回值用 `void` | `int`, `String` |
| 方法名 | 见名知意，动词开头 | `sum`, `printInfo` |
| 参数列表 | 传入的数据 | `int a, int b` |
| 返回值 | 交给调用者的结果 | `return result;` |

### 方法重载（Overload）

在同一个类中，**方法名相同，参数列表不同**的多个方法。

参数列表不同指的是：参数**个数**不同、**类型**不同、或**顺序**不同。与返回值类型和修饰符无关。

## 三、代码对比

### 3.1 不用重载 vs 用重载

```java
// ❌ 不用重载：方法名混乱，调用者要记一堆名字
public static int sumTwo(int a, int b) { return a + b; }
public static int sumThree(int a, int b, int c) { return a + b + c; }
public static double sumDouble(double a, double b) { return a + b; }

// ✅ 用重载：同一个名字，编译器自动匹配
public static int sum(int a, int b) { return a + b; }
public static int sum(int a, int b, int c) { return a + b + c; }
public static double sum(double a, double b) { return a + b; }

// 调用时只需记住一个名字
sum(10, 20);           // 输出: 30
sum(10, 20, 30);       // 输出: 60
sum(1.5, 2.5);         // 输出: 4.0
```

**原因**：重载让 API 更简洁一致，`System.out.println()` 就是最经典的重载——能打印任何类型，但你只记一个名字。

### 3.2 重载判断的正误

```java
// ✅ 参数个数不同 → 是重载
void method(int a) {}
void method(int a, int b) {}

// ✅ 参数类型不同 → 是重载
void method(int a) {}
void method(double a) {}

// ✅ 参数顺序不同 → 是重载（但不推荐，易混淆）
void method(int a, double b) {}
void method(double a, int b) {}

// ❌ 仅返回值不同 → 编译错误，不是重载
// int method(int a) {}
// double method(int a) {}

// ❌ 仅参数名不同 → 编译错误，不是重载
// void method(int a) {}
// void method(int b) {}
```

## 四、实现方式

### 4.1 方法定义与调用

```java
public class MethodDemo {
    public static void main(String[] args) {
        // 赋值调用（推荐）
        int result = sum(10, 20);
        System.out.println(result);  // 输出: 30

        // 打印调用
        System.out.println(sum(10, 20));  // 输出: 30

        // 单独调用（void 方法）
        printHello();
    }

    public static int sum(int a, int b) {
        return a + b;  // 关键：return 将结果返回给调用者
    }

    public static void printHello() {
        System.out.println("Hello!");
    }
}
```

### 4.2 return 的两种用法

```java
// 有返回值：return 带回结果
public static String getSign(int num) {
    if (num > 0) return "正数";
    if (num < 0) return "负数";
    return "零";
}

// void 方法：return 提前退出
public static void printPositive(int n) {
    if (n <= 0) return;  // 关键：void 中 return 用于提前退出
    System.out.println(n);
}
```

### 4.3 重载的匹配规则

编译器按以下优先级匹配重载方法：

```
精确匹配 → 自动类型提升 → 可变参数（最低优先级）
```

```java
public static void method(int a)   { System.out.println("int"); }
public static void method(long a)  { System.out.println("long"); }
public static void method(int... a) { System.out.println("varargs"); }

method(10);     // 精确匹配 → 输出: int
method(10L);    // 精确匹配 → 输出: long
byte b = 10;
method(b);      // 自动提升 → 输出: int（byte → int）
method(10, 20); // 前两个不匹配 → 输出: varargs
```

### 4.4 常见重载场景

```java
// 构造方法重载
class Person {
    String name;
    int age;

    Person() { this("未知", 0); }              // 无参
    Person(String name) { this(name, 0); }     // 一个参数
    Person(String name, int age) { ... }       // 全参数
}

// 工具方法重载
public static int max(int a, int b)    { return a > b ? a : b; }
public static int max(int a, int b, int c) { return max(max(a, b), c); }
public static double max(double a, double b) { return a > b ? a : b; }
```

## 五、Q&A

**Q：返回值类型不同能构成重载吗？**
A：不能。调用 `method(10)` 时，编译器无法根据返回值判断你调的是哪个方法。

**Q：void 方法可以写 return 吗？**
A：可以，但 return 后面不能跟值。`return;` 用于提前退出方法。

**Q：参数顺序不同算重载，那这种写法好吗？**
A：能编译，但不推荐。`method(int, double)` 和 `method(double, int)` 容易混淆，调用时稍不留神就调错。

**Q：重载和重写（Override）什么区别？**
A：重载发生在同一个类中，参数列表不同；重写发生在父子类之间，参数列表相同。两者无关，只是名字像。

## 六、小结表格

| 要点 | 内容 |
|------|------|
| 方法定义 | 修饰符 + 返回值类型 + 方法名 + 参数列表 + 方法体 |
| 方法调用 | 赋值调用（推荐）、打印调用、单独调用 |
| void 方法 | 无返回值，可用 `return;` 提前退出 |
| 重载条件 | 方法名相同，参数列表不同（个数/类型/顺序） |
| 非重载 | 仅返回值不同、仅修饰符不同、仅参数名不同 |
| 匹配优先级 | 精确匹配 > 自动类型提升 > 可变参数 |
| 典型应用 | 构造方法重载、工具方法重载、`println()` |
