# 逻辑运算符与优先级

## 一、问题引入

检查一个字符串是否为空，随手写了 `str != null & str.isEmpty()`——结果当 `str` 为 `null` 时，照样抛出空指针异常。问题出在哪？`&` 是"非短路"逻辑与，即使左边已经是 `false`，右边照样执行。换成 `&&` 就没事了。

再看优先级：`a > b && b < c || d == e` 到底先算哪部分？写条件时不搞清优先级，逻辑就跟着错。运算符优先级就像数学中的"先乘除后加减"——不记住规则，就得用括号。

## 二、核心概念

### 逻辑运算符

| 运算符 | 名称 | 说明 | 短路？ |
|--------|------|------|--------|
| `&&` | 短路与 | 全真才真，遇 false 停 | 是 |
| `||` | 短路或 | 一真即真，遇 true 停 | 是 |
| `&` | 逻辑与 | 全真才真，两边都算 | 否 |
| `|` | 逻辑或 | 一真即真，两边都算 | 否 |
| `!` | 逻辑非 | 取反 | — |
| `^` | 异或 | 不同为真，相同为假 | 否 |

类比：`&&` 像串联电路——一处断开，整路不通；`||` 像并联电路——一路通，整路通。

### 三元运算符

```java
条件 ? 表达式1 : 表达式2
// 条件为 true 返回表达式1，否则返回表达式2
```

### 运算符优先级（由高到低）

| 优先级 | 运算符 | 结合性 |
|--------|--------|--------|
| 1 | `()` `[]` | 左→右 |
| 2 | `++` `--` `!` `+`(正) `-`(负) | 右→左 |
| 3 | `*` `/` `%` | 左→右 |
| 4 | `+` `-` | 左→右 |
| 5 | `<` `>` `<=` `>=` | 左→右 |
| 6 | `==` `!=` | 左→右 |
| 7 | `&&` | 左→右 |
| 8 | `||` | 左→右 |
| 9 | `?:` | 右→左 |
| 10 | `=` `+=` 等 | 右→左 |

记忆口诀：**括号最优先，单目次之；乘除先于加减，关系先于逻辑；短路与高于短路或，赋值排末尾。**

## 三、代码对比

### 短路 vs 非短路

```java
String str = null;

// ❌ & 非短路与：两边都执行，空指针异常
if (str != null & str.equals("hello")) { ... }

// ✅ && 短路与：左边 false 则跳过右边
if (str != null && str.equals("hello")) { ... }
```

```java
int a = 10;
// &：两边都执行，a++ 被计算
boolean r1 = (a > 5) & (a++ > 10);  // a 变成 11

a = 10;
// &&：左边为 true 才看右边，a++ 被计算
boolean r2 = (a > 100) && (a++ > 10);  // a 仍为 10，右边被短路
```

### 三元运算符简化 if-else

```java
// ❌ 简单二选一却写了一堆
String status;
if (score >= 60) {
    status = "及格";
} else {
    status = "不及格";
}

// ✅ 一行搞定
String status = score >= 60 ? "及格" : "不及格";
```

### 嵌套三元：可读性陷阱

```java
// ❌ 过度嵌套，难以阅读
String r = a > 0 ? (b > 0 ? (c > 0 ? "全正" : "c非正") : "b非正") : "a非正";

// ✅ 复杂逻辑用 if-else
if (a > 0 && b > 0 && c > 0) {
    r = "全正";
} else if (a <= 0) {
    r = "a非正";
}
```

### 优先级误判

```java
// ❌ 以为 || 和 && 同级
boolean r = a > b || b < c && d == e;
// 实际: a > b || (b < c && d == e)   && 优先级高于 ||

// ✅ 用括号明确意图
boolean r = (a > b || b < c) && d == e;  // 如果这是你的意图
```

## 四、实现方式

### 短路保护模式

```java
// 空指针保护
if (str != null && str.length() > 0) { ... }

// 除零保护
if (divisor != 0 && num / divisor > threshold) { ... }

// 集合非空检查
if (list != null && !list.isEmpty()) { ... }
```

### 德·摩根定律简化条件

```java
// !(A && B) = !A || !B
// !(A || B) = !A && !B

// ❌ 双重否定，绕脑子
if (!(age >= 18 && income >= 5000)) { ... }

// ✅ 用德·摩根定律展开
if (age < 18 || income < 5000) { ... }
```

### 三元运算符的常见用法

```java
// 求最大值
int max = a > b ? a : b;

// 绝对值
int abs = num >= 0 ? num : -num;

// 默认值
String name = input != null ? input : "未知";

// 嵌套（一层尚可）
String grade = score >= 90 ? "优秀"
             : score >= 60 ? "及格"
             : "不及格";
```

### 拆分复杂表达式

```java
// ❌ 一行写完，难以理解
boolean valid = age >= 18 && age <= 60 && score >= 60 && score <= 100;

// ✅ 拆分为语义变量
boolean ageValid = age >= 18 && age <= 60;
boolean scoreValid = score >= 60 && score <= 100;
boolean valid = ageValid && scoreValid;
```

## 五、Q&A

**Q：实际开发中该用 `&` 还是 `&&`？**

A：99% 的场景用 `&&` 和 `||`。`&` 和 `|` 只在需要两边都执行（如位运算或刻意要求副作用）时才用。就像走路，`&&` 是看到红灯就停，`&` 是不管红绿灯都得走完。

**Q：`^`（异或）有什么用？**

A：最常见的是"切换状态"——`flag = flag ^ true` 把布尔值翻转。另外，`a ^ b ^ b == a` 这个特性可用于简单加密或交换变量。

**Q：`5 > 3 ? 10 : 20 + 1` 等于几？**

A：`10`。三元运算符优先级低于 `+`，所以 `20 + 1` 先算，但 `5>3` 为 true 直接取 10。想要 `(5>3?10:20)+1=11`，必须加括号。

**Q：为什么 `a < b < c` 编译报错？**

A：`a < b` 返回 boolean，boolean 不能和 int 比较。Java 不是数学——必须写成 `a < b && b < c`。

## 六、小结表格

| 类别 | 要点 | 备注 |
|------|------|------|
| 短路运算 | `&&` 遇 false 停，`||` 遇 true 停 | 日常首选 |
| 非短路运算 | `&` `|` 两边都执行 | 几乎不用 |
| 逻辑非 | `!` 取反 | 优先级最高 |
| 异或 | `^` 相同为 false | 用于状态切换 |
| 三元运算符 | `条件 ? A : B` | 简化简单 if-else；避免嵌套 |
| 优先级 | `!` > 算术 > 关系 > `&&` > `||` > `?:` > 赋值 | 拿不准就加括号 |
| 德·摩根 | `!(A&&B)=!A\|\|!B` | 简化复杂否定条件 |
| 最佳实践 | 用括号明确意图；拆分复杂条件；短路保护空指针 | 可读性 > 简洁 |
