# 算术与关系运算符

## 一、问题引入

计算两个整数的平均值，结果却不是预期的 `5.5` 而是 `5`——整数除法的截断问题困扰着每一个 Java 初学者。再看关系运算：用 `==` 比较两个 `String`，内容一样却返回 `false`，让人直呼"这不科学"。

这两个问题的根源：**算术运算有隐式类型规则，关系运算有值与引用之分**。本节就来弄清这些"坑"。

## 二、核心概念

### 算术运算符

| 运算符 | 说明 | 示例 | 结果 |
|--------|------|------|------|
| `+` `-` `*` | 加减乘 | `5+3` / `5-3` / `5*3` | 8 / 2 / 15 |
| `/` | 除法 | `10/3` | 3（整数截断） |
| `%` | 取模 | `7%3` | 1 |
| `++` `--` | 自增自减 | `i++` / `--i` | i+1 / i-1 |

### 赋值运算符

| 运算符 | 说明 | 等价形式 |
|--------|------|----------|
| `=` | 赋值 | — |
| `+=` `-=` `*=` `/=` `%=` | 复合赋值 | `a+=5` → `a=(a的类型)(a+5)` |

### 关系运算符

| 运算符 | 说明 | 结果类型 |
|--------|------|----------|
| `==` `!=` | 等于 / 不等于 | boolean |
| `>` `<` `>=` `<=` | 大小比较 | boolean |

## 三、代码对比

### 整数除法 vs 浮点除法

```java
// ❌ 整数除法截断，丢失小数
int avg = 11 / 2;       // 输出: 5

// ✅ 至少一个操作数为浮点，结果才保留小数
double avg = 11.0 / 2;  // 输出: 5.5
double avg = (double)11 / 2;  // 输出: 5.5
```

### 自增前缀 vs 后缀

```java
int i = 5;
// ❌ 混淆前缀后缀的行为
int a = i++;  // a=5, i=6  先用后加
int b = ++i;  // i=7, b=7  先加后用

// ✅ 单独使用时无区别，在表达式中才需区分
i++;   // 等价于 ++i，都是 i=i+1
```

记忆：**运算符在前，先运算再取值；变量在前，先取值再运算**——就像排队，先到先服务。

### 复合赋值的隐式转型

```java
short s = 10;
// ❌ s = s + 5;  // 编译错误: int 不能自动转 short
// ✅ += 自带强制转换
s += 5;           // 等价于 s = (short)(s + 5)
```

### == 比较引用 vs equals 比较内容

```java
String s1 = new String("hello");
String s2 = new String("hello");

// ❌ == 比较的是引用地址
s1 == s2;           // false，两个不同对象

// ✅ equals 比较的是内容
s1.equals(s2);      // true
```

### 浮点数不能用 == 比较

```java
double a = 0.1 + 0.2;
// ❌ 精度误差导致比较失败
a == 0.3;           // false

// ✅ 用误差范围比较
Math.abs(a - 0.3) < 1e-9;  // true
```

## 四、实现方式

### 取模的典型应用

```java
int num = 1234;
int ge = num % 10;          // 个位: 4
int shi = (num / 10) % 10;  // 十位: 3

// 判断奇偶
num % 2 == 0  // 偶数

// 判断闰年
boolean leap = (year % 4 == 0 && year % 100 != 0) || (year % 400 == 0);
```

### 自增在循环中的使用

```java
// 累加 1~100
int sum = 0;
for (int i = 1; i <= 100; i++) {  // i++ 此处前缀后缀无差别
    sum += i;
}
// 输出: 5050
```

### 关系运算的组合判断

```java
int score = 85;
// 范围判断
boolean valid = score >= 0 && score <= 100;

// 分段判断
String grade = score >= 90 ? "优秀"
             : score >= 60 ? "及格"
             : "不及格";
```

### 整数溢出防护

```java
int max = Integer.MAX_VALUE;  // 2147483647
// ❌ 溢出，无报错
int overflow = max + 1;       // -2147483648

// ✅ 用 long 防溢出
long safe = (long)max + 1;    // 2147483648
```

## 五、Q&A

**Q：`10 / 0` 和 `10.0 / 0.0` 结果一样吗？**

A：不一样。整数除零抛出 `ArithmeticException`；浮点除零返回 `Infinity`（无穷大），不抛异常。这是 IEEE 754 浮点标准的规定。

**Q：`-7 % 3` 等于几？**

A：`-1`。Java 取模结果的符号与左操作数一致——就像温度计，零下 7 度对 3 取整后剩余的度数还是负的。

**Q：`if (a = 5)` 为什么编译错误？**

A：`=` 是赋值，返回 int；`if` 需要 boolean。Java 不允许 int 隐式转 boolean，这比 C 语言更安全。

**Q：复合赋值 `byte b = 10; b += 127;` 结果是什么？**

A：`-119`。`+=` 虽然自带强制转换不会编译报错，但溢出依然发生——安全的是编译，不是运行结果。

## 六、小结表格

| 类别 | 运算符 | 关键规则 |
|------|--------|----------|
| 算术 | `+ - * / %` | 整数除法截断；`%` 符号跟左操作数 |
| 自增自减 | `++ --` | 前缀先变后用，后缀先用后变；单独使用无差别 |
| 赋值 | `= += -=` | 复合赋值含隐式强转 `a+=b` → `a=(type)(a+b)` |
| 关系 | `== != > < >= <=` | 结果为 boolean；`==` 比引用，`equals` 比内容 |
| 陷阱 | — | 整数溢出无报错；浮点数有精度误差；除零行为不同 |
| 优先级 | — | `++ --` > `* / %` > `+ -` > 关系 > 赋值 |
