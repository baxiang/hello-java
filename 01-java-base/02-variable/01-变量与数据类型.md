# 变量与数据类型

## 一、问题引入

你刚写完第一个 Java 程序，输出了一行 "Hello World"。但真实程序需要处理数据——用户年龄、商品价格、是否通过考试……这些数据存在哪里？怎么区分"18 岁"和"1.75 米"？

更麻烦的是，如果把价格 `9.9` 存成整数，小数部分直接消失；把年龄 `150` 存成 `byte`，直接溢出变负数。选错类型，数据就错了。

**变量**是存储数据的容器，**数据类型**决定容器能装什么。两者必须一起理解。

## 二、核心概念

### 变量的本质

变量就像一个**贴了标签的盒子**：标签是变量名，盒子大小由类型决定，里面装的是值。

```
变量 = 数据类型 + 变量名 + 值
```

### 8 种基本数据类型

Java 的基本类型分为四大类，共 8 种：

| 分类 | 类型 | 字节 | 取值范围 | 默认值 | 生活类比 |
|------|------|------|----------|--------|----------|
| 整数 | `byte` | 1 | -128 ~ 127 | 0 | 邮箱格子 |
| 整数 | `short` | 2 | -32768 ~ 32767 | 0 | 抽屉 |
| 整数 | `int` | 4 | -2³¹ ~ 2³¹-1（约±21亿） | 0 | 仓库 |
| 整数 | `long` | 8 | -2⁶³ ~ 2⁶³-1 | 0L | 货场 |
| 浮点 | `float` | 4 | 约 7 位有效数字 | 0.0f | 粗略秤 |
| 浮点 | `double` | 8 | 约 15 位有效数字 | 0.0 | 精密天平 |
| 字符 | `char` | 2 | 0 ~ 65535（Unicode） | '\u0000' | 名牌 |
| 布尔 | `boolean` | 1 | true / false | false | 开关 |

> 日常开发 90% 的情况只用 `int`、`double`、`boolean` 三种。

## 三、代码对比

### 变量命名：随意 vs 规范

```java
// ❌ 无意义命名
int a = 18;
String s = "张三";
double d = 99.99;

// ✅ 有意义命名
int age = 18;
String name = "张三";
double price = 99.99;
```

命名要"见名知意"——三天后看代码，`price` 一眼就懂，`d` 要猜半天。

### 浮点字面量：遗漏后缀

```java
// ❌ 编译错误：3.14 默认是 double，不能直接赋给 float
float price = 3.14;

// ✅ 加 F 后缀
float price = 3.14f;
```

### 整数溢出：类型选错

```java
// ❌ byte 装不下 200
byte score = 200;  // 编译错误

// ✅ 用 int
int score = 200;
```

## 四、实现方式

### 变量的三种定义方式

```java
// 方式一：声明并初始化（最常用）
int age = 18;

// 方式二：先声明后赋值
int score;
score = 90;

// 方式三：同时定义多个同类型变量
int x = 1, y = 2, z = 3;
```

### 整数类型

```java
byte age = 18;                   // 小范围整数
short port = 8080;               // 较少使用
int count = 1000;                // 默认整数类型，最常用
long population = 7000000000L;   // 大整数，末尾加 L // 关键：L 后缀

// 进制表示
int decimal = 10;      // 十进制
int binary = 0b1010;   // 二进制
int octal = 012;       // 八进制
int hex = 0xA;         // 十六进制
int million = 1_000_000; // 数字分隔符（Java 7+）
```

### 浮点类型

```java
float price = 99.99f;            // 单精度，加 F 后缀 // 关键：F 后缀
double pi = 3.1415926535;        // 双精度，默认浮点类型

// 精度问题
System.out.println(0.1 + 0.2);   // 输出: 0.30000000000000004
```

> 金额计算不要用 float/double，用 `BigDecimal`。

### 字符类型

```java
char grade = 'A';          // 字符字面量，单引号
char gender = '男';        // 中文也行
char unicode = '\u0041';   // Unicode 表示，即 'A'

// 转义字符
char newline = '\n';       // 换行
char tab = '\t';           // 制表符
char quote = '\'';         // 单引号
char backslash = '\\';     // 反斜杠

// char 本质是整数（Unicode 编码）
char c = 'A';
int code = c;              // 输出: 65
```

### 布尔类型

```java
boolean isStudent = true;
boolean isPassed = score >= 60;  // 布尔表达式

// ❌ boolean 不能与 int 互换
// int x = true;            // 编译错误
// boolean b = 1;           // 编译错误

// ✅ 用比较表达式
boolean b = (x == 1);
```

### 变量作用域

```java
public static void main(String[] args) {
    int x = 10;             // 作用域：整个 main 方法

    if (x > 5) {
        int y = 20;         // 作用域：if 块内
        System.out.println(y);  // ✅ 可访问
    }
    // System.out.println(y);   // 编译错误：y 超出作用域
}
```

## 五、Q&A

**Q：为什么整数默认是 `int`，小数默认是 `double`？**

A：这是 Java 的设计选择。`int` 是 32 位，对大多数场景够用且性能好；`double` 精度更高，避免 `float` 精度不足导致的误差。需要更小范围时再手动指定。

**Q：`long` 后缀用大写 `L` 还是小写 `l`？**

A：推荐大写 `L`。小写 `l` 容易和数字 `1` 混淆：`100l` 看起来像 `1001`。

**Q：`char` 和 `String` 有什么区别？**

A：`char` 是基本类型，只能存一个字符，用单引号 `'A'`；`String` 是引用类型，可以存多个字符，用双引号 `"Hello"`。

**Q：变量未赋值直接使用会怎样？**

A：编译错误。局部变量必须先赋值再使用，这是 Java 的安全机制——防止使用未定义的"垃圾值"。

## 六、小结表格

| 要点 | 说明 |
|------|------|
| 变量定义 | `类型 变量名 = 值;`，先声明后使用 |
| 整数选型 | 默认 `int`，超大数用 `long` 加 `L` 后缀 |
| 浮点选型 | 默认 `double`，用 `float` 加 `F` 后缀 |
| 字符 | `char` 用单引号，本质是 Unicode 整数 |
| 布尔 | 只有 `true`/`false`，不能与整数互换 |
| 命名规范 | 小驼峰（`studentAge`），见名知意 |
| 作用域 | 变量只在定义它的 `{}` 内有效 |
| 浮点精度 | `0.1 + 0.2 ≠ 0.3`，金额用 `BigDecimal` |
