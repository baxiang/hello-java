# 类型转换与常量

## 一、问题引入

你把用户输入的年龄 `18` 存在 `int` 里，现在要算平均年龄——总人数是 `long`，除一下？编译报错，类型不匹配。反过来，`double pi = 3.14` 想转成 `int` 取整？直接截断，变成 `3`，小数没了。

更头疼的是：计算商品税金 `price * 0.13`，`0.13` 在计算机里存不精确，结果差了一分钱。金融系统里，一分钱的偏差就是事故。

还有，代码里到处写着 `if (score >= 60)`——这个 `60` 是什么意思？及格线？加班时长？三个月后你自己都忘了。这种"魔术数字"必须用**常量**消灭。

## 二、核心概念

### 类型转换的两种方式

```
类型转换
├── 自动类型转换（小 → 大）  像小杯水倒进大杯，不会洒
└── 强制类型转换（大 → 小）  像大杯水倒进小杯，可能溢出
```

自动转换方向：

```
byte → short → int → long → float → double
                  ↑
                char
```

### 常量

常量是**值不能改变**的量，用 `final` 修饰。像刻在石头上的数字——写上去就改不了了。

```java
final 数据类型 常量名 = 值;
```

## 三、代码对比

### 自动转换：运算中的隐式提升

```java
// ❌ 误解：以为结果是 int
byte b1 = 10;
byte b2 = 20;
byte b3 = b1 + b2;  // 编译错误：byte 运算自动提升为 int

// ✅ 正确：接收为 int
int b3 = b1 + b2;   // 30
```

> byte、short、char 参与运算时，结果自动提升为 int——这是 Java 的硬性规则。

### 强制转换：精度丢失

```java
// ❌ 直接截断，不是四舍五入
double d = 3.99;
int i = (int) d;  // 输出: 3（不是 4！）

// ✅ 需要四舍五入时用 Math.round
int i = (int) Math.round(d);  // 输出: 4
```

### 整数除法的坑

```java
// ❌ 以为能得到小数结果
int a = 10, b = 3;
double result = a / b;       // 输出: 3.0（先整除再转换）

// ✅ 先转换其中一个操作数
double result = (double) a / b;  // 输出: 3.333...
double result2 = a * 1.0 / b;    // 输出: 3.333...
```

### 魔术数字 vs 常量

```java
// ❌ 魔术数字：60 是什么意思？
if (score >= 60) {
    System.out.println("及格");
}

// ✅ 常量：一目了然
final int PASS_LINE = 60;
if (score >= PASS_LINE) {
    System.out.println("及格");
}
```

### 常量命名规范

```java
// ❌ 看起来像变量
final int maxCount = 100;
final int MaxCount = 100;

// ✅ 全大写 + 下划线
final int MAX_COUNT = 100;
```

## 四、实现方式

### 自动类型转换

```java
// 小范围 → 大范围，自动完成
byte b = 10;
int i = b;          // byte → int
long l = i;         // int → long
double d = l;       // long → double

// char → int（取 Unicode 编码）
char c = 'A';
int code = c;       // 输出: 65

// 混合运算：结果为范围最大的类型
int x = 10;
double y = 3.14;
double result = x + y;  // int 自动提升为 double
```

### 强制类型转换

```java
// 语法：目标类型 变量 = (目标类型) 原值;
double pi = 3.14159;
int intPi = (int) pi;       // 输出: 3

long big = 1000L;
int small = (int) big;      // 输出: 1000

// 溢出警告！
int overflow = 130;
byte b = (byte) overflow;   // 输出: -126（溢出）
// 130 超出 byte 范围 (-128~127)
```

### 字符串与数值互转

```java
// String → 数值
int num = Integer.parseInt("123");
double d = Double.parseDouble("3.14");
long l = Long.parseLong("1000000");

// ❌ 格式不对会抛异常
// Integer.parseInt("abc");  // NumberFormatException

// 数值 → String
String s1 = 123 + "";           // 拼接空字符串
String s2 = String.valueOf(123); // valueOf 方法
String s3 = Integer.toString(123); // toString 方法
```

### 浮点数精度问题与解决

```java
// ❌ 浮点数运算不精确
double a = 0.1, b = 0.2;
System.out.println(a + b);  // 输出: 0.30000000000000004

// ✅ 用 BigDecimal 解决
import java.math.BigDecimal;
BigDecimal bd1 = new BigDecimal("0.1");  // 关键：用字符串构造
BigDecimal bd2 = new BigDecimal("0.2");
System.out.println(bd1.add(bd2));        // 输出: 0.3
```

> `BigDecimal` 必须用字符串构造，用 `new BigDecimal(0.1)` 仍然不精确。

### 溢出防护

```java
int max = Integer.MAX_VALUE;   // 2147483647
int overflow = max + 1;        // 输出: -2147483648（溢出变负数！）

// ✅ 使用更大类型
long safe = (long) max + 1;    // 输出: 2147483648
```

### final 常量

```java
// 局部常量
final int MAX_RETRY = 3;

// 类常量（最常用：static final）
public class Constants {
    public static final double PI = 3.14159;
    public static final int MAX_VALUE = 100;
}

// 使用类常量
double area = Constants.PI * r * r;
```

### final 修饰引用类型

```java
final StringBuilder sb = new StringBuilder("Hello");
sb.append(" World");             // ✅ 可以修改内容
// sb = new StringBuilder("Hi"); // 编译错误：不能修改引用

final int[] arr = {1, 2, 3};
arr[0] = 100;                    // ✅ 可以修改元素
// arr = new int[5];             // 编译错误：不能修改引用
```

> `final` 修饰引用类型时，引用不可变，但对象内容可变——就像门牌号不能改，但房子里的人可以换。

### final 必须初始化

```java
// ❌ 编译错误：final 变量未初始化
// final int x;

// ✅ 声明时初始化
final int x = 10;

// ✅ 在构造方法中初始化（成员变量适用）
public class Demo {
    final int y;
    public Demo() {
        y = 20;
    }
}
```

## 五、Q&A

**Q：`float` 的范围比 `long` 大，但精度比 `long` 低，怎么回事？**

A：浮点数用科学计数法存储——指数部分决定范围，尾数部分决定精度。`float` 的指数范围大，但尾数只有 23 位，超过 7 位有效数字就不准了。`long` 有 63 位尾数，整数精度远高于 `float`。

**Q：`(double) a / b` 和 `(double)(a / b)` 结果为什么不同？**

A：前者先转换 `a` 为 double，再除以 `b`（b 自动提升），结果是 `3.333...`；后者先做整数除法 `a/b` 得 `3`，再转 double 得 `3.0`。括号位置决定转换时机。

**Q：`static final` 和 `final` 有什么区别？**

A：`final` 是实例常量，每个对象各有一份；`static final` 是类常量，所有对象共享一份，通过类名直接访问。全局不变的值用 `static final`。

**Q：常量有什么实际好处？**

A：三点——可读性（`PASS_LINE` 比 `60` 清晰）、可维护性（改一处全局生效）、安全性（`final` 防止意外修改）。越是多人协作的项目，常量越重要。

## 六、小结表格

| 要点 | 说明 |
|------|------|
| 自动转换 | 小→大，无需额外代码，运算时自动提升 |
| 强制转换 | 大→小，需 `(类型)` 语法，可能丢精度或溢出 |
| byte/short 运算 | 结果自动提升为 `int` |
| 整数除法 | `int/int` 得 `int`，需先转 `double` 才得小数 |
| 浮点精度 | 金额用 `BigDecimal`，用字符串构造 |
| 常量 | `final` 修饰，命名全大写+下划线 |
| 类常量 | `public static final`，类名直接访问 |
| final 引用 | 引用不可变，对象内容可变 |
| 魔术数字 | 用常量替代，提高可读性与可维护性 |
